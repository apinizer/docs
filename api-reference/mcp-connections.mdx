## Endpoints

- [List MCP Connections](/api-reference/mcp-connections/list-mcp-connections) - List all MCP connections in a project
- [Get MCP Connection](/api-reference/mcp-connections/get-mcp-connection) - Retrieve a single MCP connection by name
- [Create MCP Connection](/api-reference/mcp-connections/create-mcp-connection) - Create or update (upsert) an MCP connection
- [Update MCP Connection](/api-reference/mcp-connections/update-mcp-connection) - Update an existing MCP connection
- [Delete MCP Connection](/api-reference/mcp-connections/delete-mcp-connection) - Delete an MCP connection

## Authentication

All endpoints require authentication using a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Permissions

- **Read** (list, get): `AI_DEVELOPMENT` + `VIEW` permission in the project
- **Write** (create, update, delete): `AI_DEVELOPMENT` + `MANAGE` permission in the project
- System administrators have full access

## Notes and Warnings

- **Name-based**: All references use the connection `name`, never the internal ID.
- **Upsert**: `POST /{connectionName}/` creates a new connection, or updates it if a connection with the same name already exists in the project.
- **Polymorphic body**: The request body for `POST`/`PUT` MUST include `"_class": "mcp"` so the server can deserialize it as an MCP connection.
- **Secret masking (INV-06)**: The `bearerToken` and `clientSecret` fields are never returned in read responses. Instead, boolean presence flags `bearerTokenSet` and `clientSecretSet` indicate whether each secret is configured.
- **Blank secret preserves stored value**: On update, sending a blank `bearerToken`/`clientSecret` keeps the currently stored (encrypted) value instead of clearing it.
- **Worker deploy**: A successful create/update encrypts secrets and deploys the connection to the gateway workers; delete undeploys it.
