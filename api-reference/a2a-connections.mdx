## Endpoints

- [List A2A Connections](/api-reference/a2a-connections/list-a2a-connections) - List all A2A outbound connections in a project
- [Get A2A Connection](/api-reference/a2a-connections/get-a2a-connection) - Get a single A2A outbound connection by name
- [Create A2A Connection](/api-reference/a2a-connections/create-a2a-connection) - Create (or upsert) an A2A outbound connection
- [Update A2A Connection](/api-reference/a2a-connections/update-a2a-connection) - Update an existing A2A outbound connection
- [Delete A2A Connection](/api-reference/a2a-connections/delete-a2a-connection) - Delete an A2A outbound connection and undeploy it from workers
- [Test A2A Connection](/api-reference/a2a-connections/test-a2a-connection) - Run a live AgentCard fetch probe against connection settings
- [Parse A2A Skills](/api-reference/a2a-connections/parse-a2a-skills) - Fetch and parse the remote AgentCard skills, persist them, and redeploy

## Authentication

All endpoints require authentication using a Personal API Access Token
(`Authorization: Bearer {token}`). JHipster JWTs do not work on APIops.

## Permissions

- All endpoints require the **`AI_DEVELOPMENT`** asset category — **VIEW** for reads
  (`GET`), **MANAGE** for writes (`POST`, `PUT`, `DELETE`, `test-connection`, `parse-skills`).
- Enabling **private-network egress** (`allowPrivateNetworkEgress = true`) is restricted to
  **system administrators**. A project-level `AI_DEVELOPMENT` + `MANAGE` token that is not a
  system admin cannot turn this flag on through any write endpoint (SSRF hardening).

## Notes

- **Name-based** - All references (project, connection) are by name. Runtime IDs are never
  required in the path.
- **Upsert** - `POST /{connectionName}/` creates the connection, or updates it in place if the
  name already exists.
- **Secret masking on read** - `bearerToken` and `clientSecret` are `@SecretData` fields. Read
  responses never disclose them; they are returned as boolean presence flags
  (`bearerTokenSet`, `clientSecretSet`) instead (INV-06).
- **Save = encrypt + deploy** - write operations encrypt `@SecretData` fields and push the
  connection to the workers through the owning service; delete undeploys.
