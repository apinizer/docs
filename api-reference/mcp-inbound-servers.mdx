## Endpoints

- [List MCP Inbound Servers](/api-reference/mcp-inbound-servers/list-mcp-inbound-servers) - List all MCP inbound servers in a project
- [Get MCP Inbound Server](/api-reference/mcp-inbound-servers/get-mcp-inbound-server) - Get a single MCP inbound server by name
- [Create MCP Inbound Server](/api-reference/mcp-inbound-servers/create-mcp-inbound-server) - Create an MCP inbound server (upsert)
- [Update MCP Inbound Server](/api-reference/mcp-inbound-servers/update-mcp-inbound-server) - Update an existing MCP inbound server
- [Delete MCP Inbound Server](/api-reference/mcp-inbound-servers/delete-mcp-inbound-server) - Delete an MCP inbound server and undeploy it

All endpoints are project-scoped under the base path:

```
/apiops/projects/{projectName}/mcp-inbound-servers
```

MCP inbound servers are referenced by **name** (not runtime ID). `POST /{serverName}/` is an **upsert** — it creates the server, or updates it in place if a server with the same name already exists in the project.

## Authentication

All endpoints require authentication using a Personal API Access Token
(`Authorization: Bearer YOUR_TOKEN`). JHipster session JWTs do not work on APIops.

## Permissions

- Reads (`GET`) require **`AI_DEVELOPMENT` + `VIEW`** permission in the project.
- Writes (`POST` / `PUT` / `DELETE`) require **`AI_DEVELOPMENT` + `MANAGE`** permission in the project.
