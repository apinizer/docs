---
title: "Create MCP Connection"
description: "Creates an AI Gateway MCP outbound connection in the specified project. This endpoint is an upsert — if a connection with the same name already exists it is updated in place. On save, @SecretData fields are encrypted and the connection is deployed to the workers."
---

## Endpoint

```
POST /apiops/projects/{projectName}/mcp-connections/{connectionName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| connectionName | string | Yes | MCP outbound connection name (must match `name` in the body) |

### Query Parameters

None

### Request Body

The body is a polymorphic `ConnectionConfig` document. Because MCP is a connection-config subtype,
the discriminator `"_class": "mcp"` is **required** so the payload is deserialized as an MCP
connection.

#### Full JSON Body Example - Bearer Token Auth

```json
{
  "_class": "mcp",
  "name": "github-mcp",
  "description": "GitHub MCP server",
  "enabled": true,
  "endpoint": "https://mcp.example.com/mcp",
  "transport": "HTTP_SSE",
  "mcpProtocolVersion": "2024-11-05",
  "allowPrivateNetworkEgress": false,
  "authScheme": "BEARER",
  "bearerToken": "ghp_...secret...",
  "capabilityCacheTtlSeconds": 300,
  "capabilityCacheEnabled": true
}
```

#### Full JSON Body Example - OAuth2 Client Credentials Auth

```json
{
  "_class": "mcp",
  "name": "partner-mcp",
  "description": "Partner MCP server (OAuth2)",
  "enabled": true,
  "endpoint": "https://partner.example.com/mcp",
  "transport": "HTTP_SSE",
  "mcpProtocolVersion": "2024-11-05",
  "allowPrivateNetworkEgress": false,
  "authScheme": "OAUTH2_CLIENT_CREDENTIALS",
  "tokenEndpoint": "https://partner.example.com/oauth/token",
  "clientId": "apinizer-client",
  "clientSecret": "s3cr3t-value",
  "scope": "mcp.invoke mcp.read",
  "capabilityCacheTtlSeconds": 300,
  "capabilityCacheEnabled": true
}
```

#### Request Body Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| _class | string | Yes | - | Connection-config discriminator — must be `"mcp"` |
| name | string | Yes | - | Connection name — must equal the `connectionName` path variable |
| description | string | No | - | Connection description |
| enabled | boolean | No | `true` | Whether the connection is enabled |
| endpoint | string | Yes | - | MCP server base URL (e.g. `https://mcp.example.com/mcp`) |
| transport | string | No | `HTTP_SSE` | Transport type: `HTTP_SSE` or `STDIO` (STDIO is not supported yet) |
| mcpProtocolVersion | string | No | `2024-11-05` | MCP protocol version (adapter lookup key) |
| allowPrivateNetworkEgress | boolean | No | `false` | Permit cluster-internal / private-network (RFC1918) egress. Metadata (169.254.169.254), loopback, link-local, multicast and CGNAT targets remain blocked regardless |
| authScheme | string | No | `NONE` | Auth scheme: `BEARER`, `OAUTH2_CLIENT_CREDENTIALS`, `MTLS`, or `NONE` |
| bearerToken | string | Conditional | - | Static bearer token for the `BEARER` scheme (`@SecretData`, encrypted on save). A blank value on update preserves the stored secret |
| credentialRef | string | No | - | Credential reference for the `OAUTH2_CLIENT_CREDENTIALS` scheme |
| tokenEndpoint | string | Conditional | - | OAuth2 token endpoint URL (`OAUTH2_CLIENT_CREDENTIALS`) |
| clientId | string | Conditional | - | OAuth2 client ID (`OAUTH2_CLIENT_CREDENTIALS`) |
| clientSecret | string | Conditional | - | OAuth2 client secret (`@SecretData`, encrypted on save). A blank value on update preserves the stored secret |
| scope | string | No | - | OAuth2 scope, space-delimited (`OAUTH2_CLIENT_CREDENTIALS`) |
| sslConfigRef | string | No | - | SSL config reference for the `MTLS` scheme (placeholder — handshake not yet wired) |
| capabilityCacheTtlSeconds | integer | No | `300` | Capability cache TTL in seconds (`0` = no cache) |
| capabilityCacheEnabled | boolean | No | `true` | Whether the capability cache is enabled |
| failoverList | array[string] | No | `[]` | Ordered list of MCP connection IDs used as a failover chain |
| discoveredTools | array | No | `[]` | Statically discovered tools. Normally populated by the "Discover / Reparse" flow, not set manually |

### Notes

- `name` in the body must match the `connectionName` path variable (case-insensitive), otherwise a `400` is returned.
- `_class` must be `"mcp"`; without it the body cannot be deserialized as an MCP connection.
- The response does not echo the saved object — use [Get MCP Connection](/api-reference/mcp-connections/get-mcp-connection) to read it back (with secrets masked).

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| deploymentResult | object | Deployment outcome |
| deploymentResult.success | boolean | `true` when the connection was saved and deployed |

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "MCP connection name in path (github-mcp) does not match name in body (github-mcpp)!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "MCP connection body can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## Notes and Warnings

- **Upsert Semantics**: `POST /{connectionName}/` creates the connection, or updates it in place
  if the name already exists in the project. To force an update-only operation use
  [Update MCP Connection](/api-reference/mcp-connections/update-mcp-connection).
- **Polymorphic Body**: The `"_class": "mcp"` discriminator is mandatory; omitting it causes the
  body to be rejected during deserialization.
- **Secret Handling (INV-06)**: `bearerToken` and `clientSecret` are `@SecretData` — encrypted on
  save and never returned on read. On an upsert that hits an existing connection, sending a **blank**
  `bearerToken`/`clientSecret` preserves the stored secret.
- **Private-Network Egress**: Setting `allowPrivateNetworkEgress = true` lets the connection reach
  in-cluster MCP servers (RFC1918 / IPv6 ULA). Cloud metadata (169.254.169.254), loopback,
  link-local, multicast, `0.0.0.0/8` and CGNAT (`100.64.0.0/10`) targets stay blocked always.
- **Save = Encrypt + Deploy**: On success the connection is deployed to the gateway workers.
