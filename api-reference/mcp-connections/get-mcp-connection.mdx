---
title: "Get MCP Connection"
description: "Gets a single AI Gateway MCP outbound connection by name. Secret fields (bearerToken, clientSecret) are returned masked as boolean presence flags, and the referencing API proxies are listed in usedByRefs."
---

## Endpoint

```
GET /apiops/projects/{projectName}/mcp-connections/{connectionName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| connectionName | string | Yes | MCP outbound connection name |

### Query Parameters

None

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | Single-element list containing the MCP connection object |
| resultCount | integer | Number of connections returned (always `1` on success) |

#### MCP Connection Object Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Internal connection ID (read-only; name is the API reference) |
| projectId | string | Owning project ID (read-only) |
| projectName | string | Owning project name (read-only) |
| name | string | Connection name (unique within the project) |
| description | string | Connection description |
| enabled | boolean | Whether the connection is enabled |
| endpoint | string | MCP server base URL (e.g. `https://mcp.example.com/mcp`) |
| transport | string | Transport type: `HTTP_SSE` or `STDIO` |
| mcpProtocolVersion | string | MCP protocol version (adapter lookup key), default `2024-11-05` |
| allowPrivateNetworkEgress | boolean | Whether cluster-internal / private-network (RFC1918) egress is permitted |
| authScheme | string | Auth scheme: `BEARER`, `OAUTH2_CLIENT_CREDENTIALS`, `MTLS`, or `NONE` |
| bearerTokenSet | boolean | `true` if a Bearer token is configured (the plaintext is never returned — INV-06) |
| credentialRef | string | Credential reference for the `OAUTH2_CLIENT_CREDENTIALS` scheme |
| tokenEndpoint | string | OAuth2 token endpoint URL (`OAUTH2_CLIENT_CREDENTIALS`) |
| clientId | string | OAuth2 client ID (`OAUTH2_CLIENT_CREDENTIALS`) |
| scope | string | Optional OAuth2 scope (space-delimited) |
| clientSecretSet | boolean | `true` if an OAuth2 client secret is configured (the plaintext is never returned — INV-06) |
| sslConfigRef | string | SSL config reference for the `MTLS` scheme (placeholder — handshake not yet wired) |
| capabilityCacheTtlSeconds | integer | Capability cache TTL in seconds (`0` = no cache), default `300` |
| capabilityCacheEnabled | boolean | Whether the capability cache is enabled, default `true` |
| failoverList | array[string] | Ordered list of MCP connection IDs used as a failover chain |
| credentialMappingRef | string | Credential mapping reference (reserved for future use) |
| discoveredToolCount | integer | Number of statically discovered tools for this connection |
| toolsParsedAt | integer | Epoch millis of the last successful tools parse (`null` = never parsed) |
| healthStatus | string | Last health-probe status: `HEALTHY`, `DOWN`, or `UNKNOWN` (`null` = never probed) |
| lastHealthCheckAt | string | Timestamp of the last completed health probe |
| healthMessage | string | Short error description when `healthStatus` is `DOWN`; otherwise `null` |
| usedByRefs | array[string] | Names of API proxies that reference this connection (populated only in the single-get response) |

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "MCP connection (name: github-mcp) was not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/mcp-connections/github-mcp/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Secret Masking (INV-06)**: `bearerToken` and `clientSecret` are never returned. Only the
  boolean presence flags `bearerTokenSet` and `clientSecretSet` indicate whether they are configured.
- **usedByRefs**: This field lists the API proxies referencing the connection and is populated only
  in this single-get response (the [list endpoint](/api-reference/mcp-connections/list-mcp-connections)
  omits it to avoid N-query cost).
- **Not Found**: A missing connection returns a `400 Bad Request` with a `FAILURE` status, not `404`.
- **Case Sensitivity**: Project and connection names are case-sensitive in path references; the
  name lookup itself is performed case-insensitively.
- **Permission**: Requires `AI_DEVELOPMENT` + `VIEW` in the project.
