---
title: "Delete A2A Connection"
description: "Deletes an existing AI Gateway A2A outbound connection from the specified project and undeploys it from the workers. References are resolved by name."
---

## Endpoint

```
DELETE /apiops/projects/{projectName}/a2a-connections/{connectionName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| connectionName | string | Yes | A2A outbound connection name (the connection must already exist) |

### Query Parameters

None

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| deploymentResult | object | Undeployment outcome |
| deploymentResult.success | boolean | `true` when the connection was removed and undeployed |

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A outbound connection (name: weather-agent) was not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X DELETE \
  "https://demo.apinizer.com/apiops/projects/MyProject/a2a-connections/weather-agent/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Undeploy on Delete**: The connection is removed from the database and undeployed from the
  workers in the same operation.
- **Project Scoped**: Deletion is guarded by project ownership (IDOR protection) — a connection
  can only be deleted from the project that owns it.
- **Not Found**: Deleting a connection that does not exist returns a `400 Bad Request` with a
  `FAILURE` status, not `404`.
- **In-Use Connections**: Deleting a connection that is still referenced by API proxies will break
  those proxies at runtime. Check `usedByRefs` from
  [Get A2A Connection](/api-reference/a2a-connections/get-a2a-connection) first.
- **Irreversible**: There is no soft-delete — recreate the connection with
  [Create A2A Connection](/api-reference/a2a-connections/create-a2a-connection) if needed.
