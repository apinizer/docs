---
title: "Parse A2A Skills"
description: "Discovers (or re-parses) the skills of an existing AI Gateway A2A outbound connection. Fetches the remote agent's AgentCard, persists the discovered skill list on the connection, and redeploys it to the workers. This is the API equivalent of the manual 'Discover / Reparse' action."
---

## Endpoint

```
POST /apiops/projects/{projectName}/a2a-connections/{connectionName}/parse-skills/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| connectionName | string | Yes | A2A outbound connection name (the connection must already exist) |

### Query Parameters

None

### Request Body

None. The connection's stored settings (endpoint, auth) are used to fetch the AgentCard — you do
not send a body.

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | Single-element list containing the updated connection object |
| resultCount | integer | Number of connections returned (always `1` on success) |

#### Discovered Skill Object Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Skill id as advertised by the agent (`AgentCard.skills[].id`) |
| name | string | Skill name (`AgentCard.skills[].name`) — the runtime allow-gate key |
| description | string | Human-readable skill description |
| tags | array[string] | Skill tags |

> The full connection object shape is documented in
> [Get A2A Connection](/api-reference/a2a-connections/get-a2a-connection). `discoveredSkillCount`
> and `skillsParsedAt` are updated to reflect this parse.

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A outbound connection (name: weather-agent) was not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X POST \
  "https://demo.apinizer.com/apiops/projects/MyProject/a2a-connections/weather-agent/parse-skills/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Connection Must Exist**: The named connection must already exist — parse-skills operates on the
  stored connection, not on a body you supply. Save it first with
  [Create A2A Connection](/api-reference/a2a-connections/create-a2a-connection).
- **Persist + Redeploy**: A successful parse persists the discovered skill list on the connection and
  redeploys it to the workers (deferred, after commit).
- **Runtime Skill Gate**: At runtime the `PolicyAiA2aAgentCall` policy validates each requested skill
  name against `discoveredSkills`. When the list is non-empty it is fail-closed (only listed skills
  are allowed); an empty list falls back to the explicit allow-list configured on the policy.
- **Secrets Masked**: The returned connection masks `bearerToken`/`clientSecret` as
  `bearerTokenSet`/`clientSecretSet` (INV-06).
- **AgentCard Fetch**: Discovery reaches out to the remote agent's AgentCard. A failed fetch
  (unreachable agent, auth failure) surfaces as an error response.
