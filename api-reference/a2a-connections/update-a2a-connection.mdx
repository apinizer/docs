---
title: "Update A2A Connection"
description: "Updates an existing AI Gateway A2A outbound connection in the specified project. A blank bearerToken or clientSecret preserves the stored secret. On save, @SecretData fields are encrypted and the connection is redeployed to the workers."
---

## Endpoint

```
PUT /apiops/projects/{projectName}/a2a-connections/{connectionName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| connectionName | string | Yes | A2A outbound connection name (the connection must already exist) |

### Query Parameters

None

### Request Body

The body is a polymorphic `ConnectionConfig` document. Because A2A is a connection-config subtype,
the discriminator `"_class": "a2a"` is **required**. If `name` is omitted from the body it defaults
to the `connectionName` path variable.

#### Full JSON Body Example

```json
{
  "_class": "a2a",
  "name": "weather-agent",
  "description": "External weather A2A agent (updated)",
  "enabled": true,
  "endpoint": "https://agent.example.com/a2a",
  "a2aProtocolVersion": "1.0",
  "allowPrivateNetworkEgress": false,
  "authScheme": "BEARER",
  "bearerToken": "",
  "agentCardCacheTtlSeconds": 600,
  "agentCardCacheEnabled": true
}
```

> In the example above `bearerToken` is sent blank (`""`), so the previously stored, encrypted
> bearer token is preserved unchanged.

#### Request Body Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| _class | string | Yes | - | Connection-config discriminator — must be `"a2a"` |
| name | string | No | path variable | Connection name — if present must equal the `connectionName` path variable; if blank it falls back to the path variable |
| description | string | No | - | Connection description |
| enabled | boolean | No | `true` | Whether the connection is enabled |
| endpoint | string | Yes | - | A2A agent endpoint (AgentCard well-known URL / message endpoint) |
| a2aProtocolVersion | string | No | `1.0` | A2A protocol/adapter version |
| allowPrivateNetworkEgress | boolean | No | `false` | Permit cluster-internal / private-network egress. **System-admin only** — a non-admin caller setting this to `true` is rejected |
| authScheme | string | No | `NONE` | Auth scheme: `BEARER`, `OAUTH2_CLIENT_CREDENTIALS`, `MTLS`, or `NONE` |
| bearerToken | string | No | - | Static bearer token for the `BEARER` scheme (`@SecretData`). **Blank preserves the stored secret** |
| credentialRef | string | No | - | Credential reference for the `OAUTH2_CLIENT_CREDENTIALS` scheme |
| tokenEndpoint | string | No | - | OAuth2 token endpoint URL (`OAUTH2_CLIENT_CREDENTIALS`) |
| clientId | string | No | - | OAuth2 client ID (`OAUTH2_CLIENT_CREDENTIALS`) |
| clientSecret | string | No | - | OAuth2 client secret (`@SecretData`). **Blank preserves the stored secret** |
| scope | string | No | - | OAuth2 scope, space-delimited (`OAUTH2_CLIENT_CREDENTIALS`) |
| sslConfigRef | string | No | - | SSL config reference for the `MTLS` scheme (placeholder) |
| agentCardCacheTtlSeconds | integer | No | `300` | AgentCard cache TTL in seconds (`0` = no cache) |
| agentCardCacheEnabled | boolean | No | `true` | Whether AgentCard caching is enabled |
| discoveredSkills | array | No | `[]` | Discovered skills. Normally populated by [Parse A2A Skills](/api-reference/a2a-connections/parse-a2a-skills), not set manually |

### Notes

- The connection identified by `connectionName` **must already exist**, otherwise a `400` is returned.
- If `name` is present in the body it must match the `connectionName` path variable (case-insensitive).
- `_class` must be `"a2a"`; without it the body cannot be deserialized as an A2A connection.

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| deploymentResult | object | Deployment outcome |
| deploymentResult.success | boolean | `true` when the connection was updated and redeployed |

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A outbound connection (name: weather-agent) was not found!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "Private-network egress (allowPrivateNetworkEgress) can only be enabled by a system administrator!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X PUT \
  "https://demo.apinizer.com/apiops/projects/MyProject/a2a-connections/weather-agent/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "_class": "a2a",
    "name": "weather-agent",
    "description": "External weather A2A agent (updated)",
    "enabled": true,
    "endpoint": "https://agent.example.com/a2a",
    "a2aProtocolVersion": "1.0",
    "allowPrivateNetworkEgress": false,
    "authScheme": "BEARER",
    "bearerToken": "",
    "agentCardCacheTtlSeconds": 600,
    "agentCardCacheEnabled": true
  }'
```

## Notes and Warnings

- **Update-Only**: Unlike [Create A2A Connection](/api-reference/a2a-connections/create-a2a-connection)
  (which upserts), `PUT` fails with a `400` when the named connection does not exist.
- **Secret Preservation**: Sending a blank `bearerToken`/`clientSecret` keeps the stored, encrypted
  value unchanged. Send a non-blank value to rotate the secret (INV-06).
- **Secrets Never Returned**: The response does not echo the object; read it back with
  [Get A2A Connection](/api-reference/a2a-connections/get-a2a-connection), where secrets are masked.
- **Private-Network Egress**: `allowPrivateNetworkEgress = true` is rejected for non-system-admin
  callers (SSRF hardening).
- **Save = Encrypt + Deploy**: On success the connection is redeployed to the workers (deferred,
  after commit).
