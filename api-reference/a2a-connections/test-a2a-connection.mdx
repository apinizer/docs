---
title: "Test A2A Connection"
description: "Runs a live AgentCard fetch probe against the supplied A2A outbound connection settings. This is transport-only â€” nothing is persisted, deployed, or cached. Use it to validate an endpoint and auth configuration before saving."
---

## Endpoint

```
POST /apiops/projects/{projectName}/a2a-connections/{connectionName}/test-connection/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| connectionName | string | Yes | A2A outbound connection name (must match `name` in the body, or the body `name` may be blank) |

### Query Parameters

None

### Request Body

The body is the polymorphic `ConnectionConfig` (A2A) settings to probe. The discriminator
`"_class": "a2a"` is **required**. The connection need not already exist â€” this endpoint probes the
supplied settings directly (transport-only). If `name` is omitted it falls back to the
`connectionName` path variable.

#### Full JSON Body Example

```json
{
  "_class": "a2a",
  "name": "weather-agent",
  "endpoint": "https://agent.example.com/a2a",
  "a2aProtocolVersion": "1.0",
  "allowPrivateNetworkEgress": false,
  "authScheme": "BEARER",
  "bearerToken": "eyJhbGciOi...secret..."
}
```

#### Request Body Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| _class | string | Yes | - | Connection-config discriminator â€” must be `"a2a"` |
| name | string | No | path variable | Connection name â€” if present must equal the `connectionName` path variable; blank falls back to the path variable |
| endpoint | string | Yes | - | A2A agent endpoint to probe (AgentCard well-known URL / message endpoint) |
| a2aProtocolVersion | string | No | `1.0` | A2A protocol/adapter version |
| allowPrivateNetworkEgress | boolean | No | `false` | Permit private-network egress during the probe. **System-admin only** â€” a non-admin caller setting this to `true` is rejected |
| authScheme | string | No | `NONE` | Auth scheme: `BEARER`, `OAUTH2_CLIENT_CREDENTIALS`, `MTLS`, or `NONE` |
| bearerToken | string | No | - | Static bearer token for the `BEARER` scheme (`@SecretData`) |
| tokenEndpoint | string | No | - | OAuth2 token endpoint URL (`OAUTH2_CLIENT_CREDENTIALS`) |
| clientId | string | No | - | OAuth2 client ID (`OAUTH2_CLIENT_CREDENTIALS`) |
| clientSecret | string | No | - | OAuth2 client secret (`@SecretData`) |
| scope | string | No | - | OAuth2 scope, space-delimited (`OAUTH2_CLIENT_CREDENTIALS`) |
| sslConfigRef | string | No | - | SSL config reference for the `MTLS` scheme (placeholder) |

### Notes

- Transport-only: no persist, no deploy, no cache mutation. `${env}` placeholders in the body are
  resolved before the probe runs.
- `${var}` runtime variables (non-environment) are not resolvable outside a live request and are
  left as-is.

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true,
    "detail": "A2A connection test successful"
  }
}
```

A **reachable-but-failing** probe also returns `200 OK` with `success: false` and the probe error
in `detail`:

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": false,
    "detail": "AgentCard fetch failed: 404 Not Found"
  }
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` (`SUCCESS` means the request was processed, not that the probe passed) |
| deploymentResult | object | Probe outcome |
| deploymentResult.success | boolean | `true` when the AgentCard fetch probe succeeded |
| deploymentResult.detail | string | Human-readable probe result (`"A2A connection test successful"` on success, otherwise the error) |

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A outbound connection name in path (weather-agent) does not match name in body (other-agent)!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "Private-network egress (allowPrivateNetworkEgress) can only be enabled by a system administrator!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X POST \
  "https://demo.apinizer.com/apiops/projects/MyProject/a2a-connections/weather-agent/test-connection/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "_class": "a2a",
    "name": "weather-agent",
    "endpoint": "https://agent.example.com/a2a",
    "a2aProtocolVersion": "1.0",
    "allowPrivateNetworkEgress": false,
    "authScheme": "BEARER",
    "bearerToken": "eyJhbGciOi...secret..."
  }'
```

## Notes and Warnings

- **Probe Failure vs Request Failure**: A validation error (name mismatch, egress gate) returns a
  `400` with `status: FAILURE`. A reachable-but-failing agent returns `200` with
  `deploymentResult.success = false` â€” always check `deploymentResult.success`, not just the HTTP status.
- **No Side Effects**: This endpoint never persists, deploys, or invalidates cache. It does not
  create or update the connection.
- **Private-Network Egress**: `allowPrivateNetworkEgress = true` during a probe is rejected for
  non-system-admin callers (SSRF hardening).
- **Secrets in the Body**: The `bearerToken`/`clientSecret` you send are used only for the probe and
  are never returned in the response.
