---
title: "Get A2A Connection"
description: "Gets a single AI Gateway A2A outbound connection by name. Secret fields (bearerToken, clientSecret) are returned masked as boolean presence flags, and the referencing API proxies are listed in usedByRefs."
---

## Endpoint

```
GET /apiops/projects/{projectName}/a2a-connections/{connectionName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| connectionName | string | Yes | A2A outbound connection name |

### Query Parameters

None

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | Single-element list containing the A2A connection object |
| resultCount | integer | Number of connections returned (always `1` on success) |

#### A2A Connection Object Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Internal connection ID (read-only) |
| projectId | string | Owning project ID (read-only) |
| projectName | string | Owning project name (read-only) |
| name | string | Connection name (unique within the project) |
| description | string | Connection description |
| enabled | boolean | Whether the connection is enabled |
| endpoint | string | A2A agent endpoint (AgentCard well-known URL / message endpoint) |
| a2aProtocolVersion | string | A2A protocol/adapter version (default `1.0`) |
| allowPrivateNetworkEgress | boolean | Whether private-network egress is permitted (admin-gated) |
| authScheme | string | Auth scheme: `BEARER`, `OAUTH2_CLIENT_CREDENTIALS`, `MTLS`, or `NONE` |
| bearerTokenSet | boolean | `true` if a bearer token is configured (the plaintext is never returned) |
| credentialRef | string | Credential reference for the OAuth2 client-credentials scheme |
| tokenEndpoint | string | OAuth2 token endpoint URL |
| clientId | string | OAuth2 client ID |
| scope | string | OAuth2 scope (space-delimited) |
| clientSecretSet | boolean | `true` if an OAuth2 client secret is configured (the plaintext is never returned) |
| sslConfigRef | string | SSL config reference for the mTLS scheme (placeholder) |
| agentCardCacheTtlSeconds | integer | AgentCard cache TTL in seconds (`0` = no cache; default `300`) |
| agentCardCacheEnabled | boolean | Whether AgentCard caching is enabled |
| discoveredSkillCount | integer | Number of skills discovered from the agent's AgentCard |
| discoveredSkills | array | Discovered skill objects (id, name, description, tags) |
| skillsParsedAt | integer | Epoch millis of the last successful AgentCard parse (`null` = never parsed) |
| healthStatus | string | Last health-probe status: `HEALTHY`, `DOWN`, or `UNKNOWN` (`null` = never probed) |
| lastHealthCheckAt | string | Timestamp of the last completed health probe |
| healthMessage | string | Short error description when `healthStatus` is `DOWN`; otherwise `null` |
| usedByRefs | array[string] | Names of API proxies / proxy groups that reference this connection (populated only in the single-get response) |

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A outbound connection (name: weather-agent) was not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/a2a-connections/weather-agent/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Secret Masking**: `bearerToken` and `clientSecret` are never returned. Only the boolean
  presence flags `bearerTokenSet` and `clientSecretSet` indicate whether they are configured (INV-06).
- **usedByRefs**: This field lists the proxies/proxy-groups referencing the connection and is
  populated only in this single-get response (the list endpoint omits it to avoid N-query cost).
- **Not Found**: A missing connection returns a `400 Bad Request` with a `FAILURE` status, not `404`.
- **Case Sensitivity**: Name lookups are performed case-insensitively.
