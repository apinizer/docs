---
title: "Get API Proxy Invoke Info"
description: "Returns the public invoke URL components (gateway base URL, root context, full path, accepted HTTP methods, supported schemes) of an API proxy for every environment, so a CI/CD or test caller can build the gateway call URL in one hop without hard-coding the URL prefix."
---

## Endpoint

```
GET /apiops/projects/{projectName}/apiProxies/{apiProxyName}/invoke-info/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| apiProxyName | string | Yes | API Proxy name (must exist) |

### Query Parameters

None.

### Request Body

None.

## Response

The response lists one entry per environment, so a caller can pick the environment it
targets and build the final call URL as `gatewayBaseUrl` + `fullPaths[i]`. The root
context is platform-wide and the same across environments; the gateway base URL and
supported schemes differ per environment.

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "resultList": [
    {
      "environmentName": "production",
      "gatewayBaseUrl": "https://gateway.example.com",
      "rootContext": "/apigateway",
      "fullPaths": [
        "/apigateway/ai/deepseek"
      ],
      "methods": [
        "GET",
        "POST"
      ],
      "schemes": [
        "https"
      ],
      "deployed": true
    },
    {
      "environmentName": "test",
      "gatewayBaseUrl": "https://gateway-test.example.com",
      "rootContext": "/apigateway",
      "fullPaths": [
        "/apigateway/ai/deepseek"
      ],
      "methods": [
        "GET",
        "POST"
      ],
      "schemes": [
        "https"
      ],
      "deployed": false
    }
  ],
  "resultCount": 2
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | One invoke-info entry per environment |
| resultCount | integer | Number of entries returned |

### Invoke Info Object Fields

| Field | Type | Description |
|-------|------|-------------|
| environmentName | string | Environment name |
| gatewayBaseUrl | string | Public-facing gateway URL clients call for this environment (not the internal management URL) |
| rootContext | string | Platform-wide gateway root context prefix (normalized with a leading slash, no trailing slash). Empty string when not configured |
| fullPaths | array | Full paths the proxy is exposed under, each being `rootContext` + the proxy's relative path. Combine with `gatewayBaseUrl` to build the call URL |
| methods | array | Upper-case HTTP methods the proxy accepts |
| schemes | array | Supported schemes for this environment (for example `http`, `https`) |
| deployed | boolean | Whether the proxy is currently deployed to this environment |

### Error Response (400 Bad Request)

```json
{
  "error": "bad_request",
  "error_description": "ApiProxy with name (petstore-api) is not found!"
}
```

#### Common Causes

- `apiProxyName` is empty
- API Proxy name does not exist in the project

### Error Response (401 Unauthorized)

```json
{
  "error": "unauthorized_client",
  "error_description": "Invalid token"
}
```

### Error Response (404 Not Found)

```json
{
  "error": "not_found",
  "error_description": "Project (MyProject) is not found or user does not have privilege to access it!"
}
```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/apiProxies/petstore-api/invoke-info/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Build the URL from `gatewayBaseUrl` + `fullPaths[i]`**: The root context is already included in each `fullPaths` entry, so do not add it a second time.
- **Root context varies by deployment**: The gateway root context prefix differs between installations; reading it from this endpoint avoids guessing the prefix and hitting a 404/405 from the gateway.
- **One entry per environment**: Every environment is listed; use `deployed` to tell whether the proxy is actually serving traffic in that environment.
- **Multiple paths**: A proxy may expose more than one relative path, so `fullPaths` and `methods` can contain multiple values.

## Permissions

User must have **`API_MANAGEMENT`** read access to the project.
