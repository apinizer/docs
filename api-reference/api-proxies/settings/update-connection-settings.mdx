---
title: "Update Connection Settings"
description: "Updates connection settings for an API proxy. Connection settings control timeout, redirect, connection pool, and header forwarding behavior."
---

## Endpoint

```
PATCH /apiops/projects/{projectName}/apiProxies/{apiProxyName}/settings/connection/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| apiProxyName | string | Yes | API Proxy name |

#### Request Body Fields

**Connection Settings**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| connectionSettingsEnabled | boolean | No | - | Enable custom connection settings (overrides system defaults) |
| connectTimeout | integer | No | 30 | Connection timeout in **seconds** (max time to establish a connection to the backend). Range: 1–3600 |
| readTimeout | integer | No | 30 | Read timeout in **seconds** (max time to wait for a response from the backend). Range: 1–3600 |
| connectionRequestTimeout | integer | No | 3 | Connection request timeout in **seconds** (max time to wait for a connection from the pool). Range: 1–300 |
| redirectsEnabled | boolean | No | - | Enable HTTP redirect following |
| maxRedirects | integer | No | 50 | Maximum number of redirects to follow |
| relativeRedirectsAllowed | boolean | No | - | Allow relative redirects |
| enableStreaming | boolean | No | - | Enable streaming for large responses |
| retryCount | integer | No | 0 | Number of retries for failed requests |
| failoverRetryCount | integer | No | 0 | Number of retries during failover |
| ignoreRoutingError | boolean | No | - | Ignore routing errors |
| connectionPoolManagementType | string | No | - | Connection pool management type. See [EnumConnectionPoolManagementType](#enumconnectionpoolmanagementtype) |
| customConnectionPoolSize | integer | No* | - | Custom connection pool size (required if connectionPoolManagementType=CUSTOM) |
| validateAfterInactivity | integer | No | 2000 | Stale connection check interval in **milliseconds**. Range: 0–60000 |
| idleConnectionTimeout | integer | No | 30 | Idle connection cleanup timeout in **seconds**. Range: 1–3600 |

**Header & Response Settings**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| hopByHopHeaders | array[string] | No | - | List of hop-by-hop headers to remove before forwarding to backend |
| sendUserAgentToBackend | boolean | No | - | Send User-Agent header to backend |
| userAgentValue | string | No | - | Custom User-Agent header value (used when sendUserAgentToBackend=true) |
| doNotRouteZeroValueContentLength | boolean | No | - | Do not route requests with Content-Length: 0 |
| enableDownload | boolean | No | - | Enable file download support for large response bodies |
| useBase64TextForCompressedResponse | boolean | No | false | When true, the gateway returns Base64-encoded text for a compressed response (legacy client compatibility); when false, it returns the raw byte array |

**Charset & Encoding Override Settings**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| requestCharsetOverride | string | No | null | Override request charset (e.g. `ISO-8859-9`, `windows-1252`). null = read from header |
| responseCharsetOverride | string | No | null | Override response charset (e.g. `ISO-8859-9`, `windows-1252`). null = read from header |
| requestDecompressOverride | string | No | null | Override request decompress encoding. See [EnumContentEncodingOverride](#enumcontentencodingoverride). null = auto, `NOT_APPLY` = skip decompress |
| requestCompressOverride | string | No | null | Override request compress encoding (to backend). null = auto, `NOT_APPLY` = skip compress |
| responseDecompressOverride | string | No | null | Override response decompress encoding (from backend). null = auto, `NOT_APPLY` = skip decompress |
| responseCompressOverride | string | No | null | Override response compress encoding (to client). null = auto, `NOT_APPLY` = skip compress |

**Retry Delay Settings**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| retryDelayType | string | No | - | Retry delay strategy. See [EnumRetryDelayType](#enumretrydelaytype) |
| retryFixedDelayMs | integer | No | - | Fixed delay between retries in milliseconds |
| retryInitialDelayMs | integer | No | - | Initial delay for exponential backoff in milliseconds |
| retryMaxDelayMs | integer | No | - | Maximum delay for exponential backoff in milliseconds |
| retryBackoffMultiplier | number | No | - | Backoff multiplier for exponential delay |

**Deployment (optional)**

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| deploy | boolean | No | - | If true, deploy the API Proxy after saving changes |
| deployTargetEnvironmentNameList | array[string] | No* | - | Environment names to deploy to (required when deploy=true) |

### EnumConnectionPoolManagementType

- `GENERAL` - General connection pool (shared pool)
- `CUSTOM` - Custom connection pool (requires customConnectionPoolSize)
- `NONE` - No connection pool

### EnumRetryDelayType

- `NO_DELAY` - No delay between retries
- `FIXED` - Fixed delay between retries (uses `retryFixedDelayMs`)
- `EXPONENTIAL` - Exponential backoff (uses `retryInitialDelayMs`, `retryMaxDelayMs`, `retryBackoffMultiplier`)

### EnumContentEncodingOverride

Accepted values for the `*DecompressOverride` / `*CompressOverride` fields:

- `GZIP` - gzip
- `DEFLATE` - deflate
- `BR` - brotli
- `ZSTD` - zstd
- `NOT_APPLY` - skip (de)compression
- `MIRROR_SERVER_RESPONSE` - mirror the server response encoding
- `AUTO_BY_CLIENT_ACCEPT_ENCODING` - choose automatically from the client `Accept-Encoding` header

A `null` value means automatic detection (Content-Encoding header + magic byte).

**Note:** All fields are optional. Only provided fields are updated.

## cURL Example

### Example 1: Configure Connection Timeouts

```bash
curl -X PATCH \
  "https://demo.apinizer.com/apiops/projects/MyProject/apiProxies/MyAPI/settings/connection/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "connectionSettingsEnabled": true,
    "connectTimeout": 30,
    "readTimeout": 60,
    "connectionRequestTimeout": 3,
    "retryCount": 3,
    "failoverRetryCount": 2
  }'
```

### Example 2: Configure Custom Connection Pool

```bash
curl -X PATCH \
  "https://demo.apinizer.com/apiops/projects/MyProject/apiProxies/MyAPI/settings/connection/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "connectionSettingsEnabled": true,
    "connectionPoolManagementType": "CUSTOM",
    "customConnectionPoolSize": 20,
    "connectTimeout": 60,
    "readTimeout": 120
  }'
```

### Example 3: Configure Connection Pool Stale/Idle Handling

```bash
curl -X PATCH \
  "https://demo.apinizer.com/apiops/projects/MyProject/apiProxies/MyAPI/settings/connection/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "connectionSettingsEnabled": true,
    "validateAfterInactivity": 2000,
    "idleConnectionTimeout": 30
  }'
```

### Example 4: Configure Header and Response Settings

```bash
curl -X PATCH \
  "https://demo.apinizer.com/apiops/projects/MyProject/apiProxies/MyAPI/settings/connection/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "hopByHopHeaders": ["X-Internal-Trace", "X-Debug-Info"],
    "sendUserAgentToBackend": true,
    "userAgentValue": "Apinizer-Gateway/1.0",
    "enableDownload": true
  }'
```

## Notes and Warnings

- **Timeouts**: `connectTimeout`, `readTimeout`, `connectionRequestTimeout` and `idleConnectionTimeout` are in **seconds**; only `validateAfterInactivity` is in **milliseconds**
- **Connection Pool**: `validateAfterInactivity` and `idleConnectionTimeout` help prevent stale connection issues; tune per backend load balancer keep-alive timeout
- **Connection Pool**: `CUSTOM` requires `customConnectionPoolSize` to be set
- **Retries**: `retryCount` retries on the same backend, `failoverRetryCount` tries failover backends
- **Retry Delay**: `retryDelayType=FIXED` uses `retryFixedDelayMs`; `EXPONENTIAL` uses `retryInitialDelayMs`, `retryMaxDelayMs` and `retryBackoffMultiplier`
- **Redirects**: When `redirectsEnabled=true`, HTTP redirects are followed (max `maxRedirects`)
- **Streaming**: When `enableStreaming=true`, response is streamed instead of buffered
- **Encoding Override**: Charset/encoding override fields accept `null` (auto) or an explicit value; `NOT_APPLY` skips the (de)compression step
- **Hop-by-Hop Headers**: Headers listed in `hopByHopHeaders` are removed before the request is forwarded to the backend
- **User-Agent**: When `sendUserAgentToBackend=true`, the specified `userAgentValue` is sent as the User-Agent header to the backend
- **Download Mode**: When `enableDownload=true`, the gateway handles large response bodies as file downloads
- **Deployment**: When `deploy=true`, `deployTargetEnvironmentNameList` is required to deploy the proxy after saving

User must have **`API_MANAGEMENT` + `MANAGE`** permission in the project.
