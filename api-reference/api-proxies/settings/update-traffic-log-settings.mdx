---
title: "Traffic Log Settings"
description: "Manage traffic log settings for an API proxy. Traffic log settings control what request/response data is logged and which connectors are used for log export. Settings are configured per environment."
---

## General Information

### Endpoints

#### List All Environments' Settings

```plaintext
GET /apiops/projects/{projectName}/apiProxies/{apiProxyName}/trafficLogSettings/
```

#### Get Settings for Specific Environment

```plaintext
GET /apiops/projects/{projectName}/apiProxies/{apiProxyName}/trafficLogSettings/{environmentName}/
```

#### Update Settings for Specific Environment

```plaintext
PUT /apiops/projects/{projectName}/apiProxies/{apiProxyName}/trafficLogSettings/{environmentName}/
```

#### Update Method-Level Connector Override

```plaintext
PUT /apiops/projects/{projectName}/apiProxies/{apiProxyName}/trafficLogSettings/{environmentName}/methods/{methodName}/connectors/{connectorType}/
```

#### Delete Method-Level Connector Override

```plaintext
DELETE /apiops/projects/{projectName}/apiProxies/{apiProxyName}/trafficLogSettings/{environmentName}/methods/{methodName}/connectors/{connectorType}/
```

| Field                        | Type    | Required | Default | Description                              |

| logHeaderRequestToTarget     | boolean | No       | true    | Log headers sent to target               |

| logHeaderResponseToClient    | boolean | No       | true    | Log headers sent to client response      |
| logBodyResponseToClient      | boolean | No       | true    | Log body sent to client response         |

| Field                        | Type    | Required | Default | Description                                  |
|------------------------------|---------|----------|---------|----------------------------------------------|
| enableDatabaseConnector      | boolean | No       | false   | Enable database connector for log export     |
| enableElasticsearchConnector | boolean | No       | false   | Enable Elasticsearch connector for log export |
| enableSyslogConnector        | boolean | No       | false   | Enable Syslog connector for log export       |
| enableWebhookConnector       | boolean | No       | false   | Enable Webhook connector for log export      |
| enableRabbitMqConnector      | boolean | No       | false   | Enable RabbitMQ connector for log export     |

**Note:** All fields are optional. Only provided fields are updated.

### Connectors

Connectors must be created and configured before enabling them. See [Connections API](/api-reference/connections/connections/connection-activemq.mdx) for connector management.

### Connector Log Settings Object (connectorSettingsMap values)

Each entry in `connectorSettingsMap` is keyed by connector type and contains detailed per-connector settings.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| enabled | boolean | No | false | Enable/disable this connector |
| logParamRequestFromClient | boolean | No | null | Log query parameters from client (overrides top-level) |
| logHeaderRequestFromClient | boolean | No | null | Log headers from client (overrides top-level) |
| logBodyRequestFromClient | boolean | No | null | Log body from client (overrides top-level) |
| logParamRequestToTarget | boolean | No | null | Log query parameters to target (overrides top-level) |
| logHeaderRequestToTarget | boolean | No | null | Log headers to target (overrides top-level) |
| logBodyRequestToTarget | boolean | No | null | Log body to target (overrides top-level) |
| logHeaderResponseFromTarget | boolean | No | null | Log headers from target (overrides top-level) |
| logBodyResponseFromTarget | boolean | No | null | Log body from target (overrides top-level) |
| logHeaderResponseToClient | boolean | No | null | Log headers to client (overrides top-level) |
| logBodyResponseToClient | boolean | No | null | Log body to client (overrides top-level) |
| enableMetadataIdentification | boolean | No | null | Log identification metadata |
| enableMetadataApiProxy | boolean | No | null | Log API proxy metadata |
| enableMetadataRequest | boolean | No | null | Log request metadata |
| enableMetrics | boolean | No | null | Log performance metrics |
| enableMetadataResponse | boolean | No | null | Log response metadata |
| enableSentAddress | boolean | No | null | Log routing address |
| enableBodyFromClientSize | boolean | No | null | Enable body size limit from client |
| sizeBodyFromClient | integer | No | null | Max body size from client (KB) |
| enableBodyToTargetSize | boolean | No | null | Enable body size limit to target |
| sizeBodyToTarget | integer | No | null | Max body size to target (KB) |
| enableBodyFromTargetSize | boolean | No | null | Enable body size limit from target |
| sizeBodyFromTarget | integer | No | null | Max body size from target (KB) |
| enableBodyToClientSize | boolean | No | null | Enable body size limit to client |
| sizeBodyToClient | integer | No | null | Max body size to client (KB) |
| enablePrivacy | boolean | No | false | Enable privacy masking |
| privacySalt | string | No | null | Salt for privacy hashing |

**Connector Types**: `ELASTICSEARCH`, `DATABASE`, `SYSLOG`, `WEBHOOK`, `RABBIT_MQ`, `ACTIVE_MQ`, `KAFKA`, `GRAYLOG`, `LOGBACK`

**Note:** Fields set to `null` inherit from top-level settings.

## Update Method-Level Connector Override

Adds or updates a method-level connector override for traffic log settings. This allows different log settings for specific API methods and connector types.

### Endpoint

```plaintext
PUT /apiops/projects/{projectName}/apiProxies/{apiProxyName}/trafficLogSettings/{environmentName}/methods/{methodName}/connectors/{connectorType}/
```

### Request

#### Headers

| Header        | Value            |
|---------------|------------------|
| Authorization | Bearer `{token}` |
| Content-Type  | application/json |

#### Path Parameters

| Parameter       | Type   | Required | Description                                        |
|-----------------|--------|----------|----------------------------------------------------|
| projectName     | string | Yes      | Project name                                       |
| apiProxyName    | string | Yes      | API Proxy name                                     |
| environmentName | string | Yes      | Environment name                                   |
| methodName      | string | Yes      | API method name                                    |
| connectorType   | string | Yes      | Connector type (e.g., `ELASTICSEARCH_INDEX`, `KAFKA_SEND`) |

#### Connector Types

- `ELASTICSEARCH_INDEX` - Elasticsearch connector
- `KAFKA_SEND` - Kafka connector
- `RABBITMQ_SEND` - RabbitMQ connector
- `ACTIVEMQ_SEND` - ActiveMQ connector
- `GRAYLOG_SEND` - Graylog connector
- `LOGBACK_APPEND` - Logback connector

#### Request Body

```json
{
  "enabled": true,
  "logParamRequestFromClient": true,
  "logHeaderRequestFromClient": false,
  "logBodyRequestFromClient": true,
  "logParamRequestToTarget": true,
  "logHeaderRequestToTarget": false,
  "logBodyRequestToTarget": true,
  "logHeaderResponseFromTarget": false,
  "logBodyResponseFromTarget": true,
  "logHeaderResponseToClient": false,
  "logBodyResponseToClient": true
}
```

### Response

#### Success Response (200 OK)

```json
{
  "success": true
}
```

### cURL Example

```bash
curl -X PUT \
  "https://demo.apinizer.com/apiops/projects/MyProject/apiProxies/MyAPI/trafficLogSettings/production/methods/getUsers/connectors/ELASTICSEARCH_INDEX/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "enabled": true,
    "logParamRequestFromClient": true,
    "logHeaderRequestFromClient": false,
    "logBodyRequestFromClient": true,
    "logBodyResponseFromTarget": true
  }'
```

## Notes and Warnings

- **Environment-Specific**: Traffic log settings are configured per environment
- **Environment Name**: Must match an existing environment name
- **Connectors**: Connectors must be created and configured before enabling
- **Logging Overhead**: Logging request/response bodies can impact performance
- **Data Privacy**: Be careful when logging sensitive data (passwords, tokens, etc.)
- **Trace Log**: `enabledTraceLog` enables trace logging for debugging
- **Application Log**: `enabledApplicationLog` enables application-level logging
- **Default Values**: Most log fields default to `true` if not specified
- **Automatic Deployment**: Changes are automatically deployed to the corresponding environment after save

## Permissions

User must have **`API_MANAGEMENT` + `VIEW`** permission for GET endpoints and **`API_MANAGEMENT` + `MANAGE`** permission for PUT/DELETE endpoints.
