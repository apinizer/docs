---
title: "Get Traffic Log Settings by Environment"
description: "Returns traffic log settings for a specific environment of the specified API Proxy. Includes connector settings and method-level overrides."
---

## Endpoint

```
GET /apiops/projects/{projectName}/api-proxies/{apiProxyName}/traffic-log-settings/{environmentName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer {token} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| apiProxyName | string | Yes | API Proxy name |
| environmentName | string | Yes | Environment name |

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/api-proxies/MyAPI/traffic-log-settings/production/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Permissions

User must have **`API_MANAGEMENT` + `VIEW`** permission in the project.

## Notes

- The `connectorSettingsMap` may contain both connector instance IDs (UUID format) and legacy connector type keys (e.g., `ELASTICSEARCH_INDEX`). UUID-based keys take precedence when both exist for the same connector.
- The `availableConnectors` field lists all connectors configured in the environment. Use the [Get Available Connectors](/api-reference/api-proxies/traffic-log-settings/get-connectors) endpoint for a dedicated connector discovery call.
