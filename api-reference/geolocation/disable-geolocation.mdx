---
title: "Disable Geolocation"
description: "Disables geolocation functionality in Apinizer. After disabling, geolocation features are no longer active on Gateway environments."
---
## Endpoint

```
PUT /apiops/settings/geolocation/disable
```

## Authentication

Requires a Personal API Access Token with system admin access (`sysAdmin` or `ADMIN` authority).

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

None.

### Request Body

None.

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true,
    "responseTime": 1500,
    "detailList": [
      {
        "envName": "production",
        "success": true,
        "detail": "Deployed successfully",
        "responseTime": 450
      }
    ]
  }
}
```

### Error Response (400 Bad Request)

Non-admin user with valid token:

```json
{
  "status": "FAILURE",
  "resultMessage": "Unauthorized! Only users with ADMIN role can access this resource!"
}
```

### Error Response (401 Unauthorized)

Invalid or missing API token.

```json
{
  "status": "FAILURE",
  "resultMessage": "User not found!"
}
```

## cURL Example

```bash
curl -X PUT \
  "https://demo.apinizer.com/apiops/settings/geolocation/disable" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Permissions

User must have **system admin (`sysAdmin`)** or **`ADMIN`** authority. See [Geolocation API](/api-reference/geolocation).

## Notes and Warnings

- **System admin only**:
  - Not a project-scoped endpoint

- **Does not delete MMDB**:
  - Disable turns off geolocation; the uploaded MMDB remains stored until replaced by a new upload

- **Automatic deployment**:
  - Setting is deployed to Gateway environments after disable
  - Deployment results are returned in `deploymentResult.detailList`
