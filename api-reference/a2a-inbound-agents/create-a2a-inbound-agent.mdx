---
title: "Create A2A Inbound Agent"
description: "Creates an AI Gateway A2A inbound agent in the specified project. If an agent with the same name already exists, it is updated instead (upsert). The agent exposes exactly one target AI proxy as an A2A agent."
---
## Endpoint

```
POST /apiops/projects/{projectName}/a2a-inbound-agents/{agentName}/
```

This endpoint is an **upsert**: it creates a new agent, or updates the existing one when an agent with the same name already exists in the project.

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| agentName | string | Yes | A2A inbound agent name. Must match the `name` in the request body |

### Query Parameters

None

### Request Body

#### Full JSON Body Example

```json
{
  "name": "customer-support-agent",
  "description": "Exposes the customer support AI proxy as an A2A agent",
  "active": true,
  "authType": "VIRTUAL_API_KEY",
  "virtualApiKey": "sk-a2a-8f3c1e9b2d7a4056",
  "targetProxyId": "6721a3f9e4b0c8a1d2e3f100",
  "allowPrivateNetworkEgress": false
}
```

#### Request Body Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| name | string | Yes | - | Agent name. Must equal the `agentName` path parameter (case-insensitive) |
| description | string | No | - | Optional description; surfaced in the AgentCard |
| active | boolean | No | false | Whether the agent is deployed and its `/a2a/\{id\}/...` route is served |
| authType | string | Yes | VIRTUAL_API_KEY | Inbound authentication type. Allowed: `VIRTUAL_API_KEY`, `NONE`. `OAUTH2` is rejected at persist time in the MVP. A `null` authType is rejected (fails closed) |
| virtualApiKey | string | Conditional | - | Virtual API key for `VIRTUAL_API_KEY` auth mode. Write-only (never returned). Required when `authType` is `VIRTUAL_API_KEY` |
| targetProxyId | string | Yes | - | ID of the single target AI proxy this agent exposes. Must reference an `AI`-type proxy in the same project |
| allowPrivateNetworkEgress | boolean | No | false | Permit push-notification webhooks to egress to cluster-internal (RFC1918 / IPv6-ULA) targets. Enabling requires a system administrator |

### Notes

- `name` must not be empty and must match the `agentName` path parameter.
- `virtualApiKey` is write-only: it is accepted here but never returned in read responses (INV-06).
- `allowPrivateNetworkEgress: true` on a new agent is rejected for non-admin callers (SSRF guard).

## Permissions

- User must have **`AI_DEVELOPMENT` + `MANAGE`** permission in the project.
- Enabling `allowPrivateNetworkEgress` additionally requires a **system administrator**.

## Notes and Warnings

- **Upsert semantics**:
  - If no agent with `agentName` exists, a new one is created.
  - If an agent with that name already exists, it is updated (its ID is reused).
- **Name match**: The `agentName` path parameter must match the `name` in the body, otherwise the request is rejected with `400`.
- **Write-only secret**:
  - `virtualApiKey` is accepted on create but never returned by any read endpoint.
  - It is encrypted at rest (`@SecretData`).
- **auth type constraints**:
  - `OAUTH2` is reserved for forward-compat but rejected at persist time in the current release.
  - A `null` or missing `authType` is rejected - it is never treated as `NONE`.
- **Target proxy**:
  - `targetProxyId` must reference an `AI`-type proxy in the same project (validated at save time).
  - One agent exposes exactly one AI proxy.
- **SSRF hardening**:
  - Setting `allowPrivateNetworkEgress: true` on a new agent requires a system administrator; a non-admin caller is rejected with `400`.
  - Even when enabled, loopback, link-local, cloud-metadata (169.254.169.254), any-local and multicast targets remain blocked.
