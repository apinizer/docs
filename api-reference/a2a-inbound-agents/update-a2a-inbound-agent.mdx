---
title: "Update A2A Inbound Agent"
description: "Updates an existing AI Gateway A2A inbound agent in the specified project. A blank virtualApiKey preserves the stored secret. The agent must already exist."
---
## Endpoint

```
PUT /apiops/projects/{projectName}/a2a-inbound-agents/{agentName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| agentName | string | Yes | A2A inbound agent name. If the body `name` is blank it defaults to this value; otherwise the two must match |

### Query Parameters

None

### Request Body

#### Full JSON Body Example

```json
{
  "name": "customer-support-agent",
  "description": "Updated description for the customer support A2A agent",
  "active": true,
  "authType": "VIRTUAL_API_KEY",
  "virtualApiKey": "",
  "targetProxyId": "6721a3f9e4b0c8a1d2e3f100",
  "allowPrivateNetworkEgress": false
}
```

#### Request Body Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| name | string | No | agentName | Agent name. If blank, falls back to the `agentName` path parameter; otherwise must match it (case-insensitive) |
| description | string | No | - | Optional description; surfaced in the AgentCard |
| active | boolean | No | false | Whether the agent is deployed and its `/a2a/\{id\}/...` route is served |
| authType | string | Yes | VIRTUAL_API_KEY | Inbound authentication type. Allowed: `VIRTUAL_API_KEY`, `NONE`. `OAUTH2` is rejected at persist time. A `null` authType is rejected (fails closed) |
| virtualApiKey | string | No | - | Virtual API key for `VIRTUAL_API_KEY` auth mode. Write-only (never returned). **Leave blank to preserve the stored value** |
| targetProxyId | string | Yes | - | ID of the single target AI proxy this agent exposes. Must reference an `AI`-type proxy in the same project |
| allowPrivateNetworkEgress | boolean | No | (preserved) | Permit push-notification webhooks to egress to cluster-internal targets. A non-admin caller cannot change this flag - the stored value is preserved regardless of the body |

### Notes

- The agent must already exist; otherwise `400 Bad Request` is returned.
- A **blank** `virtualApiKey` preserves the currently stored (encrypted) secret (INV-06).
- For a non-admin caller, `allowPrivateNetworkEgress` in the body is ignored and the stored value is kept unchanged.

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A inbound agent (name: customer-support-agent) was not found!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A inbound agent name in path (customer-support-agent) does not match name in body (support-agent)!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X PUT \
  "https://demo.apinizer.com/apiops/projects/MyProject/a2a-inbound-agents/customer-support-agent/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "customer-support-agent",
    "description": "Updated description for the customer support A2A agent",
    "active": true,
    "authType": "VIRTUAL_API_KEY",
    "virtualApiKey": "",
    "targetProxyId": "6721a3f9e4b0c8a1d2e3f100",
    "allowPrivateNetworkEgress": false
  }'
```

## Permissions

- User must have **`AI_DEVELOPMENT` + `MANAGE`** permission in the project.
- Changing `allowPrivateNetworkEgress` additionally requires a **system administrator**; for a non-admin caller the stored value is silently preserved.

## Notes and Warnings

- **Must exist**: The agent must already exist. To create a new agent use the [Create (upsert)](/api-reference/a2a-inbound-agents/create-a2a-inbound-agent) endpoint.
- **Blank secret preserves**:
  - Send `virtualApiKey` as an empty string (or omit it) to keep the stored key.
  - Send a new non-blank value to rotate the key.
- **Name match**: If the body `name` is non-blank it must equal the `agentName` path parameter; a blank body `name` defaults to the path value.
- **auth type constraints**:
  - `OAUTH2` is rejected at persist time in the current release.
  - A `null` or missing `authType` is rejected - it is never treated as `NONE`.
- **SSRF hardening**:
  - A non-admin caller cannot turn `allowPrivateNetworkEgress` on or off - the persisted value is preserved regardless of the request body.
  - Even when enabled, loopback, link-local, cloud-metadata (169.254.169.254), any-local and multicast targets remain blocked.
