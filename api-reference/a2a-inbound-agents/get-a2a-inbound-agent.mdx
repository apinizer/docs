---
title: "Get A2A Inbound Agent"
description: "Retrieves a single AI Gateway A2A inbound agent by name from the specified project. The write-only virtualApiKey is never returned."
---
## Endpoint

```
GET /apiops/projects/{projectName}/a2a-inbound-agents/{agentName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| agentName | string | Yes | A2A inbound agent name |

### Query Parameters

None

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "resultList": [
    {
      "id": "6721a3f9e4b0c8a1d2e3f4a5",
      "name": "customer-support-agent",
      "projectId": "6721a3f9e4b0c8a1d2e3f001",
      "description": "Exposes the customer support AI proxy as an A2A agent",
      "active": true,
      "authType": "VIRTUAL_API_KEY",
      "targetProxyId": "6721a3f9e4b0c8a1d2e3f100",
      "allowPrivateNetworkEgress": false,
      "createdBy": "admin",
      "createdDate": "2026-07-07T09:15:00.000+00:00",
      "lastModifiedBy": "admin",
      "lastModifiedDate": "2026-07-10T14:22:30.000+00:00"
    }
  ],
  "resultCount": 1
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | Single-element list containing the requested agent object |
| resultCount | integer | Number of agents returned (`1` on success) |

#### Agent Object Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Internal agent ID |
| name | string | Agent name (unique within the project) |
| projectId | string | Owning project ID |
| description | string | Optional description (surfaced in the AgentCard) |
| active | boolean | Whether the agent is deployed and its `/a2a/\{id\}/...` route is served |
| authType | string | Inbound authentication type: `VIRTUAL_API_KEY`, `OAUTH2`, or `NONE` |
| targetProxyId | string | ID of the single target AI proxy this agent exposes |
| allowPrivateNetworkEgress | boolean | Whether push-notification webhooks may egress to cluster-internal targets (admin-gated) |
| createdBy | string | User who created the agent |
| createdDate | string | Creation timestamp (ISO 8601) |
| lastModifiedBy | string | User who last modified the agent |
| lastModifiedDate | string | Last modification timestamp (ISO 8601) |

The write-only `virtualApiKey` is **never** included in the response (INV-06).

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A inbound agent (name: customer-support-agent) was not found!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A inbound agent name can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/a2a-inbound-agents/customer-support-agent/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Permissions**: Requires `AI_DEVELOPMENT` + `VIEW` permission in the project.
- **Name lookup**: The agent is matched by name (case-insensitive) scoped to the project. A non-existent name returns `400 Bad Request`.
- **Secret masking**: `virtualApiKey` is a write-only field and is never returned in the response.
- **Single element**: On success the agent is returned as the sole element of `resultList` with `resultCount` of `1`.
