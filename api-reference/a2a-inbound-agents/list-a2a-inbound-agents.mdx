---
title: "List A2A Inbound Agents"
description: "Lists all AI Gateway A2A inbound agents in the specified project. The write-only virtualApiKey is never returned."
---
## Endpoint

```
GET /apiops/projects/{projectName}/a2a-inbound-agents/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |

### Query Parameters

None

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "resultList": [
    {
      "id": "6721a3f9e4b0c8a1d2e3f4a5",
      "name": "customer-support-agent",
      "projectId": "6721a3f9e4b0c8a1d2e3f001",
      "description": "Exposes the customer support AI proxy as an A2A agent",
      "active": true,
      "authType": "VIRTUAL_API_KEY",
      "targetProxyId": "6721a3f9e4b0c8a1d2e3f100",
      "allowPrivateNetworkEgress": false,
      "createdBy": "admin",
      "createdDate": "2026-07-07T09:15:00.000+00:00",
      "lastModifiedBy": "admin",
      "lastModifiedDate": "2026-07-10T14:22:30.000+00:00"
    }
  ],
  "resultCount": 1
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | List of A2A inbound agent objects |
| resultCount | integer | Total number of agents returned |

#### Agent Object Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Internal agent ID |
| name | string | Agent name (unique within the project) |
| projectId | string | Owning project ID |
| description | string | Optional description (surfaced in the AgentCard) |
| active | boolean | Whether the agent is deployed and its `/a2a/\{id\}/...` route is served |
| authType | string | Inbound authentication type: `VIRTUAL_API_KEY`, `OAUTH2`, or `NONE` |
| targetProxyId | string | ID of the single target AI proxy this agent exposes |
| allowPrivateNetworkEgress | boolean | Whether push-notification webhooks may egress to cluster-internal targets (admin-gated) |
| createdBy | string | User who created the agent |
| createdDate | string | Creation timestamp (ISO 8601) |
| lastModifiedBy | string | User who last modified the agent |
| lastModifiedDate | string | Last modification timestamp (ISO 8601) |

The write-only `virtualApiKey` is **never** included in read responses (INV-06).

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Project with name (MyProject) is not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/a2a-inbound-agents/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Permissions**: Requires `AI_DEVELOPMENT` + `VIEW` permission in the project.
- **Secret masking**: `virtualApiKey` is a write-only field and is never returned - it is omitted from every read response, not returned as ciphertext.
- **Empty list**: If the project has no A2A inbound agents, an empty `resultList` with `resultCount` of `0` is returned.
- **Name-based**: Agents are addressed by name in single-agent endpoints; use `name` values from this list.
