---
title: "Delete A2A Inbound Agent"
description: "Deletes an existing AI Gateway A2A inbound agent from the specified project and undeploys it from the workers."
---
## Endpoint

```
DELETE /apiops/projects/{projectName}/a2a-inbound-agents/{agentName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| agentName | string | Yes | A2A inbound agent name |

### Query Parameters

None

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A inbound agent (name: customer-support-agent) was not found!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "A2A inbound agent name can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X DELETE \
  "https://demo.apinizer.com/apiops/projects/MyProject/a2a-inbound-agents/customer-support-agent/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Permissions

- User must have **`AI_DEVELOPMENT` + `MANAGE`** permission in the project.

## Notes and Warnings

- **Must exist**: The agent must exist in the project; deleting a non-existent name returns `400 Bad Request`.
- **Undeploy**: Deletion removes the agent from the database and undeploys its `/a2a/\{id\}/...` route from the workers.
- **Project-scoped**: The delete is guarded by the project ID (IDOR protection) - an agent can only be deleted from the project that owns it.
- **Irreversible**: Deletion is permanent; recreate the agent with the [Create](/api-reference/a2a-inbound-agents/create-a2a-inbound-agent) endpoint if needed.
