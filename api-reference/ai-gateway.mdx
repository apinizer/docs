## Overview

Every AI Gateway configuration asset is manageable through APIops. Endpoints follow the standard
APIops conventions:

- **Name-based** references (project, asset). Runtime IDs are never required in the path.
- **Secret masking on read** вЂ” `apiKey`/`apiSecret`/`bearerToken`/`virtualApiKey`/credential references
  are returned as boolean presence flags only; plaintext/ciphertext secrets are never disclosed.
- **Save = encrypt + deploy** вЂ” write operations encrypt `@SecretData` fields and push to the workers
  through the owning service; delete undeploys.
- **Upsert** вЂ” `POST /{name}/` creates the asset, or updates it in place if the name already exists.

## Project-scoped resources

Base path: `/apiops/projects/{projectName}/...` вЂ” authenticated with `AI_DEVELOPMENT`
(VIEW for reads, MANAGE for writes), except AI Budgets which use `AI_BUDGETS`.

### LLM Providers вЂ” `/llm-providers`
`GET /` В· `GET /{providerName}/` В· `POST /{providerName}/` В· `PUT /{providerName}/` В· `DELETE /{providerName}/`

### MCP Outbound Connections вЂ” `/mcp-connections`
`GET /` В· `GET /{connectionName}/` В· `POST /{connectionName}/` В· `PUT /{connectionName}/` В· `DELETE /{connectionName}/`

### Model Catalog вЂ” `/llm-models`
`GET /` В· `GET /{modelName}/` В· `POST /{modelName}/` В· `PUT /{modelName}/` В· `DELETE /{modelName}/`

### Virtual Keys вЂ” `/virtual-keys`
`GET /` В· `GET /{virtualKeyName}/` В· `POST /{virtualKeyName}/` В· `PUT /{virtualKeyName}/` В· `DELETE /{virtualKeyName}/`
- `POST /{virtualKeyName}/rotate/` вЂ” rotates the backing credential. Uses a grace window (Pattern-B):
  a new active row is created as `<name>-rotated` while the previous row is retired but kept as a
  rollback safety-net until `graceMinutes` elapse. Body: `{ "newCredentialRef": "...", "graceMinutes": 60 }`.

### VectorDB Connections вЂ” `/vector-dbs`
`GET /` В· `GET /{vectorDbName}/` В· `POST /{vectorDbName}/` В· `PUT /{vectorDbName}/` В· `DELETE /{vectorDbName}/`
- `POST /{vectorDbName}/test-connection/` вЂ” validates connectivity against the target vector store
  (in-process manager test), returns a deployment result.

### Knowledge Bases вЂ” `/knowledge-bases`
`GET /` В· `GET /{name}/` В· `POST /{name}/` В· `PUT /{name}/` В· `DELETE /{name}/`
- `POST /{name}/reindex/` вЂ” rebuilds the vector index for the knowledge base.

#### Knowledge Base Documents вЂ” `/knowledge-bases/{kbName}/documents`
- `POST /` вЂ” upload a document (`multipart/form-data`, `file` part). Chunked + embedded on ingest.
- `GET /` вЂ” list documents in the knowledge base.
- `GET /{documentId}/` вЂ” get a single document. *(Documents are referenced by id вЂ” filenames are not unique.)*
- `DELETE /{documentId}/` вЂ” remove a document and its vectors.

### MCP Inbound Servers вЂ” `/mcp-inbound-servers`
`GET /` В· `GET /{serverName}/` В· `POST /{serverName}/` В· `PUT /{serverName}/` В· `DELETE /{serverName}/`

### A2A Outbound Connections вЂ” `/a2a-connections`
`GET /` В· `GET /{connectionName}/` В· `POST /{connectionName}/` В· `PUT /{connectionName}/` В· `DELETE /{connectionName}/`
- `POST /{connectionName}/test-connection/` вЂ” probes the remote agent.
- `POST /{connectionName}/parse-skills/` вЂ” fetches and parses the remote AgentCard skills.

### A2A Inbound Agents вЂ” `/a2a-inbound-agents`
`GET /` В· `GET /{agentName}/` В· `POST /{agentName}/` В· `PUT /{agentName}/` В· `DELETE /{agentName}/`

### AI Global Policies вЂ” `/ai-global-policies`
- `GET /?type={policyType}` вЂ” list `global=true` AI policies (optionally filtered by policy type).
- `GET /{policyName}/` вЂ” get a single global AI policy.
- `DELETE /{policyName}/` вЂ” delete a global AI policy (fetch-then-verify project ownership + the
  `global=true` invariant, then fan out the removal to every referencing proxy/group).
- `GET /{policyName}/usage/` вЂ” list proxies/proxy-groups referencing the policy.

> **Creating / updating** a global AI policy is done through the generic
> [export/import pipeline](/api-reference/api-proxies), not through this resource вЂ” an AI global
> policy is a plain `Policy` document, so it flows through the same path every other global policy
> type uses. Reads are serialized through the typed policy DTOs, so AI connection references are
> carried name-based.

### AI Budgets / Quota вЂ” `/ai-budgets` *(permission: `AI_BUDGETS`)*
- `GET /scope-rules/` вЂ” flat list of budget scope rules.
- `GET /hierarchy/` вЂ” budget hierarchy tree rooted at the project.
- `GET /hierarchy/{nodeType}/{nodeId}/effective-limit/` вЂ” effective limit preview for a node
  (`nodeType`: `CREDENTIAL` | `CREDENTIAL_ORGANIZATION` | `API_PROXY` | `PROJECT`).
- `PUT /` вЂ” update an owner token budget. The body `targetProjectId` is forced to the path project
  (cross-tenant guard).

## Admin (global) resources

Base path: `/apiops/settings/...` вЂ” these are platform-level catalogs/settings and require an
**ADMIN** API token (sysAdmin), matching the classic UI. A project-scoped `AI_DEVELOPMENT` token is
NOT sufficient.

### AI Gateway Settings вЂ” `/apiops/settings/ai-gateway`
- `GET /` вЂ” read the singleton AI Gateway settings.
- `PUT /` вЂ” update the settings (timeouts, retry, response-body caps, `maxToolTurns`, default trace mode).

### AI Privacy (PII) Presets вЂ” `/apiops/settings/ai-privacy-presets`
`GET /` В· `GET /{presetName}/` В· `POST /{presetName}/` В· `PUT /{presetName}/` В· `DELETE /{presetName}/`
- Optional `?projectId=` selects the scope (default admin/global). Built-in presets are read-only.

### AI Prompt-Guard Presets вЂ” `/apiops/settings/ai-prompt-guard-presets`
`GET /` В· `GET /{presetName}/` В· `POST /{presetName}/` В· `PUT /{presetName}/` В· `DELETE /{presetName}/`
- Same scope/immutability rules as privacy presets.

## Authentication

All endpoints require authentication using a Personal API Access Token
(`Authorization: Bearer {token}` or `apinizer-token: {token}`). JHipster JWTs do not work on APIops.

## Permissions

| Resource group | Permission |
| --- | --- |
| LLM/MCP/Model/VirtualKey/VectorDB/KB/A2A/Global-Policy | `AI_DEVELOPMENT` (VIEW / MANAGE) |
| AI Budgets / Quota | `AI_BUDGETS` (VIEW / MANAGE) |
| Presets & Gateway Settings | **Admin only** (sysAdmin) |

APIops never widens the privilege surface relative to the UI: presets and gateway settings require
system-admin exactly as they do in the Manager UI; A2A private-network egress remains admin-gated.
