---
title: "Export Key"
description: "Exports a key definition as a ZIP file containing the key definition and sensitive cryptographic material, decrypted for portability across instances."
---

## Endpoint

```
GET /apiops/projects/{projectName}/keys/{keyName}/export/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| keyName | string | Yes | Key name |

### Query Parameters

None.

## Response

### Success Response (200 OK)

The response is a ZIP file containing the key definition.

### Headers

- `Content-Type: application/octet-stream`
- `Content-Disposition: attachment; filename="keyExportFile.zip"`
- `Content-Length: {file-size}`

### Response Body

- Binary ZIP file containing:
  - Key definition in JSON format
  - Sensitive fields (key content and key password) are decrypted for portability

### Error Response (400 Bad Request)

```json
{
  "error": "bad_request",
  "error_description": "projectName value can not be empty!"
}
```

or

```json
{
  "error": "bad_request",
  "error_description": "keyName value can not be empty!"
}
```

or

```json
{
  "error": "bad_request",
  "error_description": "Key (name: my-key) is not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "error": "unauthorized_client",
  "error_description": "Invalid token"
}
```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/keys/my-key/export/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  --output keyExportFile.zip
```

## Permissions

User must have **`SECRETS` + `EXPORT_IMPORT`** permission in the project.

## Notes and Warnings

- **Sensitive Material**:
  - Export contains sensitive cryptographic material
  - Store the exported ZIP file securely
  - Do not share publicly or via unsecured channels
  - Treat with the same care as the original key file

- **Portability**:
  - The key definition and sensitive fields (key content, key password) are decrypted in the export
  - This allows the exported ZIP to be portable across different Apinizer instances
  - The matching import endpoint will re-encrypt these fields on the target instance

- **Key Types**:
  - Supports export of private keys, public keys, and secret keys
  - All key types are handled consistently during export and import

- **Import Compatibility**:
  - The exported ZIP file can be imported into another project or instance using the Import Key endpoint
  - Import will re-encrypt sensitive fields according to the target instance's encryption configuration
