---
title: "Export Keystore"
description: "Exports a keystore definition as a ZIP file containing the keystore definition and sensitive cryptographic material, decrypted for portability across instances."
---

## Endpoint

```
GET /apiops/projects/{projectName}/keystores/{keystoreName}/export/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| keystoreName | string | Yes | Keystore name |

### Query Parameters

None.

## Response

### Success Response (200 OK)

The response is a ZIP file containing the keystore definition.

### Headers

- `Content-Type: application/octet-stream`
- `Content-Disposition: attachment; filename="keystoreExportFile.zip"`
- `Content-Length: {file-size}`

### Response Body

- Binary ZIP file containing:
  - Keystore definition in JSON format
  - Sensitive fields (keystore file content and keystore password) are decrypted for portability

### Error Response (400 Bad Request)

```json
{
  "error": "bad_request",
  "error_description": "projectName value can not be empty!"
}
```

or

```json
{
  "error": "bad_request",
  "error_description": "keystoreName value can not be empty!"
}
```

or

```json
{
  "error": "bad_request",
  "error_description": "Keystore (name: my-keystore) is not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "error": "unauthorized_client",
  "error_description": "Invalid token"
}
```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/keystores/my-keystore/export/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  --output keystoreExportFile.zip
```

## Permissions

User must have **`SECRETS` + `EXPORT_IMPORT`** permission in the project.

## Notes and Warnings

- **Sensitive Material**:
  - Export contains sensitive cryptographic material
  - Store the exported ZIP file securely
  - Do not share publicly or via unsecured channels
  - Treat with the same care as the original keystore file

- **Portability**:
  - The keystore definition and sensitive fields (keystore file content, keystore password) are decrypted in the export
  - This allows the exported ZIP to be portable across different Apinizer instances
  - The matching import endpoint will re-encrypt these fields on the target instance

- **Import Compatibility**:
  - The exported ZIP file can be imported into another project or instance using the Import Keystore endpoint
  - Import will re-encrypt sensitive fields according to the target instance's encryption configuration
