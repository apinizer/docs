---
title: "Import Keystore"
description: "Imports a keystore definition from a ZIP file. Sensitive fields are re-encrypted on the target instance. An optional override flag allows replacing existing keystores with the same name."
---

## Endpoint

```
POST /apiops/projects/{projectName}/keystores/import/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | multipart/form-data | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name where the keystore will be imported |

### Query Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| override | boolean | No | false | When true, an existing keystore with the same name will be overwritten; otherwise import fails if keystore exists |

### Form Data

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| keystoreExportFile | file | Yes | ZIP file produced by the Export Keystore endpoint. Must have `.zip` extension |

### Notes

- File must be a valid ZIP archive produced by the Export Keystore endpoint
- File must end with `.zip` extension (case-insensitive)
- Keystore names are resolved by name (APIops convention)
- References within the export are resolved by name on import

## cURL Examples

### Example 1: Basic Import

```bash
curl -X POST \
  "https://demo.apinizer.com/apiops/projects/MyProject/keystores/import/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "keystoreExportFile=@keystoreExportFile.zip"
```

### Example 2: Import with Override

```bash
curl -X POST \
  "https://demo.apinizer.com/apiops/projects/MyProject/keystores/import/?override=true" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "keystoreExportFile=@keystoreExportFile.zip"
```

or

```bash
curl -X POST \
  "https://demo.apinizer.com/apiops/projects/MyProject/keystores/import/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "keystoreExportFile=@keystoreExportFile.zip" \
  -F "override=true"
```

## Permissions

User must have **`SECRETS` + `EXPORT_IMPORT`** permission in the project.

## Notes and Warnings

- **Override Behavior**:
  - When `override=false` (default), import fails if a keystore with the same name already exists
  - When `override=true`, an existing keystore with the same name will be replaced
  - Use override with caution as it will overwrite all existing keystore configuration

- **Encryption**:
  - Sensitive fields in the export are decrypted for portability
  - During import, these fields are re-encrypted according to the target instance's encryption configuration
  - The target instance must have proper encryption keys configured

- **Name Resolution**:
  - Keystore names are resolved by name, not by ID
  - If a keystore with the same name exists and override is not enabled, import will fail

- **File Format**:
  - File must be a valid ZIP archive produced by the Export Keystore endpoint
  - File must end with `.zip` extension (case-insensitive)
  - ZIP file must contain valid keystore definition JSON

- **Project Validation**:
  - Project must exist
  - User must have access to the project
  - User must have `SECRETS` + `EXPORT_IMPORT` permission
