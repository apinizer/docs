---
title: "Import Key"
description: "Imports a key definition from a ZIP file. Sensitive fields are re-encrypted on the target instance. An optional override flag allows replacing existing keys with the same name."
---

## Endpoint

```
POST /apiops/projects/{projectName}/keys/import/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | multipart/form-data | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name where the key will be imported |

### Query Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| override | boolean | No | false | When true, an existing key with the same name will be overwritten; otherwise import fails if key exists |

### Form Data

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| keyExportFile | file | Yes | ZIP file produced by the Export Key endpoint. Must have `.zip` extension |

### Notes

- File must be a valid ZIP archive produced by the Export Key endpoint
- File must end with `.zip` extension (case-insensitive)
- Key names are resolved by name (APIops convention)
- References within the export are resolved by name on import

## cURL Examples

### Example 1: Basic Import

```bash
curl -X POST \
  "https://demo.apinizer.com/apiops/projects/MyProject/keys/import/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "keyExportFile=@keyExportFile.zip"
```

### Example 2: Import with Override

```bash
curl -X POST \
  "https://demo.apinizer.com/apiops/projects/MyProject/keys/import/?override=true" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "keyExportFile=@keyExportFile.zip"
```

or

```bash
curl -X POST \
  "https://demo.apinizer.com/apiops/projects/MyProject/keys/import/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "keyExportFile=@keyExportFile.zip" \
  -F "override=true"
```

## Permissions

User must have **`SECRETS` + `EXPORT_IMPORT`** permission in the project.

## Notes and Warnings

- **Override Behavior**:
  - When `override=false` (default), import fails if a key with the same name already exists
  - When `override=true`, an existing key with the same name will be replaced
  - Use override with caution as it will overwrite all existing key configuration

- **Encryption**:
  - Sensitive fields in the export are decrypted for portability
  - During import, these fields are re-encrypted according to the target instance's encryption configuration
  - The target instance must have proper encryption keys configured

- **Name Resolution**:
  - Key names are resolved by name, not by ID
  - If a key with the same name exists and override is not enabled, import will fail

- **Key Types**:
  - Supports import of private keys, public keys, and secret keys
  - All key types are handled consistently during import

- **File Format**:
  - File must be a valid ZIP archive produced by the Export Key endpoint
  - File must end with `.zip` extension (case-insensitive)
  - ZIP file must contain valid key definition JSON

- **Project Validation**:
  - Project must exist
  - User must have access to the project
  - User must have `SECRETS` + `EXPORT_IMPORT` permission
