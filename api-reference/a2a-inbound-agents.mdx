## Endpoints

- [List A2A Inbound Agents](/api-reference/a2a-inbound-agents/list-a2a-inbound-agents) - List all A2A inbound agents in a project
- [Get A2A Inbound Agent](/api-reference/a2a-inbound-agents/get-a2a-inbound-agent) - Retrieve a single A2A inbound agent by name
- [Create A2A Inbound Agent](/api-reference/a2a-inbound-agents/create-a2a-inbound-agent) - Create (or update, if the name exists) an A2A inbound agent
- [Update A2A Inbound Agent](/api-reference/a2a-inbound-agents/update-a2a-inbound-agent) - Update an existing A2A inbound agent
- [Delete A2A Inbound Agent](/api-reference/a2a-inbound-agents/delete-a2a-inbound-agent) - Delete an A2A inbound agent and undeploy it from workers

## Authentication

All endpoints require authentication using a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Permissions

- **Read** (`GET`) - Requires **`AI_DEVELOPMENT` + `VIEW`** permission in the project
- **Write** (`POST` / `PUT` / `DELETE`) - Requires **`AI_DEVELOPMENT` + `MANAGE`** permission in the project
- **Private-network egress** - Enabling `allowPrivateNetworkEgress` additionally requires a **system administrator** (SSRF guard); a non-admin caller cannot turn it on

## Notes and Warnings

- **One agent = one AI proxy**: An A2A inbound agent references exactly one target AI proxy (`targetProxyId`) in the same project. The proxy is exposed as an A2A agent over JSON-RPC 2.0 on the route `/a2a/{agentId}/...`.
- **Name-based**: The resource is addressed by agent name in the URL path. `POST /{agentName}/` is an upsert - it creates a new agent, or updates the existing one when the name already exists.
- **Write-only secret**: `virtualApiKey` is accepted on create/update but is **never returned** in read responses (INV-06). A blank `virtualApiKey` on update preserves the stored value.
- **SSRF hardening**: The A2A push-notification webhook URL is supplied by an untrusted external A2A client, so private-network egress stays blocked unless a system admin explicitly enables `allowPrivateNetworkEgress` on the agent.
