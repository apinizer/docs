---
title: "Create Virtual Key"
description: "Creates an AI Gateway virtual key (provider credential mapping) in the specified project. This is an upsert â€” if a virtual key with the same name already exists, it is updated in place."
---

## Endpoint

```
POST /apiops/projects/{projectName}/virtual-keys/{virtualKeyName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| virtualKeyName | string | Yes | Virtual key name. Must match the `name` in the request body |

### Query Parameters

None

### Request Body

#### Full JSON Body Example

```json
{
  "name": "openai-team-key",
  "llmProviderRef": "66a0f1a2c3d4e5f6a7b8c9d0",
  "scope": "PROJECT",
  "scopeRef": "660000000000000000000001",
  "credentialRef": "66c0f1a2c3d4e5f6a7b8c9d0",
  "priority": 3,
  "validFrom": "2026-07-01T00:00:00Z",
  "validUntil": null,
  "isActive": true,
  "metadata": {
    "region": "eu",
    "tier": "premium"
  }
}
```

#### Request Body Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| name | string | Yes | - | Virtual key name. Must match `virtualKeyName` in the path (case-insensitive) |
| llmProviderRef | string | Yes | - | Reference (by id) to the LLM provider this key credentials. Must belong to the same project |
| scope | string | Yes | - | Consumer resolution scope: `CREDENTIAL`, `CREDENTIAL_ORGANIZATION`, `PROJECT`, or `GLOBAL` |
| scopeRef | string | Conditional | - | Reference resolved per `scope`: credential id (`CREDENTIAL`), organization id (`CREDENTIAL_ORGANIZATION`), project id (`PROJECT`), or `null` (`GLOBAL`) |
| credentialRef | string | Yes | - | Id of the vault `Credential` holding the real provider key. Must belong to the same project. Masked on read as `credentialRefSet` |
| priority | integer | No | scope default | Resolution priority; lower is tried first. Auto-filled from `scope` when omitted (`CREDENTIAL`=1, `CREDENTIAL_ORGANIZATION`=2, `PROJECT`=3, `GLOBAL`=4) |
| validFrom | string | No | null | ISO-8601 validity start (`null` = active immediately) |
| validUntil | string | No | null | ISO-8601 validity end (`null` = no expiry) |
| isActive | boolean | No | true | Whether the mapping is active |
| inactiveReason | string | No | null | Reason when inactive (audit label) |
| metadata | object | No | null | Free-form string key/value metadata |

### Notes

- `virtualKeyName` in the path must equal `name` in the body, otherwise a `400 Bad Request` is returned.
- **Upsert**: if a virtual key with this name already exists in the project, it is updated in place (its internal id is reused).
- `llmProviderRef` and `credentialRef` must reference assets in the same project (cross-project binding is rejected).
- Do not send a `_class` field. The internal `id` and `projectId` are derived from the path/lookup and cannot be repointed to another project on update.

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Virtual key name in path (openai-team-key) does not match name in body (other-name)!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "Virtual key body can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## Notes and Warnings

- **Permission**: Requires `AI_DEVELOPMENT` + `MANAGE` in the project.
- **Upsert semantics**: `POST /{virtualKeyName}/` creates the virtual key, or updates it if the name already exists â€” there is no separate "already exists" error.
- **Credential reference required**: `credentialRef` must be supplied â€” reads mask it as `credentialRefSet`, so a masked GET response cannot be replayed verbatim into a create; re-supply the vault credential id.
- **Cross-project guard**: The service rejects a `llmProviderRef` or `credentialRef` that resolves to a different project (IDOR/BOLA guard).
- **Save = deploy**: A successful save also pushes the mapping to the workers; the `deploymentResult` reflects the deploy outcome.
