---
title: "List Virtual Keys"
description: "Lists all AI Gateway virtual keys (provider credential mappings) in the specified project. The backing credential reference is masked as a boolean presence flag."
---

## Endpoint

```
GET /apiops/projects/{projectName}/virtual-keys/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |

### Query Parameters

None

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "resultList": [
    {
      "id": "66b0f1a2c3d4e5f6a7b8c9d0",
      "projectId": "660000000000000000000001",
      "name": "openai-team-key",
      "llmProviderRef": "66a0f1a2c3d4e5f6a7b8c9d0",
      "scope": "PROJECT",
      "scopeRef": "660000000000000000000001",
      "credentialRefSet": true,
      "priority": 3,
      "validFrom": "2026-07-01T00:00:00Z",
      "validUntil": null,
      "isActive": true,
      "inactiveReason": null,
      "metadata": {
        "region": "eu",
        "tier": "premium"
      },
      "lastResolvedAt": "2026-07-12T09:31:44Z",
      "resolveCount": 128
    }
  ],
  "resultCount": 1
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | List of virtual key objects |
| resultCount | integer | Total number of virtual keys returned |

#### Virtual Key Object Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Internal identifier (read-only) |
| projectId | string | Owning project id |
| name | string | Virtual key name (unique within the project) |
| llmProviderRef | string | Reference (by id) to the LLM provider this key credentials |
| scope | string | Consumer resolution scope: `CREDENTIAL`, `CREDENTIAL_ORGANIZATION`, `PROJECT`, or `GLOBAL` |
| scopeRef | string | Reference resolved per `scope` (credential id / organization id / project id / `null` for `GLOBAL`) |
| credentialRefSet | boolean | `true` if a backing vault credential is configured. The credential id itself is masked (INV-06) and never returned |
| priority | integer | Resolution priority; lower is tried first (`CREDENTIAL`=1, `CREDENTIAL_ORGANIZATION`=2, `PROJECT`=3, `GLOBAL`=4) |
| validFrom | string | ISO-8601 validity start (`null` = active immediately) |
| validUntil | string | ISO-8601 validity end (`null` = no expiry) |
| isActive | boolean | Whether the mapping is active |
| inactiveReason | string | Reason when inactive (e.g. `rotated`, `compromised`, `expired`, `manual_disabled`) |
| metadata | object | Free-form string key/value metadata |
| lastResolvedAt | string | ISO-8601 timestamp of the last resolution |
| resolveCount | integer | Number of times the mapping has been resolved |

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Project (name: MyProject) was not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",

```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/virtual-keys/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Permission**: Requires `AI_DEVELOPMENT` + `VIEW` in the project.
- **Masked credential**: `credentialRef` is never returned — only the boolean `credentialRefSet` flag (INV-06). The vault credential holding the real provider key is never disclosed.
- **Empty list**: If the project has no virtual keys, an empty `resultList` and `resultCount: 0` are returned.
- **Case sensitivity**: Project and virtual key names are matched case-insensitively for lookup but stored as entered.
