---
title: "Delete Virtual Key"
description: "Deletes an existing AI Gateway virtual key (provider credential mapping) from the specified project and undeploys it from the workers."
---

## Endpoint

```
DELETE /apiops/projects/{projectName}/virtual-keys/{virtualKeyName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| virtualKeyName | string | Yes | Virtual key name. Must exist in the project |

### Query Parameters

None

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Virtual key (name: openai-team-key) was not found!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "Virtual key name can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X DELETE \
  "https://demo.apinizer.com/apiops/projects/MyProject/virtual-keys/openai-team-key/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Permission**: Requires `AI_DEVELOPMENT` + `MANAGE` in the project.
- **Removes and undeploys**: Deletion removes the mapping from the database and undeploys it from the workers.
- **Not found**: Deleting a non-existent name returns a `400 Bad Request` with `Virtual key (name: ...) was not found!`.
- **Irreversible**: The vault `Credential` that the virtual key referenced is not deleted — only the mapping is removed. Consumers relying on this mapping will fall back to a broader-scope mapping (or fail to resolve if none exists).
