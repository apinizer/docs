---
title: "Rotate Virtual Key"
description: "Rotates the vault credential backing an existing AI Gateway virtual key. A new active mapping is created pointing at the new credential, while the previous mapping is retired but kept resolvable as a rollback safety net for a grace window."
---

## Endpoint

```
POST /apiops/projects/{projectName}/virtual-keys/{virtualKeyName}/rotate/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| virtualKeyName | string | Yes | Virtual key name to rotate. Must exist in the project |

### Query Parameters

None

### Request Body

#### Full JSON Body Example

```json
{
  "newCredentialRef": "66d0f1a2c3d4e5f6a7b8c9d0",
  "graceMinutes": 60
}
```

#### Request Body Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| newCredentialRef | string | Yes | - | Id of the vault `Credential` the rotated mapping should resolve to. Must be non-blank |
| graceMinutes | integer | No | null | Minutes the previous (retired) mapping remains resolvable as a rollback safety net, deprioritized behind the new one. `null` or `<= 0` = immediate cutover (no grace). Maximum `10080` (7 days) |

### Notes

- The virtual key named `virtualKeyName` must already exist, otherwise a `400 Bad Request` is returned.
- `graceMinutes` above `10080` (7 days) is rejected with `graceMinutes exceeds 10080!`.
- **Pattern-B rotation**: rotation does not swap `credentialRef` in place. A new mapping row is created (named `<virtualKeyName>-rotated`) pointing at `newCredentialRef`; the original mapping is retired (deprioritized) but kept resolvable for the grace window.

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "newCredentialRef can not be empty!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "graceMinutes exceeds 10080!"
}
```

or

```json
{
  "status": "FAILURE",

```

```json
{
  "status": "FAILURE",

```

## Notes and Warnings

- **Permission**: Requires `AI_DEVELOPMENT` + `MANAGE` in the project.
- **Pattern-B semantics**: Rotation creates a **new** mapping row named `<virtualKeyName>-rotated` pointing at `newCredentialRef`. The original row is **retired** (deprioritized), not deleted â€” it stays resolvable for `graceMinutes` as a rollback safety net, then ages out on its own via its validity check.
- **Name shift**: After rotation, the original `virtualKeyName` now resolves to the retired row; the active mapping lives under `<virtualKeyName>-rotated`. Update any automation that addresses the key by name, or rotate again referencing the new name.
- **Grace window cap**: `graceMinutes` is capped at `10080` (7 days). Omitting it (or `<= 0`) performs an immediate cutover.
- **Both rows deploy**: The rotation pushes both the new and the retired mapping to the workers; the `deploymentResult` reflects the combined deploy outcome.
- **Credential is not exposed**: `newCredentialRef` is a vault `Credential` id â€” the underlying provider key is never disclosed by this API (INV-06).
