---
title: "Update Virtual Key"
description: "Updates an existing AI Gateway virtual key (provider credential mapping) in the specified project. The virtual key must already exist; use Create for upsert behavior."
---

## Endpoint

```
PUT /apiops/projects/{projectName}/virtual-keys/{virtualKeyName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| virtualKeyName | string | Yes | Virtual key name. Must exist in the project |

### Query Parameters

None

### Request Body

#### Full JSON Body Example

```json
{
  "name": "openai-team-key",
  "llmProviderRef": "66a0f1a2c3d4e5f6a7b8c9d0",
  "scope": "PROJECT",
  "scopeRef": "660000000000000000000001",
  "credentialRef": "66c0f1a2c3d4e5f6a7b8c9d0",
  "priority": 3,
  "validFrom": "2026-07-01T00:00:00Z",
  "validUntil": "2026-12-31T23:59:59Z",
  "isActive": true,
  "metadata": {
    "region": "eu",
    "tier": "premium"
  }
}
```

#### Request Body Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| name | string | No | path value | Virtual key name. If blank, falls back to `virtualKeyName` from the path. If present, must match the path (case-insensitive) |
| llmProviderRef | string | Yes | - | Reference (by id) to the LLM provider this key credentials. Must belong to the same project |
| scope | string | Yes | - | Consumer resolution scope: `CREDENTIAL`, `CREDENTIAL_ORGANIZATION`, `PROJECT`, or `GLOBAL` |
| scopeRef | string | Conditional | - | Reference resolved per `scope` (credential id / organization id / project id / `null` for `GLOBAL`) |
| credentialRef | string | Yes | - | Id of the vault `Credential` holding the real provider key. Must belong to the same project. Masked on read as `credentialRefSet` |
| priority | integer | No | scope default | Resolution priority; lower is tried first. Auto-filled from `scope` when omitted |
| validFrom | string | No | null | ISO-8601 validity start (`null` = active immediately) |
| validUntil | string | No | null | ISO-8601 validity end (`null` = no expiry) |
| isActive | boolean | No | true | Whether the mapping is active. Setting `false` undeploys the mapping from the workers |
| inactiveReason | string | No | null | Reason when inactive (audit label) |
| metadata | object | No | null | Free-form string key/value metadata |

### Notes

- The virtual key named `virtualKeyName` must already exist, otherwise a `400 Bad Request` is returned.
- If `name` is present in the body it must match `virtualKeyName` (case-insensitive); a blank `name` falls back to the path value.
- The internal `id` and `projectId` are preserved from the existing record and cannot be repointed to another project (IDOR/BOLA guard).
- Do not send a `_class` field.

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Virtual key (name: openai-team-key) was not found!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "Virtual key name in path (openai-team-key) does not match name in body (other-name)!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X PUT \
  "https://demo.apinizer.com/apiops/projects/MyProject/virtual-keys/openai-team-key/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "openai-team-key",
    "llmProviderRef": "66a0f1a2c3d4e5f6a7b8c9d0",
    "scope": "PROJECT",
    "scopeRef": "660000000000000000000001",
    "credentialRef": "66c0f1a2c3d4e5f6a7b8c9d0",
    "priority": 3,
    "validUntil": "2026-12-31T23:59:59Z",
    "isActive": true
  }'
```

## Notes and Warnings

- **Permission**: Requires `AI_DEVELOPMENT` + `MANAGE` in the project.
- **Must exist**: Unlike Create (upsert), `PUT` fails with `Virtual key (name: ...) was not found!` if the name does not exist.
- **Re-supply `credentialRef`**: Reads mask the reference as `credentialRefSet`, and the update persists the body as sent — supply the vault credential id explicitly. Sending a blank `credentialRef` clears the reference rather than preserving the stored value.
- **Deactivation undeploys**: Setting `isActive: false` undeploys the mapping from all running environments; setting it back to `true` redeploys.
- **Save = deploy**: A successful update also pushes the mapping to the workers; the `deploymentResult` reflects the deploy outcome.
