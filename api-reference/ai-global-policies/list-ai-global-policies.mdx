---
title: "List AI Global Policies"
description: "Retrieves all global (global=true) AI policies defined in the specified project, optionally filtered by policy type. Policies are serialized through the typed PolicyDTO with AI connection references resolved to names."
---

## Endpoint

```
GET /apiops/projects/{projectName}/ai-global-policies/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| type | string | No | AI policy type filter. Accepts either the type string (e.g. `ai-pii-mask`) or the enum name (e.g. `AI_PII_MASK`). When omitted, all 11 AI policy collections are scanned. An **unknown** value is silently ignored and also falls back to all collections. |

Accepted `type` values: `ai-pii-mask`, `ai-prompt-guard`, `ai-token-rate-limit`, `ai-semantic-cache`,
`ai-rag-injection`, `ai-mcp-tool-call`, `a2a-agent-call`, `ai-dlp-guard`, `ai-loop-guard`,
`ai-topic-guard`, `ai-context-integrity`.

## Response

### Success Response (200 OK)

Each element of `resultList` is a polymorphic `PolicyDTO`. The `type` field is the discriminator; the
remaining fields depend on the policy type. The example below shows a PII-mask policy and a
semantic-cache policy (whose `vectorDbName` / `embeddingProviderName` are resolved from raw IDs to
names).

```json
{
  "status": "SUCCESS",
  "resultList": [
    {
      "type": "ai-pii-mask",
      "name": "global-pii-mask",
      "description": "Mask PII across all AI proxies",
      "active": true,
      "buildCustomMessage": false,
      "privacyDefinitions": [
        {
          "elementName": "email",
          "operation": "MASK",
          "builtInPiiType": "EMAIL",
          "regexPattern": null
        }
      ],
      "carryBufferSize": 256
    },
    {
      "type": "ai-semantic-cache",
      "name": "global-semantic-cache",
      "description": "Shared semantic cache",
      "active": true,
      "buildCustomMessage": false,
      "enabled": true,
      "ttlSeconds": 3600,
      "cacheKeyStrategy": "PROXY_ISOLATED",
      "cacheConnectionTimeoutInSeconds": 3,
      "maxCacheableBytes": 262144,
      "vectorEnabled": true,
      "vectorDbName": "prod-qdrant",
      "embeddingProviderName": "openai-embeddings",
      "embeddingModelId": "text-embedding-3-small",
      "embeddingDimension": 1536,
      "similarityThreshold": 0.95,
      "vectorTopK": 1,
      "vectorCollectionName": "semantic_cache"
    }
  ],
  "resultCount": 2
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | List of global AI policy objects (polymorphic `PolicyDTO`) |
| resultCount | integer | Total number of policies returned |

#### Policy Object Fields (common)

| Field | Type | Description |
|-------|------|-------------|
| type | string | Policy type discriminator (e.g. `ai-pii-mask`, `ai-semantic-cache`) |
| name | string | Policy name (unique within the project's global set) |
| description | string | Policy description |
| active | boolean | Whether the policy is active |
| policyCondition | object | Optional condition tree gating policy execution (`null` when unconditional) |
| errorMessageList | array | Custom error messages attached to the policy (`null` when none) |
| buildCustomMessage | boolean | Whether a custom response message is built |
| customMessageTemplate | string | Custom message template (when `buildCustomMessage` is `true`) |
| customMessageContentType | string | Custom message content type |
| customMessageHttpStatusCode | integer | Custom message HTTP status code |

Type-specific fields extend this base. For example, `ai-pii-mask` adds `privacyDefinitions` and
`carryBufferSize`; `ai-semantic-cache` adds the cache/vector-layer fields shown above, where
**`vectorDbName`** and **`embeddingProviderName`** are the name-resolved AI connection references
(the raw connection IDs are never serialized).

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Project with name (MyProject) is not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

### List all global AI policies

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/ai-global-policies/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### List only PII-mask global policies

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/ai-global-policies/?type=ai-pii-mask" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Cross-collection aggregate**: A global AI policy is a `Policy` document with `global=true` in one
  of 11 `PolicyAi*` collections. This endpoint aggregates all of them (or a single collection when
  `type` is supplied).
- **Unknown type falls back to all**: An unrecognized `type` value is not an error - it is logged and
  ignored, and the query scans all 11 collections.
- **Name-based AI references**: For `ai-semantic-cache` / `ai-rag-injection` the LLM and VectorDB
  references are returned as `vectorDbName` / `embeddingProviderName`; for `ai-mcp-tool-call` the MCP
  reference is returned by name. Raw connection IDs are never exposed.
- **Secrets never disclosed (INV-06)**: Policy documents carry no provider secrets. The referenced AI
  connection configs hold the `@SecretData` fields, which are never included here.
- **Empty List**: If the project has no global AI policies, an empty `resultList` is returned with
  `resultCount` = `0`.
- **Case Sensitivity**: Project names are case-sensitive in path references.
- **Permission**: Requires `AI_DEVELOPMENT` + `VIEW` in the project.
