---
title: "Delete AI Global Policy"
description: "Deletes a global AI policy by name and fans out the removal to every referencing proxy and proxy-group. Only global=true policies are removable here; the delete re-verifies project ownership (IDOR guard) and the global invariant before removing."
---

## Endpoint

```
DELETE /apiops/projects/{projectName}/ai-global-policies/{policyName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| policyName | string | Yes | Global AI policy name to delete |

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| type | string | No | Optional AI policy type hint to restrict the lookup to a single collection (e.g. `ai-pii-mask` or `AI_PII_MASK`). When omitted, all 11 collections are searched. An unknown value is silently ignored. |

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| deploymentResult | object | Result of the delete + fan-out redeploy |
| deploymentResult.success | boolean | `true` when the policy was removed and the redeploy succeeded |

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Global AI policy (name: global-pii-mask) was not found!"
}
```

The delete also rejects requests that violate the ownership or global-policy invariants:

```json
{
  "status": "FAILURE",
  "resultMessage": "Global AI policy (name: global-pii-mask) does not belong to project MyProject!"
}
```

```json
{
  "status": "FAILURE",
  "resultMessage": "Policy (name: global-pii-mask) is not a global policy!"
}
```

or, when the policy name is blank:

```json
{
  "status": "FAILURE",
  "resultMessage": "Global AI policy name can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X DELETE \
  "https://demo.apinizer.com/apiops/projects/MyProject/ai-global-policies/global-pii-mask/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Global-only**: Only a policy with `global=true` is removable through this endpoint. A local
  (`global=false`) policy attached to a specific proxy is never deleted here - it returns
  `... is not a global policy!`.
- **IDOR guard (fetch-then-verify)**: Before deleting, the endpoint re-fetches the policy and asserts
  it belongs to the authenticated project. A policy owned by another project returns
  `... does not belong to project ...!`.
- **Cascading removal**: The delete fans out across the 11 `PolicyAi*` collections (first match wins)
  and triggers `GlobalPolicyUpdateService` to strip the embedded copy of the policy from every
  referencing proxy and proxy-group, then redeploys.
- **Check usage first**: Use [Get AI Global Policy Usage](/api-reference/ai-global-policies/get-ai-global-policy-usage)
  to see which proxies/proxy-groups reference the policy before removing it, since the delete detaches
  it from all of them.
- **Permission**: Requires `AI_DEVELOPMENT` + `MANAGE` in the project (stricter than the read
  endpoints, which require only VIEW).
