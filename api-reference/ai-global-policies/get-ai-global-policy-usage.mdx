---
title: "Get AI Global Policy Usage"
description: "Returns the API proxies and proxy-groups that reference a given global AI policy by name. Each result is a lightweight reference (asset type + name), reported as a reference count, not runtime traffic statistics."
---

## Endpoint

```
GET /apiops/projects/{projectName}/ai-global-policies/{policyName}/usage/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| policyName | string | Yes | Global AI policy name |

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| type | string | No | Optional AI policy type hint to restrict the policy lookup to a single collection (e.g. `ai-topic-guard` or `AI_TOPIC_GUARD`). When omitted, all 11 collections are searched. An unknown value is silently ignored. |

## Response

### Success Response (200 OK)

`resultList` contains one reference object per API proxy and proxy-group that uses the policy.

```json
{
  "status": "SUCCESS",
  "resultList": [
    {
      "type": "API_PROXY",
      "name": "chat-proxy"
    },
    {
      "type": "API_PROXY",
      "name": "summarize-proxy"
    },
    {
      "type": "API_PROXY_GROUP",
      "name": "ai-gateway-group"
    }
  ],
  "resultCount": 3
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | List of usage-reference objects |
| resultCount | integer | Total number of referencing assets |

#### Usage Reference Object Fields

| Field | Type | Description |
|-------|------|-------------|
| type | string | Referencing asset type: `API_PROXY` or `API_PROXY_GROUP` |
| name | string | Name of the referencing proxy or proxy-group |

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Global AI policy (name: global-topic-guard) was not found!"
}
```

or, when the policy name is blank:

```json
{
  "status": "FAILURE",
  "resultMessage": "Global AI policy name can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/ai-global-policies/global-topic-guard/usage/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Reference count, not runtime stats**: This endpoint reports which proxies/proxy-groups embed a
  copy of the policy; it does not report request volume, hit rate or any traffic metrics.
- **Two asset types only**: `type` is either `API_PROXY` or `API_PROXY_GROUP`. Proxies are listed
  first, then proxy-groups.
- **Empty usage**: If nothing references the policy, an empty `resultList` is returned with
  `resultCount` = `0` - the policy is safe to delete without detaching it from any asset.
- **Delete impact**: Deleting the policy detaches it from every asset listed here (via
  `GlobalPolicyUpdateService`). Review this list before calling
  [Delete AI Global Policy](/api-reference/ai-global-policies/delete-ai-global-policy).
- **Permission**: Requires `AI_DEVELOPMENT` + `VIEW` in the project.
