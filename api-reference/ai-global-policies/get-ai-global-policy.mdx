---
title: "Get AI Global Policy"
description: "Retrieves a single global AI policy by name, scoped to the project's global=true set across all 11 PolicyAi* collections. The policy is serialized through the typed PolicyDTO with AI connection references resolved to names."
---

## Endpoint

```
GET /apiops/projects/{projectName}/ai-global-policies/{policyName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| policyName | string | Yes | Global AI policy name (matched case-insensitively within the project's `global=true` set) |

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| type | string | No | Optional AI policy type hint to restrict the lookup to a single collection (e.g. `ai-semantic-cache` or `AI_SEMANTIC_CACHE`). When omitted, all 11 collections are searched. An unknown value is silently ignored (search falls back to all collections). |

## Response

### Success Response (200 OK)

The single matched policy is returned as the only element of `resultList`. The response is a
polymorphic `PolicyDTO`; the `type` field is the discriminator. The example below is a semantic-cache
policy, whose `vectorDbName` / `embeddingProviderName` are the name-resolved AI connection references
(raw connection IDs are never serialized).

```json
{
  "status": "SUCCESS",
  "resultList": [
    {
      "type": "ai-semantic-cache",
      "name": "global-semantic-cache",
      "description": "Shared semantic cache",
      "active": true,
      "buildCustomMessage": false,
      "enabled": true,
      "ttlSeconds": 3600,
      "cacheKeyStrategy": "PROXY_ISOLATED",
      "cacheConnectionTimeoutInSeconds": 3,
      "maxCacheableBytes": 262144,
      "vectorEnabled": true,
      "vectorDbName": "prod-qdrant",
      "embeddingProviderName": "openai-embeddings",
      "embeddingModelId": "text-embedding-3-small",
      "embeddingDimension": 1536,
      "similarityThreshold": 0.95,
      "vectorTopK": 1,
      "vectorCollectionName": "semantic_cache"
    }
  ],
  "resultCount": 1
}
```

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | Single-element list containing the matched global AI policy (polymorphic `PolicyDTO`) |
| resultCount | integer | Number of policies returned (`1` on success) |

#### Policy Object Fields (common)

| Field | Type | Description |
|-------|------|-------------|
| type | string | Policy type discriminator (e.g. `ai-semantic-cache`) |
| name | string | Policy name |
| description | string | Policy description |
| active | boolean | Whether the policy is active |
| policyCondition | object | Optional condition tree gating policy execution (`null` when unconditional) |
| errorMessageList | array | Custom error messages attached to the policy (`null` when none) |
| buildCustomMessage | boolean | Whether a custom response message is built |
| customMessageTemplate | string | Custom message template (when `buildCustomMessage` is `true`) |
| customMessageContentType | string | Custom message content type |
| customMessageHttpStatusCode | integer | Custom message HTTP status code |

Type-specific fields extend this base (for the example above, the semantic-cache fields
`enabled`, `ttlSeconds`, `vectorEnabled`, `vectorDbName`, `embeddingProviderName`, etc.).

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Global AI policy (name: global-semantic-cache) was not found!"
}
```

or, when the policy name is blank:

```json
{
  "status": "FAILURE",
  "resultMessage": "Global AI policy name can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/ai-global-policies/global-semantic-cache/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

With a type hint to restrict the search to one collection:

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/ai-global-policies/global-semantic-cache/?type=ai-semantic-cache" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Name match is case-insensitive**: The policy name is compared case-insensitively within the
  project's `global=true` set; the path itself is still name-based.
- **Name-based AI references**: For `ai-semantic-cache` / `ai-rag-injection` the LLM and VectorDB
  references are returned as `vectorDbName` / `embeddingProviderName`; for `ai-mcp-tool-call` the MCP
  reference is returned by name. Raw connection IDs are never exposed.
- **Secrets never disclosed (INV-06)**: Policy documents carry no provider secrets; the referenced AI
  connection configs hold the `@SecretData` fields, which are never included here.
- **Not found**: A non-existent policy name yields a `400 Bad Request` with a
  `... was not found!` message.
- **Permission**: Requires `AI_DEVELOPMENT` + `VIEW` in the project.
