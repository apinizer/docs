---
title: "Update MCP Inbound Server"
description: "Updates an existing AI Gateway MCP inbound server in the specified project. A blank virtualApiKey preserves the stored secret. The server is re-encrypted and redeployed to the workers."
---

## Endpoint

```
PUT /apiops/projects/{projectName}/mcp-inbound-servers/{serverName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |
| Content-Type | application/json | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| serverName | string | Yes | Existing MCP inbound server name (must match `name` in the body if provided) |

### Request Body

#### Full JSON Body Example

```json
{
  "name": "customer-tools",
  "description": "Exposes customer REST proxies as MCP tools (updated)",
  "active": true,
  "authType": "VIRTUAL_API_KEY",
  "virtualApiKey": "",
  "exposedTools": [
    {
      "proxyId": "660aabbccddee00112233445",
      "toolName": "get_customer",
      "toolDescription": "Fetch a customer by id",
      "apiMethodHttpMethod": "GET",
      "apiMethodPath": "/customers/{id}",
      "requiredScopes": []
    }
  ]
}
```

#### Request Body Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| name | string | No | path value | Server name; if blank, falls back to the `serverName` path segment. If provided, must match the path (case-insensitive) |
| description | string | No | - | Optional description shown in the MCP manifest |
| active | boolean | No | false | Whether to deploy the server and serve the `/aimcp/\{id\}/...` route |
| authType | string | No | VIRTUAL_API_KEY | Inbound auth mechanism: `VIRTUAL_API_KEY`, `OAUTH2`, or `NONE` |
| virtualApiKey | string | No | (preserved) | Bearer key for `VIRTUAL_API_KEY` auth. **Blank/omitted preserves the stored value** |
| oauth2Settings | object | Conditional | - | OAuth2/JWT validation settings. Required when `authType = OAUTH2` (fail-closed if missing) |
| toolOutputFilter | object | No | null | Tool-call output filter (`null`/`enabled=false` = passthrough) |
| exposedTools | array | No | [] | Proxies to expose as MCP tools |

The nested `oauth2Settings`, `toolOutputFilter` and `exposedTools` fields are identical to those documented on the [Create MCP Inbound Server](/api-reference/mcp-inbound-servers/create-mcp-inbound-server) page.

### Notes

- The server must already exist; a non-existent `serverName` returns `400` with a `not found` message.
- If the body `name` is blank it defaults to the path `serverName`; if provided it must match the path (case-insensitive).
- Sending a blank/omitted `virtualApiKey` preserves the previously stored encrypted secret (INV-06).
- The `projectId` is always taken from the path project — any value sent in the body is ignored.

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "MCP inbound server (name: customer-tools) was not found!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "MCP inbound server name in path (customer-tools) does not match name in body (other-name)!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

#### Causes

- Missing or invalid Authorization header
- Token expired or revoked
- User lacks `AI_DEVELOPMENT` + `MANAGE` permission in the project

## cURL Example

```bash
curl -X PUT \
  "https://demo.apinizer.com/apiops/projects/MyProject/mcp-inbound-servers/customer-tools/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "customer-tools",
    "description": "Exposes customer REST proxies as MCP tools (updated)",
    "active": true,
    "authType": "VIRTUAL_API_KEY",
    "virtualApiKey": "",
    "exposedTools": [
      {
        "proxyId": "660aabbccddee00112233445",
        "toolName": "get_customer",
        "apiMethodHttpMethod": "GET",
        "apiMethodPath": "/customers/{id}"
      }
    ]
  }'
```

## Notes and Warnings

- **Server Must Exist**: `PUT` fails if the named server does not exist. To create-or-update, use the [upsert POST](/api-reference/mcp-inbound-servers/create-mcp-inbound-server).
- **Secret Preservation**: A blank or omitted `virtualApiKey` keeps the stored encrypted value — you do not need to resend the secret on every update (INV-06).
- **Name Match**: If `name` is provided in the body it must equal the path `serverName` (case-insensitive); a blank body name defaults to the path value.
- **OAuth2 Fail-Closed**: When `authType = OAUTH2`, a missing or incomplete `oauth2Settings` renders the server misconfigured and every inbound request is denied.
- **Redeploy on Save**: A successful update re-encrypts secrets and redeploys the server to the workers.
