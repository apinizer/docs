---
title: "List MCP Inbound Servers"
description: "Lists all AI Gateway MCP inbound servers in the specified project. The virtualApiKey secret field is never returned."
---

## Endpoint

```
GET /apiops/projects/{projectName}/mcp-inbound-servers/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |

### Query Parameters

None

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | List of MCP inbound server objects |
| resultCount | integer | Total number of servers returned |

#### MCP Inbound Server Object Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Server ID (read-only) |
| name | string | Server name (unique within the project) |
| projectId | string | Owning project ID |
| description | string | Optional description shown in the MCP manifest |
| active | boolean | Whether the server is deployed and serving the `/aimcp/\{id\}/...` route |
| authType | string | Inbound auth mechanism: `VIRTUAL_API_KEY`, `OAUTH2`, or `NONE` |
| oauth2Settings | object | OAuth2/JWT validation settings (present only for `authType = OAUTH2`) |
| toolOutputFilter | object | Tool-call output filter config (`null` = disabled) |
| exposedTools | array | Proxies exposed as MCP tools |
| createdBy | string | User who created the server |
| createdDate | string (date-time) | Creation timestamp |
| lastModifiedBy | string | User who last modified the server |
| lastModifiedDate | string (date-time) | Last modification timestamp |

> **`virtualApiKey` is never present in read responses.** It is a write-only `@SecretData` field — accepted on create/update and always omitted (never returned as plaintext or ciphertext) on read (INV-06).

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "Project with name (MyProject) is not found!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

#### Causes

- Missing or invalid Authorization header
- Token expired or revoked
- User lacks `AI_DEVELOPMENT` + `VIEW` permission in the project

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/mcp-inbound-servers/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Secret Masking**: The `virtualApiKey` field is write-only and is never returned in list responses (INV-06).
- **Empty List**: If the project has no MCP inbound servers, an empty `resultList` is returned with `resultCount: 0`.
- **Name-based**: Servers are identified by name within the project; runtime IDs are informational only.
- **Case Sensitivity**: Project and server names are case-sensitive in the path; server name matching on write is case-insensitive.
