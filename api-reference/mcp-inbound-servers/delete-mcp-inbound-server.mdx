---
title: "Delete MCP Inbound Server"
description: "Deletes an AI Gateway MCP inbound server from the specified project and undeploys it from the workers."
---

## Endpoint

```
DELETE /apiops/projects/{projectName}/mcp-inbound-servers/{serverName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| serverName | string | Yes | MCP inbound server name to delete |

### Query Parameters

None

## Response

### Success Response (200 OK)

```json
{
  "status": "SUCCESS",
  "deploymentResult": {
    "success": true
  }
}
```

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "MCP inbound server (name: customer-tools) was not found!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "MCP inbound server name can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

#### Causes

- Missing or invalid Authorization header
- Token expired or revoked
- User lacks `AI_DEVELOPMENT` + `MANAGE` permission in the project

## cURL Example

```bash
curl -X DELETE \
  "https://demo.apinizer.com/apiops/projects/MyProject/mcp-inbound-servers/customer-tools/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Server Must Exist**: Deleting a non-existent server name returns `400 Bad Request` with a `not found` message (not `404`).
- **Undeploy**: Deletion removes the server from the database and undeploys its `/aimcp/\{id\}/...` route from every worker.
- **Project Scoped**: The deletion is guarded by the path project (IDOR protection) — you can only delete servers owned by the named project.
- **Irreversible**: There is no soft-delete; the server document and its exposed-tool references are permanently removed.
