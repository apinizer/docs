---
title: "Get MCP Inbound Server"
description: "Retrieves a single AI Gateway MCP inbound server by name. The virtualApiKey secret field is never returned."
---

## Endpoint

```
GET /apiops/projects/{projectName}/mcp-inbound-servers/{serverName}/
```

## Authentication

Requires a Personal API Access Token.

### Header

```
Authorization: Bearer YOUR_TOKEN
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| Authorization | Bearer \{token\} | Yes |

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| projectName | string | Yes | Project name |
| serverName | string | Yes | MCP inbound server name |

### Query Parameters

None

#### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| status | string | Response status: `SUCCESS` or `FAILURE` |
| resultList | array | List containing the single matched server object |
| resultCount | integer | Number of servers returned (`1` on success) |

#### MCP Inbound Server Object Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Server ID (read-only) |
| name | string | Server name (unique within the project) |
| projectId | string | Owning project ID |
| description | string | Optional description shown in the MCP manifest |
| active | boolean | Whether the server is deployed and serving the `/aimcp/\{id\}/...` route |
| authType | string | Inbound auth mechanism: `VIRTUAL_API_KEY`, `OAUTH2`, or `NONE` |
| oauth2Settings | object | OAuth2/JWT validation settings (present only for `authType = OAUTH2`; see fields below) |
| toolOutputFilter | object | Tool-call output filter config (`null` = disabled; see fields below) |
| exposedTools | array | Proxies exposed as MCP tools (see fields below) |
| createdBy | string | User who created the server |
| createdDate | string (date-time) | Creation timestamp |
| lastModifiedBy | string | User who last modified the server |
| lastModifiedDate | string (date-time) | Last modification timestamp |

#### oauth2Settings Fields

| Field | Type | Description |
|-------|------|-------------|
| jwksUrl | string | Remote JWKS endpoint used to validate inbound Bearer JWTs |
| issuer | string | Expected `iss` claim (exact match) |
| audience | string | Expected `aud` claim (the token's `aud` list must contain it) |
| requiredScopes | array[string] | Server-level scope gate; the token `scope` claim must contain ALL entries (empty = no gate) |
| clockSkewSeconds | integer | Allowed clock skew for `exp`/`nbf` (`null` = 0, strict) |

#### toolOutputFilter Fields

| Field | Type | Description |
|-------|------|-------------|
| enabled | boolean | Master switch (`null`/`false` = passthrough) |
| maskPii | boolean | Mask the 5 built-in PII types in tool-call output |
| denyPatterns | array[string] | Regex denylist; any match blocks the whole response |

#### exposedTools Fields

| Field | Type | Description |
|-------|------|-------------|
| proxyId | string | ApiProxy ID reference resolved at tool-call time |
| toolName | string | MCP tool name override (auto-generated from proxy name if null) |
| toolDescription | string | Tool description override (proxy description if null) |
| overrideSchema | string | Manual JSON-RPC `inputSchema` override (used mainly for SOAP/WSDL proxies) |
| apiMethodHttpMethod | string | Optional HTTP method pinning the tool to one operation |
| apiMethodPath | string | Optional REST endpoint path pinning the tool to one operation |
| requiredScopes | array[string] | Per-tool scope gate (enforced only when `authType = OAUTH2`) |

> **`virtualApiKey` is never present in read responses.** It is a write-only `@SecretData` field — accepted on create/update and always omitted (never returned as plaintext or ciphertext) on read (INV-06).

### EnumStatus

- `SUCCESS` - Operation successful
- `FAILURE` - Operation failed

### Error Response (400 Bad Request)

```json
{
  "status": "FAILURE",
  "resultMessage": "MCP inbound server (name: customer-tools) was not found!"
}
```

or

```json
{
  "status": "FAILURE",
  "resultMessage": "MCP inbound server name can not be empty!"
}
```

### Error Response (401 Unauthorized)

```json
{
  "status": "FAILURE",
  "resultMessage": "Token is not valid!"
}
```

#### Causes

- Missing or invalid Authorization header
- Token expired or revoked
- User lacks `AI_DEVELOPMENT` + `VIEW` permission in the project

## cURL Example

```bash
curl -X GET \
  "https://demo.apinizer.com/apiops/projects/MyProject/mcp-inbound-servers/customer-tools/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Notes and Warnings

- **Secret Masking**: The `virtualApiKey` field is write-only and is never returned (INV-06).
- **Single Result**: The matched server is returned as the only element of `resultList`.
- **Not Found**: A non-existent server name returns `400 Bad Request` with a `not found` message (not `404`).
- **Case Sensitivity**: The `serverName` path segment is matched case-insensitively.
