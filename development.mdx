# Yerel Geliştirme

Apinizer dokümantasyonunu yerel olarak görüntülemek ve düzenlemek için bu rehberi takip edin.

<Info>
**Gereksinimler**:
- Node.js sürüm 18 veya üzeri
- npm veya yarn paket yöneticisi
- Git
</Info>

## Mintlify CLI Kurulumu

Apinizer dokümantasyonu Mintlify kullanılarak oluşturulmuştur. Yerel geliştirme için Mintlify CLI'yi kurmanız gerekir.

<Steps>
  <Step title="Mintlify CLI'yi yükleyin">

```bash
npm i -g mint
```
    
    Kurulumun başarılı olduğunu doğrulamak için:
    
    ```bash
    mint --version
    ```
  </Step>

  <Step title="Repository'yi klonlayın">
    
    ```bash
    git clone https://github.com/apinizer/docs.git
    cd docs
    ```
</Step>

  <Step title="Yerel önizlemeyi başlatın">

    Dokümantasyon dizininde (docs.json dosyasının bulunduğu dizin) şu komutu çalıştırın:

```bash
mint dev
```

    Yerel önizleme `http://localhost:3000` adresinde kullanılabilir olacaktır.

    <Tip>Değişiklikleriniz otomatik olarak tarayıcıda yenilenir.</Tip>
</Step>
</Steps>

## Özel Port Kullanımı

Varsayılan olarak Mintlify port 3000'i kullanır. Farklı bir port kullanmak için `--port` bayrağını kullanabilirsiniz:

```bash
mint dev --port 3333
```

Eğer seçtiğiniz port zaten kullanılıyorsa, Mintlify bir sonraki kullanılabilir portu otomatik olarak kullanacaktır:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify CLI Güncelleme

Her CLI sürümü belirli bir Mintlify sürümüyle ilişkilidir. Yerel önizlemeniz production sürümüyle uyumlu değilse, CLI'yi güncelleyin:

```bash
npm update -g mint
```

Veya tam yeniden kurulum için:

```bash
npm remove -g mint
npm i -g mint
```

## Bağlantı Doğrulama

CLI, dokümantasyonunuzdaki bağlantıları doğrulamanıza yardımcı olabilir. Kırık bağlantıları bulmak için:

```bash
mint broken-links
```

## Dokümantasyon Düzenleme

### Dosya Yapısı

- Dokümantasyon dosyaları `.mdx` formatındadır
- Ana konfigürasyon dosyası `docs.json` içindedir
- Türkçe dokümantasyon `tr/` dizini altındadır
- Görseller `images/` dizininde saklanır

### MDX Formatı

Apinizer dokümantasyonu MDX formatını kullanır. MDX, Markdown'ın genişletilmiş bir versiyonudur ve React bileşenlerini destekler.

### Kod Formatlama

IDE'nizde MDX'i tanımak ve formatlamak için uzantılar kullanmanızı öneririz:

- **VSCode kullanıcıları için**:
  - [MDX VSCode extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) - Syntax highlighting için
  - [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) - Kod formatlama için

## Sorun Giderme

<AccordionGroup>
  <Accordion title='Hata: "sharp" modülü yüklenemedi (darwin-arm64 runtime)'>
    
    Bu hata genellikle eski bir Node.js sürümünden kaynaklanır. Şu adımları deneyin:
    
    1. Mevcut CLI sürümünü kaldırın: `npm remove -g mint`
    2. Node.js'i v18 veya üzeri bir sürüme yükseltin
    3. CLI'yi yeniden yükleyin: `npm i -g mint`
  </Accordion>

  <Accordion title="Sorun: Bilinmeyen bir hata oluştu">
    
    Çözüm: Cihazınızın kök dizinine gidin ve `~/.mintlify` klasörünü silin. Ardından `mint dev` komutunu tekrar çalıştırın.
  </Accordion>

  <Accordion title="Sorun: Sayfa 404 hatası veriyor">
    
    `docs.json` dosyasının bulunduğu dizinde çalıştırdığınızdan emin olun. Ayrıca `docs.json` dosyasının geçerli bir JSON formatında olduğunu kontrol edin.
  </Accordion>

  <Accordion title="Sorun: Değişiklikler görünmüyor">
  
    - Tarayıcıyı yenileyin (Ctrl+R veya Cmd+R)
    - Mintlify CLI'yi güncelleyin: `npm update -g mint`
    - `~/.mintlify` klasörünü temizleyin ve tekrar deneyin
  </Accordion>
</AccordionGroup>

## Faydalı Komutlar

```bash
# Yerel önizlemeyi başlat
mint dev

# Özel port ile başlat
mint dev --port 3333

# Kırık bağlantıları kontrol et
mint broken-links

# CLI sürümünü kontrol et
mint --version

# CLI'yi güncelle
npm update -g mint
```
