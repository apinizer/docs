Bu rehber Apinizer Integration katmanının — Manager tarafından kaydedilen task flow'ları çalıştıran Quartz scheduler'ı — standalone Linux paketinin kurulumunu anlatır.

<Info>
Installer sudo yetkisi gerektirir; interaktif root login gerekmez.
</Info>

## Gereksinimler

- Linux x86-64 (glibc) — Ubuntu 20.04+, RHEL/Rocky/Alma 8+
- sudo yetkisi
- Manager'ın kullandığı **aynı** MongoDB
- Açık port: `8092/tcp` (Manager → Integration REST)

## 1. İndir + Aç

```bash
VERSION=2026.04.5
curl -fSLO "https://packages.apinizer.com/apinizer-packages/integration/${VERSION}/apinizer-integration-${VERSION}-linux-x64.tar.gz"
curl -fSLO "https://packages.apinizer.com/apinizer-packages/integration/${VERSION}/checksums.sha256"
sha256sum -c checksums.sha256

sudo mkdir -p /opt
sudo tar -xzf apinizer-integration-${VERSION}-linux-x64.tar.gz -C /opt
sudo mv /opt/apinizer-integration-${VERSION} /opt/apinizer-integration
```

## 2. Yapılandır

`conf/apinizer-integration.env`'i edit edin:

| Anahtar | Açıklama |
| --- | --- |
| `SPRING_DATA_MONGODB_URI` | MongoDB connection string (Manager ile aynı instance) |
| `SPRING_DATA_MONGODB_DATABASE` | Veritabanı adı (Manager DB ile aynı olmalı) |
| `INTEGRATION_TIMEZONE` | IANA zone id (örn. `Europe/Istanbul`). Manager ve Worker ile eşleşmeli, aksi takdirde cron yanlış saatte fire eder. |

## 3. Installer + Şifreleme

```bash
sudo /opt/apinizer-integration/bin/apiintegration-install.sh
sudo -u apinizer /opt/apinizer-integration/bin/apiintegration-encrypt.sh
```

## 4. Servisi başlat

```bash
sudo systemctl start apinizer-apiintegration
sudo systemctl status apinizer-apiintegration
sudo journalctl -u apinizer-apiintegration -f
```

Beklenen log:

```
Integration timezone set to: Europe/Istanbul
Initializing Custom MongoDB Job Store
Quartz Scheduler Prod Environment Setup started
MongoDB connection established for Quartz Job Store
Tomcat started on port 8092 (http)
```

## 5. Çoklu node / HA

Birden fazla Integration node'u aynı MongoDB veritabanını paylaşabilir — otomatik clustered Quartz scheduler oluştururlar. Ek standalone ayar gerekmez, sadece aynı `SPRING_DATA_MONGODB_URI` + `SPRING_DATA_MONGODB_DATABASE`'i set edin.

<Tip>
Zamana duyarlı flow'lar için en az iki node çalıştırın. Tek node ile bir restart scheduled run'ları bloklar.
</Tip>

## 6. Manager'ı Integration'a bağla

Manager UI'da `Admin → Integration Servers` altına bu node'un hostname/IP + port'unu (`8092`) ekleyin. Manager job register/unregister ve cache-refresh event'lerini REST üzerinden push eder.

## Manuel kontrol

```bash
sudo -u apinizer /opt/apinizer-integration/bin/apiintegration-start.sh           # background
sudo -u apinizer /opt/apinizer-integration/bin/apiintegration-start.sh -f        # foreground
sudo -u apinizer /opt/apinizer-integration/bin/apiintegration-stop.sh
```

<Info>
Task flow'lar dakikalar süren işler içeriyorsa systemd unit'teki `TimeoutStopSec=`'i artırın (varsayılan 120 s). Quartz `waitForJobsToCompleteOnShutdown=true` ile yapılandırılmıştır; SIGTERM temiz drain tetikler — timeout drain'in maksimum süresini belirler.
</Info>

## Sorun Giderme

### Quartz job beklenen zamanda fire olmadı

`INTEGRATION_TIMEZONE` Manager'daki cron expression timezone'una göre kontrol edin. Çoklu node'lar farklı timezone'larla tutarsız fire üretir.

### `Locked by another scheduler` ile exit

Önceki süreç trigger ortasında SIGKILL aldı. Yeni süreç lock-recovery sweep'i bir `clusterCheckin` cycle (default 7.5 s) içinde temizler.
