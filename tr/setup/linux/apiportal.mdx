Bu rehber Apinizer API Portal standalone paketinin Linux sunucuya kurulumunu anlatır. Portal stateless'tır ve veriyi Manager REST API'sinden çeker — doğrudan MongoDB bağlantısı yoktur.

<Info>
Installer sudo yetkisi gerektirir; interaktif root login gerekmez.
</Info>

## Gereksinimler

- Linux x86-64 (glibc) — Ubuntu 20.04+, RHEL/Rocky/Alma 8+
- sudo yetkisine sahip bir operatör
- `openssl`, `systemd`, `tar`, `curl`
- Erişilebilir bir Apinizer Manager ve geçerli bir Manager API key

## 1. Manager'da Portal'ı kaydet

Kuruluma başlamadan önce Manager UI'da Portal kaydını yapın:

1. **Portal → Portals → Create** — üretilen **Portal ID**'yi not edin.
2. **Portal → API Keys → Generate** — API key'i kopyalayın (sadece bir kez gösterilir).

## 2. İndir + Aç

```bash
VERSION=2026.04.5
curl -fSLO "https://packages.apinizer.com/apinizer-packages/apiportal/${VERSION}/apinizer-apiportal-${VERSION}-linux-x64.tar.gz"
curl -fSLO "https://packages.apinizer.com/apinizer-packages/apiportal/${VERSION}/checksums.sha256"
sha256sum -c checksums.sha256

sudo mkdir -p /opt
sudo tar -xzf apinizer-apiportal-${VERSION}-linux-x64.tar.gz -C /opt
sudo mv /opt/apinizer-apiportal-${VERSION} /opt/apinizer-portal
```

## 3. Yapılandır

`conf/application.env`'i edit edin:

```bash
sudo vi /opt/apinizer-portal/conf/application.env
```

Zorunlu alanlar:

| Anahtar | Açıklama |
| --- | --- |
| `API_PORTAL_ID` | 1. adımdan gelen Portal kimliği |
| `API_PORTAL_MANAGEMENT_API_BASE_URL` | Manager REST API base URL |
| `API_PORTAL_MANAGEMENT_API_KEY` | 1. adımdan gelen API key |

Opsiyonel: `SERVER_PORT=8080`, `SSL_ENABLED=false` (in-container TLS için `SSL_KEY_STORE*` alanlarını kullanın).

## 4. Installer + Şifreleme

```bash
sudo /opt/apinizer-portal/bin/apiportal-install.sh
sudo -u apinizer /opt/apinizer-portal/bin/apiportal-encrypt.sh
```

Encrypt scripti `API_PORTAL_MANAGEMENT_API_KEY`'i (ve yapılandırılmışsa diğer hassas alanları) `ENC(...)` ile şifreler.

## 5. Servisi başlat

```bash
sudo systemctl start apinizer-apiportal
sudo systemctl status apinizer-apiportal
sudo journalctl -u apinizer-apiportal -f
```

## 6. Doğrula

```bash
curl -fsS http://127.0.0.1:8080/management/health
```

Tarayıcıda Portal URL'ini açın. İlk sayfa katalogu Manager'dan çeker — yanlış URL/key 502 banner verir, kurulum çökmez.

## Çoklu replika

Portal stateless. Aynı host'ta farklı portlar veya farklı host'larda load balancer arkasında:

- Her replikaya **aynı** `API_PORTAL_ID`'yi verin.
- LB önünde sticky session gerekmez.

## Manuel kontrol

```bash
sudo -u apinizer /opt/apinizer-portal/bin/apiportal-start.sh           # background
sudo -u apinizer /opt/apinizer-portal/bin/apiportal-start.sh -f        # foreground
sudo -u apinizer /opt/apinizer-portal/bin/apiportal-stop.sh
```

## Sorun Giderme

### Servis döngüde restart, log'da `API_PORTAL_ID is not set`

Zorunlu env var boş — `API_PORTAL_ID`, `API_PORTAL_MANAGEMENT_API_BASE_URL`, `API_PORTAL_MANAGEMENT_API_KEY` üçünü de doldurun.

### Ana sayfa `502 Bad Gateway`

Portal Manager REST API'ye erişemiyor. Manager URL ve API key'i doğrulayın; servis host'undan Manager'a ağ erişimini test edin.
