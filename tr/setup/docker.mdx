Apinizer modülleri Docker konteynerleri olarak resmi `apinizercloud/*` imajları ile çalıştırılabilir. Her imaj Eclipse Temurin OpenJDK 25 ve gömülü Spring Boot uygulamasını taşır — host'ta Java yüklü olması gerekmez. Konteyner başlangıcında JVM otomatik olarak bellek limitini okur ve heap, direct memory ve metaspace değerlerini ayarlar.

<Info>
Docker akışında hassas değerler (MongoDB URI, API key'ler) plaintext environment variable'lar olarak gönderilir — konteyner, sanal sunucu paketindeki boot-time `ENC(...)` decrypt sürecini çalıştırmaz. Güvenli aktarım için Docker Secret (Compose / Swarm) veya Kubernetes Secret mekanizmalarını kullanın.
</Info>

## Kurulum Akışı

Apinizer Docker kurulumu aşağıdaki sırayı izler:

1. MongoDB bağlantı hazırlığı (önceden kurulu olmalı)
2. **API Manager kurulumu** (yönetim konsolu)
3. **Gateway kurulumu** (API Gateway runtime — zorunlu)
4. Manager UI'da Gateway'i Environment olarak tanımlama
5. Opsiyonel: Cache, Integration, API Portal kurulumları

## Genel Gereksinimler

Tüm Apinizer modüllerinin başarılı şekilde çalışması için gerekli ortam koşulları:

- **Docker Engine 24+** ve **docker-compose v2+** kurulu bir Linux x86-64 host (veya Docker Desktop)
- **CPU**: Modül başına en az 2 çekirdek; üretim için 4+ çekirdek önerilir
- **RAM**: Modül başına en az 2 GB; üretim için 4–8 GB önerilir
- **Disk**: Her modül imajı ~1 GB; log volume'ları için ek alan
- **Ağ**: MongoDB'ye ve modüller arası TCP bağlantı; istemcilerden portal erişimi (varsa)
- **MongoDB**: Önceden kurulu ve erişilebilir; üretim için replica set önerilir
- **Java**: Gerekmez — her imajda OpenJDK 25 gömülü

<Warning>
MongoDB kurulumu bu belgede kapsam dışıdır. Apinizer Docker kurulumuna başlamadan önce erişilebilir bir MongoDB instance'ı hazır olmalıdır. Detay için [MongoDB Kurulumu](/tr/setup/mongodb) sayfasına bakın.
</Warning>

<Note>
Karşılaşılabilecek Docker-spesifik sorunlar için [Sorun Giderme](/tr/setup/sorun-giderme) sayfasına bakın.
</Note>

## API Manager Kurulumu

Apinizer Yönetim Konsolu (**API Manager**), platform merkezidir: REST API, iş mantığı, Angular UI ve MongoDB üzerinde proxy/policy/environment kayıtlarını barındırır. Her Docker kuruluşunun **en az bir API Manager örneği** olmalıdır.

### 1. İmajı Çek

```bash
VERSION=2026.04.5
docker pull apinizercloud/apimanager:${VERSION}
```

Tag konvansiyonu: `2026.04.5` (major.minor.patch). Son sürüm için [Docker — Genel Bakış](/tr/setup/docker/genel-bakis) sayfasına bakın.

### 2. Yapılandır

Zorunlu environment variable'lar:

| Değişken | Açıklama |
|----------|----------|
| `SPRING_DATA_MONGODB_URI` | MongoDB connection string (örn. `mongodb://user:pass@host:25080/?replicaSet=rs0&authSource=admin`) |
| `SPRING_DATA_MONGODB_DATABASE` | Veritabanı adı (örn. `apinizer`) |
| `SPRING_PROFILES_ACTIVE` | `prod` (üretim profili) |

Opsiyonel:

| Değişken | Varsayılan | Açıklama |
|----------|-----------|---------|
| `SERVER_PORT` | `8080` | HTTP portu |
| `JAVA_OPTS` | boş | Ek JVM bayrakları |
| `jvmOverride` | `false` | `true` yapıldığında JAVA_OPTS tier varsayılanlarını ezer |
| `JHIPSTER_SLEEP` | `0` | Başlangıç gecikme (MongoDB aynı compose'da başlatılıyorsa kullanışlı) |

### 3. docker run (Tek Konteyner)

```bash
docker run -d \
  --name apinizer-apimanager \
  -p 8080:8080 \
  --memory=4g \
  -e SPRING_DATA_MONGODB_URI='mongodb://user:pass@mongo:25080/?authSource=admin' \
  -e SPRING_DATA_MONGODB_DATABASE='apinizer' \
  -e SPRING_PROFILES_ACTIVE=prod \
  -v apimanager-logs:/app/logs \
  --restart unless-stopped \
  apinizercloud/apimanager:${VERSION}
```

<Tip>
`--memory=4g` limitini her zaman açıkça belirtin. Vermezseniz JVM host'un tüm belleğini görür ve heap'i olması gerekenden büyütebilir.
</Tip>

### 4. docker-compose (Önerilen)

`docker-compose.yml` dosyası oluşturun:

```yaml
services:
  apimanager:
    image: apinizercloud/apimanager:2026.04.5
    container_name: apinizer-apimanager
    restart: unless-stopped
    mem_limit: 4g
    ports:
      - "8080:8080"
    environment:
      SPRING_PROFILES_ACTIVE: prod
      SPRING_DATA_MONGODB_DATABASE: apinizer
      JAVA_OPTS: "-XX:+ExitOnOutOfMemoryError"
    env_file:
      - ./secrets/apimanager.env       # SPRING_DATA_MONGODB_URI=… içerir
    volumes:
      - apimanager-logs:/app/logs
    healthcheck:
      test: ["CMD-SHELL", "wget -qO- http://127.0.0.1:8080/management/health | grep -q '\"status\":\"UP\"'"]
      interval: 30s
      timeout: 5s
      retries: 5
      start_period: 90s

volumes:
  apimanager-logs:
```

Başlatma:

```bash
docker compose up -d
docker compose logs -f apimanager
```

### 5. Doğrula

```bash
curl -fsS http://127.0.0.1:8080/management/health
```

Beklenen: `200 OK` ve `"status":"UP"` JSON. İlk başlangıçta Mongock migration'ları çalışır — veritabanı boyutuna göre 30–90 saniye sürebilir.

### 6. Operasyon

| Eylem | Komut |
|-------|-------|
| Durum | `docker ps --filter name=apimanager` |
| Log (follow) | `docker logs -f apinizer-apimanager` |
| Yeniden başlat | `docker restart apinizer-apimanager` |
| Durdur | `docker stop apinizer-apimanager` |
| Shell | `docker exec -it apinizer-apimanager sh` |

### 7. Kaynak Tavsiyesi

| Yük | `mem_limit` | `cpus` |
|-----|-----------|--------|
| Smoke test / geliştirme | 2 GB | 2 |
| Üretim (≤200 aktif proxy) | 4 GB | 4 |
| Büyük kurulum / çok admin | 6–8 GB | 4–8 |

<Note>
API Manager kurulumunda sorun varsa [Sorun Giderme](/tr/setup/sorun-giderme) sayfasında Docker sekmesine bakın.
</Note>

## Gateway Kurulumu

API Gateway (**Worker**), Apinizer'ın runtime modülüdür. Gelen HTTP/HTTPS/gRPC/WebSocket isteklerini sonlandırır, deploy edilmiş proxy policy pipeline'larını çalıştırır ve trafik log'u yazar. Her kuruluşun **en az bir Gateway** olmalıdır.

<Info>
Worker imajı **Ubuntu noble (glibc)** üzerine kuruludur, Alpine değildir — çünkü `brotli4j` native compression library'sinin musl build'i yok.
</Info>

### 1. İmajı Çek

```bash
docker pull apinizercloud/worker:${VERSION}
```

### 2. Yapılandır

Zorunlu:

| Değişken | Açıklama |
|----------|----------|
| `SPRING_DATA_MONGODB_URI` | MongoDB connection string |
| `SPRING_DATA_MONGODB_DATABASE` | MongoDB veritabanı adı |
| `APINIZER_ENVIRONMENT_NAME` | Environment adı — Manager UI'daki Environment ile **tam eşleşmeli** (case-sensitive) |
| `WORKER_TIMEZONE` | Timezone offset (örn. `+03:00`) |

Önerilen:

| Değişken | Varsayılan | Açıklama |
|----------|-----------|---------|
| `JAVA_OPTS` | boş | Üretim: `"-server -XX:MaxRAMPercentage=75.0 -Dhttp.maxConnections=4096 -Dlog4j.formatMsgNoLookups=true"` |

Opsiyonel (Undertow tuning):

| Değişken | Varsayılan | Açıklama |
|----------|-----------|---------|
| `tuneWorkerThreads` | `1024` | XNIO worker pool boyutu |
| `tuneWorkerMaxThreads` | `4096` | Worker pool hard cap |
| `tuneIoThreads` | `4` | I/O selector'lar (vCPU ile eşitleyin) |
| `tuneBufferSize` | `16384` | Read/write buffer (bayt) |
| `tuneDirectBuffers` | `true` | Off-heap buffer kullan |
| `tuneBacklog` | `10000` | TCP listen backlog |

### 3. docker run

```bash
docker run -d \
  --name apinizer-worker \
  -p 8091:8091 \
  -p 8443:8443 \
  --memory=8g \
  --cpus=4 \
  -e SPRING_DATA_MONGODB_URI='mongodb://user:pass@mongo:25080/?authSource=admin' \
  -e SPRING_DATA_MONGODB_DATABASE='apinizer' \
  -e APINIZER_ENVIRONMENT_NAME='prod-dc1' \
  -e WORKER_TIMEZONE='+03:00' \
  -e JAVA_OPTS='-server -XX:MaxRAMPercentage=75.0 -Dhttp.maxConnections=4096 -Dlog4j.formatMsgNoLookups=true' \
  -v worker-logs:/app/logs \
  --restart unless-stopped \
  --ulimit nofile=1048576:1048576 \
  apinizercloud/worker:${VERSION}
```

<Tip>
Worker yüksek throughput için `--ulimit nofile=1048576:1048576` kritiktir.
</Tip>

### 4. docker-compose

```yaml
services:
  worker:
    image: apinizercloud/worker:2026.04.5
    container_name: apinizer-worker
    restart: unless-stopped
    mem_limit: 8g
    cpus: 4
    ports:
      - "8091:8091"
      - "8443:8443"
    environment:
      SPRING_DATA_MONGODB_DATABASE: apinizer
      APINIZER_ENVIRONMENT_NAME: prod-dc1
      WORKER_TIMEZONE: "+03:00"
      JAVA_OPTS: "-server -XX:MaxRAMPercentage=75.0 -Dhttp.maxConnections=4096 -Dlog4j.formatMsgNoLookups=true"
      tuneWorkerThreads: "1024"
      tuneIoThreads: "4"
    env_file:
      - ./secrets/worker.env
    ulimits:
      nofile:
        soft: 1048576
        hard: 1048576
    volumes:
      - worker-logs:/app/logs
    healthcheck:
      test: ["CMD-SHELL", "wget -qO- http://127.0.0.1:8091/server-status | grep -q OK || exit 1"]
      interval: 30s
      timeout: 5s
      retries: 5
      start_period: 60s

volumes:
  worker-logs:
```

### 5. Doğrula

```bash
curl -fsS http://127.0.0.1:8091/server-status
```

Beklenen boot log'u (containerdan):

```
ApiGateway standalone=true (env: prod-dc1)
HTTP listener bound: 0.0.0.0:8091
HTTPS listener bound: 0.0.0.0:8443    (Environment HTTPS açıksa)
Loaded N proxies for environment prod-dc1
```

### 6. Operasyon

| Eylem | Komut |
|-------|-------|
| Durum | `docker ps --filter name=worker` |
| Log | `docker logs -f apinizer-worker` |
| Yeniden başlat | `docker restart apinizer-worker` |
| Durdur (drain) | `docker stop -t 60 apinizer-worker` |

<Warning>
Konteyneri durdururken cömert stop timeout kullanın (`docker stop -t 60`) — in-flight istekler drain olabilsin.
</Warning>

### 7. Kaynak Tavsiyesi

| Yük | `mem_limit` | `cpus` |
|-----|-----------|--------|
| Smoke / staging (&lt;200 rps) | 2 GB | 2 |
| Üretim (1–3 krps/Worker) | 4–6 GB | 4 |
| Yüksek throughput (5+ krps) | 8 GB | 4–8 |

<Note>
Gateway kurulumunda sorun varsa [Sorun Giderme](/tr/setup/sorun-giderme) sayfasına bakın.
</Note>

## Manager Üzerinden Gateway'i Environment Olarak Tanımlama

Gateway konteyneri başarıyla çalışır duruma geldikten sonra, Manager UI'da bu Gateway'i bir **Environment** (ortam) olarak tanımlamanız gerekir.

### Adımlar

1. Manager UI'na gidin: `http://<Manager_Host>:8080`
2. **Yönetici** → **Sunucu Yönetimi** → **Gateway Runtime'ları** → **Yeni** yolunu izleyin
3. Aşağıdaki forma bilgileri girin:

<Frame>
  <img
    src="/images/yonetici/envVm.png"
    alt="Standalone Gateway Environment tanımlama formu"
  />
</Frame>

### Form Alanları

| Alan | Değer |
|------|-------|
| **Platform** | Standalone (Docker / VM / Bare metal) seçin |
| **Ortam Tipi** | Production / Test / Development |
| **İletişim Protokolü Türü** | HTTP / HTTPS / gRPC / WebSocket |
| **Ortam Adı** | Gateway konteynerinin `APINIZER_ENVIRONMENT_NAME` ile **birebir aynı** (örn. `prod-dc1`) |
| **Anahtar (Key)** | API URL prefix (örn. `prod`, `stg`) |
| **Erişim Adresi** | Gateway'in dış erişim URL'i (örn. `http://gateway.example.com:8091`) |

### Gateway Node Listesi

Standalone seçildiğinde form üzerinde **Gateway Node Listesi** tablosu gösterilir. Her node için:

| Alan | Açıklama |
|------|----------|
| **Adı (Name)** | Tanımlayıcı isim (örn. `gateway-1`, `worker-prod-01`) |
| **Gateway Yönetim API URL** | `http://<HOST_IP>:8091` (Manager'dan erişilebilir olmalı) |
| **Cache Yönetim API URL** | `http://<CACHE_HOST>:8092` (Cache kullanıyorsanız; yoksa boş bırakın) |

<Warning>
Node URL'leri Manager pod'ından **erişilebilir** olmalıdır. Docker bridge network'ünde service adı ile referans verin veya host IP'yi açıkça yazın.
</Warning>

### Ortamı Yayınla

**Kaydet** butonuna basın. Apinizer **Docker deployment yapmaz** — Gateway zaten konteyner olarak çalışıyor. Manager sadece bağlantı metadatasını kaydeder ve Gateway health endpoint'ini test eder.

<Info>
Yayım sonrası API Proxy'leri bu Environment'a deploy edebilirsiniz.
</Info>

## All-in-One docker-compose Örneği

Tüm Apinizer modüllerini ve değerlendirme için bir MongoDB sidecar'ını tek bir `docker-compose.yml` ile aynı host'ta ayağa kaldırmak isterseniz aşağıdaki örneği kullanabilirsiniz. Hedef kullanım: değerlendirme, PoC, geliştirici makinesi, iç demo.

<Warning>
Bu compose olduğu gibi **üretim için uygun değildir**. Üretimde:
- MongoDB'yi ayrı bir replica set cluster'a ayırın
- Plaintext secret'ları Docker Secrets veya bir secret manager ile değiştirin
- TLS sonlandırmasını bir reverse proxy'de yapın
- Kaynakları trafiğinize göre boyutlandırın
</Warning>

### Dizin Yapısı

```
apinizer-stack/
├── docker-compose.yml
├── .env                      # Versiyon + MongoDB credential
└── secrets/
    ├── apimanager.env        # SPRING_DATA_MONGODB_URI=…
    ├── apiworker.env
    ├── apicache.env
    ├── apiintegration.env
    └── apiportal.env
```

### `.env` Dosyası

```dotenv
VERSION=2026.04.5
MONGO_ROOT_USER=apinizer
MONGO_ROOT_PASSWORD=changeme-please
TIMEZONE_OFFSET=+03:00
TIMEZONE_IANA=Europe/Istanbul
```

### `docker-compose.yml`

```yaml
name: apinizer

services:
  # MongoDB (değerlendirme için tek node; üretim replica set)
  mongo:
    image: mongo:7
    container_name: apinizer-mongo
    restart: unless-stopped
    command: ["mongod", "--port", "25080"]
    environment:
      MONGO_INITDB_ROOT_USERNAME: ${MONGO_ROOT_USER}
      MONGO_INITDB_ROOT_PASSWORD: ${MONGO_ROOT_PASSWORD}
    volumes:
      - mongo-data:/data/db
    ports:
      - "25080:25080"
    healthcheck:
      test: ["CMD", "mongosh", "--port", "25080", "--quiet", "--eval", "db.adminCommand('ping').ok"]
      interval: 10s
      timeout: 5s
      retries: 10
      start_period: 20s

  # API Manager
  apimanager:
    image: apinizercloud/apimanager:${VERSION}
    container_name: apinizer-apimanager
    restart: unless-stopped
    depends_on:
      mongo:
        condition: service_healthy
    mem_limit: 4g
    ports:
      - "8080:8080"
    environment:
      SPRING_PROFILES_ACTIVE: prod
      SPRING_DATA_MONGODB_DATABASE: apinizer
    env_file:
      - ./secrets/apimanager.env
    volumes:
      - apimanager-logs:/app/logs
    healthcheck:
      test: ["CMD-SHELL", "wget -qO- http://127.0.0.1:8080/management/health | grep -q '\"status\":\"UP\"'"]
      interval: 30s
      timeout: 5s
      retries: 5
      start_period: 90s

  # Cache (Hazelcast)
  cache:
    image: apinizercloud/cache:${VERSION}
    container_name: apinizer-cache
    restart: unless-stopped
    depends_on:
      mongo:
        condition: service_healthy
    mem_limit: 4g
    ports:
      - "5701:5701"
      - "8090:8090"
    environment:
      SPRING_DATA_MONGODB_DATABASE: apinizer
      CACHE_QUOTA_TIMEZONE: "${TIMEZONE_OFFSET}"
      HAZELCAST_CLUSTER_NAME: apnz-hz-cluster
    env_file:
      - ./secrets/apicache.env
    volumes:
      - cache-logs:/app/logs

  # Worker (Gateway)
  worker:
    image: apinizercloud/worker:${VERSION}
    container_name: apinizer-worker
    restart: unless-stopped
    depends_on:
      apimanager:
        condition: service_healthy
      cache:
        condition: service_started
    mem_limit: 8g
    cpus: 4
    ports:
      - "8091:8091"
      - "8443:8443"
    environment:
      SPRING_DATA_MONGODB_DATABASE: apinizer
      APINIZER_ENVIRONMENT_NAME: prod-dc1
      WORKER_TIMEZONE: "${TIMEZONE_OFFSET}"
      JAVA_OPTS: "-server -XX:MaxRAMPercentage=75.0 -Dhttp.maxConnections=4096 -Dlog4j.formatMsgNoLookups=true"
    env_file:
      - ./secrets/apiworker.env
    ulimits:
      nofile:
        soft: 1048576
        hard: 1048576
    volumes:
      - worker-logs:/app/logs

  # Integration (Quartz)
  integration:
    image: apinizercloud/integration:${VERSION}
    container_name: apinizer-integration
    restart: unless-stopped
    depends_on:
      apimanager:
        condition: service_healthy
    mem_limit: 2g
    stop_grace_period: 120s
    ports:
      - "8092:8092"
    environment:
      SPRING_DATA_MONGODB_DATABASE: apinizer
      INTEGRATION_TIMEZONE: "${TIMEZONE_IANA}"
      SPRING_PROFILES_ACTIVE: prod
    env_file:
      - ./secrets/apiintegration.env
    volumes:
      - integration-logs:/app/logs

  # API Portal
  apiportal:
    image: apinizercloud/apiportal:${VERSION}
    container_name: apinizer-apiportal
    restart: unless-stopped
    depends_on:
      apimanager:
        condition: service_healthy
    mem_limit: 2g
    ports:
      - "8081:8080"
    environment:
      API_PORTAL_ID: portal-prod-01
      API_PORTAL_MANAGEMENT_API_BASE_URL: http://apimanager:8080
      SPRING_PROFILES_ACTIVE: prod
    env_file:
      - ./secrets/apiportal.env
    volumes:
      - apiportal-logs:/app/logs

volumes:
  mongo-data:
  apimanager-logs:
  cache-logs:
  worker-logs:
  integration-logs:
  apiportal-logs:
```

### Başlatma Sırası

```bash
# 1. MongoDB'yi başlat ve healthy olmasını bekle
docker compose up -d mongo
docker compose logs -f mongo   # "ready to accept connections" bekleyin

# 2. Manager'ı başlat
docker compose up -d apimanager
docker compose logs -f apimanager   # "Started ApinizerManagerApp" bekleyin (~2-3 dakika)

# 3. Manager UI'da manuel adımlar:
#    a) http://localhost:8080 açın (varsayılan admin şifresi log'da görünür)
#    b) "Sunucu Yönetimi → Gateway Runtime'ları → Yeni" ile "prod-dc1" Environment oluşturun
#    c) "Portal → Portals → Create" ile Portal kaydedin
#    d) "Portal → API Keys → Generate" ile API key oluşturun
#    e) secrets/apiportal.env dosyasına API_PORTAL_MANAGEMENT_API_KEY=<kopyalanan-key> ekleyin

# 4. Geri kalanı başlat
docker compose up -d cache worker integration apiportal
```

<Tip>
Portal ve Worker, Manager'ın bootstrap edilmiş olmasına bağlı (Environment yaratıldı, Portal kaydedildi). Compose'un `depends_on`'u sadece healthcheck'i bekler — Manager UI'daki manuel adımlar 2. ve 4. adımlar arasında yapılmalıdır.
</Tip>

### Operasyon

```bash
docker compose ps              # Durum
docker compose logs -f apimanager
docker compose restart worker
docker compose down            # Durdur (volume'ları sakla)
docker compose down -v         # Durdur ve tüm volume'ları SİL
```

    ## Apinizer Cache Kurulumu

    API Cache (**Hazelcast** tabanlı), dağıtık bir data grid'tir: quota sayaçları, OIDC session'ları, circuit-breaker durumu ve client-flow banner'larını Worker'lar arasında paylaşır. Çoklu Worker veya quota/OIDC kullanan kurulumlar için opsiyoneldir.

    ### 1. İmajı Çek

    ```bash
    docker pull apinizercloud/cache:${VERSION}
    ```

    ### 2. Yapılandır

    Zorunlu:

    | Değişken | Açıklama |
    |----------|----------|
    | `SPRING_DATA_MONGODB_URI` | MongoDB connection string |
    | `SPRING_DATA_MONGODB_DATABASE` | MongoDB veritabanı (Manager/Worker ile aynı) |
    | `CACHE_QUOTA_TIMEZONE` | Quota window timezone (örn. `+03:00`). Worker ile eşleşmeli. |

    Opsiyonel:

    | Değişken | Varsayılan | Açıklama |
    |----------|-----------|---------|
    | `HAZELCAST_CLUSTER_NAME` | `apnz-hz-cluster` | Cluster adı — multi-tenant senaryolarda override edin |
    | `APINIZER_CACHE_CLUSTER_MEMBERS` | boş | Çoklu node için comma-separated peer listesi |

    ### 3. docker run (Tek Node)

    ```bash
    docker run -d \
      --name apinizer-cache \
      -p 5701:5701 \
      -p 8090:8090 \
      --memory=4g \
      -e SPRING_DATA_MONGODB_URI='mongodb://user:pass@mongo:25080/?authSource=admin' \
      -e SPRING_DATA_MONGODB_DATABASE='apinizer' \
      -e CACHE_QUOTA_TIMEZONE='+03:00' \
      -v cache-logs:/app/logs \
      --restart unless-stopped \
      apinizercloud/cache:${VERSION}
    ```

    Beklenen boot log'u:

    ```
    Cache running in STANDALONE mode (cluster: apnz-hz-cluster)
    Members {size:1, ver:1} [ Member [127.0.0.1]:5701 - ... ]
    ```

    ### 4. Çoklu Node Cluster

    Her node'da **aynı** peer listesi ve **aynı** cluster adı olmalı:

    ```yaml
    services:
      cache-1:
        image: apinizercloud/cache:2026.04.5
        networks:
          apinizer:
            ipv4_address: 172.30.0.11
        environment: &cache-env
          SPRING_DATA_MONGODB_DATABASE: apinizer
          CACHE_QUOTA_TIMEZONE: "+03:00"
          HAZELCAST_CLUSTER_NAME: apnz-hz-cluster
          APINIZER_CACHE_CLUSTER_MEMBERS: 172.30.0.11,172.30.0.12,172.30.0.13
        env_file: [./secrets/cache.env]
        mem_limit: 4g

      cache-2:
        image: apinizercloud/cache:2026.04.5
        networks:
          apinizer:
            ipv4_address: 172.30.0.12
        environment: *cache-env
        env_file: [./secrets/cache.env]
        mem_limit: 4g

      cache-3:
        image: apinizercloud/cache:2026.04.5
        networks:
          apinizer:
            ipv4_address: 172.30.0.13
        environment: *cache-env
        env_file: [./secrets/cache.env]
        mem_limit: 4g

    networks:
      apinizer:
        driver: bridge
        ipam:
          config:
            - subnet: 172.30.0.0/24
    ```

    <Tip>
    TCP/IP discovery simetriktir — her node'a aynı peer listesini dağıtın.
    </Tip>

    ### 5. Doğrula

    ```bash
    curl -fsS http://127.0.0.1:8090/management/health
    docker logs apinizer-cache | grep -E 'Members \{'
    ```

    | Eylem | Komut |
    |-------|-------|

    ## Apinizer Integration Kurulumu

    **(Opsiyonel)** Integration, Quartz tabanlı task-flow scheduler'ıdır. Manager'da tanımlanan zamanlanmış akışları çalıştırır. **Yalnızca** task-flow / scheduled job ihtiyacı olan kurulumlar için gereklidir.

    ### 1. İmajı Çek

    ```bash
    docker pull apinizercloud/integration:${VERSION}
    ```

    ### 2. Yapılandır

    Zorunlu:

    | Değişken | Açıklama |
    |----------|----------|
    | `SPRING_DATA_MONGODB_URI` | MongoDB connection string |
    | `SPRING_DATA_MONGODB_DATABASE` | MongoDB veritabanı (Manager ile aynı) |
    | `INTEGRATION_TIMEZONE` | IANA zone id (örn. `Europe/Istanbul`) |

    Opsiyonel:

    | Değişken | Varsayılan | Açıklama |
    |----------|-----------|---------|
    | `SERVER_PORT` | `8092` | HTTP portu |

    ### 3. docker run

    ```bash
    docker run -d \
      --name apinizer-integration \
      -p 8092:8092 \
      --memory=2g \
      --stop-timeout=120 \
      -e SPRING_DATA_MONGODB_URI='mongodb://user:pass@mongo:25080/?authSource=admin' \
      -e SPRING_DATA_MONGODB_DATABASE='apinizer' \
      -e INTEGRATION_TIMEZONE='Europe/Istanbul' \
      -v integration-logs:/app/logs \
      --restart unless-stopped \
      apinizercloud/integration:${VERSION}
    ```

    <Warning>
    `--stop-timeout=120` (compose'da `stop_grace_period: 120s`) kullanın — Quartz job'ları temiz drain yapabilir.
    </Warning>

    ### 4. docker-compose

    ```yaml
    services:
      integration:
        image: apinizercloud/integration:2026.04.5
        container_name: apinizer-integration
        restart: unless-stopped
        mem_limit: 2g
        stop_grace_period: 120s
        ports:
          - "8092:8092"
        environment:
          SPRING_DATA_MONGODB_DATABASE: apinizer
          INTEGRATION_TIMEZONE: Europe/Istanbul
          SPRING_PROFILES_ACTIVE: prod
        env_file:
          - ./secrets/integration.env
        volumes:
          - integration-logs:/app/logs
        healthcheck:
          test: ["CMD-SHELL", "wget -qO- http://127.0.0.1:8092/management/health | grep -q '\"status\":\"UP\"'"]
          interval: 30s
          timeout: 5s
          retries: 5
          start_period: 60s

    volumes:
      integration-logs:
    ```

    ### 5. Çoklu Node / HA

    Birden fazla Integration konteyneri aynı MongoDB veritabanını paylaşabilir — otomatik Quartz cluster oluştururlar. Zamana duyarlı flow'larınız varsa 2+ replika çalıştırın.

    ### 6. Doğrula

    ```bash
    curl -fsS http://127.0.0.1:8092/management/health
    ```

    ### 7. Operasyon

    | Eylem | Komut |
    |-------|-------|
    | Durum | `docker ps --filter name=integration` |
    | Log | `docker logs -f apinizer-integration` |
    | Yeniden başlat | `docker restart -t 120 apinizer-integration` |

    Integration kurulumunda sorun varsa [Sorun Giderme](/tr/setup/sorun-giderme) sayfasına bakın.

  <Tab title="SSL ile API Manager">
    ## API Manager'i SSL ile Başlatma

    API Manager'ın HTTPS ile çalışması için bir PKCS#12 keystore hazırlayın ve yapılandırma değişkenlerini ayarlayın.

    ### 1. Keystore Oluştur (veya mevcut sertifikayı dönüştür)

    JKS'den PKCS#12'ye:

    ```bash
    keytool -importkeystore \
      -srckeystore api-manager.jks \
      -destkeystore api-manager.p12 \
      -srcstoretype JKS \
      -deststoretype PKCS12 \
      -srcstorepass <eski-sifre> \
      -deststorepass <yeni-sifre> \
      -srcalias manager-cert \
      -destalias manager-cert
    ```

    ### 2. docker-compose'a Ekle

    ```yaml
    services:
      apimanager:
        image: apinizercloud/apimanager:2026.04.5
        # ... diğer ayarlar ...
        environment:
          SSL_ENABLED: "true"
          SSL_KEY_STORE: /run/keystore/api-manager.p12
          SSL_KEY_STORE_TYPE: PKCS12
          SSL_KEY_STORE_PASSWORD: <keystore-sifre>
          SSL_KEY_ALIAS: manager-cert
          SSL_PROTOCOLS: "TLSv1.2,TLSv1.3"
          SERVER_PORT: "8443"
        volumes:
          - ./secrets/api-manager.p12:/run/keystore/api-manager.p12:ro
        ports:
          - "8443:8443"
    ```

    ### 3. Doğrula

    ```bash
    curl -kfS https://127.0.0.1:8443/management/health
    ```

    <Tip>
    Üretimde reverse proxy (nginx, HAProxy, AWS ALB) kullanarak TLS terminasyonunu proxy katmanında yapmanız önerilir. Bu konteyner içi TLS, daha basit kurulumlar için kullanılabilir.
    </Tip>
