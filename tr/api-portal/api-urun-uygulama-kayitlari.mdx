<Info>
Uygulama kayıtları varsayılan olarak herkese açık (Public) olarak tanımlanır ve ekstra onay gerektirmez.

Uygulama kayıt isteklerini admin tarafından yönetmek için ilgili ayarı etkinleştirmek üzere [API Ürünü Erişim Ayarları](/tr/api-portal/onay-istekleri#api-urunu-erisim-ayarlari) bağlantısını ziyaret edebilirsiniz.

Portal üzerinden uygulama kaydının nasıl yapılacağına dair detaylı bilgi için [Portal Arayüzünde Kullanımı](/tr/api-portal/onay-istekleri#portal-arayuzunde-kullanimi) bağlantısını ziyaret edebilirsiniz.
</Info>

## Filtreleme Seçenekleri

Sayfada üç ana filtreleme kriteri bulunur:

| Filtreleme Kriteri | Açıklama | Detaylar |
|-------------------|----------|----------|
| **1. Durumlara Göre Ara (Search by Status)** | Kayıt taleplerini durumlarına göre filtrelemek için kullanılır. | - **Approved (Onaylandı):** Admin tarafından onaylanmış, aktif kayıtları gösterir.<br/>- **Waiting (Bekliyor):** Henüz admin onayı bekleyen kayıtları gösterir.<br/>- **Rejected (Reddedildi):** Admin tarafından reddedilmiş kayıtları gösterir. |
| **2. Hesap / Geliştiriciye Göre Ara (Search by Account/Developer)** | Belirli bir kullanıcıya ait kayıtları görüntülemek için kullanılır. | Kullanıcının **e-posta adresi** veya **ad bilgisi** üzerinden arama yapılabilir. Örneğin: `test.user@apinizer.com` veya Test User. |
| **3. API Ürüne Göre Ara (Search by API Product)** | Belirli bir API ürününe ait kayıtları filtrelemek için kullanılır. | Örneğin sadece **Calculator API Product**'ın uygulama kayıtlarını görüntülemek için kullanılır. |

## Kayıt Listesi

Sayfada her kayıt için aşağıdaki bilgiler görüntülenir:

| Alan | Açıklama / İşlem |
|------|------------------|
| **API Ürünü** *(API Product)* | Bu alan, bir API ürününe ait bilgileri gösterir. Kayıt yapılan API ürününü ifade eder.<br/><br/>Örneğin, "Calculator API Product", bir hesaplama API'sinin adıdır. |
| **Hesap/Geliştirici** *(Account/Developer)* | Bu alan, API ürünlerine başvuran kullanıcıların e-posta adresi ve adını gösterir. Kullanıcılar, portal'a üye olarak API'leri kullanabilmek için kayıt olurlar ve her başvuru, kullanıcının e-posta adresi ve adı ile ilişkilendirilir.<br/><br/>Örneğin, "test.user@apinizer.com (Test User)" gibi bir formatta, kullanıcıya ait e-posta adresi ve isim bilgisi görüntülenir. |
| **Kurum** *(Organization)* | Kullanıcıların bağlı olduğu organizasyonu belirtir. Portal'e üye olan her kullanıcı bir kuruma bağlıdır. Bu organizasyon, kullanıcının çalıştığı şirket veya grup olabilir.<br/><br/>Örneğin, "Organization-3" ve "Organization-1" gibi organizasyon adları burada görüntülenir. |
| **Application** *(Uygulama)* | API başvurusu sırasında kullanılan uygulama adıdır. Bu alan, API'yi kullanacak olan uygulamanın ismini belirtir.<br/><br/>Örneğin, "App-1" veya "test" gibi uygulama isimleri görülebilir. Kullanıcı, API ürününe başvururken hangi uygulama için kayıt yapıyorsa, o uygulama adı burada görüntülenir. |
| **Plan** *(Plan)* | API kullanımı için seçilen abonelik planının detayları burada yer alır. Plan, API kullanımına dair çeşitli limitler (örneğin, çağrı limitleri, fiyatlandırma) içerir.<br/><br/>Örneğin, "Throttling for 100 calls per min & $100,00 per month" gibi bir açıklama, planın içeriğini ve ücretlendirmesini gösterir. Bu, API'nin ne kadar kullanılabileceğini ve ne kadar ücretlendirileceğini belirtir. |
| **Durum** *(Status)* | Kullanıcının API başvurusu sırasında geçirdiği onay sürecinin durumunu gösterir. Bu, kullanıcı e-posta adresini doğruladıktan sonra aktif olur. Durum seçenekleri şunlardır:<br/><br/>- **Confirmation Needed (Hesap Onayı Gerekli):** Kullanıcı e-posta adresini doğruladıktan sonra bu durum aktif olur. Kullanıcı, hesabının onaylanması için ek bir adım atmalıdır.<br/>- **Approved (Onaylandı):** Kullanıcı, başvurduğu API ürününe kabul edilmiş ve onaylanmıştır.<br/>- **Rejected (Reddedildi):** Kullanıcı başvurusu reddedilmiş, API kullanımına izin verilmemiştir. |

**Durum** sütunundaki butonlar üzerinden:

* Uygulama kaydının **detay bilgilerini görüntüleyebilirsiniz**.
* **Bekliyor** (Waiting) durumundaki kayıtları **onaylayabilir** veya **reddedebilirsiniz**.
* **Onaylandı** (Approved) durumundaki kayıtları **iptal edebilirsiniz**.
* **Reddedildi** (Rejected) durumundaki kayıtları **yeniden değerlendirme için tekrar işleme alabilirsiniz**.
* Her durum değişikliğinde kullanıcıya **otomatik bilgilendirme e-postası** gönderilir.

<Info>
Otomatik bilgilendirme e-postalarını özelleştirmek için [E-posta Şablonları](/tr/api-portal/api-portal#e-posta-bildirimleri) linkini inceleyebilirsiniz.
</Info>
