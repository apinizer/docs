**Ne İşe Yarar?**

* Kurum bazında başarılı / hatalı / bloklanan istek sayılarını ve hata oranını gösterir
* Dönemde en sık görülen HTTP durum kodlarını (Baskın Hata Kodları) kurum bazında listeler
* Genel platform sağlığını ve hangi kurumların daha fazla hata/blok ile karşılaştığını tek ekrandan karşılaştırmanızı sağlar

**Kimler Kullanır?**

* **Portal Yöneticileri:** Genel hata/blok oranındaki değişimi izlemek için
* **Destek Ekipleri:** Şikâyet eden bir kurumun trafiğinde ne tür hatalar/bloklar yaşandığını teşhis etmek için
* **Ürün Sahipleri:** Bir API ürününe bağlı entegrasyon sorunlarının kurum bazında yaygınlığını görmek için

## Ekrana Erişim ve Yetkiler

* Admin panelinde sol menüde **Portal → Raporlar → Hata / Blok Analizi** yolunu izleyin.
* Sadece yetkili roller (Portal Manager, Admin, Portal Business) raporu görüntüleyebilir.

## Filtreleme

| Filtre | Açıklama |
|--------|----------|
| **API Ürünü** | Tek veya birden çok API ürünü seçerek raporu daraltabilirsiniz. Boş bırakılırsa tüm ürünler dahil edilir. |
| **Kurum** | Tek veya birden çok kurum seçerek raporu daraltabilirsiniz. Boş bırakılırsa tüm kurumlar dahil edilir. |
| **Hesap** | Belirli bir portal hesabına göre daraltma yapabilirsiniz. |
| **Tarih Aralığı** | Hazır kısayollar veya özel başlangıç/bitiş tarihi ile dönemi belirleyebilirsiniz. |

## Rapor Nasıl Okunur?

Üst kısımdaki özet kartlar, seçilen dönem için platform genelindeki toplamları gösterir:

| Özet Kartı | Açıklama |
|------------|----------|
| **Toplam İstek** | Seçilen dönemdeki toplam istek sayısı. |
| **Genel Başarı %** | Başarılı isteklerin toplam isteklere oranı. |
| **Hatalı** | Hata ile sonuçlanan toplam istek sayısı. |
| **Bloklanan** | Gateway politikaları (kota, hız sınırlama, yetkilendirme) tarafından reddedilen toplam istek sayısı. |

Alttaki tabloda kurum bazında kırılım yer alır:

| Sütun | Açıklama |
|-------|----------|
| **Kurum** | İstek trafiği gösterilen kurumun adı. |
| **Toplam** | Kuruma ait toplam istek sayısı. |
| **Başarılı** | Başarıyla sonuçlanan istek sayısı. |
| **Hatalı** | Hata ile sonuçlanan istek sayısı. |
| **Bloklanan** | Gateway politikaları tarafından reddedilen istek sayısı. |
| **Hata Oranı %** | Hatalı isteklerin toplam isteklere oranı. |
| **Baskın Hata Kodları** | Dönemde en sık görülen HTTP durum kodları; 2xx başarılı (yeşil), 3xx yönlendirme (gri), 4xx istemci hatası (turuncu), 5xx sunucu hatası (kırmızı) renkleriyle işaretlenir. |

**Kullanım Senaryoları:**

* Hata oranı ortalamanın belirgin biçimde üzerinde olan bir kurumu tespit ederek destek ekibine yönlendirir.
* Bloklanan istek sayısı yüksek bir kurumda kota veya hız sınırlama planının yetersiz kaldığını fark eder.
* Baskın hata kodlarına bakarak bir entegrasyon sorununun kimlik doğrulama mı (4xx) yoksa altyapı kaynaklı mı (5xx) olduğunu ayırt eder.
