Filtreleme ve Arama Özellikleri
1. Giriş Tarihi ve Saat Aralığını Belirleme
Kullanıcıların giriş veya işlem aktivitelerini belirli bir tarih aralığında filtrelemek için tarih aralığı belirleyebilirsiniz.
<Steps>
  <Step title="Başlangıç Tarihi">
    **From (Başlangıç Tarihi)** alanına filtrelemek istediğiniz başlangıç tarihini giriniz
  </Step>
  <Step title="Bitiş Tarihi">
    **To (Bitiş Tarihi)** alanına filtrelemek istediğiniz bitiş tarihini giriniz
  </Step>
  <Step title="Sonuçları Görüntüle">
    Sadece belirttiğiniz tarih aralığı içindeki kayıtlar görüntülenecektir
  </Step>
</Steps>
<Tip>
  Tarih aralığı seçerken performans için maksimum 90 günlük periyot önerilir.
</Tip>
2. E-posta Adresine Göre Arama
Belirli bir kullanıcının aktivitelerini incelemek için e-posta bazlı arama yapabilirsiniz.
<ParamField path="email" type="string">
  **Search by email** alanına kullanıcının e-posta adresini girerek arama yapabilirsiniz. Bu filtreleme ile sadece ilgili kullanıcıya ait kayıtlar listelenecektir.
</ParamField>
Örnek Kullanım:
ahmet.yilmaz@example.com
3. IP Adresine Göre Arama
Giriş yapılan IP adresine göre filtrelemek için IP bazlı arama özelliğini kullanabilirsiniz.
<ParamField path="ipAddress" type="string">
  **Search by IP address** alanına ilgili IP adresini giriniz. Bu sayede belirli bir IP'den yapılan aktiviteler görüntülenir.
</ParamField>
Örnek Kullanım:
192.168.1.100
<Warning>
  Güvenlik ihlallerini tespit etmek için aynı IP adresinden çok sayıda başarısız giriş denemesine dikkat edin.
</Warning>
Tablo Kolonları
Audit Logs tablosunda aşağıdaki bilgiler görüntülenir:
Status (Durum)
Tabloda yer alan Status sütunu, yapılan işlemin sonucunu belirtir:
<ResponseField name="SUCCESS" type="badge">
  **Başarılı:** İşlem başarıyla tamamlanmıştır
</ResponseField>
<ResponseField name="FAILURE" type="badge">
  **Başarısız:** İşlem başarısız olmuştur
Başarısız giriş denemeleri, yetki hataları veya diğer işlem hataları bu durumda görünür.
</ResponseField>
Message (Mesaj)
Başarısız işlemler ile ilgili hata veya uyarı mesajları bu sütunda yer alır.
Örnek Mesajlar:
<AccordionGroup>
  <Accordion title="Hesap Kilitlenmesi" icon="lock">
```
    Account locked due to multiple failed login attempts
```
    Çok sayıda başarısız giriş denemesi nedeniyle hesap kilitlenmiştir.
  </Accordion>
  <Accordion title="Hatalı Şifre" icon="key">
```
    Invalid password
```
    Girilen şifre hatalıdır.
  </Accordion>
  <Accordion title="Kullanıcı Bulunamadı" icon="user-slash">
```
    User not found
```
    Belirtilen e-posta adresine sahip kullanıcı bulunamadı.
  </Accordion>
  <Accordion title="Yetki Hatası" icon="shield-xmark">
```
    Insufficient permissions
```
    İşlem için yeterli yetki bulunmamaktadır.
  </Accordion>
</AccordionGroup>

<ParamField path="email" type="string">
  **Email:** İşlem yapılan kullanıcının e-posta adresi
</ParamField>

</ParamField>
Kullanım Senaryoları
Senaryo 1: Kullanıcı Aktivite Kontrolü
<Steps>
  <Step title="Tarih Aralığı Seçimi">
    Belirli bir kullanıcının giriş aktivitelerini kontrol etmek için:
    * **From:** `2025-12-16`
    * **To:** `2026-01-17`
  </Step>
  <Step title="Kullanıcı Filtreleme">
    **Search by email** alanına kullanıcının e-posta adresini girin
  </Step>
  <Step title="Sonuçları İncele">
    Başarılı ve başarısız girişleri "Status" kolonundan kontrol edin
  </Step>
</Steps>
Senaryo 2: Başarısız Giriş Analizi
<Steps>
  <Step title="Durum Filtresi">
    **Status** kolonundan sadece `FAILURE` durumundaki kayıtları filtreleyin
  </Step>
  <Step title="Hata Mesajlarını İncele">
    **Message** sütunundan başarısızlık nedenlerini analiz edin
  </Step>
  <Step title="Gerekli Aksiyonu Al">
    * Hesap kilitlenmişse kullanıcıya bilgi verin
    * Şüpheli aktivite varsa güvenlik önlemleri alın
  </Step>
</Steps>
Senaryo 3: Güvenlik İhlali Tespiti
<Steps>
  <Step title="IP Bazlı Arama">
    Şüpheli bir IP adresinden yapılan tüm girişleri görüntüleyin
  </Step>
  <Step title="Başarısız Deneme Sayısı">
    Aynı IP'den çok sayıda başarısız giriş denemesi olup olmadığını kontrol edin
  </Step>
  <Step title="Güvenlik Önlemleri">
    Gerekirse IP adresini engelleyin veya ek güvenlik katmanları ekleyin
  </Step>
</Steps>
<Warning>
  **Güvenlik Uyarısı:** Aynı IP adresinden kısa sürede çok sayıda başarısız giriş denemesi, potansiyel bir brute force saldırısı işareti olabilir.
</Warning>

Raporlama ve Analiz
Audit logs verilerini kullanarak aşağıdaki analizleri yapabilirsiniz:

Kullanım İstatistikleri: En aktif kullanıcılar ve giriş zamanları
Güvenlik Analizi: Başarısız giriş oranları ve trendler
Coğrafi Dağılım: Farklı IP adreslerinden gelen erişimler
Performans Metrikleri: Giriş başarı oranları ve sistem sağlığı

<Tip>
  Düzenli raporlar oluşturarak güvenlik ve kullanım trendlerini takip edebilirsiniz.
</Tip>
Sonuç
Bu doküman sayesinde API Developer Portal aktiviteleri etkili şekilde izlenebilir ve yönetilebilir. Audit logs özelliği ile:
✅ Kullanıcı aktivitelerini detaylı şekilde takip edebilirsiniz
✅ Güvenlik ihlallerini hızlıca tespit edebilirsiniz
✅ Sistem kullanımını analiz edebilirsiniz
✅ Compliance gereksinimlerini karşılayabilirsiniz
