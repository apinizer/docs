Varsayılan olarak, **API Ürün Uygulama Kaydı** ve **Hesap/Geliştirici** talepleri otomatik olarak onaylanır. Bu modüllerin manuel onay sürecine dahil edilmesi için **API Ürünü Erişim Ayarları** ve **Onay İstekleri** yapılandırmalarının gerçekleştirilmesi gerekmektedir.

## API Ürünü Erişim Ayarları

Kullanıcılar doğrudan API'ye erişim sağlamazlar; önce bir uygulama (Application) oluştururlar ve bu uygulama üzerinden API ürününe erişim talep ederler.

API ürünleri varsayılan olarak herkese açık (public) yapılandırılmıştır. Bu durumda, bir kullanıcı uygulaması üzerinden API'ye erişim talep ettiğinde, istek otomatik olarak onaylanır ve kullanıcı hemen erişim sağlar.

Ancak ürün sahibi, **Apinizer API Manager > Yönetim (Administration) > Portal > API Ürünleri (API Products)** sayfasında **yapılandır (configure)** ile her bir ürün için bulunan **"Access requests from applications" (Uygulamalardan gelen erişim istekleri)** ayarını değiştirebilmektedir. Bu ayar aşağıdaki resimde gösterilmiştir:

<Info>
Bu ayar ürün (Product) bazlı yapılandırılır ve her API ürünü bağımsız olarak yönetilebilir. Bir üründeki değişiklik diğer ürünleri etkilemez.
</Info>

<img

* **Seçenek işaretli olduğunda:** Uygulamalardan gelen erişim istekleri otomatik olarak onaylanır (varsayılan durum).
* **Seçenek işaretli olmadığında:** Kullanıcılar otomatik erişim sağlayamaz. Bu durumda kullanıcı, uygulaması üzerinden erişim isteğinde bulunur ve ürün sahibinin manuel onayını beklemesi gerekir.

Bu ayar sayesinde ürün sahibi, API ürününe kimlerin erişebileceğini kontrollü bir şekilde yönetebilir.

Yapılacak işlemler aşağıda sırasıyla açıklanmıştır.

<Steps>
  <Step title="API Portal Sayfasına Giriş ve Ürün Seçimi">
    API'yı kullanmak isteyen kullanıcı, **Portal > APIs** sayfasına giriş yapar. Bu sayfada kategorilere göre (Education, Store, Entertainment vb.) gruplanmış API ürünleri listelenir. Kullanıcı kullanmak istediği API ürününü seçer ve "Explore API" butonuna tıklar.
  </Step>
  <Step title="API Ürün Detay Sayfası ve Uygulama Oluşturma">
    API ürün detay sayfasında "Applications" sekmesine tıklar. Eğer kayıtlı bir uygulama yoksa, "Bu API ürünü için kayıtlı uygulamanız yok" mesajı görünür.

    "Create New App" butonuna tıklayarak açılan pop-up'ta:
    * Uygulama adını ve açıklamasını girer
    * "Save" butonuna tıklayarak uygulamayı oluşturur
  </Step>
  <Step title="Kayıt olma">
    "Kayıt için bir uygulama ve plan seçin" popup'ında:
    * Oluşturduğu uygulamayı seçer
    * Plan seçer (örn: Plus)
    * "Register" butonuna tıklar
  </Step>
  <Step title="Kayıt Durumunu Görüntüleme">
    "Applications" sekmesinde kayıt durumu görüntülenir:
    * **Onay bekliyor:** Ürün sahibinin onayı beklenir
    * **Onaylandı:** API kullanıma hazır
    * **Reddedildi:** Erişim talebi reddedildi
  </Step>
</Steps>

<Info>
Kullanıcı tarafından bir erişim isteği oluşturulduğunda, bu istek Apinizer arayüzü üzerinde **Apinizer API Manager > Yönetim (Administration) > Portal > Onay İstekleri (Approval Requests)** sayfasına iletilir. Bu sayfada ürün sahipleri, kendilerine gelen erişim isteklerini görüntüleyebilir, inceleyebilir ve uygun gördükleri takdirde onay veya ret işlemini gerçekleştirebilirler. Aşağıda bu sayfa detaylı açıklanmıştır.
</Info>

## Onay İstekleri API Ürün Uygulama Kaydı

API ürününe erişim için otomatik onay kapalı olduğunda, kullanıcıların kayıt talepleri; **Apinizer API Manager > Yönetim (Administration) > Portal > Onay İstekleri (Approval Requests)** Sayfasında listelenir.

<img
  src="/images/portal/onay_istekleri_liste.png"
  alt="API Ürün Uygulama Kayıtları Ekranı"
  width="900"
  className="mr-auto"
/>

| Alan | Açıklama |
|------|----------|
| **API Ürün**<br/>*(API Product)* | Hangi API ürününe erişim talep edildiği |
| **Hesap/Geliştirici**<br/>*(Account/Developer)* | Talebi yapan kullanıcının e-posta adresi ve adı |
| **Kurum**<br/>*(Organization)* | Kullanıcının bağlı olduğu kuruluş |
| **Uygulama**<br/>*(Application)* | Hangi uygulama ile kayıt yapıldığı |
| **Plan**<br/>*(Plan)* | Seçilen abonelik planı ve detayları |
| **Durum**<br/>*(Status)* | Talebin durumu (Waiting, Approved, Rejected) |

<img
  src="/images/portal/onay_istekleri_detay.png"
  alt="API Ürün Uygulama Kayıtları Ekranı"
  width="900"
  className="mr-auto"
/>

<Steps>
  <Step title="Onay Bekleyen Talebi Seçme">
    Status sütunundaki "Waiting" (Bekliyor) butonuna tıklanır
  </Step>
  <Step title="Talep Detaylarını Görüntüleme">
    "Confirm the operation" popup'ı açılır ve talep detayları görüntülenir:
    * Account/Developer Information: Kullanıcı bilgileri (email, ad, organizasyon, telefon)
    * API Product Information: API ürün adı ve başlığı
    * Plan Information: Uygulama adı ve seçilen plan ve detayları
  </Step>
  <Step title="Karar Verme">
    Admin, talebi değerlendirir ve karar verir:
    * **Approve:** Talebi onaylar, kullanıcı API'ye erişim sağlar.
    * **Reject:** Talebi reddeder, kullanıcı erişim sağlayamaz.
  </Step>
</Steps>

<Info>
Her iki durumda da kullanıcıya otomatik olarak bilgilendirme e-postası gönderilir.
</Info>

## Otomatik Onay Ayarı

API Portal varsayılan olarak otomatik onay olarak yapılandırılmıştır. Bu süreci yönetmek için aşağıdaki adımlar uygulanır.

Portal'e yeni üye olmak isteyen kullanıcıların onay sürecini **Apinizer API Manager > Yönetim (Administration) > Portal > Ayarlar (Settings) > Portal > Güvenlik (Security)** sekmesindeki **"Hesap/Geliştirici Otomatik Onayla (Auto Approve Account/Developers)"** ayarını kontrol eder.

Bu ayar açıksa, admin doğrulamasına gerek kalmadan mail doğrulaması ile direkt hesaba giriş yapabilir.

Bu ayar kapalıysa, önce mail adresini doğrular daha sonra kayıt talebi **Apinizer API Manager > Yönetim (Administration) > Onay İstekleri (Approval Requests) > Hesap/Geliştirici (Account/Developer)** ekranına gelir.

<img
  src="/images/portal/onay_istekleri_otomatik_onay.png"
  alt="API Ürün Uygulama Kayıtları Ekranı"
  width="900"
  className="mr-auto"
/>

| Alan | Açıklama / İşlem |
|------|------------------|
| **Hesapları/Geliştiricileri Otomatik Onaylayın**<br/>*(Auto Approve Account/Developers)*<br/>*Ayar Kapalı (Not Active):* | Kullanıcı kayıt formunu doldurur.<br/>Sistеm kullanıcıya doğrulama e-postası gönderir.<br/>Kullanıcı e-posta onayını tamamlar.<br/>Kayıt talebi **Apinizer API Manager > Yönetim (Administration) > Onay İstekleri (Approval Requests) > Hesap/Geliştirici (Account/Developer)** sekmesinde listelenir.<br/>Admin, talebi manuel olarak onaylar veya reddeder. |
| **Hesapları/Geliştiricileri Otomatik Onaylayın**<br/>*(Auto Approve Account/Developers)*<br/>*Ayar Açık (Active):* | Kullanıcı kayıt formunu doldurur.<br/>E-posta doğrulamasından sonra hesap otomatik olarak onaylanır.<br/>Kullanıcı hemen Portal'e giriş yapabilir.<br/>Admin onayına gerek kalmaz.<br/>**Apinizer API Manager > Yönetim (Administration) > Onay İstekleri (Approval Requests) > Hesap/Geliştirici (Account/Developer)** sekmesinde talep görünmez. Otomatik onaylanır. |

## Onay İstekleri Hesap/geliştirici

Otomatik onay kapalıyken, admin **Apinizer API Manager > Yönetim (Administration) > Onay İstekleri (Approval Requests) > Hesap/Geliştirici (Account/Developer)** sekmesinde bekleyen kayıt taleplerini görüntüler. Her talep için şu bilgiler listelenir:

<img
  src="/images/portal/onay_istekleri_hesap_gelistirici.png"
  alt="API Ürün Uygulama Kayıtları Ekranı"
  width="900"
  className="mr-auto"
/>

| Alan | Açıklama / İşlem |
|------|------------------|
| **Email**<br/>*(Email)* | Portal'e üye olan kullanıcının mail adresi görüntülenir. |
| **Adı Soyadı**<br/>*(Full Name)* | Portal'e üye olan kullanıcının adı soyadı görüntülenir. |
| **Kurum**<br/>*(Organization)* | Portal'e üye olan kullanıcının bağlı olduğu kurum görüntülenir. |
| **Telefon Numarası**<br/>*(Phone)* | Portal'e üye olan kullanıcının telefon numarası görüntülenir. |
| **Onay Durumu**<br/>*(status)* | Kullanıcı e-posta adresini doğruladıktan sonra "Hesap Onayı Gerekli" kısmı enable olur. Daha sonra "Onaylandı" veya "Reddedildi" durumlarından biri seçilir.<br/>*(Confirmation Needed, Approved, Rejected)* |

Admin, Kullanıcı mail adresini doğruladıktan sonra **Status** butonuna tıklayarak kullanıcı kaydını onaylar veya reddeder.

<Info>
Her iki durumda da kullanıcıya otomatik olarak bilgilendirme e-postası gönderilir.

Otomatik bilgilendirme e-postalarını özelleştirmek için [E-posta Şablonları](/tr/api-portal/api-portal#e-posta-bildirimleri) linkini inceleyebilirsiniz.
</Info>
