---

**Ne İşe Yarar?**

* Bir API ürününe kayıtlı olduğu halde seçilen eşikten uzun süredir istek göndermeyen abonelikleri listeler
* **Hiç istek atmamış** abonelikleri (onboarding tamamlanmamış) ile **sessizleşmiş** abonelikleri (daha önce kullanılmış ama sonradan durmuş) ayırt eder
* Onboarding sürecinde takılan veya kullanımı bırakan (churn) kurum/hesapları erken tespit etmenizi sağlar

**Kimler Kullanır?**

* **Portal Yöneticileri:** Onboarding tamamlanmayan abonelikleri tespit edip takip etmek için
* **Destek / Müşteri Başarı Ekipleri:** Sessizleşen kurum ve hesaplara proaktif olarak ulaşmak için
* **İş Geliştirme Ekipleri:** Churn riski taşıyan hesapları erken aşamada fark etmek için

## Ekrana Erişim ve Yetkiler

* Admin panelinde sol menüde **Portal → Raporlar → Aktif Olmayan Aboneler** yolunu izleyin.
* Sadece yetkili roller (Portal Manager, Admin, Portal Business) raporu görüntüleyebilir.

## Filtreleme

| Filtre | Açıklama |
|--------|----------|
| **API Ürünü** | Tek veya birden çok API ürünü seçerek raporu daraltabilirsiniz. Boş bırakılırsa tüm ürünler dahil edilir. |
| **Kurum** | Tek veya birden çok kurum seçerek raporu daraltabilirsiniz. Boş bırakılırsa tüm kurumlar dahil edilir. |
| **Hesap** | Belirli bir portal hesabına göre daraltma yapabilirsiniz. |
| **Eşik (gün)** | Sessizlik eşiğini **30**, **60** veya **90** gün olarak seçebilirsiniz (varsayılan: 30 gün). |

## Rapor Nasıl Okunur?

Üst kısımdaki özet kartlar şu bilgileri gösterir:

| Özet Kartı | Açıklama |
|------------|----------|
| **Toplam Sessiz Abonelik** | Seçilen eşiği aşan toplam abonelik sayısı. |
| **Sessizleşmiş** | Daha önce trafik üretmiş ancak son istekten bu yana eşik süresi aşılmış abonelik sayısı. |
| **Hiç İstek Atmamış** | Onaylandığından bu yana hiç trafik üretmemiş abonelik sayısı. |

Tablodaki sütunlar:

| Sütun | Açıklama |
|-------|----------|
| **API Ürünü / Kurum / Hesap / Uygulama** | Aboneliğin ait olduğu API ürünü, kurum, hesap ve uygulama bilgisi. |
| **Kayıt Tarihi** | Aboneliğin onaylandığı tarih. |
| **Son İstek** | Aboneliğin gönderdiği son isteğin tarihi; hiç istek gönderilmemişse boş bırakılır. |
| **Sessiz Gün** | Hiç istek atmamış aboneliklerde kayıt tarihinden, sessizleşmiş aboneliklerde son istekten bu yana geçen gün sayısı. |
| **Sinyal** | Aboneliğin **Sessizleşmiş** mi yoksa **Hiç istek atmamış** mı olduğunu gösterir. |

**Kullanım Senaryoları:**

* Hiç istek atmamış abonelikleri listeleyerek onboarding sürecinde destek gereken hesapları belirler.
* Sessizleşmiş abonelikleri düzenli olarak gözden geçirerek müşteri başarı ekibinin proaktif iletişimini tetikler.
* Eşik süresini kısaltıp uzatarak (30/60/90 gün) erken uyarı ile kesin churn sinyali arasında denge kurar.
