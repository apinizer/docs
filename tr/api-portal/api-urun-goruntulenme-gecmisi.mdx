## API Ürün Görüntülenme Geçmişi Nedir?

**API Ürün Görüntülenme Geçmişi**, API Portal üzerinde yayınlanan API ürünlerinin kullanıcılar tarafından görüntülenme sıklığını raporlayan bir analiz ekranıdır. Bu ekran, API dokümantasyonlarının erişim istatistiklerini ve kullanıcıların ürünlere olan ilgisini ölçmek amacıyla kullanılır.

### Bu Sayfanın Amacı

Bu sayfa sayesinde:

- **İlgi Düzeyi Analizi:** Hangi API ürünlerinin daha çok görüntülendiğini öğrenebilirsiniz
- **Kullanıcı Segmentasyonu:** Trafiğin anonim ziyaretçilerden mi yoksa kayıtlı kullanıcılardan mı geldiğini anlayabilirsiniz
- **Trend Analizi:** Zaman içindeki görüntülenme trendlerini takip edebilirsiniz
- **Dokümantasyon Optimizasyonu:** Hangi ürünlerin dokümantasyonuna daha fazla ihtiyaç olduğunu belirleyebilirsiniz
- **Tanıtım Etki Analizi:** Yapılan duyuruların veya yeni özelliklerin trafiğe etkisini ölçebilirsiniz

#### API Product Name (API Ürün Adı)

**Açıklama:**
Bu kolon, portal'da yayınlanan API ürününün tam adını gösterir. Her satır, belirli bir API ürününün görüntülenme kaydını temsil eder.

**Özellikler:**
- API ürün adı, Portal → API Products bölümünde tanımlanan adla aynıdır
- Aynı API ürünü için farklı kullanıcı tipleri (Public/Kayıtlı) ayrı satırlarda gösterilir
- API ürün adına tıklayarak ilgili API ürün detay sayfasına gidebilirsiniz

**Kullanım:**
- En popüler API ürünlerini belirlemek için View Count'a göre sıralayın
- Belirli bir API ürününün performansını takip etmek için filtreleyin

#### Account Name (Hesap Adı)

**Açıklama:**
Bu kolon, API ürün sayfasını görüntüleyen kullanıcının kimliğini gösterir. İki farklı değer alabilir:

**1. Public (Herkese Açık):**
- Giriş yapmamış, anonim ziyaretçileri temsil eder
- Portal'a kayıt olmamış veya giriş yapmamış kullanıcılar
- Bu ziyaretçiler API dokümantasyonunu görüntüleyebilir ancak API'yi kullanamazlar

**2. Email Adresi (Kayıtlı Kullanıcı):**
- Portal'a kayıt olmuş ve giriş yapmış kullanıcıları temsil eder
- Örnek: `developer@example.com`, `john.doe@company.com`
- Bu kullanıcılar API'yi kullanabilir ve uygulama kaydı yapabilirler

**Kullanım Senaryoları:**

<AccordionGroup>
  <Accordion title="Trafik Kaynağı Analizi">
    **Public vs. Kayıtlı Kullanıcı Karşılaştırması:**
    
    - **Public trafiği yüksekse:** Portal tanıtımı iyi çalışıyor, ancak kayıt oranı düşük olabilir
    - **Kayıtlı kullanıcı trafiği yüksekse:** Mevcut geliştiriciler aktif olarak API'leri inceliyor
    
    **Aksiyon:**
    - Public trafiği yüksekse, kayıt sürecini optimize edin
    - Kayıtlı kullanıcı trafiği yüksekse, API kullanım oranlarını analiz edin
  </Accordion>
  
  <Accordion title="Kullanıcı Segmentasyonu">
    **Farklı Kullanıcı Gruplarının İlgi Alanları:**
    
    - Hangi API ürünlerine anonim ziyaretçiler daha çok bakıyor?
    - Hangi API ürünlerine kayıtlı kullanıcılar daha çok bakıyor?
    
    **Aksiyon:**
    - Anonim ziyaretçilerin ilgilendiği API'ler için daha iyi tanıtım yapın
    - Kayıtlı kullanıcıların ilgilendiği API'ler için daha detaylı dokümantasyon hazırlayın
  </Accordion>
</AccordionGroup>

#### View Count (Görüntülenme Sayısı)

**Açıklama:**
Bu kolon, seçilen zaman dilimi içinde ilgili API ürün sayfasının kaç kez görüntülendiğini gösterir. Her sayfa görüntüleme bir kez sayılır.

**Özellikler:**
- Sayısal değer olarak gösterilir (örn: `150`, `1,234`, `5,678`)
- Aynı kullanıcının birden fazla görüntülemesi ayrı ayrı sayılır
- Zaman periyoduna göre toplam görüntülenme sayısını gösterir

**Kullanım:**
- View Count'a göre sıralama yaparak en popüler API ürünlerini belirleyin
- Trend analizi için farklı zaman periyotlarını karşılaştırın
- Büyüme oranlarını hesaplayın

<Info>
View Count, sayfa görüntülenme sayısını gösterir. Bu, API'nin gerçek kullanım sayısı (API çağrıları) değildir. Gerçek API trafiği için Analytics modülünü kullanın.
</Info>

## Kullanım Senaryoları ve Analiz Örnekleri

API Ürün Görüntülenme Geçmişi sayfasını kullanarak çeşitli analizler ve stratejik kararlar alabilirsiniz. Aşağıda, en yaygın kullanım senaryoları detaylı şekilde açıklanmıştır.

### Senaryo 1: İlgi Düzeyi Analizi ve Popülerlik Belirleme

Hangi API ürünlerinin daha çok ilgi gördüğünü belirlemek ve ürün portföyünü buna göre yönetmek için:

<Steps>
  <Step title="Zaman Periyodu Seçimi">
    Analiz etmek istediğiniz zaman dilimini seçin:
    - **Günlük:** Son 24 saat veya belirli bir gün
    - **Haftalık:** Son 7 gün
    - **Aylık:** Son 30 gün veya belirli bir ay
    
    <Tip>Uzun vadeli trendleri görmek için aylık periyodu kullanın.</Tip>
  </Step>
  <Step title="View Count'a Göre Sıralama">
    Tabloyu View Count kolonuna göre sıralayın (yüksekten düşüğe):
    - En yüksek görüntülenme sayısına sahip API ürünleri en üstte görünür
    - Bu, en popüler API ürünlerini gösterir
  </Step>
  <Step title="Popülerlik Analizi">
    Sonuçları analiz edin:
    
    **En Popüler API'ler:**
    - Hangi API ürünleri en çok görüntüleniyor?
    - Bu API'lerin dokümantasyonu yeterli mi?
    - Bu API'ler için ek kaynaklar gerekli mi?
    
    **Az Popüler API'ler:**
    - Hangi API ürünleri daha az görüntüleniyor?
    - Bu API'lerin tanıtımı yeterli mi?
    - Dokümantasyon iyileştirmesi gerekli mi?
  </Step>
  <Step title="Stratejik Kararlar">
    Analiz sonuçlarına göre aksiyon alın:
    
    **Popüler API'ler İçin:**
    - Daha detaylı dokümantasyon hazırlayın
    - Örnek kod ve tutorial'lar ekleyin
    - Destek kaynaklarını artırın
    
    **Az Popüler API'ler İçin:**
    - Tanıtım ve pazarlama çalışmaları yapın
    - Dokümantasyonu gözden geçirin ve iyileştirin
    - Kullanım örnekleri ve case study'ler ekleyin
  </Step>
</Steps>

### Senaryo 2: Kullanıcı Segmentasyonu ve Trafik Analizi

Portal trafiğinin kaynak dağılımını analiz etmek ve kullanıcı segmentasyonu yapmak için:

<Steps>
  <Step title="Account Name Filtreleme">
    Tabloyu Account Name kolonuna göre filtreleyin veya gruplayın:
    
    **Public (Anonim) Trafiği:**
    - Account Name = "Public" olan kayıtları inceleyin
    - Toplam Public görüntülenme sayısını hesaplayın
    
    **Kayıtlı Kullanıcı Trafiği:**
    - Account Name = Email adresi olan kayıtları inceleyin
    - Toplam kayıtlı kullanıcı görüntülenme sayısını hesaplayın
  </Step>
  <Step title="Trafik Oranı Hesaplama">
    Her iki trafik türünün oranını hesaplayın:
    
    **Örnek Hesaplama:**
    - Toplam görüntülenme: 10,000
    - Public görüntülenme: 7,000 (%70)
    - Kayıtlı kullanıcı görüntülenme: 3,000 (%30)
    
    **Yorumlama:**
    - Public trafiği yüksekse: Portal tanıtımı iyi, ancak kayıt oranı düşük olabilir
    - Kayıtlı kullanıcı trafiği yüksekse: Mevcut geliştiriciler aktif
  </Step>
  <Step title="Segment Bazlı Analiz">
    Her segment için ayrı analiz yapın:
    
    **Public Trafiği Analizi:**
    - Hangi API'ler anonim ziyaretçiler tarafından daha çok görüntüleniyor?
    - Bu API'ler için tanıtım materyalleri yeterli mi?
    - Kayıt sürecini optimize etmek gerekli mi?
    
    **Kayıtlı Kullanıcı Trafiği Analizi:**
    - Hangi API'ler mevcut geliştiriciler tarafından daha çok görüntüleniyor?
    - Bu API'ler için dokümantasyon yeterli mi?
    - API kullanım oranları ile görüntülenme oranları uyumlu mu?
  </Step>
  <Step title="Optimizasyon Stratejisi">
    Segment analizine göre optimizasyon yapın:
    
    **Public Trafiği İçin:**
    - Portal ana sayfasını optimize edin
    - API tanıtım sayfalarını iyileştirin
    - Kayıt sürecini basitleştirin
    - Kayıt teşvik kampanyaları düzenleyin
    
    **Kayıtlı Kullanıcı Trafiği İçin:**
    - Dokümantasyon kalitesini artırın
    - Örnek kod ve tutorial'lar ekleyin
    - API kullanım rehberleri hazırlayın
    - Destek kaynaklarını genişletin
  </Step>
</Steps>

### Senaryo 3: Dokümantasyon Optimizasyonu

Görüntülenme sayısı yüksek ancak kullanım oranı düşük ürünlerin dokümantasyon içeriklerini gözden geçirmek için:

<Steps>
  <Step title="Yüksek Görüntülenme, Düşük Kullanım Tespiti">
    View Count yüksek ancak API kullanımı düşük olan ürünleri belirleyin:
    
    **Kriterler:**
    - View Count: Yüksek (örn: 1,000+)
    - API Kullanımı: Düşük (Analytics'ten kontrol edin)
    - Uygulama Kayıtları: Az sayıda
    
    **Yorumlama:**
    - Kullanıcılar API'yi görüntülüyor ancak kullanmıyor
    - Dokümantasyon yetersiz veya kafa karıştırıcı olabilir
    - Örnek kod veya kullanım rehberi eksik olabilir
  </Step>
  <Step title="Dokümantasyon İncelemesi">
    İlgili API ürününün dokümantasyonunu inceleyin:
    
    **Kontrol Edilecekler:**
    - Dokümantasyon açık ve anlaşılır mı?
    - Örnek kod ve kullanım senaryoları var mı?
    - API endpoint'leri detaylı açıklanmış mı?
    - Hata yönetimi ve troubleshooting bilgisi var mı?
    - Quick start guide mevcut mu?
  </Step>
  <Step title="İyileştirme Planı">
    Dokümantasyon iyileştirmeleri için plan yapın:
    
    **Eklenebilecekler:**
    - Daha detaylı API açıklamaları
    - Adım adım kullanım rehberleri
    - Örnek kod ve snippet'ler
    - Common use case'ler
    - Troubleshooting bölümü
    - FAQ (Sık Sorulan Sorular)
  </Step>
  <Step title="İyileştirme Sonrası Takip">
    İyileştirmelerden sonra sonuçları takip edin:
    
    - Görüntülenme sayısı değişti mi?
    - API kullanım oranı arttı mı?
    - Uygulama kayıtları arttı mı?
    - Kullanıcı geri bildirimleri nasıl?
  </Step>
</Steps>

### Senaryo 4: Tanıtım Etki Analizi

Yapılan duyuru veya yayınlanan yeni özelliklerin portal trafiğine olan etkisini ölçmek için:

<Steps>
  <Step title="Öncesi ve Sonrası Karşılaştırması">
    Duyuru veya yeni özellik öncesi ve sonrası verilerini karşılaştırın:
    
    **Öncesi Dönem:**
    - Duyuru öncesi 7 günlük görüntülenme sayıları
    - Günlük ortalama görüntülenme
    
    **Sonrası Dönem:**
    - Duyuru sonrası 7 günlük görüntülenme sayıları
    - Günlük ortalama görüntülenme
    
    **Hesaplama:**
    - Artış oranı = ((Sonrası - Öncesi) / Öncesi) × 100
  </Step>
  <Step title="Etki Analizi">
    Duyurunun etkisini analiz edin:
    
    **Yüksek Etki İşaretleri:**
    - Görüntülenme sayısında belirgin artış
    - İlgili API ürünlerinde artış
    - Yeni kullanıcı kayıtlarında artış
    
    **Düşük Etki İşaretleri:**
    - Görüntülenme sayısında değişiklik yok
    - Kullanıcı ilgisinde artış yok
  </Step>
  <Step title="Kampanya Optimizasyonu">
    Analiz sonuçlarına göre kampanyayı optimize edin:
    
    **Yüksek Etki Varsa:**
    - Benzer kampanyalar düzenleyin
    - Başarılı kanalları daha fazla kullanın
    - İçeriği genişletin
    
    **Düşük Etki Varsa:**
    - Kampanya kanallarını değiştirin
    - Mesajı gözden geçirin
    - Hedef kitleyi yeniden değerlendirin
  </Step>
</Steps>

## Teknik Notlar ve Önemli Uyarılar

<Warning>
  **Görüntülenme (View) ve Kullanım (Runtime) Ayrımı**

  Bu rapordaki veriler, API ürününün dokümantasyon sayfasına yapılan ziyaretleri (**View Count**) ifade eder. Bu sayılar, API uç noktalarına yapılan teknik istekleri (**Request**) kapsamaz.

  **Fark:**
  - **View Count:** Dokümantasyon sayfasına yapılan ziyaret sayısı
  - **API Request Count:** API endpoint'lerine yapılan gerçek çağrı sayısı

  **Gerçek API Trafiği İçin:**
  * Gerçek API trafiği, hata oranları ve performans metrikleri için **Analytics** modülü kullanılmalıdır.
  * Anlık istek/yanıt detayları için **Traffic Logs** ekranı incelenmelidir.
  * API kullanım istatistikleri için **API Management → Analytics** bölümüne bakılmalıdır.
</Warning>

<Info>
  **Veri Güncelleme:**
  
  Görüntülenme verileri gerçek zamanlı olarak güncellenir. Ancak, toplam sayıların hesaplanması için kısa bir gecikme olabilir (genellikle birkaç dakika).
</Info>

<Tip>
  **En İyi Uygulamalar:**
  
  - Düzenli olarak (haftalık veya aylık) görüntülenme raporlarını inceleyin
  - Trend analizi için farklı zaman periyotlarını karşılaştırın
  - View Count ile API kullanım oranlarını birlikte analiz edin
  - Kullanıcı geri bildirimleri ile görüntülenme verilerini birleştirin
</Tip>
