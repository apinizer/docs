## Veri Analizi ve Filtreleme

Bu rapor, API ürünlerinin görünürlüğünü ve erişim detaylarını analiz etmenizi sağlar. Verileri doğru yorumlamak için aşağıdaki filtreleme seçenekleri ve sütun açıklamaları referans alınmalıdır.

### Zaman Periyotları

Raporlanan verilerin zaman aralığı, sol üstteki butonlar aracılığıyla belirlenir:

<CardGroup cols={3}>
  <Card title="Daily (Günlük)" icon="calendar-day">
    Günlük bazda görüntülenme sayılarını listeler. Anlık trafik değişimlerini izlemek için kullanılır.
  </Card>
  <Card title="Weekly (Haftalık)" icon="calendar-week">
    Son 7 günü kapsayan verileri gösterir. Haftalık kullanım trendlerini analiz etmek için uygundur.
  </Card>
  <Card title="Monthly (Aylık)" icon="calendar">
    Aylık görüntülenme toplamlarını sunar. Uzun vadeli kullanım ve ilgi değişimlerini gösterir.
  </Card>
</CardGroup>

### Tablo Alanları

Raporda yer alan veri alanlarının teknik açıklamaları şöyledir:

| Sütun | Açıklama | Kullanım Amacı |
| :--- | :--- | :--- |
| **API Product Name** | Görüntülenen API ürününün adıdır. | En yüksek erişim sayısına sahip ürünlerin tespiti. |
| **Account Name** | Sayfayı görüntüleyen kullanıcı kimliğidir. <br/> • `Public`: Anonim (giriş yapmamış) ziyaretçiler. <br/> • `Email`: Kayıtlı ve oturum açmış kullanıcılar. | Trafiğin anonim ziyaretçilerden mi yoksa mevcut geliştiricilerden mi geldiğinin ayrımı. |
| **View Count** | İlgili zaman dilimindeki toplam sayfa görüntülenme sayısı. | Ürün dokümantasyonuna gösterilen ilginin sayısal büyüklüğü. |

## Teknik Notlar

<Warning>
  **Görüntülenme (View) ve Kullanım (Runtime) Ayrımı**

  Bu rapordaki veriler, API ürününün dokümantasyon sayfasına yapılan ziyaretleri (**View Count**) ifade eder. Bu sayılar, API uç noktalarına yapılan teknik istekleri (**Request**) kapsamaz.

  * Gerçek API trafiği, hata oranları ve performans metrikleri için **Analytics** modülü kullanılmalıdır.
  * Anlık istek/yanıt detayları için **Traffic Logs** ekranı incelenmelidir.
</Warning>
