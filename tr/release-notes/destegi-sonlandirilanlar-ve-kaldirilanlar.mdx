## Sürüm 2025.11.0

<span style={{ display: 'inline-block', backgroundColor: '#e5e7eb', color: '#4b5563', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>KALDIRILAN ÖZELLIKLER</span>

**DİKKAT!** Management API'lerde Create/Update API Proxy API'lerinde relativePath alanı yerine clientRoute alanı getirildi. Proxy management API'leri CI/CD süreçlerinde kullananların bu alana dikkat etmesi gerekmektedir. [bkz.](https://apinizer.github.io/management-api-docs/02-api-reference/04-api-proxies/crud/create-api-proxy-from-url/)

## Sürüm 2025.04.0

<span style={{ display: 'inline-block', backgroundColor: '#e5e7eb', color: '#4b5563', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>KALDIRILAN ÖZELLIKLER</span>

**DİKKAT!** Uygulama üzerinden uygulamanın güncellenmesi özelliği kaldırıldı.

**DİKKAT!** Opsgenie ürününün Jira tarafından sonlandırılması sebebiyle bu ürün ile entegrasyon kaldırıldı.

## Versiyon 2025.01.4

<span style={{ display: 'inline-block', backgroundColor: '#e5e7eb', color: '#4b5563', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>KALDIRILACAK ÖZELLIKLER</span>

* Bir sonraki release versiyonda, Uygulama üzerinden uygulamanın güncellenmesi özelliği kaldırılacaktır.
* Bir sonraki release versiyonda, Opsgenie entegrasyonu kaldırılacaktır.
* Bir sonraki release versiyonda, Client Banner özelliği, Routing bölümünden kaldırılacaktır. Bu özellik artık doğrudan politika üzerinden yönetilecektir.

## Sürüm 2024.09.0

<span style={{ display: 'inline-block', backgroundColor: '#e5e7eb', color: '#4b5563', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>KALDIRILACAK ÖZELLIKLER</span>

**DİKKAT!** Log kayıtlarının veri sahibi tarafından kontrol edilerek silinmesi gerekliliği olduğundan, Monitör Ekranlarında, Uygulama Loglarında, API Integration, Raporlar sayfalarında log kayıtlarının retention süresi ile otomatik silinmesi kaldırıldı. Bu kayıtların veri tabanından silinebilmesi için purge scriptleri hazırlandı. [Bkz.](/tr/operations/genel-bakis#veritabani-buyume-yonetimi)

**ÖNEMLİ!** Elasticsearch indekslerini Apinizer ile ekran üzerinden otomatik yedeklemek için kullanılan snapshot politikası ekranları, snapshot politikasının kullanım gereksinimleri sebebiyle kaldırıldı. Bu işlemlerin manuel olarak yapılabilmesi için gerekli scriptler hazırlandı. [Bkz.](/tr/setup/elasticsearch/yedekleme-politikasi)

**DİKKAT!** Eski revizyon mantığı ile ilgili tüm özellikler kullanım zorluğu sebebiyle kaldırıldı. Artık tamamen Versiyon 2023.06.1'de uygulanmaya başlanan "Deployment History" uygulamasına geçildi. Apinizer Yönetim API'lerinde yer alan revizyon parametreleri de kaldırıldı. [Bkz.](/tr/develop/yukleme-ve-surum-yonetimi)

## Versiyon 2024.05.3

<span style={{ display: 'inline-block', backgroundColor: '#e5e7eb', color: '#4b5563', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>KALDIRILAN ÖZELLIKLER</span>

**DİKKAT** API Proxy Grup'ta çoklu API Proxy seçimine izin verilmesi sağlandı. Bu ayar ile eskiden mevcut olan API Proxy Gruptaki politikaların API Proxy'lere uygulanması özelliği kaldırıldı. [Bkz.](/tr/develop/yonlendirme/http-yonlendirme)

Bundan sonra API Proxy Grubundaki politikalar sadece API Proxy Gruba gelen isteklere uygulanacak, API Proxy'e doğrudan istek geldiyse ve bu API Proxy, API Proxy Gruba dahilse, API Proxy Gruptaki politikalar bu durumda işletilmeyecek.

## Versiyon 2024.01.1

<span style={{ display: 'inline-block', backgroundColor: '#e5e7eb', color: '#4b5563', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>KALDIRILAN ÖZELLIKLER</span>

## Versiyon 2023.03.1

<span style={{ display: 'inline-block', backgroundColor: '#e5e7eb', color: '#4b5563', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>KALDIRILAN ÖZELLIKLER</span>

* Swagger ve Open API tipinde API Proxy oluşturmak için Spec'den Yükle seçeneği kaldırıldı.
* JOSE Doğrulama politikası JWT 3.Taraf (3rd Party) politikasını içerdiği için JWT 3.Taraf politikası kaldırıldı.
* APNZ-3472 ile yapılan düzenlemeden dolayı API Proxy Grup bazında log ayarlarını konfigüre etme özelliği kaldırıldı.
* APNZ-3472 ile yapılan düzenlemeden dolayı Proje Log Ayarları ekranı kaldırıldı.
* APNZ-3472 ile yapılan düzenlemeden dolayı Genel Ayarlar bazında log ayarlarının önem derecesini ayarlama özelliği kaldırıldı.
