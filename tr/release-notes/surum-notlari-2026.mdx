## Sürüm 2026.04.0

<Note>
Bu sürüm, platformun tüm bileşenlerinin Java 25'e geçişini, kapsamlı güvenlik sıkılaştırmalarını ve çok sayıda yeni özelliği içeren ana sürümdür.
</Note>
<span style={{ display: 'inline-block', backgroundColor: '#fecaca', color: '#991b1b', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>KALDIRILAN ÖZELLİKLER</span>

- **Temizleme Görevleri aracılığıyla denetim logları ve ACL denetim loglarının otomatik silinmesi kaldırıldı.** Bu loglar uyumluluk açısından kritik öneme sahiptir ve otomatik olarak silinmemelidir. Manuel temizlik scriptleri veritabanı büyüme yönetimi dokümantasyonunda mevcuttur.
- **Genel Ayarlar'da denetim ve giriş logları için veritabanına yazımı devre dışı bırakma seçeneği kaldırıldı.** Bu loglar artık her zaman veritabanına kaydedilmektedir.

<span style={{ display: 'inline-block', backgroundColor: '#86efac', color: '#056d4f', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>ÖNE ÇIKAN YENİ ÖZELLİKLER</span>

- ### Java 25 Geçişi (APNZ-5897)
Apinizer platformunun tüm bileşenleri (API Manager, Gateway Worker, Cache Server, Integration, API Portal) Java 25'e yükseltildi. Sanal thread desteği, modern JDK özellikleri ve performans iyileştirmeleri sağlandı.

<Warning>

    **Script Politikası — javax → jakarta Otomatik Dönüşüm:** Bu sürümden itibaren, Groovy script politikalarında kullanılan Jakarta EE namespace'leri (örn. `javax.servlet.*`, `javax.persistence.*`, `javax.xml.bind.*`, `javax.mail.*` vb.) derleme öncesinde otomatik olarak `jakarta.*` karşılıklarına dönüştürülmektedir. Mevcut script'leriniz bu sürüme geçişte otomatik olarak migrate edilecektir. **JDK standart kütüphane paketleri** (`javax.crypto.*`, `javax.net.ssl.*`, `javax.xml.parsers.*`, `javax.xml.transform.*`, `javax.script.*`) bu dönüşümden **etkilenmez** ve olduğu gibi kullanılmaya devam eder.

</Warning>

- ### Güvenlik Sıkılaştırmaları (APNZ-5736, APNZ-5737, APNZ-5738)
Güvenlik taramasında tespit edilen açık içeren bağımlılıklar en güncel sürümlerine yükseltildi. Cross-Site Scripting (XSS), Insecure Direct Object References (IDOR) ve Stack Trace Leak güvenlik açıkları giderildi. innerHTML temizliği, endpoint bazlı proje yetki kontrolü ve hata mesajlarından teknik detay sızıntısı önlendi.

- ### Elasticsearch 7, 8 ve 9 Tam Desteği (APNZ-5642, APNZ-5728, APNZ-5898)
Elasticsearch istemci altyapısı yeniden yapılandırılarak ES 7.x, 8.x ve 9.x sürümleri tam desteklenir hale getirildi. ILM, index template, failover ve tüm analitik ekranları her üç sürümde de test edildi.

- ### Message Builder Politikası (APNZ-5783)
İstek ve yanıt mesajlarını dinamik olarak oluşturmayı sağlayan yeni bir politika eklendi. JEXL ifade desteği, JSONPath, context variable ve koşullu mantık ile esnek mesaj şablonları oluşturulabilir. [Bkz.](/tr/develop/politikalar/mesaj-olusturucu)

- ### Hata Mesajı Özelleştirme (APNZ-5798, APNZ-5940, APNZ-5888)
Politikalardaki hata mesajlarının dönüş formatları (JSON/XML) özelleştirilebilir hale getirildi. Her hata tipi için neden bu hatanın oluşabileceği bilgisi eklendi. Tanımsız veya beklenmeyen hatalar için de format seçimi sağlandı. Tekrarlayan ve yakın anlamlı hata mesajları birleştirilerek sadeleştirildi. [Bkz.](/tr/admin/system-settings/hata-mesajlari)

- ### API Promotion Modülü (APNZ-5851)
Farklı ortamlar arasında API tanımlarının taşınması ve yönetilmesini sağlayan API Promotion modülü eklendi. [Bkz.](/tr/admin/api-promotion/genel-bakis)

- ### Gelişmiş Loglama Özellikleri (APNZ-5942, APNZ-6033, APNZ-6020)
API Proxy bazlı log ayarları sekmesi eklendi. Log politikası oluşturuldu. Log ayarlarındaki metadata alanları yeniden tasarlandı ve açıklayıcı uyarı mesajları eklendi. [Bkz.](/tr/develop/api-proxy-konfigurasyonu/api-trafik-log-ayarlari)

- ### API Manager Uygulaması Yetki Sistemi 
Apinizer API Manager uygulamasındaki yetki sistemi yenilendi. Yetkilendirme artık **asset kategorisi + aksiyon** modeline dayalı olarak çalışmaktadır. Benzer işlevlere sahip bileşenler asset kategorileri altında gruplandırılmış (API Yönetimi, Bağlantılar, Kimlik ve Erişim Kontrolü vb.) ve her kategori için belirli aksiyonlar (Görüntüle, Yönet, Dağıt/Kaldır, Çalıştır, Dışa/İçe Aktar) tanımlanmıştır. Bu sayede daha esnek, detaylı ve güvenli bir yetki yönetimi sağlanmıştır. [Bkz.](/tr/admin/user-access-management/yetkilendirme-matrisi) 

- ### Cache Server'ın Gateway'den Bağımsız Namespace'te Tanımlanması
Apinizer uygulaması altyapısında gateway işlevini gören worker pod'ları ile cache işlemleri için kullanılan Hazelcast pod'larının aynı namespace içinde olması zorunluluğu kaldırıldı. Artık worker pod'ları ve cache pod'ları ayrı namespace'lerde yönetilebilir. Böylece daha esnek bir altyapı yapılandırması sağlanmıştır. [Bkz.](/tr/admin/server-management/dagitik-cache)

- ### Çoklu API Portal Desteği
Tek bir Apinizer kurulumu üzerinden birden fazla bağımsız API Portal yönetebilme desteği eklendi. Farklı iş birimleri veya müşteri grupları için özelleştirilmiş portal deneyimleri sunulabilmesi sağlandı. [Bkz.](/tr/api-portal/api-multi-portal-kullanimi)

- ### Gelişmiş Yönlendirme (Routing) Özellikleri
API Gateway yönlendirme yetenekleri; **Sticky Session (Oturum Sadakati)**, **Exponential Retry Delay** ve **Aktif Health Check** mekanizmaları ile zenginleştirildi. Bu sayede trafik yönetimi daha esnek, dayanıklı ve akıllı hale getirildi. [Bkz.](/tr/develop/yonlendirme/http-yonlendirme)

- ### APIOps Management API Geliştirmeleri
Apinizer'ın tüm yönetimsel işlevlerinin APIOps Management API'ler üzerinden yapılabilmesi vizyonu doğrultusunda; Credential Secrets yönetimi, Keys/Keystores işlemleri ve JWK yapılandırmaları için tam API desteği eklendi. Ayrıca seritifikadan, public key'den, private key'den ve keystore'dan JWK oluşturma ve güncelleme yeteneği eklendi. [Bkz.](/api-reference/credentials/secrets)

<span style={{ display: 'inline-block', backgroundColor: '#86efac', color: '#056d4f', padding: '0.25rem 0.75rem', borderRadius: '9999px', fontSize: '0.875rem', fontWeight: '500' }}>YENİ ÖZELLİKLER</span>

- ### Upstream Sticky Session (Oturum Sadakati)
İstemcilerin her zaman aynı backend adresine yönlendirilmesini sağlayan Sticky Session desteği eklendi. Cookie-based, IP Hash ve Hybrid yöntemleri ile session state yönetimi gerektiren backend yapılarının Gateway üzerinden verimli çalışması sağlandı. [Bkz.](/tr/develop/yonlendirme/kalici-oturum)

- ### Upstream Aktif Health Check Mekanizması
Backend servislerinin sağlık durumlarını periyodik olarak kontrol eden Aktif Health Check mekanizması eklendi. Hatalı çalışan backend adreslerinin trafikten otomatik olarak çıkarılması ve düzeldiğinde tekrar trafiğe alınması sağlandı. [Bkz.](/tr/develop/yonlendirme/tekrar-deneme-ve-yuk-devretme#active-health-check)

- ### Upstream Üstel Gecikmeli Yeniden Deneme (Exponential Retry Delay)
Hata durumlarında yapılan retry işlemleri arasına, her denemede artan süreler koymayı sağlayan "Exponential Backoff" özelliği eklendi. Bu sayede backend servislerinin yoğun yük altında toparlanması için gerekli süre tanınmış oldu. [Bkz.](/tr/develop/yonlendirme/tekrar-deneme-ve-yuk-devretme#retry-mekanizmasi)

- ### Environment Diagnostic Ekranı
API Gateway ortamlarının çalışma durumlarını, kaynak tüketimlerini ve sağlık metriklerini izlemeyi sağlayan Diagnostic ekranı eklendi. [Bkz.](/tr/admin/server-management/environment-diagnostics)

- **APIOps Management API ile sertifika, public key, private key veya keystore güncellendiğinde eğer ilişkili JWK varsa bu JWK'ların da güncellenebilmesi veya ilişkinin silinmesi özelliği eklendi.** [Bkz.](/api-reference/credentials/secrets)
