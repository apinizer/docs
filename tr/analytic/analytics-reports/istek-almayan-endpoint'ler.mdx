<Info>

</Info>

## Filtreler

Bu rapor, belirli kriterlere göre istek almayan veya az istek alan endpoint'leri tespit etmenize olanak sağlar. Aşağıdaki adımları takip ederek filtreleri yapılandırabilirsiniz:

<Steps>
  <Step title="Ortam Seçimi">
    Raporu çalıştırmak için öncelikle bir ortam seçmeniz gerekmektedir. Ortam seçimi zorunludur ve metrikler seçilen ortama göre filtrelenir.

    <Info>
    Bir API Proxy birden fazla ortama yüklenebildiği için metrikleri de yüklendiği ortama göre değerlendirmek gerekir.
    </Info>
  </Step>

  <Step title="Proje Seçimi (Yönetici)">
    Yönetici kullanıcılar için birden fazla proje seçebilme imkanı sunulmaktadır. Proje seçimi yapıldığında, seçilen projelerdeki API Proxy'ler otomatik olarak yüklenir.

    <Note>
    Bu filtre sadece yönetici kullanıcılar için görünür.
    </Note>
  </Step>

  <Step title="API Proxy Seçimi">
    Analiz edilmek istenen API Proxy'leri çoklu seçim yaparak belirleyebilirsiniz. Seçim yapılmadığında tüm API Proxy'ler analiz edilir.
  </Step>

  <Step title="Durum Filtresi">
    API Proxy'lerin durumlarına göre filtreleme yapabilirsiniz:

    - **Tümü**: Tüm durumlardaki API Proxy'ler
    - **Yüklü (Deployed)**: Aktif olarak çalışan API Proxy'ler
    - **Yüklenmemiş (Undeployed)**: Pasif durumdaki API Proxy'ler
  </Step>

  <Step title="Minimum Kullanım Sayısı Eşiği">
    Endpoint'lerin kullanım durumunu belirlemek için minimum kullanım sayısı eşiği tanımlayabilirsiniz. Belirlenen değerin altında istek alan endpoint'ler listelenir.

    Bu eşik sayesinde, tamamen kullanılmayan veya çok az kullanılan endpoint'leri tespit edebilirsiniz.
  </Step>

  <Step title="Maksimum Kullanım Sayısı Eşiği">
    Endpoint'lerin kullanım durumunu belirlemek için maksimum kullanım sayısı eşiği tanımlayabilirsiniz. Belirlenen değerin üzerinde istek alan endpoint'ler listelenir.

    Bu eşik sayesinde, belirli bir kullanım aralığındaki endpoint'leri tespit edebilirsiniz.
  </Step>

  <Step title="Tarih Aralığı Seçimi">
    Analiz edilmek istenen tarih aralığını seçebilirsiniz. Tarih aralığı seçimi yapıldığında, sadece bu aralıktaki istekler analiz edilir.
  </Step>
</Steps>

## Rapor Tablosu

Rapor sonuçları aşağıdaki kolonlarda gösterilir:

| Alan | Açıklama |
|------|----------|
| Proje (*Project*) (Yönetici) | Endpoint'in ait olduğu proje adı (sadece yönetici kullanıcılar için görünür) |
| API Proxy Adı (*API Proxy*) | Endpoint'in ait olduğu API Proxy'nin adı |
| Metot Adı (*Method/Endpoint*) | Endpoint'in metot bilgisi (HTTP metodu ve path bilgisi) |
| Başarılı (*Successful*) | Başarılı olarak tamamlanan istek sayısı |
| Engellenen (*Blocked*) | Güvenlik politikaları veya rate limiting nedeniyle engellenen istek sayısı |
| Hata (*Error*) | Hata ile sonuçlanan istek sayısı |
| Toplam (*Total*) | Toplam istek sayısı (başarılı + engellenen + hata) |

<Note>
Elasticsearch entegrasyonu aktif olmayan ortamlar için rapor çalıştırılamaz. Bu durumda ekranda uyarı mesajı gösterilir.
</Note>
