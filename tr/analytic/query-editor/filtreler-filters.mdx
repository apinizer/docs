## Konfigürasyon Alanları

Filtre konfigürasyonu için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan                                 | Açıklama                                                                                                   |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------- |
| **Ad** _(Name)_                      | Oluşturulan filtre için filtre adı bilgisidir.                                                             |
| **Açıklama** _(Description)_         | Oluşturulan filtre ile ilgili yönetimi kolaylaştırmak için açıklama yazılabilir.                           |
| **Proxyler** _(Proxies)_             | Filtrenin uygulanacağı Proxyler seçilir. Herhangi bir Proxy seçilmemiş ise filtre tüm Proxylere uygulanır. |
| **Tarih Aralığı** _(Date Range)_     | Filtrenin çalışacağı tarih aralığı seçilir.                                                                |
| **İşlem Sonuç Tipi** _(Result Type)_ | Log kayıtlarını filtrelemek için istek işleminin sonuç tip bilgileri seçilebilir.                          |
| **Sıralama** _(Order List)_          | Sorgu sonucunun sıralanması için log kaydından yer alan alanlar üzerinden sıralama koşulları eklenebilir.  |

## Filtre Kriterleri

Filtre tanımı aşağıdaki kriterleri içerebilir:

<CardGroup cols={2}>
  <Card title="Zaman Aralığı" icon="calendar">
    Filtrenin çalışacağı tarih aralığını belirler
  </Card>
  <Card title="API Proxy Filtreleme" icon="server">
    Belirli API Proxy'lerde arama yapmayı sağlar
  </Card>
  <Card title="Sonuç Tipi Filtreleme" icon="filter">
    İsteğin sonuç tipine göre kısıtlama (başarılı, başarısız, bloklanmış)
  </Card>
  <Card title="Sıralama" icon="sort">
    Sorgu sonucunun sıralanması için log kaydı alanları üzerinden sıralama koşulları
  </Card>
</CardGroup>

<Info>
  Herhangi bir Proxy seçilmemiş ise filtre tüm Proxylere uygulanır.
</Info>
