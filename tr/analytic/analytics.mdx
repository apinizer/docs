## Analytics Nedir?

Analytics modülü, Apinizer API Gateway üzerinde çalışan API Proxy'lerin performansını, kullanımını ve trafiğini analiz etmek için kapsamlı araçlar sunan bir modüldür. Bu modül sayesinde API çağrıları hakkında toplanan metrik verilerini görselleştirebilir, log verilerini analiz edebilir, özel raporlar ve panolar oluşturabilirsiniz.

### Temel İşlevler

Analytics modülü aşağıdaki temel işlevleri yerine getirir:

- **Performans İzleme**: API'lerin yükünü ve performansını izleme, durum tespiti yapma
- **Gelişmiş Filtreleme**: Log verilerine gelişmiş filtreler uygulama ve sorgulama
- **Görselleştirme**: Metrikleri grafikler ve panolar ile görselleştirme
- **Raporlama**: Özel raporlar oluşturma ve otomatik rapor üretme
- **Gerçek Zamanlı İzleme**: API trafiğini gerçek zamanlı olarak izleme
- **Tracing**: İstek akışını adım adım izleme ve debug yapma
- **Veri Yönetimi**: Log verilerini yönetme ve taşıma

### Kullanım Senaryoları

- API performans metriklerini izleme ve analiz etme
- Hata oranlarını ve hata tiplerini analiz etme
- Endpoint kullanım istatistiklerini görüntüleme
- İstemci bazlı trafik analizi yapma
- IP bazlı trafik ve güvenlik analizi
- Zaman içinde trend analizi yapma
- Özel raporlar oluşturma ve paylaşma
- Performans sorunlarını tespit etme ve debug yapma

## Modül Detayları

### Pano (Dashboard)

[Pano (Dashboard)](/tr/analytic/dashboard) modülü, API Proxy'lerinizin performansını ve kullanımını görselleştirmenizi sağlar. Özelleştirilebilir widget'lar, çeşitli metrik türleri ve esnek zaman aralığı seçimi sunar.

**Özellikler:**
- Özelleştirilebilir dashboard widget'ları
- Çeşitli performans metrikleri (trafik, yanıt süresi, hata oranı)
- Esnek zaman aralığı seçimi
- Karşılaştırmalı analiz
- Custom dashboard oluşturma

### API Trafik Log Ayarları

[API Trafik Log Ayarları](/tr/analytic/api-traffic-log-ayarlari) modülü, API Proxy trafik loglarını yapılandırmanızı sağlar. Proje ve ortam bazlı log ayarları yönetimi yapabilirsiniz.

**Özellikler:**
- Proje ve ortam bazlı log ayarları
- Mesaj bölgelerine göre loglama kontrolü (Header, Body, Parameter)
- Elasticsearch entegrasyonu
- Entegrasyon hedeflerine göre log gönderimi

### Gerçek Zamanlı İzleme

[Gerçek Zamanlı İzleme](/tr/analytic/api-traffic-real-time-monitoring) modülü, API trafiğini gerçek zamanlı olarak izlemenizi sağlar.

**Özellikler:**
- Canlı trafik metrikleri
- Gerçek zamanlı istek/yanıt izleme
- Anlık performans göstergeleri

### Sorgu Editörü

[Sorgu Editörü](/tr/analytic/query-editor/genel-bakis) modülü, log kayıtları üzerinde sorgu ve filtre tanımları yapmanızı sağlar.

**Alt Modüller:**
- [Sorgular (Queries)](/tr/analytic/query-editor/sorgular-queries): Sorgu oluşturma ve yönetimi
- [Filtreler (Filters)](/tr/analytic/query-editor/filtreler-filters): Filtre oluşturma ve yönetimi

**Kullanım Alanları:**
- Anomali Dedektörü
- Rapor Üretici
- Grafik Oluşturucu

### Raporlar

[Raporlar](/tr/analytic/analytics-reports/raporlar) modülü, hazır raporlar ve özel rapor üretme imkanı sunar.

**Hazır Raporlar:**
- API Trafiği (Zamana Göre)
- API Trafiği & Zaman Metrikleri
- Endpoint Trafiği & Zaman Metrikleri
- İstemci Trafiği & Zaman Metrikleri
- IP Trafiği & Zaman Metrikleri
- API Proxy Oluşturma Raporu
- API Endpoint Oluşturma Raporu
- [Rapor Üretici](/tr/analytic/analytics-reports/rapor-uretici): Özel rapor oluşturma

### Tracing

[Tracing](/tr/analytic/tracing) modülü, API Proxy'de gelen isteklerin nasıl işlendiğini adım adım izlemenizi sağlar.

**Özellikler:**
- Trace başlatma ve yönetimi
- Politika bazlı execution izleme
- İstek/yanıt inceleme
- Performans analizi
- Timeline ve Tree görünümleri

### Veri Yönetimi

[Gönderilmemiş API Trafik Loglarını Taşı](/tr/analytic/gonderilmemis-api-trafik-loglarini-tasi) modülü, gönderilmemiş log verilerini taşımanızı sağlar.

**Özellikler:**
- Gönderilmemiş log verilerini tespit etme
- Log verilerini Elasticsearch'e taşıma
- Veri kaybını önleme

## Veri Yapısı

API trafik log kayıtlarının veri yapısı hakkında detaylı bilgi için [API Trafiği Log Kaydı Veri Yapısı](/tr/analytic/api-trafigi-log-kaydi-veri-yapisi) sayfasına bakabilirsiniz.

## Önemli Notlar

<Warning>
- Apinizer Yönetim Konsolunda ilgili API Trafiğinin sorgulanması ve analitik olarak görselleştirilmesi ancak ilgili ortama Elasticsearch Konnektörü eklenmesi ile mümkün olmaktadır.
- Eğer birden fazla Elasticsearch Konnektörü tanımlanması durumunda Apinizer Yönetim Konsolunda **ilk** eklenen Elasticsearch Konnektöründeki veriler görüntülenebilir.
</Warning>

<Note>
- Proje menüsü altında yer alan bölümde sadece proje altında yer alan metrikler üzerinde raporlama yapılabilir.
- Admin menüsü altında yer alan bölümde tüm metrikler üzerinde raporlama yapılabilir.
- API Proxy Trafiği (Zamana Göre) ve API Proxy Oluşturma Raporu sayfaları sadece Admin'e özeldir.
</Note>
