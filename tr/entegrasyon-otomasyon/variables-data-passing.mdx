Variables & Data Passing, Task Flow'larda task'lar arasında veri paylaşımını sağlayan temel mekanizmadır. Bu özellik sayesinde bir task'ın ürettiği veriyi diğer task'lar kullanabilir ve karmaşık veri dönüşümleri yapabilirsiniz.

## Genel Bakış

Variables & Data Passing özelliği şunları sağlar:

* Task'lar arasında veri paylaşımı
* Veri dönüşümü ve mapping
* Dinamik değer üretimi
* Veri validasyonu
* Veri saklama ve yönetimi

## Variable Nedir?

Variable, Task Flow içinde saklanan ve task'lar arasında paylaşılan veri değerleridir. Variable'lar sayesinde:

* Bir task'ın çıktısını diğer task'larda kullanabilirsiniz
* Veriyi dönüştürebilir ve işleyebilirsiniz
* Dinamik değerler üretebilirsiniz

## Variable Tipleri

API Integrator şu variable tiplerini destekler:

### String

Metin verileri için kullanılır.

**Örnek:**
```
name = "John Doe"
email = "john@example.com"
```

### Number

Sayısal veriler için kullanılır.

**Örnek:**
```
count = 100
price = 99.99
```

### Boolean

Mantıksal değerler için kullanılır.

**Örnek:**
```
isActive = true
isDeleted = false
```

### Object

JSON objeleri için kullanılır.

**Örnek:**
```
user = {
  "id": 123,
  "name": "John",
  "email": "john@example.com"
}
```

### Array

Dizi verileri için kullanılır.

**Örnek:**
```
users = [
  {"id": 1, "name": "John"},
  {"id": 2, "name": "Jane"}
]
```

## Variable Tanımlama

Variable tanımlamak için:

### Global Variables

Task Flow seviyesinde tanımlanan variable'lar:

1. Task Flow tasarım ekranında **Variables** sekmesine gidin
2. **Add Variable** butonuna tıklayın
3. Variable bilgilerini girin

**Ayarlar:**
* **Name**: Variable adı (unique olmalı)
* **Type**: Variable tipi
* **Default Value**: Varsayılan değer
* **Description**: Açıklama

**Örnek:**
```
Name: userId
Type: Number
Default Value: 0
Description: Kullanıcı ID'si
```

### Local Variables

Task seviyesinde tanımlanan variable'lar:

1. Task'ı seçin
2. **Variables** sekmesine gidin
3. **Add Local Variable** butonuna tıklayın

Local variable'lar sadece ilgili task içinde geçerlidir.

## Variable Kullanımı

Variable'ları task'larda kullanmak için `${variableName}` syntax'ını kullanın.

### Basit Kullanım

```
${userId}
${userName}
${isActive}
```

### Nested Object Access

Object içindeki property'lere erişmek için:

```
${user.id}
${user.name}
${user.address.city}
```

### Array Access

Array elemanlarına erişmek için:

```
${users[0]}
${users[0].name}
${users.length}
```

### Expression Evaluation

Expression'ları değerlendirmek için:

```
${count + 1}
${price * quantity}
${firstName + " " + lastName}
```

## Data Passing

Task'lar arasında veri geçişi için:

### Output Mapping

Bir task'ın çıktısını variable'a atamak için:

1. Task'ı seçin
2. **Output** sekmesine gidin
3. Output mapping'i yapılandırın

**Örnek:**
```
Task Output: {"id": 123, "name": "John"}
Mapping:
  userId = ${output.id}
  userName = ${output.name}
```

### Input Mapping

Bir task'ın girdisini variable'lardan almak için:

1. Task'ı seçin
2. **Input** sekmesine gidin
3. Input mapping'i yapılandırın

**Örnek:**
```
Task Input:
{
  "userId": ${userId},
  "userName": ${userName}
}
```

### Data Flow

Veri akışı örneği:

```
Task 1 Output → Variable: userData
              → Task 2 Input: ${userData}
              → Task 2 Output → Variable: result
                                → Task 3 Input: ${result}
```

## Data Transformation

Veri dönüşümü için:

### Transform Task

Transform task kullanarak veri dönüşümü yapabilirsiniz:

1. **Transform Task** ekleyin
2. Input ve output formatlarını belirleyin
3. Mapping kurallarını tanımlayın

**Örnek:**
```
Input:
{
  "first_name": "John",
  "last_name": "Doe",
  "email_address": "john@example.com"
}

Mapping:
firstName = first_name
lastName = last_name
email = email_address

Output:
{
  "firstName": "John",
  "lastName": "Doe",
  "email": "john@example.com"
}
```

### Script Transformation

Script task kullanarak karmaşık dönüşümler yapabilirsiniz:

```javascript
// Input: userData
var transformed = {
    fullName: userData.firstName + " " + userData.lastName,
    email: userData.email.toLowerCase(),
    timestamp: new Date().toISOString()
};
return transformed;
```

## Variable Scoping

Variable'ların kapsamı (scope):

### Global Scope

Task Flow seviyesinde tanımlanan variable'lar tüm task'larda kullanılabilir.

### Local Scope

Task seviyesinde tanımlanan variable'lar sadece ilgili task içinde geçerlidir.

### Shadowing

Local variable, aynı isimdeki global variable'ı gölgeler (shadow).

**Örnek:**
```
Global: userName = "John"
Local (Task 1): userName = "Jane"
Task 1 içinde: ${userName} = "Jane"
Diğer task'larda: ${userName} = "John"
```

## Predefined Variables

Sistem tarafından sağlanan önceden tanımlı variable'lar:

### Execution Variables

* `${executionId}`: Execution ID'si
* `${startTime}`: Başlangıç zamanı
* `${currentTime}`: Mevcut zaman
* `${timestamp}`: Timestamp

### System Variables

* `${environment}`: Ortam (dev, test, prod)
* `${userId}`: Kullanıcı ID'si
* `${userName}`: Kullanıcı adı

### Utility Variables

* `${randomString}`: Rastgele string
* `${randomNumber}`: Rastgele sayı
* `${uuid}`: UUID

## Data Validation

Veri doğrulama için:

### Type Validation

Variable tipini doğrulamak için:

```
Type: Number
Value: "123" → Error: Type mismatch
Value: 123 → OK
```

### Value Validation

Değer doğrulaması için:

```
Min: 0
Max: 100
Value: 150 → Error: Value out of range
Value: 50 → OK
```

### Custom Validation

Özel doğrulama için script kullanın:

```javascript
if (value.length < 3) {
    throw new Error("Value must be at least 3 characters");
}
```

## Best Practices

### Variable Naming

* **Descriptive**: Açıklayıcı isimler kullanın
* **Consistent**: Tutarlı naming convention kullanın
* **Avoid Conflicts**: Çakışmalardan kaçının

**Örnek:**
```
Good: userId, userName, userEmail
Bad: var1, temp, data
```

### Data Passing

* **Minimize**: Gereksiz veri geçişinden kaçının
* **Validate**: Veriyi doğrulayın
* **Transform Early**: Dönüşümleri erken yapın

### Performance

* **Cache**: Sık kullanılan verileri cache'leyin
* **Lazy Loading**: Gerektiğinde yükleyin
* **Batch Processing**: Batch işleme yapın

## Örnek Senaryolar

### Senaryo 1: Basit Data Passing

```
Task 1: Get User
Output: {"id": 123, "name": "John"}
Variable: userData = Task 1 Output

Task 2: Send Email
Input: {"userId": ${userData.id}, "userName": ${userData.name}}
```

### Senaryo 2: Data Transformation

```
Task 1: Fetch Data
Output: {"first_name": "John", "last_name": "Doe"}

Transform Task:
Input: Task 1 Output
Mapping: firstName = first_name, lastName = last_name
Output: {"firstName": "John", "lastName": "Doe"}

Task 2: Process Data
Input: Transform Task Output
```

### Senaryo 3: Conditional Data Passing

```
Decision Task: Check Status
→ [status == "success"] → Variable: result = successData
→ [status == "error"] → Variable: result = errorData

Task: Process Result
Input: ${result}
```
