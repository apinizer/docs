* **Değişken Tanımlama**: Task Flow içinde değişkenler tanımlama

## Değişken Tipleri

### Input Variables (Girdi Değişkenleri)

Task Flow'a dışarıdan gelen veriler:

* **Tanım**: Task Flow başlatıldığında dışarıdan alınan veriler
* **Kaynak**: API isteği, Scheduled Job, Webhook vb.
* **Kullanım**: Task Flow içinde işlenecek veriler

**Örnek:**
```json
{
  "customerId": "12345",
  "orderId": "ORD-001",
  "amount": 1000.00
}
```

### Output Variables (Çıktı Değişkenleri)

Task Flow'dan dışarıya gönderilen veriler:

* **Tanım**: Task Flow tamamlandığında dönen veriler
* **Hedef**: API yanıtı, Sonraki Task Flow, Database vb.
* **Kullanım**: İşlem sonuçları

**Örnek:**
```json
{
  "status": "success",
  "message": "Order processed successfully",
  "transactionId": "TXN-12345"
}
```

### Local Variables (Yerel Değişkenler)

Task Flow içinde kullanılan geçici değişkenler:

* **Tanım**: Task Flow içinde hesaplanan veya dönüştürülen veriler
* **Kapsam**: Sadece Task Flow içinde geçerli
* **Kullanım**: Ara hesaplamalar, dönüştürülmüş veriler

**Örnek:**
```javascript
var totalAmount = orderAmount + taxAmount;
var formattedDate = formatDate(currentDate);
```

## Değişken Tanımlama

### Input Variable Tanımlama

Input variable tanımlamak için:

1. Task Flow özelliklerinde **Input Variables** sekmesine gidin
2. **Add Variable** butonuna tıklayın
3. Değişken bilgilerini girin:
   * **Name**: Değişken adı
   * **Type**: Değişken tipi (String, Number, Boolean, Object, Array)
   * **Required**: Zorunlu mu?
   * **Default Value**: Varsayılan değer
   * **Description**: Açıklama

### Output Variable Tanımlama

Output variable tanımlamak için:

1. Task Flow özelliklerinde **Output Variables** sekmesine gidin
2. **Add Variable** butonuna tıklayın
3. Değişken bilgilerini girin:
   * **Name**: Değişken adı
   * **Type**: Değişken tipi
   * **Source**: Değişkenin kaynağı (görev veya hesaplama)
   * **Description**: Açıklama

### Local Variable Tanımlama

Local variable tanımlamak için:

1. Script Connector veya Transformation görevinde değişken tanımlayın
2. Değişken adını belirleyin
3. Değişken değerini hesaplayın veya atayın

## Veri Aktarımı

### Görevler Arası Veri Aktarımı

Görevler arasında veri aktarımı:

* **Otomatik Aktarım**: Önceki görevin çıktısı otomatik olarak sonraki göreve aktarılır
* **Manuel Mapping**: Veri mapping ile belirli alanlar aktarılır
* **Transformation**: Veri dönüştürme ile aktarım

### Veri Mapping

Veri mapping, bir görevin çıktısını diğer görevin girdisine eşleştirme işlemidir:

**Örnek Mapping:**
```
Source Task Output:
{
  "customer_id": "12345",
  "customer_name": "John Doe",
  "order_total": 1000.00
}

Target Task Input:
{
  "id": ${customer_id},
  "name": ${customer_name},
  "amount": ${order_total}
}
```

### Field Mapping

Alan bazlı veri eşleştirme:

* **Direct Mapping**: Aynı isimli alanlar otomatik eşleşir
* **Custom Mapping**: Özel alan eşleştirmeleri
* **Transformation Mapping**: Dönüştürme ile eşleştirme

## Veri Dönüştürme

### Format Dönüştürme

Veri formatı dönüştürme:

* **JSON ↔ XML**: JSON ve XML arası dönüştürme
* **String ↔ Number**: String ve sayı arası dönüştürme
* **Date Format**: Tarih formatı dönüştürme
* **Encoding**: Karakter kodlama dönüştürme

### Veri Zenginleştirme

Veri zenginleştirme işlemleri:

* **Data Enrichment**: Harici kaynaklardan veri ekleme
* **Data Aggregation**: Veri toplama ve birleştirme
* **Data Validation**: Veri doğrulama ve temizleme

### Script ile Dönüştürme

Script Connector kullanarak özel dönüştürme:

```javascript
// JSON dönüştürme
var jsonData = JSON.parse(xmlData);

// Veri zenginleştirme
var enrichedData = {
  ...originalData,
  "timestamp": new Date().toISOString(),
  "processedBy": "TaskFlow"
};

// Veri filtreleme
var filteredData = dataArray.filter(item => item.status === "active");
```

## Veri Doğrulama

### Input Validation

Girdi doğrulama:

* **Required Fields**: Zorunlu alan kontrolü
* **Data Type Validation**: Veri tipi kontrolü
* **Range Validation**: Değer aralığı kontrolü
* **Pattern Validation**: Format kontrolü (regex)

**Örnek Validation:**
```javascript
if (!input.customerId || input.customerId.length < 5) {
  throw new Error("Invalid customer ID");
}

if (input.amount <= 0 || input.amount > 1000000) {
  throw new Error("Amount out of range");
}
```

### Output Validation

Çıktı doğrulama:

* **Schema Validation**: Çıktı şeması kontrolü
* **Data Completeness**: Veri bütünlüğü kontrolü
* **Format Validation**: Format kontrolü

## Değişken Kapsamı (Scope)

### Global Scope

Task Flow seviyesinde değişkenler:

* **Input Variables**: Tüm Task Flow'da erişilebilir
* **Output Variables**: Tüm Task Flow'da set edilebilir
* **Task Flow Variables**: Task Flow içinde tanımlanan değişkenler

### Local Scope

Görev seviyesinde değişkenler:

* **Task Variables**: Sadece ilgili görevde geçerli
* **Script Variables**: Script içinde tanımlanan değişkenler

## Veri Aktarımı Örnekleri

### Senaryo 1: Basit Veri Aktarımı

```
Input Variable: customerId
│
├─ Database Connector: Müşteri bilgilerini çek
│   Output: customerData
│
├─ Script Connector: Veriyi dönüştür
│   Input: customerData
│   Output: transformedData
│
└─ HTTP Connector: API'ye gönder
    Input: transformedData
    Output: apiResponse
```

### Senaryo 2: Karmaşık Veri Mapping

```
Input Variables:
- orderId
- items[]
- customerId

│
├─ Parallel Task:
│   ├─ Database Connector: Müşteri bilgileri
│   │   Input: customerId
│   │   Output: customerInfo
│   │
│   └─ Database Connector: Ürün bilgileri
│       Input: items[].productId
│       Output: productInfo[]
│
├─ Script Connector: Verileri birleştir
│   Input: customerInfo, productInfo[], items[]
│   Output: enrichedOrder
│
└─ HTTP Connector: Siparişi gönder
    Input: enrichedOrder
    Output: orderResponse
```

## En İyi Uygulamalar

### Değişken İsimlendirme

* **Açıklayıcı İsimler**: Değişkenlere açıklayıcı isimler verin
* **Naming Convention**: Tutarlı bir isimlendirme kuralı kullanın
* **Avoid Abbreviations**: Kısaltmalardan kaçının

### Veri Yönetimi

* **Minimize Data Transfer**: Gereksiz veri transferini önleyin
* **Data Filtering**: Sadece gerekli verileri aktarın
* **Data Caching**: Uygun yerlerde önbellekleme kullanın

### Güvenlik

* **Sensitive Data**: Hassas verileri değişkenlerde saklamayın
* **Input Validation**: Tüm girdileri doğrulayın
* **Output Sanitization**: Çıktıları temizleyin

### Performans

* **Large Data**: Büyük veriler için streaming kullanın
* **Lazy Loading**: Verileri gerektiğinde yükleyin
* **Data Compression**: Uygun yerlerde veri sıkıştırma

## Hata Yönetimi

### Değişken Hataları

Değişken hatalarını yönetme:

* **Undefined Variables**: Tanımsız değişken kontrolü
* **Type Mismatch**: Tip uyumsuzluğu kontrolü
* **Null/Undefined Checks**: Null/undefined kontrolü

### Veri Aktarımı Hataları

Veri aktarımı hatalarını yönetme:

* **Mapping Errors**: Mapping hatalarını yakalama
* **Transformation Errors**: Dönüştürme hatalarını yakalama
* **Validation Errors**: Doğrulama hatalarını yakalama
