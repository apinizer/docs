Monitoring ve debugging, güvenilir Task Flow'ları sürdürmek için gereklidir. Apinizer, çalıştırmayı izleme, log görüntüleme, değişken değerlerini takip etme ve sorunları debug etme için kapsamlı araçlar sağlar.

## Genel Bakış

Monitoring ve debugging araçları, Task Flow davranışını anlamanıza, sorunları tespit etmenize ve performansı optimize etmenize yardımcı olur. Bu araçlar, çalıştırma akışı, değişken durumları ve hata koşulları hakkında görünürlük sağlar.

## Çalıştırma İzleme

### Gerçek Zamanlı İzleme

Aktif Task Flow çalışmalarını izleyin:

**Aktif Çalıştırma Görünümü:**

* **Çalıştırma Durumu**: Mevcut çalıştırma durumu
* **Mevcut Task**: Şu anda hangi task çalışıyor
* **Geçen Süre**: Çalıştırma ne kadar süredir devam ediyor
* **İlerleme**: Görsel ilerleme göstergesi
* **Değişken Değerleri**: Mevcut değişken değerleri

**Erişim:**

1. **Taskflow Monitor**'a gidin
2. **Aktif Çalışmalar** bölümünü görüntüleyin
3. Detayları görüntülemek için bir çalışmaya tıklayın

### Çalıştırma Geçmişi

Geçmiş Task Flow çalışmalarını görüntüleyin:

**Geçmiş Görünümü:**

* **Çalıştırma Listesi**: Tüm geçmiş çalışmalar
* **Filtre Seçenekleri**: Durum, tarih, Task Flow'a göre filtreleme
* **Arama**: Çalıştırma ID'si veya Task Flow adına göre arama
* **Sıralama Seçenekleri**: Tarih, süre, duruma göre sıralama

**Çalıştırma Detayları:**

* **Çalıştırma Özeti**: Genel çalıştırma bilgisi
* **Task Zaman Çizelgesi**: Task çalıştırmanın sıralı görünümü
* **Değişken Durumları**: Her adımdaki değişken değerleri
* **Hata Bilgisi**: Çalıştırma başarısız olduysa hata detayları

## Çalıştırma Logları

### Log Seviyeleri

Çeşitli bilgi türleri için farklı log seviyeleri:

* **DEBUG**: Detaylı debugging bilgisi
* **INFO**: Genel bilgilendirme mesajları
* **WARN**: Uyarı mesajları
* **ERROR**: Hata mesajları
* **FATAL**: Kritik hatalar

### Log Bileşenleri

**Task Çalıştırma Logları:**

```
[2024-01-15 10:30:00] INFO  Task Flow başladı: ProcessOrder
[2024-01-15 10:30:01] INFO  Task başladı: DatabaseConnector
[2024-01-15 10:30:01] DEBUG Sorgu çalıştırıldı: SELECT * FROM orders WHERE id = '12345'
[2024-01-15 10:30:01] INFO  Task tamamlandı: DatabaseConnector (süre: 150ms)
[2024-01-15 10:30:02] INFO  Task başladı: HTTPConnector
[2024-01-15 10:30:02] DEBUG HTTP isteği: POST https://api.example.com/orders
[2024-01-15 10:30:03] INFO  Task tamamlandı: HTTPConnector (süre: 1000ms)
[2024-01-15 10:30:03] INFO  Task Flow tamamlandı: ProcessOrder (süre: 3000ms)
```

**Değişken Değişiklik Logları:**

```
[2024-01-15 10:30:01] DEBUG Değişken ayarlandı: orderId = "12345"
[2024-01-15 10:30:01] DEBUG Değişken ayarlandı: order = {"id": "12345", "status": "pending"}
[2024-01-15 10:30:02] DEBUG Değişken güncellendi: order.status = "processing"
[2024-01-15 10:30:03] DEBUG Değişken ayarlandı: result = {"status": "success"}
```

**Hata Logları:**

```
[2024-01-15 10:30:02] ERROR Task başarısız: HTTPConnector
[2024-01-15 10:30:02] ERROR Hata türü: ConnectionError
[2024-01-15 10:30:02] ERROR Hata mesajı: api.example.com'a bağlanılamadı
[2024-01-15 10:30:02] ERROR Stack trace: ...
[2024-01-15 10:30:02] ERROR Değişken bağlamı: {orderId: "12345", order: {...}}
```

### Log Filtreleme

Çeşitli kriterlere göre logları filtreleyin:

**Filtre Seçenekleri:**

* **Log Level**: Log seviyesine göre filtreleme
* **Task**: Belirli task'a göre filtreleme
* **Zaman Aralığı**: Zaman damgasına göre filtreleme
* **Arama**: Log içeriğinde arama
* **Sadece Hata**: Sadece hata loglarını göster

## Değişken Takibi

### Değişken Durum Takibi

Çalıştırma boyunca değişken değerlerini takip edin:

**Değişken Durumları:**

* **Başlangıç Değerleri**: Task Flow başlangıcındaki değişkenler
* **Her Task'tan Sonra**: Her task'tan sonraki değişken değerleri
* **Karar Noktalarında**: Decision node'lardaki değişken değerleri
* **Final Değerler**: Task Flow sonundaki değişkenler

**Değişken Durumlarını Görüntüleme:**

1. Çalıştırma detaylarını açın
2. **Değişken Durumları** sekmesine gidin
3. Task veya adımı seçin
4. O noktadaki değişken değerlerini görüntüleyin

### Değişken Geçmişi

Değişken değer değişikliklerini görüntüleyin:

**Değişken Geçmişi:**

```
orderId:
  - Başlangıç: undefined
  - Task 1'den sonra: "12345"
  - Task 2'den sonra: "12345" (değişmedi)
  - Task 3'ten sonra: "12345" (değişmedi)

order:
  - Başlangıç: undefined
  - Task 1'den sonra: {"id": "12345", "status": "pending"}
  - Task 2'den sonra: {"id": "12345", "status": "processing"}
  - Task 3'ten sonra: {"id": "12345", "status": "completed"}
```

### Değişken İnceleme

Değişken değerlerini detaylı olarak inceleyin:

**İnceleme Özellikleri:**

* **Nesneleri Genişlet**: İç içe nesne özelliklerini görüntüleyin
* **Dizileri Genişlet**: Dizi öğelerini görüntüleyin
* **Tip Bilgisi**: Değişken tiplerini görüntüleyin
* **Boyut Bilgisi**: Nesne/dizi boyutlarını görüntüleyin
* **Arama**: Değişken değerleri içinde arama

## Debug Modu

### Debug Modunu Etkinleştirme

Detaylı çalıştırma bilgileri için debug modunu etkinleştirin:

**Debug'ı Etkinleştir:**

1. Task Flow'u tasarımcıda açın
2. **Debug** butonuna tıklayın
3. Debug ayarlarını yapılandırın
4. Çalıştırmayı başlatın

**Debug Ayarları:**

* **Breakpoints**: Belirli task'larda breakpoint'ler ayarlayın
* **Step Mode**: Her seferinde bir adım çalıştırın
* **Variable Watch**: Belirli değişkenleri izleyin
* **Log Level**: Log seviyesini DEBUG'a ayarlayın

### Adım Adım Debugging

Task Flow'u adım adım çalıştırın:

**Adım Kontrolleri:**

* **Step Over**: Mevcut task'ı çalıştırın ve bir sonrakine geçin
* **Step Into**: Task detaylarına girin
* **Step Out**: Mevcut task'tan çıkın
* **Continue**: Normal çalıştırmaya devam edin
* **Stop**: Çalıştırmayı durdurun

**Breakpoints:**

* Belirli task'larda breakpoint'ler ayarlayın
* Çalıştırma breakpoint'lerde duraklar
* Breakpoint'te değişkenleri inceleyin
* Breakpoint'ten devam edin veya adım atın

### Değişken İzleme

Çalıştırma sırasında belirli değişkenleri izleyin:

**İzleme Değişkenleri:**

* İzleme listesine değişkenler ekleyin
* Değişken değerlerini gerçek zamanlı görüntüleyin
* Değişken değişikliklerini oluştukları anda görün
* Değişken değerlerine dayalı koşullu breakpoint'ler ayarlayın

## Performans İzleme

### Çalıştırma Metrikleri

Task Flow performansını izleyin:

**Temel Metrikler:**

* **Çalıştırma Süresi**: Toplam çalıştırma süresi
* **Task Süreleri**: Bireysel task çalıştırma süreleri
* **Throughput**: Zaman dilimi başına çalıştırma sayısı
* **Başarı Oranı**: Başarılı çalışmaların yüzdesi
* **Hata Oranı**: Başarısız çalışmaların yüzdesi

### Performans Analizi

Performans bottleneck'lerini analiz edin:

**Analiz Araçları:**

* **Yavaş Task'lar**: En uzun süren task'ları tespit edin
* **Kaynak Kullanımı**: CPU ve bellek kullanımını izleyin
* **Zaman Çizelgesi Görünümü**: Çalıştırmanın görsel zaman çizelgesi
* **Karşılaştırma**: Çalıştırma sürelerini karşılaştırın

**Performans Raporu:**

```
Task Flow: ProcessOrder
Toplam Çalıştırma Süresi: 3000ms

Task Dağılımı:
  - DatabaseConnector: 150ms (%5)
  - HTTPConnector: 1000ms (%33)
  - ScriptConnector: 500ms (%17)
  - EmailConnector: 1350ms (%45)

Bottleneck: EmailConnector (toplam sürenin %45'i)
```

## Hata Analizi

### Hata Takibi

Hataları takip edin ve analiz edin:

**Hata Bilgisi:**

* **Hata Sıklığı**: Hataların ne sıklıkla oluştuğu
* **Hata Türleri**: Hata türleri
* **Hata Konumları**: Hataların nerede oluştuğu
* **Hata Trendleri**: Zaman içinde hata trendleri

### Hata Detayları

Detaylı hata bilgilerini görüntüleyin:

**Hata Detayları:**

* **Hata Mesajı**: Detaylı hata mesajı
* **Hata Türü**: Hata türü
* **Hata Kodu**: Varsa hata kodu
* **Stack Trace**: Tam stack trace
* **Bağlam**: Hata anındaki çalıştırma bağlamı
* **Değişkenler**: Hata anındaki değişken değerleri

### Hata Desenleri

Hata desenlerini tespit edin:

**Desen Analizi:**

* **Yaygın Hatalar**: En sık görülen hatalar
* **Hata Kümeleri**: Birlikte oluşan hatalar
* **Zaman Desenleri**: Günün saatine göre hatalar
* **Task Desenleri**: Task türüne göre hatalar

## Debugging Teknikleri

### Sorunları İzole Etme

Sorun alanlarını daraltın:

**İzolasyon Adımları:**

1. **Başarısız Task'ı Tespit Edin**: Hangi task'ın başarısız olduğunu belirleyin
2. **Girdi Değişkenlerini Kontrol Edin**: Girdi değişken değerlerini doğrulayın
3. **Konnektör Yapılandırmasını Kontrol Edin**: Konnektör ayarlarını doğrulayın
4. **Dış Sistemleri Kontrol Edin**: Dış sistem kullanılabilirliğini doğrulayın
5. **Bireysel Test Edin**: Başarısız task'ı izole olarak test edin

### Test Senaryoları

Çeşitli senaryoları test edin:

**Test Durumları:**

* **Happy Path**: Normal çalıştırma akışı
* **Hata Senaryoları**: Çeşitli hata koşulları
* **Edge Cases**: Sınır koşulları
* **Load Testing**: Yüksek yük senaryoları
* **Eşzamanlı Çalıştırma**: Birden fazla eşzamanlı çalıştırma

### Log Analizi

Sorunları tespit etmek için logları analiz edin:

**Analiz Adımları:**

1. **İlgili Logları Filtreleyin**: Zaman, task veya hataya göre filtreleyin
2. **Desenleri Tespit Edin**: Loglarda desenler arayın
3. **Olayları İlişkilendirin**: İlgili olayları ilişkilendirin
4. **Çalıştırmayı İzleyin**: Çalıştırma akışını izleyin
5. **Kök Nedeni Belirleyin**: Kök nedeni belirleyin

## Sorun Giderme Rehberi

### Yaygın Sorunlar

**Sorun: Task Flow Zaman Aşımına Uğruyor**

* Task çalıştırma sürelerini kontrol edin
* Dış sistem yanıt sürelerini doğrulayın
* Uygunsa timeout ayarlarını artırın
* Yavaş task'ları optimize edin

**Sorun: Değişken Ayarlanmamış**

* Değişken adı yazımını kontrol edin
* Değişken kapsamını doğrulayın
* Task çıktı eşlemesini kontrol edin
* Değişkenin kullanılmadan önce ayarlandığını doğrulayın

**Sorun: Konnektör Başarısız**

* Bağlantı yapılandırmasını doğrulayın
* Dış sistem kullanılabilirliğini kontrol edin
* Kimlik bilgilerini doğrulayın
* Ağ bağlantısını kontrol edin

**Sorun: Beklenmeyen Davranış**

* Debug modunu etkinleştirin
* Her adımda değişken değerlerini kontrol edin
* Çalıştırma loglarını gözden geçirin
* Bireysel task'ları test edin
