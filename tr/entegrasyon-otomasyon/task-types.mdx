Task Flow Builder'da üç temel görev tipi bulunur: Sequential (Sıralı), Parallel (Paralel) ve Decision (Karar). Her görev tipinin kendine özgü kullanım senaryoları ve yapılandırma seçenekleri vardır.

## Genel Bakış

Task Flow Builder'da görev tipleri:

* **Sequential Task**: Görevlerin sırayla çalıştırıldığı görev tipi
* **Parallel Task**: Görevlerin paralel olarak çalıştırıldığı görev tipi
* **Decision Task**: Koşullu dallanma sağlayan görev tipi

## Sequential Task (Sıralı Görev)

### Tanım

Sequential Task, içindeki görevlerin belirli bir sırayla, birbirini takip ederek çalıştırıldığı görev tipidir. Bir görev tamamlandıktan sonra bir sonraki görev başlar.

### Kullanım Senaryoları

Sequential Task şu durumlarda kullanılır:

* **Bağımlı İşlemler**: Bir görev diğerinin çıktısına bağımlıysa
* **Sıralı Veri İşleme**: Verilerin belirli bir sırada işlenmesi gerekiyorsa
* **Adım Adım İşlemler**: İşlemlerin adım adım yapılması gerekiyorsa

### Örnek Senaryo

```
1. Database Connector: Müşteri bilgilerini çek
2. Script Connector: Veriyi dönüştür
3. HTTP Connector: Dönüştürülmüş veriyi API'ye gönder
4. Email Connector: İşlem sonucunu e-posta ile bildir
```

### Yapılandırma

Sequential Task yapılandırması:

* **Task Name**: Görev adı
* **Description**: Açıklama
* **Sub-tasks**: Alt görevler listesi
* **Error Handling**: Hata yönetimi stratejisi

### Avantajlar

* **Basitlik**: Anlaşılması ve yönetilmesi kolay
* **Bağımlılık Yönetimi**: Görevler arası bağımlılıklar net
* **Debug Kolaylığı**: Hata ayıklama kolay

### Dezavantajlar

* **Performans**: Tüm görevler sırayla çalıştığı için daha yavaş olabilir
* **Kaynak Kullanımı**: Kaynaklar tam kapasite kullanılmaz

## Parallel Task (Paralel Görev)

### Tanım

Parallel Task, içindeki görevlerin aynı anda, paralel olarak çalıştırıldığı görev tipidir. Tüm görevler başlar ve tamamlanması beklenir.

### Kullanım Senaryoları

Parallel Task şu durumlarda kullanılır:

* **Bağımsız İşlemler**: Görevler birbirinden bağımsızsa
* **Performans Optimizasyonu**: İşlem süresini kısaltmak için
* **Paralel API Çağrıları**: Birden fazla API'ye aynı anda istek gönderme

### Örnek Senaryo

```
Paralel olarak:
├─ HTTP Connector: Müşteri API'sine istek gönder
├─ HTTP Connector: Ürün API'sine istek gönder
└─ HTTP Connector: Sipariş API'sine istek gönder

Tüm istekler tamamlandıktan sonra:
└─ Script Connector: Sonuçları birleştir
```

### Yapılandırma

Parallel Task yapılandırması:

* **Task Name**: Görev adı
* **Description**: Açıklama
* **Sub-tasks**: Alt görevler listesi
* **Max Concurrent Tasks**: Maksimum eşzamanlı görev sayısı
* **Error Handling**: Hata yönetimi stratejisi
* **Wait for All**: Tüm görevlerin tamamlanmasını bekleme

### Avantajlar

* **Performans**: İşlem süresi önemli ölçüde kısalır
* **Kaynak Kullanımı**: Kaynaklar daha verimli kullanılır
* **Ölçeklenebilirlik**: Yüksek yük altında daha iyi performans

### Dezavantajlar

* **Karmaşıklık**: Yönetimi daha karmaşık olabilir
* **Kaynak Tüketimi**: Daha fazla kaynak gerektirebilir
* **Hata Yönetimi**: Paralel hataların yönetimi daha zor

### Best Practices

* **Bağımsızlık**: Görevlerin birbirinden bağımsız olduğundan emin olun
* **Kaynak Limitleri**: Maksimum eşzamanlı görev sayısını sınırlayın
* **Timeout**: Her görev için timeout belirleyin
* **Error Handling**: Hata durumlarını düzgün yönetin

## Decision Task (Karar Görevi)

### Tanım

Decision Task, belirli koşullara göre farklı akışların çalıştırılmasını sağlayan görev tipidir. IF-THEN-ELSE mantığıyla çalışır.

### Kullanım Senaryoları

Decision Task şu durumlarda kullanılır:

* **Koşullu İşlemler**: Farklı koşullara göre farklı işlemler yapılması gerekiyorsa
* **Veri Doğrulama**: Veri doğrulamasına göre farklı akışlar
* **Hata Yönetimi**: Hata durumlarına göre farklı işlemler

### Örnek Senaryo

```
Decision Task: Müşteri durumu kontrolü
│
├─ IF: Müşteri aktif
│   └─ Sequential Task:
│       ├─ HTTP Connector: Müşteri bilgilerini güncelle
│       └─ Email Connector: Güncelleme bildirimi gönder
│
└─ ELSE: Müşteri pasif
    └─ Sequential Task:
        ├─ Database Connector: Müşteriyi pasif olarak işaretle
        └─ Email Connector: Uyarı e-postası gönder
```

### Yapılandırma

Decision Task yapılandırması:

* **Task Name**: Görev adı
* **Description**: Açıklama
* **Condition**: Koşul ifadesi
* **True Branch**: Koşul doğruysa çalışacak görevler
* **False Branch**: Koşul yanlışsa çalışacak görevler
* **Multiple Conditions**: Çoklu koşul desteği (Switch-Case)

### Koşul İfadeleri

Decision Task'te kullanabileceğiniz koşul ifadeleri:

* **Değişken Karşılaştırmaları**: `variable == "value"`
* **Sayısal Karşılaştırmalar**: `count > 10`
* **Boolean Kontrolleri**: `isActive == true`
* **Null Kontrolleri**: `data != null`
* **String İşlemleri**: `message.contains("error")`
* **Karmaşık İfadeler**: `(a > 10) && (b < 20)`

### Avantajlar

* **Esneklik**: Farklı senaryolara göre akış yönetimi
* **Kod Azaltma**: Kod yazmadan koşullu mantık
* **Okunabilirlik**: Görsel olarak anlaşılır

### Dezavantajlar

* **Karmaşıklık**: Çok fazla dallanma karmaşık olabilir
* **Bakım**: Koşulların güncellenmesi gerekebilir

### Best Practices

* **Basit Koşullar**: Koşulları mümkün olduğunca basit tutun
* **Dokümantasyon**: Koşul mantığını dokümante edin
* **Test**: Tüm dalları test edin
* **Default Branch**: Her zaman bir varsayılan dal ekleyin

## Görev Tiplerini Birleştirme

### Karmaşık Senaryolar

Görev tiplerini birleştirerek karmaşık senaryolar oluşturabilirsiniz:

```
Sequential Task: Ana İşlem
│
├─ Parallel Task: Veri Toplama
│   ├─ HTTP Connector: Müşteri bilgileri
│   ├─ HTTP Connector: Ürün bilgileri
│   └─ Database Connector: Sipariş geçmişi
│
├─ Decision Task: Veri Kontrolü
│   ├─ IF: Tüm veriler mevcut
│   │   └─ Sequential Task: İşleme
│   └─ ELSE: Veri eksik
│       └─ Email Connector: Hata bildirimi
│
└─ Sequential Task: Sonuç İşleme
    ├─ Script Connector: Veri birleştirme
    └─ HTTP Connector: Sonucu gönderme
```

### Performans Optimizasyonu

Görev tiplerini doğru kullanarak performansı optimize edebilirsiniz:

* **Bağımsız görevleri paralel çalıştırın**
* **Bağımlı görevleri sıralı tutun**
* **Koşullu dallanmaları minimize edin**

## Hata Yönetimi

### Sequential Task Hata Yönetimi

Sequential Task'te hata oluştuğunda:

* **Stop on Error**: Hata durumunda durdurma
* **Continue on Error**: Hata durumunda devam etme
* **Retry**: Başarısız görevi tekrar deneme

### Parallel Task Hata Yönetimi

Parallel Task'te hata yönetimi:

* **Wait for All**: Tüm görevlerin tamamlanmasını bekleme
* **Fail Fast**: İlk hatada durdurma
* **Partial Success**: Kısmi başarıyı kabul etme

### Decision Task Hata Yönetimi

Decision Task'te hata yönetimi:

* **Condition Error**: Koşul değerlendirme hatası
* **Default Branch**: Hata durumunda varsayılan dal
* **Error Handling**: Hata yakalama stratejisi

### Yapılandırma

* **Açıklayıcı İsimler**: Görevlere açıklayıcı isimler verin
* **Dokümantasyon**: Görev mantığını dokümante edin
* **Versiyonlama**: Değişiklikleri versiyonlayın

### Test

* **Unit Test**: Her görev tipini ayrı ayrı test edin
* **Integration Test**: Görev tiplerini birlikte test edin
