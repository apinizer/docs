Task Flow'larda farklı görev türleri ve akış kontrol mekanizmaları kullanarak karmaşık entegrasyon senaryolarını gerçekleştirebilirsiniz. Her task türü belirli bir işlevi yerine getirir ve entegrasyon akışınızın davranışını kontrol eder.

## Genel Bakış

Task Flow'larda kullanabileceğiniz temel task türleri:

* **Sequential Tasks**: Sıralı çalıştırma
* **Parallel Tasks**: Paralel çalıştırma
* **Decision Tasks**: Koşullu akış kontrolü
* **Loop Tasks**: Döngüsel işlemler
* **Connector Tasks**: Dış sistemlere bağlantı
* **Script Tasks**: Özel mantık çalıştırma

## Sequential Tasks (Sıralı Task'lar)

Sequential task'lar varsayılan çalıştırma türüdür. Task'lar tanımlandıkları sırayla çalıştırılır.

### Özellikler

* **Sıralı Çalıştırma**: Task'lar birbiri ardına çalıştırılır
* **Bağımlılık**: Bir task tamamlanmadan diğeri başlamaz
* **Veri Geçişi**: Her task'ın çıktısı bir sonraki task'ın girdisi olur
* **Hata Yönetimi**: Bir task başarısız olursa akış durur (yapılandırmaya göre)

### Kullanım Senaryosu

```
1. Database'den veri çek (Database Connector)
2. Veriyi dönüştür (Script Task)
3. HTTP API'ye gönder (HTTP Connector)
4. Sonucu logla (Logback Connector)
```

### Yapılandırma

Sequential task'lar için özel yapılandırma gerekmez. Task'ları sırayla bağladığınızda otomatik olarak sequential çalışır.

## Parallel Tasks (Paralel Task'lar)

Parallel task'lar, birden fazla task'ı aynı anda çalıştırmanızı sağlar. Bu, performansı artırır ve toplam çalışma süresini azaltır.

### Özellikler

* **Eşzamanlı Çalıştırma**: Birden fazla task aynı anda çalışır
* **Bağımsızlık**: Task'lar birbirinden bağımsızdır
* **Birleşme Noktası**: Tüm task'lar tamamlandığında akış devam eder
* **Hata Yönetimi**: Bir task başarısız olsa bile diğerleri çalışmaya devam eder

### Kullanım Senaryosu

```
Parallel Start
├─→ Email gönder (Email Connector)
├─→ SMS gönder (HTTP Connector)
└─→ Bildirim gönder (Notification Connector)
Parallel End → Sonuçları birleştir
```

### Yapılandırma

Paralel çalıştırma için:

1. **Parallel** kontrol elemanını ekleyin
2. Paralel çalıştırılacak task'ları Parallel bloğuna bağlayın
3. **Merge** noktasını ekleyin
4. Tüm task'lar tamamlandığında akış devam eder

### Avantajlar

* **Performans**: Toplam çalışma süresi en uzun task'ın süresi kadardır
* **Ölçeklenebilirlik**: Birden fazla kaynağı aynı anda kullanır
* **Esneklik**: Farklı sistemlere paralel erişim

### Dikkat Edilmesi Gerekenler

* **Kaynak Kullanımı**: Paralel task'lar daha fazla kaynak kullanır
* **Bağımlılık**: Task'lar arasında veri bağımlılığı olmamalıdır
* **Hata Yönetimi**: Paralel task'larda hata yönetimi daha karmaşıktır

## Decision Tasks (Karar Task'ları)

Decision task'lar, koşullu akış kontrolü sağlar. Belirli koşullara göre farklı dallara yönlendirme yapılır.

### Özellikler

* **Koşullu Akış**: Koşullara göre farklı dallara yönlendirme
* **Boolean Logic**: True/False mantığı
* **Complex Conditions**: Karmaşık koşullar tanımlama
* **Multiple Branches**: Birden fazla dal desteği

### Kullanım Senaryosu

```
Veri kontrolü (Script Task)
    ↓
Decision: Veri geçerli mi?
    ├─→ [True] → Veritabanına kaydet (Database Connector)
    └─→ [False] → Hata logla (Logback Connector)
```

### Yapılandırma

Decision task oluşturmak için:

1. **Decision** kontrol elemanını ekleyin
2. Koşulu tanımlayın:
   * **Condition**: Koşul ifadesi
   * **Variables**: Kullanılacak değişkenler
   * **Operators**: Operatörler (==, !=, >, <, AND, OR)
3. True ve False dallarını bağlayın

### Koşul Örnekleri

```javascript
// Basit koşul
${variable.status} == "active"

// Karmaşık koşul
${variable.count} > 100 AND ${variable.type} == "premium"

// String kontrolü
${variable.email}.contains("@")

// Null kontrolü
${variable.data} != null
```

### Multiple Branches (Çoklu Dallar)

Birden fazla dal için:

1. **Switch** kontrol elemanını kullanın
2. Her dal için koşul tanımlayın
3. Varsayılan dal (default) tanımlayın

```
Switch: ${variable.type}
├─→ "type1" → Task 1
├─→ "type2" → Task 2
├─→ "type3" → Task 3
└─→ default → Default Task
```

## Loop Tasks (Döngü Task'ları)

Loop task'lar, belirli bir koşul sağlandığı sürece task'ları tekrarlı olarak çalıştırır.

### Özellikler

* **Tekrarlı Çalıştırma**: Task'lar belirli bir koşul sağlandığı sürece tekrarlanır
* **Koşul Kontrolü**: Her iterasyonda koşul kontrol edilir
* **Break Condition**: Döngüden çıkma koşulu
* **Continue Condition**: Bir sonraki iterasyona geçme koşulu

### Döngü Türleri

#### For Loop

Belirli sayıda tekrar için:

```
For i = 1 to 10
    → Task çalıştır
End For
```

#### While Loop

Koşul sağlandığı sürece:

```
While ${variable.count} < 100
    → Task çalıştır
    → ${variable.count}++
End While
```

#### ForEach Loop

Koleksiyon elemanları üzerinde:

```
ForEach item in ${variable.list}
    → Task çalıştır (item ile)
End ForEach
```

### Kullanım Senaryosu

```
Dosya listesi al (FTP List Connector)
    ↓
ForEach dosya in liste
    ├─→ Dosyayı oku (FTP Read Connector)
    ├─→ İşle (Script Task)
    └─→ Veritabanına kaydet (Database Connector)
End ForEach
```

### Yapılandırma

Loop task oluşturmak için:

1. **Loop** kontrol elemanını ekleyin
2. Döngü türünü seçin (For, While, ForEach)
3. Döngü koşulunu tanımlayın
4. Döngü içindeki task'ları bağlayın
5. Break/Continue koşullarını tanımlayın

### Dikkat Edilmesi Gerekenler

* **Sonsuz Döngü**: Sonsuz döngü oluşmaması için dikkatli olun
* **Performans**: Çok fazla iterasyon performansı etkiler
* **Timeout**: Döngü için timeout ayarlayın
* **Hata Yönetimi**: Döngü içindeki hataları yönetin

## Connector Tasks

Connector task'lar, dış sistemlere bağlantı kurmak için kullanılır. 15+ farklı konnektör türü mevcuttur.

### Konnektör Türleri

* **Database Connector**: Veritabanı işlemleri
* **HTTP/REST Connector**: HTTP API çağrıları
* **Message Queue Connectors**: Kafka, RabbitMQ, ActiveMQ
* **Email Connector**: E-posta gönderme
* **FTP Connector**: Dosya transferi
* **Elasticsearch Connector**: Arama sorguları
* **Script Connector**: JavaScript/Groovy script'leri
* Ve daha fazlası...

### Kullanım

Connector task'ları kullanmak için:

1. İlgili konnektörü ekleyin
2. Bağlantı bilgilerini yapılandırın
3. Parametreleri ayarlayın
4. Input/Output mapping'i yapın

## Script Tasks

Script task'lar, özel mantık çalıştırmak için kullanılır. JavaScript veya Groovy script'leri yazabilirsiniz.

### Özellikler

* **JavaScript/Groovy**: İki dil desteği
* **Değişken Erişimi**: Task Flow değişkenlerine erişim
* **Veri Dönüştürme**: Veri dönüştürme işlemleri
* **Hesaplamalar**: Karmaşık hesaplamalar

### Kullanım Senaryosu

```javascript
// Veri dönüştürme
var transformedData = {
    name: input.name.toUpperCase(),
    age: parseInt(input.age),
    timestamp: new Date().toISOString()
};

// Koşul kontrolü
if (input.amount > 1000) {
    output.priority = "high";
} else {
    output.priority = "normal";
}

// Döngü işlemi
var sum = 0;
for (var i = 0; i < input.items.length; i++) {
    sum += input.items[i].price;
}
output.total = sum;
```

## Task Kombinasyonları

Farklı task türlerini birleştirerek karmaşık senaryolar oluşturabilirsiniz:

### Senaryo 1: Paralel ve Sequential Kombinasyonu

```
Parallel Start
├─→ Task 1
└─→ Task 2
Parallel End
    ↓
Task 3 (Sequential)
    ↓
Task 4 (Sequential)
```

### Senaryo 2: Decision ve Loop Kombinasyonu

```
Loop: ForEach item
    ↓
Decision: item geçerli mi?
    ├─→ [True] → İşle
    └─→ [False] → Atla
End Loop
```

## En İyi Uygulamalar

### Task Seçimi

* **Sequential**: Task'lar arasında bağımlılık varsa
* **Parallel**: Task'lar bağımsız ve performans önemliyse
* **Decision**: Koşullu akış gerekiyorsa
* **Loop**: Tekrarlı işlemler varsa

### Performans

* Mümkün olduğunca paralel çalıştırma kullanın
* Gereksiz task'ları kaldırın
* Loop'larda timeout ayarlayın
* Batch processing kullanın

### Hata Yönetimi

* Her task türü için uygun hata yönetimi yapın
* Retry mekanizması kullanın
* Fallback stratejileri tanımlayın
* Detaylı loglama yapın
