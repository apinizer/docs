Task Flow Tasarımı, kod yazmadan görsel olarak entegrasyon iş akışları oluşturmanızı sağlar. Görsel tasarımcıyı kullanarak farklı konnektörleri bağlayabilir, veri dönüşümlerini yapılandırabilir ve karmaşık entegrasyon senaryoları oluşturabilirsiniz.

## Genel Bakış

Task Flow Tasarımı, Apinizer'da Task Flow'lar oluşturmak için görsel tasarımcıdır. Konnektörler, değişkenler ve akış kontrol elemanları kullanarak entegrasyon iş akışları oluşturmak için drag-and-drop arayüzü sağlar.

## Task Flow Tasarımcısına Erişim

Task Flow Tasarımcısına erişmek için:

1. Ana menüden **Entegrasyon ve Otomasyon**'a gidin
2. **API Integrator (Task Flow Builder)**'ı seçin
3. **Yeni Task Flow**'a tıklayın veya düzenlemek için mevcut bir Task Flow seçin

## Task Flow Bileşenleri

### Başlangıç Noktası

Her Task Flow bir Başlangıç noktası ile başlar:

* **Girdi Değişkenleri**: Task Flow için girdi değişkenlerini tanımlayın
* **Girdi Doğrulama**: Doğrulama kurallarını yapılandırın
* **Varsayılan Değerler**: İsteğe bağlı girdiler için varsayılan değerleri ayarlayın

### Konnektör Noktaları

Konnektör noktaları gerçekleştirilecek eylemleri temsil eder:

* **Database Connector**: SQL sorguları çalıştırın
* **HTTP/REST Connector**: HTTP istekleri yapın
* **Message Queue Connectors**: Mesaj gönderin/alın
* **Email Connector**: E-posta gönderin
* **FTP Connector**: Dosya işlemleri
* **Script Connector**: Özel mantık çalıştırma
* Ve daha fazlası...

### Akış Kontrol Noktaları

Çalışma akışını kontrol edin:

* **Decision Node**: Koşullu dallanma (IF-THEN-ELSE)
* **Loop Node**: Koleksiyonlar üzerinde yineleme
* **Parallel Node**: Task'ları paralel olarak çalıştırın
* **Wait Node**: Çalışmayı geciktirin
* **Error Handler**: Hataları yönetin

### Bitiş Noktası

Task Flow bir Bitiş noktası ile sonlanır:

* **Çıktı Değişkenleri**: Çıktı değişkenlerini tanımlayın
* **Yanıt Formatı**: Yanıt formatını yapılandırın
* **Durum Kodları**: HTTP durum kodlarını ayarlayın

## Task Flow Oluşturma

### Adım 1: Yeni Task Flow Oluşturma

1. **Yeni Task Flow**'a tıklayın
2. Task Flow adı ve açıklamasını girin
3. Task Flow türünü seçin (API, Scheduled Job veya Sub-Flow)
4. **Oluştur**'a tıklayın

### Adım 2: Başlangıç Noktasını Yapılandırma

Başlangıç noktasını yapılandırın:

**Girdi Değişkenleri:**

```json
{
  "customerId": {
    "type": "string",
    "required": true,
    "description": "Müşteri ID"
  },
  "orderId": {
    "type": "string",
    "required": false,
    "default": null
  }
}
```

**Girdi Doğrulama:**

* Zorunlu alan doğrulaması
* Tip doğrulaması (string, number, boolean, object, array)
* Format doğrulaması (email, URL, regex)
* Özel doğrulama kuralları

### Adım 3: Konnektör Noktaları Ekleme

Akışınıza konnektör noktaları ekleyin:

1. Konnektör paletinden bir konnektörü sürükleyin
2. Canvas'a bırakın
3. Konnektör ayarlarını yapılandırın:
   * **Bağlantı**: Bağlantı seçin veya oluşturun
   * **Eylem**: Gerçekleştirilecek eylemi seçin
   * **Parametreler**: Eylem parametrelerini yapılandırın
   * **Girdi Eşleme**: Girdi değişkenlerini eşleyin
   * **Çıktı Eşleme**: Çıktı değişkenlerini eşleyin

**Örnek: Database Connector Yapılandırması**

* **Bağlantı**: Veritabanı bağlantısını seçin
* **Eylem**: Sorgu Çalıştır
* **Sorgu**: `SELECT * FROM customers WHERE id = :customerId`
* **Girdi Eşleme**: `customerId` değişkenini eşleyin
* **Çıktı Eşleme**: Sonucu `customer` değişkeninde saklayın

### Adım 4: Akış Kontrolünü Yapılandırma

Akış kontrol noktaları ekleyin:

**Decision Node:**

* **Koşul**: Koşul ifadesini tanımlayın
* **True Yolu**: Koşul true ise çalıştırılacak task'lar
* **False Yolu**: Koşul false ise çalıştırılacak task'lar

**Örnek Koşul:**

```javascript
customer.status === 'active' && customer.balance > 0
```

**Loop Node:**

* **Koleksiyon**: Dizi/koleksiyon içeren değişken
* **Öğe Değişkeni**: Mevcut öğe için değişken adı
* **İndeks Değişkeni**: Mevcut indeks için değişken adı
* **Döngü Gövdesi**: Her öğe için çalıştırılacak task'lar

### Adım 5: Veri Dönüştürmeyi Yapılandırma

Konnektörler arasında veri dönüştürün:

**Değişken Eşleme:**

* Bir konnektörün çıktısını diğerinin girdisine eşleyin
* Veri formatını dönüştürün (JSON, XML, CSV)
* İç içe değerleri çıkarın
* Birden fazla değişkeni birleştirin

**Dönüştürme Örnekleri:**

```javascript
// İç içe değer çıkarma
fullName = customer.firstName + ' ' + customer.lastName

// Dizi dönüştürme
emailList = customers.map(c => c.email).join(',')

// Koşullu dönüştürme
status = order.total > 1000 ? 'premium' : 'standard'
```

### Adım 6: Hata Yönetimini Yapılandırma

Hata yönetimi ekleyin:

**Error Handler Node:**

* **Hata Türleri**: Yakalanacak hata türleri
* **Hata Eylemleri**: Hata durumunda yapılacak eylemler
* **Retry Mantığı**: Retry denemelerini yapılandırın
* **Fallback**: Hata durumunda alternatif akış

**Hata Yönetimi Stratejileri:**

* **Retry**: Başarısız task'ı tekrar dene
* **Skip**: Başarısız task'ı atla ve devam et
* **Fail**: Çalışmayı durdur ve hata döndür
* **Fallback**: Alternatif akışı çalıştır

### Adım 7: Bitiş Noktasını Yapılandırma

Bitiş noktasını yapılandırın:

**Çıktı Değişkenleri:**

```json
{
  "result": {
    "type": "object",
    "value": {
      "customer": "${customer}",
      "order": "${order}",
      "status": "success"
    }
  }
}
```

**Yanıt Yapılandırması:**

* **Durum Kodu**: HTTP durum kodu (200, 201, 400, vb.)
* **Yanıt Formatı**: JSON, XML veya özel format
* **Header'lar**: Özel yanıt header'ları

## Task Flow Türleri

### API Task Flow

Task Flow'u REST API olarak açın:

* **HTTP Metodları**: GET, POST, PUT, DELETE, PATCH
* **Path Parametreleri**: Path parametrelerini tanımlayın
* **Query Parametreleri**: Query parametrelerini tanımlayın
* **Request Body**: Request body şemasını yapılandırın
* **Response**: Response şemasını yapılandırın

### Scheduled Task Flow

Task Flow'u zamanlanmış olarak çalıştırın:

* **Cron Expression**: Zamanlamayı tanımlayın
* **Tetikleme Koşulları**: Ek tetikleme koşulları
* **Çalışma Bağlamı**: Çalışma ortamı

### Sub-Flow

Yeniden kullanılabilir Task Flow bileşeni:

* **Girdi Parametreleri**: Girdi parametrelerini tanımlayın
* **Çıktı Parametreleri**: Çıktı parametrelerini tanımlayın
* **Yeniden Kullanılabilirlik**: Diğer Task Flow'larda kullanın

## Gelişmiş Özellikler

### Değişkenler ve İfadeler

Task Flow boyunca değişkenler ve ifadeler kullanın:

**Değişken Türleri:**

* **String**: Metin değerleri
* **Number**: Sayısal değerler
* **Boolean**: True/false değerleri
* **Object**: JSON nesneleri
* **Array**: Değer koleksiyonları

**İfade Sözdizimi:**

```javascript
// Değişken referansı
${variableName}

// İç içe özellik erişimi
${customer.address.city}

// Dizi erişimi
${items[0].name}

// İfade değerlendirmesi
${customer.balance + order.total}
```

### Özel Script'ler

Özel mantık için Script Connector kullanın:

* **JavaScript**: JavaScript kodu yazın
* **Groovy**: Groovy script'leri yazın
* **Değişkenlere Erişim**: Task Flow değişkenlerini okuyun ve yazın
* **Değer Döndürme**: Task Flow'a değerler döndürün

### Paralel Çalıştırma

Birden fazla task'ı paralel olarak çalıştırın:

* **Parallel Node**: Birden fazla dalı aynı anda çalıştırın
* **Senkronizasyon**: Tüm paralel task'ların tamamlanmasını bekleyin
* **Hata Yönetimi**: Paralel dallardaki hataları yönetin

## Task Flow'ları Test Etme

### Test Çalıştırması

Dağıtmadan önce Task Flow'ları test edin:

1. **Test** butonuna tıklayın
2. Girdi değerlerini sağlayın
3. Task Flow'u çalıştırın
4. Çalıştırma sonuçlarını gözden geçirin
5. Her adımda değişken değerlerini kontrol edin

### Debug Modu

Detaylı çalıştırma bilgileri için debug modunu etkinleştirin:

* **Adım adım çalıştırma**: Her seferinde bir adım çalıştırın
* **Değişken inceleme**: Her adımda değişken değerlerini görüntüleyin
* **Çalıştırma logları**: Detaylı çalıştırma logları
* **Hata detayları**: Kapsamlı hata bilgisi
