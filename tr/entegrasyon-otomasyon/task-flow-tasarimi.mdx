Task Flow Tasarımı, API Integrator'da görsel bir arayüz kullanarak entegrasyon akışları oluşturmanızı sağlar. Kod yazmadan, drag-and-drop yöntemiyle görevler ve konnektörler ekleyerek karmaşık entegrasyon senaryolarını tasarlayabilirsiniz.

## Genel Bakış

Task Flow Builder, görsel bir tasarım aracıdır. Bu araç sayesinde:

* Görsel akış diyagramları oluşturma
* Görevler ve konnektörler ekleme
* Akış kontrolü yapılandırma
* Veri dönüştürme işlemleri tanımlama
* Hata yönetimi stratejileri belirleme

yapabilirsiniz.

## Task Flow Builder Arayüzü

### Tasarım Alanı

Task Flow Builder'ın ana tasarım alanı:

* **Canvas**: Akış diyagramının çizildiği alan
* **Palette**: Görevler ve konnektörlerin bulunduğu araç çubuğu
* **Properties Panel**: Seçili öğenin özelliklerini düzenleme paneli
* **Toolbar**: Kaydetme, çalıştırma gibi işlemler için araç çubuğu

### Görev Ekleme

Görev eklemek için:

1. Palette'den istediğiniz görev tipini seçin
2. Canvas üzerinde istediğiniz konuma sürükleyin
3. Görev özelliklerini Properties Panel'den yapılandırın

### Konnektör Ekleme

Konnektör eklemek için:

1. Palette'den istediğiniz konnektör tipini seçin
2. Canvas üzerinde bir göreve sürükleyin
3. Konnektör özelliklerini yapılandırın
4. Bağlantı bilgilerini (Connection) seçin

## Görev Tipleri

Task Flow Builder'da kullanabileceğiniz görev tipleri:

### Sequential Task

Görevlerin sırayla çalıştırıldığı görev tipidir:

* **Kullanım**: Görevlerin belirli bir sırada çalışması gerektiğinde
* **Örnek**: Önce veritabanından veri çekme, sonra HTTP isteği gönderme

### Parallel Task

Görevlerin paralel olarak çalıştırıldığı görev tipidir:

* **Kullanım**: Bağımsız görevlerin aynı anda çalışması gerektiğinde
* **Örnek**: Birden fazla API'ye aynı anda istek gönderme

### Decision Task

Koşullu dallanma sağlayan görev tipidir:

* **Kullanım**: Farklı koşullara göre farklı akışların çalışması gerektiğinde
* **Örnek**: Veri kontrolü yapıp başarılı/başarısız durumlarına göre farklı işlemler yapma

Daha fazla bilgi için [Task Types](/tr/entegrasyon-otomasyon/task-types) sayfasına bakın.

## Konnektör Yapılandırması

### Konnektör Seçimi

Kullanılabilir konnektörler:

* **Database Connector**: Veritabanı işlemleri
* **HTTP/REST Connector**: HTTP istekleri
* **Email Connector**: E-posta gönderme
* **FTP Connector**: Dosya transferi
* **Message Queue Connectors**: Mesaj kuyruğu işlemleri
* **Script Connector**: Özel script çalıştırma
* Ve daha fazlası...

### Bağlantı Yapılandırması

Her konnektör için bir bağlantı (Connection) seçmeniz gerekir:

1. Konnektör özelliklerinde **Connection** alanını açın
2. Önceden tanımlanmış bağlantılardan birini seçin
3. Veya yeni bir bağlantı oluşturun

Daha fazla bilgi için [Bağlantı Yönetimi](/tr/entegrasyon-otomasyon/baglanti-yonetimi/connection-types-overview) sayfasına bakın.

## Veri Dönüştürme

### Değişkenler

Task Flow içinde veri taşımak için değişkenler kullanılır:

* **Input Variables**: Task Flow'a gelen veriler
* **Output Variables**: Task Flow'dan çıkan veriler
* **Local Variables**: Task Flow içinde kullanılan geçici değişkenler

### Veri Mapping

Görevler arasında veri aktarımı:

* **Field Mapping**: Alan bazlı veri eşleştirme
* **Transformation**: Veri dönüştürme işlemleri
* **Data Enrichment**: Veri zenginleştirme

Daha fazla bilgi için [Variables & Data Passing](/tr/entegrasyon-otomasyon/variables-data-passing) sayfasına bakın.

## Akış Kontrolü

### Koşullu Dallanma

Decision Task kullanarak koşullu dallanma:

* **IF-THEN-ELSE**: Koşullu akış kontrolü
* **Switch-Case**: Çoklu koşul kontrolü
* **Conditional Routing**: Koşullu yönlendirme

### Döngüler

Tekrarlayan işlemler için:

* **For Loop**: Belirli sayıda tekrar
* **While Loop**: Koşul sağlandığı sürece tekrar
* **ForEach**: Koleksiyon üzerinde döngü

## Hata Yönetimi

### Try-Catch Blokları

Hata yakalama için:

* **Try Block**: Hata oluşabilecek görevler
* **Catch Block**: Hata durumunda çalışacak görevler
* **Finally Block**: Her durumda çalışacak görevler

### Retry Mekanizması

Başarısız görevler için tekrar deneme:

* **Retry Count**: Tekrar deneme sayısı
* **Retry Interval**: Tekrar deneme aralığı
* **Exponential Backoff**: Üstel artan bekleme süresi

Daha fazla bilgi için [Error Handling & Retries](/tr/entegrasyon-otomasyon/error-handling-retries) sayfasına bakın.

## Task Flow Yapılandırması

### Genel Ayarlar

Task Flow genel ayarları:

* **Name**: Task Flow adı
* **Description**: Açıklama
* **Version**: Versiyon bilgisi
* **Tags**: Etiketler

### Input/Output Yapılandırması

Task Flow'un girdi ve çıktıları:

* **Input Schema**: Girdi şeması tanımlama
* **Output Schema**: Çıktı şeması tanımlama
* **Validation**: Girdi doğrulama kuralları

### Çalıştırma Ayarları

Task Flow çalıştırma ayarları:

* **Timeout**: Maksimum çalışma süresi
* **Retry Policy**: Yeniden deneme politikası
* **Error Handling**: Hata yönetimi stratejisi

## API Olarak Yayınlama

### REST API Olarak Yayınlama

Task Flow'u REST API olarak yayınlamak için:

1. Task Flow özelliklerinde **Publish as API** seçeneğini aktif edin
2. Endpoint path'i belirleyin
3. HTTP metodlarını seçin (GET, POST, PUT, DELETE)
4. Request/Response formatını yapılandırın

### Webhook Olarak Yayınlama

Task Flow'u webhook olarak yayınlamak için:

1. Task Flow özelliklerinde **Publish as Webhook** seçeneğini aktif edin
2. Webhook URL'ini yapılandırın
3. Authentication ayarlarını yapın

## En İyi Uygulamalar

### Tasarım Prensipleri

* **Modülerlik**: Görevleri küçük ve yeniden kullanılabilir parçalara bölün
* **Açıklık**: Görev ve değişkenlere açıklayıcı isimler verin
* **Dokümantasyon**: Task Flow'ları dokümante edin
* **Versiyonlama**: Değişiklikleri versiyonlayın

### Performans

* **Paralel İşlemler**: Bağımsız görevleri paralel çalıştırın
* **Veri Filtreleme**: Gereksiz veri transferini önleyin
* **Connection Pooling**: Bağlantı havuzlarını kullanın
* **Caching**: Uygun yerlerde önbellekleme kullanın

### Güvenlik

* **Secret Management**: Hassas bilgileri Secret Manager'da saklayın
* **Input Validation**: Girdi doğrulaması yapın
* **Error Messages**: Hata mesajlarında hassas bilgi göstermeyin
* **Access Control**: Task Flow erişim kontrolünü yapılandırın

## Kullanım Senaryoları

### Senaryo 1: Veri Senkronizasyonu

Kaynak sistemden hedef sisteme veri senkronizasyonu:

1. Database Connector ile kaynak sistemden veri çekme
2. Script Connector ile veri dönüştürme
3. HTTP Connector ile hedef sisteme veri gönderme
4. Email Connector ile başarı/hata bildirimi

### Senaryo 2: Event-Driven Integration

Mesaj kuyruğundan gelen event'leri işleme:

1. Kafka Connector ile event alma
2. Decision Task ile event tipine göre yönlendirme
3. Farklı sistemlere HTTP Connector ile event gönderme
4. Elasticsearch Connector ile loglama

### Senaryo 3: Dosya İşleme

FTP'den dosya alıp işleme:

1. FTP Connector ile dosya alma
2. Script Connector ile dosya içeriğini parse etme
3. Database Connector ile veritabanına kaydetme
4. Email Connector ile işlem sonrası bildirim
