API Integrator (Task Flow Builder), karmaşık entegrasyon senaryolarınızı görsel olarak tasarlamanızı ve yönetmenizi sağlayan güçlü bir araçtır. Bu araç sayesinde farklı sistemler arasındaki veri akışlarını, iş mantığını ve hata yönetimini görsel bir arayüz üzerinden kolayca tanımlayabilirsiniz.

## Genel Bakış

API Integrator, özellikle şu durumlarda kullanılır:

* Birden fazla sistem arasında veri entegrasyonu yapmak
* Karmaşık iş akışlarını görsel olarak tasarlamak
* Farklı veri kaynaklarından veri toplamak ve işlemek
* Hata yönetimi ve yeniden deneme mekanizmaları eklemek
* İş akışlarını izlemek ve performansını optimize etmek

## API Integrator'a Erişim

API Integrator'a erişmek için:

1. Ana menüden **Entegrasyon ve Otomasyon** bölümüne gidin
2. **API Integrator** seçeneğine tıklayın
3. Task Flow Builder arayüzü açılır

## Task Flow Nedir?

Task Flow, birbirine bağlı görevlerin (task) sıralı veya paralel olarak çalıştırıldığı bir iş akışıdır. Her task flow şu bileşenlerden oluşur:

* **Tasks**: Yapılacak işlemler (API çağrıları, veritabanı sorguları, vb.)
* **Connections**: Task'lar arasındaki bağlantılar
* **Variables**: Task'lar arasında paylaşılan veriler
* **Error Handling**: Hata durumlarında yapılacak işlemler

## Task Flow Oluşturma

Yeni bir Task Flow oluşturmak için:

1. **New Task Flow** butonuna tıklayın
2. Task Flow adını ve açıklamasını girin
3. Task Flow tasarım ekranına yönlendirilirsiniz

### Temel Task Flow Yapısı

Bir Task Flow genellikle şu yapıda olur:

```
Start → Task 1 → Task 2 → Task 3 → End
```

Daha karmaşık senaryolarda:

```
Start → Decision Task → [True] → Task A
                      → [False] → Task B
                      → End
```

## Task Types

API Integrator, farklı türde task'lar sunar:

### Sequential Tasks

Task'lar sırayla çalıştırılır. Bir task tamamlandıktan sonra bir sonraki task başlar.

**Kullanım Senaryosu:**
* Veritabanından veri çekme → Veriyi işleme → API'ye gönderme

### Parallel Tasks

Birden fazla task aynı anda paralel olarak çalıştırılır.

**Kullanım Senaryosu:**
* Farklı API'lerden aynı anda veri çekme
* Birden fazla veritabanı sorgusu çalıştırma

### Decision Tasks

Koşullu dallanma sağlar. Belirli bir koşula göre farklı task'lara yönlendirme yapar.

**Kullanım Senaryosu:**
* Veri doğrulama → Başarılı ise kaydetme → Başarısız ise hata gönderme

## Variables ve Data Passing

Task'lar arasında veri paylaşımı için variables kullanılır.

### Variable Tanımlama

Variables tanımlamak için:

1. Task Flow tasarım ekranında **Variables** sekmesine gidin
2. **Add Variable** butonuna tıklayın
3. Variable adı, tipi ve varsayılan değerini girin

### Variable Tipleri

* **String**: Metin verileri
* **Number**: Sayısal veriler
* **Boolean**: Mantıksal değerler
* **Object**: JSON objeleri
* **Array**: Dizi verileri

### Data Passing

Bir task'tan diğerine veri aktarmak için:

1. Task'ın output'unu bir variable'a atayın
2. Sonraki task'ta bu variable'ı kullanın

**Örnek:**
```
Task 1 Output: {"userId": 123}
Variable: userData = Task 1 Output
Task 2 Input: ${userData.userId}
```

## Error Handling ve Retries

Task Flow'larda hata yönetimi kritik öneme sahiptir.

### Error Handling Stratejileri

* **Stop on Error**: Hata durumunda akışı durdur
* **Continue on Error**: Hata durumunda devam et
* **Retry**: Belirli sayıda yeniden dene
* **Fallback**: Alternatif bir task çalıştır

### Retry Yapılandırması

Retry mekanizması yapılandırmak için:

1. Task'ın **Error Handling** sekmesine gidin
2. **Enable Retry** seçeneğini aktif edin
3. Retry sayısını ve aralığını belirleyin

**Örnek:**
```
Max Retries: 3
Retry Interval: 5 seconds
Backoff Strategy: Exponential
```

## Monitoring ve Debugging

Task Flow'larınızı izlemek ve hata ayıklamak için çeşitli araçlar mevcuttur.

### TaskFlow Monitor

TaskFlow Monitor, çalışan Task Flow'ların durumunu gerçek zamanlı olarak izlemenizi sağlar.

**Özellikler:**
* Çalışan Task Flow'ların listesi
* Her task'ın durumu (Running, Success, Failed)
* Execution time ve resource usage
* Error logs ve stack traces

### Debugging

Task Flow'larda hata ayıklama için:

1. **Debug Mode**'u aktif edin
2. Task Flow'u çalıştırın
3. Her task'ın input/output'unu inceleyin
4. Variable değerlerini kontrol edin

## Performance Optimization

Task Flow'larınızın performansını optimize etmek için:

### Best Practices

* **Parallel Tasks**: Mümkün olduğunca paralel task'lar kullanın
* **Connection Pooling**: Veritabanı bağlantılarını pool'layın
* **Caching**: Sık kullanılan verileri cache'leyin
* **Batch Processing**: Büyük veri setlerini batch'ler halinde işleyin

### Performance Monitoring

* Task execution time'larını izleyin
* Resource usage'ı kontrol edin
* Bottleneck'leri tespit edin
* Optimize edin

## Kullanım Senaryoları

### Senaryo 1: E-posta Bildirim Sistemi

Bir kullanıcı kaydı olduğunda e-posta gönderme:

```
Start → Database: Kullanıcı bilgilerini çek
     → Email: Hoş geldin e-postası gönder
     → End
```

### Senaryo 2: Veri Senkronizasyonu

İki sistem arasında veri senkronizasyonu:

```
Start → API 1: Verileri çek
     → Transform: Veriyi dönüştür
     → API 2: Veriyi gönder
     → Database: Log kaydı
     → End
```

### Senaryo 3: Hata Yönetimi ile İş Akışı

Hata yönetimi içeren bir iş akışı:

```
Start → API Call
     → Decision: Başarılı mı?
     → [Yes] → Database: Kaydet → End
     → [No] → Retry (3 kez)
           → [Still Failed] → Email: Hata bildirimi → End
```
