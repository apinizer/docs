Task Flow Tasarımı, API Integrator'ın kalbidir. Bu arayüz sayesinde karmaşık entegrasyon senaryolarınızı görsel olarak tasarlayabilir, task'ları yapılandırabilir ve bunlar arasındaki bağlantıları tanımlayabilirsiniz.

## Genel Bakış

Task Flow tasarım ekranı şu özellikleri sunar:

* Drag-and-drop ile kolay tasarım
* Çoklu task türleri desteği
* Görsel bağlantı yönetimi
* Variable yönetimi
* Hata yönetimi yapılandırması
* Real-time validation
* Preview ve test özellikleri

## Task Flow Tasarım Ekranına Erişim

Task Flow tasarım ekranına erişmek için:

1. **API Integrator** sayfasına gidin
2. Yeni Task Flow oluşturmak için **New Task Flow** butonuna tıklayın
3. Veya mevcut bir Task Flow'u düzenlemek için **Edit** butonuna tıklayın

## Tasarım Arayüzü

Tasarım ekranı şu bölümlerden oluşur:

### Toolbar

Üst kısımdaki toolbar'da şu araçlar bulunur:

* **Save**: Değişiklikleri kaydet
* **Save As**: Farklı kaydet
* **Undo/Redo**: Geri al/Yinele
* **Zoom In/Out**: Yakınlaştır/Uzaklaştır
* **Fit to Screen**: Ekrana sığdır
* **Validate**: Doğrula
* **Preview**: Önizleme
* **Run**: Çalıştır
* **Settings**: Ayarlar

### Task Palette

Sol taraftaki palette'te mevcut task türleri listelenir:

* **Start**: Başlangıç noktası
* **End**: Bitiş noktası
* **API Call**: API çağrısı
* **Database**: Veritabanı işlemi
* **Email**: E-posta gönderme
* **Script**: Script çalıştırma
* **Decision**: Karar verme
* **Parallel**: Paralel işlem
* **Delay**: Gecikme
* **Transform**: Veri dönüşümü

### Canvas

Ortadaki canvas alanında Task Flow'unuzu tasarlarsınız:

* Task'ları sürükleyip bırakabilirsiniz
* Task'lar arasında bağlantılar oluşturabilirsiniz
* Task'ları seçip özelliklerini düzenleyebilirsiniz

### Properties Panel

Sağ taraftaki properties panel'de:

* Seçili task'ın özellikleri
* Variable'lar
* Error handling ayarları
* Genel Task Flow ayarları

## Task Ekleme

Task eklemek için:

1. **Task Palette**'ten bir task türü seçin
2. Task'ı **Canvas**'a sürükleyip bırakın
3. Task otomatik olarak eklenir

### Task Yapılandırması

Bir task'ı yapılandırmak için:

1. Task'a çift tıklayın veya sağ tıklayıp **Properties** seçin
2. Properties panel açılır
3. Gerekli ayarları yapın

**Genel Ayarlar:**
* **Name**: Task adı
* **Description**: Açıklama
* **Type**: Task türü
* **Enabled**: Aktif/Pasif

**Task-Specific Ayarlar:**
Her task türü için farklı ayarlar mevcuttur. Örneğin:

**API Call Task:**
* **URL**: API endpoint URL'i
* **Method**: HTTP metodu (GET, POST, PUT, DELETE)
* **Headers**: Request header'ları
* **Body**: Request body
* **Authentication**: Kimlik doğrulama

**Database Task:**
* **Connection**: Veritabanı bağlantısı
* **Query Type**: Sorgu türü (Select, Insert, Update, Delete)
* **Query**: SQL sorgusu
* **Parameters**: Sorgu parametreleri

## Bağlantı Oluşturma

Task'lar arasında bağlantı oluşturmak için:

1. İlk task'ın çıkış noktasına tıklayın
2. İkinci task'ın giriş noktasına sürükleyin
3. Bağlantı oluşturulur

### Bağlantı Özellikleri

Bağlantı özelliklerini düzenlemek için:

1. Bağlantıya çift tıklayın
2. Properties panel açılır

**Bağlantı Ayarları:**
* **Condition**: Koşul (opsiyonel)
* **Label**: Etiket
* **Style**: Stil (Normal, Success, Error)

### Conditional Connections

Koşullu bağlantılar oluşturmak için:

1. **Decision Task** kullanın
2. Decision task'tan çıkan bağlantılara koşul ekleyin

**Örnek:**
```
Decision Task: Check Status
→ [status == "success"] → Success Task
→ [status == "error"] → Error Task
```

## Variable Yönetimi

Task Flow içinde variable'ları yönetmek için:

### Variable Tanımlama

1. **Variables** sekmesine gidin
2. **Add Variable** butonuna tıklayın
3. Variable bilgilerini girin

**Variable Ayarları:**
* **Name**: Variable adı
* **Type**: Variable tipi (String, Number, Boolean, Object, Array)
* **Default Value**: Varsayılan değer
* **Scope**: Kapsam (Global, Local)

### Variable Kullanımı

Task'larda variable kullanmak için:

**Input Variable:**
```
${variableName}
```

**Nested Variable:**
```
${object.property}
```

**Array Variable:**
```
${array[0]}
```

**Örnek:**
```json
{
  "userId": "${user.id}",
  "email": "${user.email}",
  "timestamp": "${timestamp}"
}
```

## Error Handling Yapılandırması

Her task için error handling yapılandırabilirsiniz:

### Error Handling Stratejileri

1. Task'ı seçin
2. **Error Handling** sekmesine gidin
3. Stratejiyi seçin

**Stratejiler:**
* **Stop on Error**: Hata durumunda durdur
* **Continue on Error**: Hata durumunda devam et
* **Retry**: Yeniden dene
* **Fallback**: Alternatif task çalıştır

### Retry Yapılandırması

Retry yapılandırmak için:

* **Max Retries**: Maksimum deneme sayısı
* **Retry Interval**: Deneme aralığı (saniye)
* **Backoff Strategy**: Backoff stratejisi (Linear, Exponential)

**Örnek:**
```
Max Retries: 3
Retry Interval: 5 seconds
Backoff Strategy: Exponential
```

### Fallback Task

Fallback task tanımlamak için:

1. **Fallback Task** seçeneğini aktif edin
2. Fallback task'ı seçin
3. Fallback task, ana task başarısız olduğunda çalışır

## Validation

Task Flow'unuzu doğrulamak için:

1. **Validate** butonuna tıklayın
2. Validation sonuçları gösterilir

**Validation Kontrolleri:**
* Tüm task'lar bağlı mı?
* Gerekli alanlar doldurulmuş mu?
* Variable'lar doğru tanımlanmış mı?
* Döngüsel bağımlılık var mı?
* Syntax hataları var mı?

## Preview ve Test

### Preview

Task Flow'unuzu önizlemek için:

1. **Preview** butonuna tıklayın
2. Task Flow'un görsel önizlemesi gösterilir

### Test

Task Flow'unuzu test etmek için:

1. **Run** butonuna tıklayın
2. Test parametrelerini girin
3. **Run** butonuna tıklayın
4. Execution sonuçları gösterilir

**Test Modları:**
* **Dry Run**: Gerçek çalıştırma yapmadan test et
* **Full Run**: Gerçek çalıştırma yap

## Best Practices

### Tasarım

* **Modular Design**: Modüler tasarım yapın
* **Clear Naming**: Açık ve anlaşılır isimler kullanın
* **Documentation**: Her task'ı dokümante edin
* **Reusability**: Yeniden kullanılabilir task'lar oluşturun

### Performance

* **Parallel Tasks**: Mümkün olduğunca paralel task'lar kullanın
* **Efficient Queries**: Verimli sorgular yazın
* **Caching**: Cache mekanizmaları kullanın
* **Batch Processing**: Batch işleme yapın

### Error Handling

* **Comprehensive**: Kapsamlı hata yönetimi yapın
* **Retry Logic**: Retry mantığı ekleyin
* **Fallback**: Fallback mekanizmaları kullanın
* **Logging**: Detaylı loglama yapın

## Örnek Senaryolar

### Senaryo 1: Basit API Chain

```
Start → API Call 1 → API Call 2 → End
```

### Senaryo 2: Conditional Flow

```
Start → Decision → [True] → Task A
                 → [False] → Task B
                 → End
```

### Senaryo 3: Parallel Processing

```
Start → Parallel → Task A ──┐
              → Task B ──┼→ Merge → End
              → Task C ──┘
```

### Senaryo 4: Error Handling ile

```
Start → API Call
     → Decision: Success?
     → [Yes] → End
     → [No] → Retry (3x)
           → [Still Failed] → Fallback Task → End
```
