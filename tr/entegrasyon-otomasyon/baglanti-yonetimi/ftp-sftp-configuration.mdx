## Genel Bakış

FTP/SFTP Configuration sayesinde:

* FTP ve SFTP sunucularına bağlantı oluşturabilirsiniz
* Güvenli SFTP bağlantıları kullanabilirsiniz
* Authentication yapılandırabilirsiniz
* Dosya transfer işlemleri gerçekleştirebilirsiniz

## FTP Connection

### Connection Configuration

FTP bağlantısı oluşturma:

1. **Connection Management** sayfasına gidin
2. **FTP/SFTP Configuration** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. **FTP** tipini seçin
5. Bağlantı bilgilerini girin

### Basic Configuration

Temel yapılandırma:

```yaml
Name: ftp-production
Protocol: FTP
Host: ftp.example.com
Port: 21
Username: ftp-user
Password: ${ftpPassword} (Secret Manager)
Passive Mode: true
```

### Passive Mode

Pasif mod yapılandırması:

* **Passive Mode**: Pasif mod (önerilen)
* **Active Mode**: Aktif mod
* **Port Range**: Port aralığı (pasif mod için)

### SSL/TLS Configuration

SSL/TLS yapılandırması:

```yaml
Use SSL: true
SSL Protocol: TLSv1.2
Certificate Validation: true
```

## SFTP Connection

### Connection Configuration

SFTP bağlantısı oluşturma:

1. **Connection Management** sayfasına gidin
2. **FTP/SFTP Configuration** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. **SFTP** tipini seçin
5. Bağlantı bilgilerini girin

### Basic Configuration

Temel yapılandırma:

```yaml
Name: sftp-production
Protocol: SFTP
Host: sftp.example.com
Port: 22
Username: sftp-user
Password: ${sftpPassword} (Secret Manager)
```

### SSH Key Authentication

SSH anahtarı kimlik doğrulama:

```yaml
Authentication: SSH Key
SSH Key: ${sshPrivateKey} (Secret Manager)
SSH Key Passphrase: ${keyPassphrase} (Secret Manager)
```

### SSH Configuration

SSH yapılandırması:

```yaml
SSH Configuration:
  Host Key Verification: true
  Known Hosts: /path/to/known_hosts
  Compression: false
  Connection Timeout: 30000ms
```

## Security Configuration

### Authentication Methods

Kimlik doğrulama yöntemleri:

* **Password**: Şifre tabanlı
* **SSH Key**: SSH anahtarı tabanlı
* **Certificate**: Sertifika tabanlı

### Security Best Practices

Güvenlik en iyi uygulamaları:

* **Use SFTP**: SFTP kullanın (FTP yerine)
* **SSH Key Authentication**: SSH anahtarı kullanın
* **Secure Credentials**: Güvenli kimlik bilgileri
* **Network Security**: Ağ güvenliği

## Connection Pooling

### Pool Configuration

Havuz yapılandırması:

```yaml
Connection Pool:
  Min Size: 3
  Max Size: 10
  Connection Timeout: 30000ms
  Idle Timeout: 300000ms
```

## Testing

### Test Connection

Bağlantı testi:

1. Bağlantı yapılandırmasında **Test Connection** butonuna tıklayın
2. Bağlantı testi çalıştırılır
3. Test sonucu görüntülenir

### Test File Operations

Dosya işlemi testi:

* **List Directory**: Dizin listeleme
* **Upload Test File**: Test dosyası yükleme
* **Download Test File**: Test dosyası indirme
* **Delete Test File**: Test dosyası silme
