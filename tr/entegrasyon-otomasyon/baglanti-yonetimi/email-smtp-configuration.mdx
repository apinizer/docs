## Genel Bakış

Email SMTP Configuration sayesinde:

* SMTP sunucularına bağlantı oluşturabilirsiniz
* SSL/TLS güvenli bağlantılar kullanabilirsiniz
* Authentication yapılandırabilirsiniz
* Çoklu SMTP sunucu desteği kullanabilirsiniz

## Connection Configuration

### Creating SMTP Connection

SMTP bağlantısı oluşturma:

1. **Connection Management** sayfasına gidin
2. **Email SMTP Configuration** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin

### Basic Configuration

Temel yapılandırma:

```yaml
Name: smtp-production
SMTP Host: smtp.example.com
SMTP Port: 587
Username: user@example.com
Password: ${smtpPassword} (Secret Manager)
From Address: noreply@example.com
```

### SSL/TLS Configuration

SSL/TLS yapılandırması:

```yaml
Use TLS: true
Use SSL: false
TLS Version: TLSv1.2
SSL Protocols: TLSv1.2, TLSv1.3
```

### Authentication Methods

Kimlik doğrulama yöntemleri:

* **PLAIN**: Düz metin kimlik doğrulama
* **LOGIN**: Login kimlik doğrulama
* **CRAM-MD5**: CRAM-MD5 kimlik doğrulama
* **DIGEST-MD5**: DIGEST-MD5 kimlik doğrulama

## Port Configuration

### Common Ports

Yaygın portlar:

* **25**: SMTP (genellikle bloklu)
* **465**: SMTPS (SSL/TLS)
* **587**: SMTP (STARTTLS)
* **2525**: Alternatif SMTP portu

### Port Selection

Port seçimi:

* **587**: STARTTLS için önerilen port
* **465**: SSL/TLS için önerilen port
* **25**: Genellikle sunucu-sunucu iletişimi için

## Security Configuration

### SSL/TLS Settings

SSL/TLS ayarları:

* **Use TLS**: TLS kullanımı (STARTTLS)
* **Use SSL**: SSL kullanımı (SMTPS)
* **TLS Version**: TLS versiyonu
* **Certificate Validation**: Sertifika doğrulama

### Authentication Security

Kimlik doğrulama güvenliği:

* **Secure Password Storage**: Güvenli şifre saklama (Secret Manager)
* **Password Encryption**: Şifre şifreleme
* **Access Control**: Erişim kontrolü

## Connection Pooling

### Pool Configuration

Havuz yapılandırması:

```yaml
Connection Pool:
  Min Size: 5
  Max Size: 20
  Connection Timeout: 30000ms
  Idle Timeout: 600000ms
```

### Benefits

Havuzlama avantajları:

* **Performance**: Performans artışı
* **Resource Efficiency**: Kaynak verimliliği
* **Connection Reuse**: Bağlantı yeniden kullanımı

## Testing

### Test Connection

Bağlantı testi:

1. Bağlantı yapılandırmasında **Test Connection** butonuna tıklayın
2. Test e-postası gönderilir
3. Test sonucu görüntülenir

### Test Email

Test e-postası:

* **To**: Test e-posta adresi
* **Subject**: Test e-posta konusu
* **Body**: Test e-posta içeriği
