Elasticsearch Configuration, Elasticsearch cluster'larına bağlantı oluşturmanızı sağlar. Bu sayfa, Elasticsearch bağlantı yapılandırmasını detaylı olarak açıklar.

## Genel Bakış

Elasticsearch Configuration sayesinde:

* Elasticsearch cluster'larına bağlantı oluşturabilirsiniz
* Çoklu node desteği kullanabilirsiniz
* Authentication yapılandırabilirsiniz
* Monitoring ve health check yapabilirsiniz

## Connection Configuration

### Creating Elasticsearch Connection

Elasticsearch bağlantısı oluşturma:

1. **Connection Management** sayfasına gidin
2. **Elasticsearch Configuration** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin

### Basic Configuration

Temel yapılandırma:

```yaml
Name: elasticsearch-production
Hosts:
  - elasticsearch1.example.com:9200
  - elasticsearch2.example.com:9200
  - elasticsearch3.example.com:9200
Protocol: HTTP
Authentication: None
```

### Authentication Configuration

Kimlik doğrulama yapılandırması:

```yaml
Authentication:
  Type: Basic
  Username: elastic
  Password: ${elasticsearchPassword} (Secret Manager)
```

### API Key Authentication

API key kimlik doğrulama:

```yaml
Authentication:
  Type: API Key
  API Key: ${elasticsearchApiKey} (Secret Manager)
```

## SSL/TLS Configuration

### SSL Configuration

SSL yapılandırması:

```yaml
SSL: true
SSL Protocol: TLSv1.2
Certificate Validation: true
Truststore: /path/to/truststore.jks
Keystore: /path/to/keystore.jks
```

### Certificate Configuration

Sertifika yapılandırması:

* **Truststore**: Güven deposu
* **Keystore**: Anahtar deposu
* **Certificate Validation**: Sertifika doğrulama

## Connection Pooling

### Pool Configuration

Havuz yapılandırması:

```yaml
Connection Pool:
  Min Size: 5
  Max Size: 20
  Connection Timeout: 30000ms
  Socket Timeout: 30000ms
  Max Retries: 3
```

## Health Check

### Cluster Health

Cluster sağlık kontrolü:

* **Cluster Status**: Cluster durumu
* **Node Status**: Node durumu
* **Index Status**: İndeks durumu
* **Shard Status**: Shard durumu

### Health Check Configuration

Sağlık kontrolü yapılandırması:

```yaml
Health Check:
  Enabled: true
  Interval: 60000ms
  Timeout: 10000ms
```
