API Çağrısı ile bir endpoint tetiklenebilir, endpointten dönen veri akış içinde kullanılabilir, veya önceki eylemin sonucu, API çağrısı ile bir başka endpointe gönderilebilir.

## Konfigürasyon

API Çağrısı yapılandırması ayarlarını içeren görsele aşağıda yer verilmiştir:

API Çağrısı yapılandırması konfigürasyonu için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan | Açıklama |
|------|----------|
| **Metot** (Method) | API çağrısı yapılacak endpoint'in HTTP metodu seçilir. Zorunludur. |
| **URL** | API çağrısı yapılacak endpoint bilgisi girilir. Zorunludur. |
| **Zaman Aşımı** (Timeout) | İsteğin ne kadar süre sonra zaman aşımına uğrayacağı bilgisidir. Zorunludur. |
| **Sertifikayı Aktive Et** (Enable Certificate) | İstekle birlikte sertifikanın gidip gitmeyeceği yönetilir. |
| **Sertifika** (Certificate) | Sertifika aktif edildiyse bu alan zorunludur. Eğer sertifika listesinde istenilen değer yoksa (Sertifika Ekle) Add Certificate linkinden yeni bir tane oluşturulabilir. |
| **Başlıklar** (Headers) | İsteğin başlık bilgileri girilir. |
| **Gövde** (Body) | İsteğin gövde değeri girilir. |

<Info>
Başlıklar ve Gövde alanlarında değişkenler kullanılabilir.
</Info>
