HTTP/REST Connector, Task Flow'larda HTTP/REST API'lerine istek göndermenizi sağlar. GET, POST, PUT, DELETE gibi HTTP metodlarını destekler ve authentication, header yönetimi gibi özellikler sunar.

## Genel Bakış

HTTP/REST Connector özellikleri:

* **HTTP Methods**: GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS
* **Authentication**: Basic, Bearer, OAuth, API Key
* **Request/Response Handling**: İstek/yanıt yönetimi
* **Error Handling**: Hata yönetimi

## HTTP Methods

### GET

Veri okuma istekleri:

**Basit GET:**
```
GET https://api.example.com/users/123
```

**Query Parameters ile GET:**
```
GET https://api.example.com/users?status=active&limit=10
```

**Headers ile GET:**
```
GET https://api.example.com/users
Headers:
  Authorization: Bearer token123
  Accept: application/json
```

### POST

Veri oluşturma istekleri:

**JSON Body ile POST:**
```
POST https://api.example.com/users
Headers:
  Content-Type: application/json
Body:
{
  "name": "John Doe",
  "email": "john@example.com"
}
```

**Form Data ile POST:**
```
POST https://api.example.com/users
Headers:
  Content-Type: application/x-www-form-urlencoded
Body:
  name=John Doe&email=john@example.com
```

### PUT

Veri güncelleme istekleri:

**JSON Body ile PUT:**
```
PUT https://api.example.com/users/123
Headers:
  Content-Type: application/json
Body:
{
  "name": "Jane Doe",
  "email": "jane@example.com"
}
```

### DELETE

Veri silme istekleri:

**Basit DELETE:**
```
DELETE https://api.example.com/users/123
```

### PATCH

Kısmi güncelleme istekleri:

**JSON Body ile PATCH:**
```
PATCH https://api.example.com/users/123
Headers:
  Content-Type: application/json
Body:
{
  "email": "newemail@example.com"
}
```

## Request Configuration

### URL Configuration

URL yapılandırması:

* **Base URL**: Temel URL (Connection'dan veya manuel)
* **Path**: Endpoint path'i
* **Query Parameters**: Query parametreleri
* **Path Parameters**: Path parametreleri

**Örnek:**
```
Base URL: https://api.example.com
Path: /users/:userId/orders
Path Parameters:
  userId: 123
Query Parameters:
  status: active
  limit: 10
Final URL: https://api.example.com/users/123/orders?status=active&limit=10
```

### Headers

Header yapılandırması:

* **Standard Headers**: Standart HTTP header'ları
* **Custom Headers**: Özel header'lar
* **Dynamic Headers**: Dinamik header'lar (değişkenlerden)

**Örnek Headers:**
```
Content-Type: application/json
Accept: application/json
Authorization: Bearer ${accessToken}
X-Request-ID: ${requestId}
X-API-Key: ${apiKey}
```

### Request Body

İstek gövdesi yapılandırması:

* **JSON**: JSON formatında gövde
* **XML**: XML formatında gövde
* **Form Data**: Form data formatında gövde
* **Raw**: Ham metin formatında gövde
* **Dynamic Body**: Dinamik gövde (değişkenlerden)

**JSON Örneği:**
```json
{
  "name": "${customerName}",
  "email": "${customerEmail}",
  "address": {
    "street": "${street}",
    "city": "${city}"
  }
}
```

## Authentication

### Basic Authentication

Temel kimlik doğrulama:

```
Username: ${username}
Password: ${password} (Secret Manager'dan)
```

### Bearer Token Authentication

Bearer token kimlik doğrulama:

```
Authorization: Bearer ${accessToken}
```

Token'ı Task Flow değişkenlerinden veya önceki bir API çağrısından alabilirsiniz.

### API Key Authentication

API key kimlik doğrulama:

**Header'da:**
```
X-API-Key: ${apiKey}
```

**Query Parameter'da:**
```
?api_key=${apiKey}
```

### OAuth 2.0 Authentication

OAuth 2.0 kimlik doğrulama:

* **Client Credentials**: Client credentials flow
* **Authorization Code**: Authorization code flow
* **Refresh Token**: Refresh token kullanımı

## Response Handling

### Response Status Codes

Yanıt durum kodları:

* **2xx**: Başarılı yanıtlar
* **4xx**: İstemci hataları
* **5xx**: Sunucu hataları

### Response Body Parsing

Yanıt gövdesi ayrıştırma:

* **JSON Parsing**: JSON yanıtları ayrıştırma
* **XML Parsing**: XML yanıtları ayrıştırma
* **Text Parsing**: Metin yanıtları ayrıştırma

### Response Variable Mapping

Yanıt değişken eşleştirme:

**JSON Response:**
```json
{
  "id": 123,
  "name": "John Doe",
  "email": "john@example.com"
}
```

Task Flow değişkenlerine:
- `response.id` → `userId`
- `response.name` → `userName`
- `response.email` → `userEmail`

## Error Handling

### HTTP Error Handling

HTTP hata yönetimi:

* **4xx Errors**: İstemci hataları (400, 401, 403, 404, vb.)
* **5xx Errors**: Sunucu hataları (500, 502, 503, vb.)
* **Network Errors**: Ağ hataları
* **Timeout Errors**: Zaman aşımı hataları

### Retry Configuration

Yeniden deneme yapılandırması:

* **Retry on 5xx**: 5xx hatalarında yeniden deneme
* **Retry on Timeout**: Zaman aşımında yeniden deneme
* **Retry Count**: Yeniden deneme sayısı
* **Retry Interval**: Yeniden deneme aralığı

### Error Response Handling

Hata yanıtı yönetimi:

* **Error Message Extraction**: Hata mesajı çıkarma
* **Error Logging**: Hata loglama
* **Error Notification**: Hata bildirimi
* **Fallback Actions**: Yedek eylemler

## Advanced Features

### Request/Response Transformation

İstek/yanıt dönüştürme:

* **JSON to XML**: JSON'dan XML'e dönüştürme
* **XML to JSON**: XML'den JSON'a dönüştürme
* **Data Mapping**: Veri eşleştirme
* **Field Transformation**: Alan dönüştürme

### Request/Response Logging

İstek/yanıt loglama:

* **Request Logging**: İstek loglama
* **Response Logging**: Yanıt loglama
* **Header Logging**: Header loglama
* **Body Logging**: Gövde loglama

### Timeout Configuration

Zaman aşımı yapılandırması:

* **Connection Timeout**: Bağlantı zaman aşımı
* **Read Timeout**: Okuma zaman aşımı
* **Total Timeout**: Toplam zaman aşımı

## Usage Examples

### Example 1: GET Request

```
HTTP/REST Connector:
- Method: GET
- URL: https://api.example.com/users/${userId}
- Headers:
    Authorization: Bearer ${token}
    Accept: application/json
- Output: userData
```

### Example 2: POST Request

```
HTTP/REST Connector:
- Method: POST
- URL: https://api.example.com/orders
- Headers:
    Content-Type: application/json
    Authorization: Bearer ${token}
- Body:
  {
    "customerId": "${customerId}",
    "items": ${orderItems},
    "totalAmount": ${totalAmount}
  }
- Output: orderResponse
```

### Example 3: PUT Request

```
HTTP/REST Connector:
- Method: PUT
- URL: https://api.example.com/users/${userId}
- Headers:
    Content-Type: application/json
    Authorization: Bearer ${token}
- Body:
  {
    "name": "${newName}",
    "email": "${newEmail}"
  }
- Output: updateResponse
```

### Example 4: Error Handling

```
HTTP/REST Connector:
- Method: GET
- URL: https://api.example.com/data
- Retry Policy:
    Max Retries: 3
    Retry on: 5xx, Timeout
    Interval: 5 seconds
- On Error:
    Log Error
    Send Notification
    Use Fallback Data
```

## Best Practices

### Request Design

* **Use Appropriate Methods**: Uygun HTTP metodlarını kullanın
* **Idempotent Operations**: İdempotent işlemler için PUT kullanın
* **RESTful Design**: RESTful tasarım prensiplerini takip edin

### Security

* **Use HTTPS**: HTTPS kullanın
* **Secure Credentials**: Kimlik bilgilerini Secret Manager'da saklayın
* **Token Management**: Token yönetimini düzgün yapın
* **Input Validation**: Girdi doğrulama yapın

### Performance

* **Connection Pooling**: Bağlantı havuzlama kullanın
* **Request Timeout**: İstek zaman aşımı ayarlayın
* **Retry Strategy**: Uygun retry stratejisi kullanın
* **Caching**: Uygun yerlerde önbellekleme kullanın

### Error Handling

* **Handle All Status Codes**: Tüm durum kodlarını yönetin
* **Meaningful Error Messages**: Anlamlı hata mesajları
* **Retry Transient Errors**: Geçici hatalarda yeniden deneme
* **Log Errors**: Hataları loglayın
