## Yapılandırma

### Bağlantı Seçimi

HTTP bağlantısı seçin veya oluşturun:

**HTTP Bağlantı Özellikleri:**

* **Base URL**: API endpoint'leri için base URL
* **Timeout**: İstek zaman aşımı
* **Retry Configuration**: Retry ayarları
* **SSL/TLS**: SSL/TLS yapılandırması

## HTTP Metodları

### GET İsteği

API'den veri alın:

**Yapılandırma:**

* **URL**: İstek URL'i
* **Headers**: İstek başlıkları
* **Query Parameters**: URL sorgu parametreleri

**Örnek:**

```json
{
  "method": "GET",
  "url": "https://api.example.com/customers/${customerId}",
  "headers": {
    "Authorization": "Bearer ${token}",
    "Accept": "application/json"
  },
  "queryParameters": {
    "include": "orders",
    "limit": "10"
  }
}
```

### POST İsteği

Yeni kaynaklar oluşturun:

**Yapılandırma:**

* **URL**: İstek URL'i
* **Headers**: İstek başlıkları
* **Body**: İstek gövdesi

**Örnek:**

```json
{
  "method": "POST",
  "url": "https://api.example.com/orders",
  "headers": {
    "Authorization": "Bearer ${token}",
    "Content-Type": "application/json"
  },
  "body": {
    "customerId": "${customerId}",
    "items": "${orderItems}",
    "total": "${orderTotal}"
  }
}
```

### PUT İsteği

Mevcut kaynakları güncelleyin:

**Yapılandırma:**

* **URL**: İstek URL'i
* **Headers**: İstek başlıkları
* **Body**: İstek gövdesi

**Örnek:**

```json
{
  "method": "PUT",
  "url": "https://api.example.com/orders/${orderId}",
  "headers": {
    "Authorization": "Bearer ${token}",
    "Content-Type": "application/json"
  },
  "body": {
    "status": "completed"
  }
}
```

### DELETE İsteği

Kaynakları silin:

**Yapılandırma:**

* **URL**: İstek URL'i
* **Headers**: İstek başlıkları

**Örnek:**

```json
{
  "method": "DELETE",
  "url": "https://api.example.com/orders/${orderId}",
  "headers": {
    "Authorization": "Bearer ${token}"
  }
}
```

## İstek Yapılandırması

### Headers

İstek başlıklarını yapılandırın:

**Yaygın Başlıklar:**

* **Authorization**: Kimlik doğrulama token'ı
* **Content-Type**: İstek gövdesi içerik türü
* **Accept**: Kabul edilebilir yanıt içerik türleri
* **User-Agent**: Client user agent
* **Custom Headers**: Özel başlık değerleri

**Örnek:**

```json
{
  "headers": {
    "Authorization": "Bearer ${token}",
    "Content-Type": "application/json",
    "Accept": "application/json",
    "X-Request-ID": "${requestId}",
    "X-API-Key": "${apiKey}"
  }
}
```

### Query Parameters

URL'ye sorgu parametreleri ekleyin:

**Örnek:**

```json
{
  "queryParameters": {
    "page": "1",
    "limit": "10",
    "sort": "created_date",
    "order": "desc"
  }
}
```

### İstek Gövdesi

İstek gövdesini yapılandırın:

**JSON Gövdesi:**

```json
{
  "body": {
    "name": "${customerName}",
    "email": "${customerEmail}",
    "status": "active"
  }
}
```

**Form Data:**

```json
{
  "body": {
    "name": "${customerName}",
    "email": "${customerEmail}"
  },
  "contentType": "application/x-www-form-urlencoded"
}
```

**XML Gövdesi:**

```json
{
  "body": "<customer><name>${customerName}</name></customer>",
  "contentType": "application/xml"
}
```

## Kimlik Doğrulama

### Bearer Token

Bearer token kimlik doğrulaması kullanın:

**Yapılandırma:**

```json
{
  "authentication": {
    "type": "bearer",
    "token": "${accessToken}"
  }
}
```

### Basic Authentication

Basic kimlik doğrulaması kullanın:

**Yapılandırma:**

```json
{
  "authentication": {
    "type": "basic",
    "username": "${username}",
    "password": "${password}"
  }
}
```

### API Key

API key kimlik doğrulaması kullanın:

**Yapılandırma:**

```json
{
  "authentication": {
    "type": "apikey",
    "key": "${apiKey}",
    "location": "header",
    "name": "X-API-Key"
  }
}
```

### OAuth 2.0

OAuth 2.0 kimlik doğrulaması kullanın:

**Yapılandırma:**

```json
{
  "authentication": {
    "type": "oauth2",
    "tokenUrl": "https://auth.example.com/token",
    "clientId": "${clientId}",
    "clientSecret": "${clientSecret}",
    "grantType": "client_credentials"
  }
}
```

## Yanıt Yönetimi

### Yanıt Eşleme

Yanıt verilerini değişkenlere eşleyin:

**Yanıt Yapısı:**

```json
{
  "statusCode": "${response.statusCode}",
  "headers": "${response.headers}",
  "body": "${response.body}"
}
```

**Gövde Eşleme:**

```json
{
  "customer": "${response.body.customer}",
  "orders": "${response.body.orders}",
  "total": "${response.body.total}"
}
```

### Hata Yönetimi

HTTP hatalarını yönetin:

**Hata Durum Kodları:**

* **4xx**: Client hataları (400, 401, 403, 404, vb.)
* **5xx**: Server hataları (500, 502, 503, vb.)

**Hata Yönetimi:**

```javascript
if (response.statusCode >= 400) {
  // Hatayı yönet
  error = {
    "statusCode": response.statusCode,
    "message": response.body.message,
    "details": response.body
  };
}
```

## Retry Mantığı

### Retry Yapılandırması

Retry mantığını yapılandırın:

**Retry Ayarları:**

* **Max Retries**: Maksimum retry denemeleri
* **Retry Interval**: Retry'lar arasındaki süre
* **Retry on Status Codes**: Retry yapılacak durum kodları
* **Retry Strategy**: Retry stratejisi (fixed, exponential backoff)

**Örnek:**

```json
{
  "retry": {
    "maxAttempts": 3,
    "interval": 1000,
    "retryOnStatusCodes": [500, 502, 503, 504],
    "strategy": "exponentialBackoff"
  }
}
```

## Yaygın Desenler

### API Çağrı Deseni

Dış API'yi çağırın:

```
Task Flow:
  ├─ Prepare Request Data
  ├─ Call API (HTTP/REST Connector)
  └─ Process Response
```

### Zincirleme API Çağrıları

Birden fazla API çağrısı yapın:

```
Task Flow:
  ├─ Call API 1 (HTTP/REST Connector)
  ├─ Process Response 1
  ├─ Call API 2 (HTTP/REST Connector)
  └─ Process Response 2
```

### Paralel API Çağrıları

Paralel API çağrıları yapın:

```
Task Flow:
  ├─ Parallel Branch
  │   ├─ Call API 1 (HTTP/REST Connector)
  │   └─ Call API 2 (HTTP/REST Connector)
  └─ Merge Results
```
