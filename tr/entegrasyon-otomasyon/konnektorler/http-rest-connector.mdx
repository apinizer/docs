HTTP/REST Connector, Task Flow'larda HTTP/HTTPS API çağrıları yapmanızı sağlayan konnektördür. Bu connector sayesinde REST API'leri çağırabilir, web servisleri entegre edebilir ve API istekleri/yanıtlarını yönetebilirsiniz.

## Genel Bakış

HTTP/REST Connector şu özellikleri sunar:

* HTTP metodları (GET, POST, PUT, DELETE, PATCH)
* Request/Response yönetimi
* Authentication (Basic, Bearer, OAuth2)
* Header yönetimi
* Query parameters
* Request body (JSON, XML, Form Data)

## HTTP/REST Connector Kullanımı

HTTP/REST Connector'ı Task Flow'da kullanmak için:

1. Task Flow tasarım ekranında **API Call Task** ekleyin
2. HTTP metodunu seçin
3. URL'i belirleyin
4. Headers ve body'yi yapılandırın
5. Authentication ayarlarını yapın

## HTTP Metodları

### GET

Veri okuma:

```
Method: GET
URL: https://api.example.com/users/${userId}
```

### POST

Veri oluşturma:

```
Method: POST
URL: https://api.example.com/users
Body: {"name": "${userName}", "email": "${userEmail}"}
```

### PUT

Veri güncelleme:

```
Method: PUT
URL: https://api.example.com/users/${userId}
Body: {"name": "${userName}"}
```

### DELETE

Veri silme:

```
Method: DELETE
URL: https://api.example.com/users/${userId}
```

### PATCH

Kısmi güncelleme:

```
Method: PATCH
URL: https://api.example.com/users/${userId}
Body: {"status": "active"}
```

## Request Yapılandırması

### URL

URL yapılandırması:

```
Base URL: https://api.example.com
Path: /users/${userId}
Full URL: https://api.example.com/users/123
```

### Headers

Header yapılandırması:

```
Content-Type: application/json
Accept: application/json
Authorization: Bearer ${token}
X-API-Key: ${apiKey}
```

### Query Parameters

Query parametreleri:

```
URL: https://api.example.com/users?status=active&limit=10
```

veya

```
Query Parameters:
  status = active
  limit = 10
```

### Request Body

**JSON:**
```json
{
  "name": "${userName}",
  "email": "${userEmail}",
  "status": "active"
}
```

**XML:**
```xml
<user>
  <name>${userName}</name>
  <email>${userEmail}</email>
</user>
```

**Form Data:**
```
name=${userName}
email=${userEmail}
```

## Authentication

### Basic Authentication

```
Username: ${username}
Password: ${password}
```

### Bearer Token

```
Authorization: Bearer ${token}
```

### OAuth2

```
Grant Type: client_credentials
Client ID: ${clientId}
Client Secret: ${clientSecret}
Token URL: https://auth.example.com/token
```

## Response Yapılandırması

### Response Handling

```
Status Code: 200
Response Body: ${responseData}
```

### Error Handling

```
Status Code: 4xx, 5xx
Error Handling: Retry or Fallback
```

## Variable Kullanımı

Variable'ları request'te kullanın:

```
URL: https://api.example.com/users/${userId}
Headers:
  Authorization: Bearer ${token}
Body:
  {
    "name": "${userName}",
    "email": "${userEmail}"
  }
```

## Best Practices

### Güvenlik

* **HTTPS**: HTTPS kullanın
* **Authentication**: Güvenli kimlik doğrulama kullanın
* **Credentials**: Kimlik bilgilerini güvenli saklayın
* **Input Validation**: Girdi doğrulaması yapın

### Performance

* **Connection Pooling**: Bağlantı havuzlama kullanın
* **Timeout**: Timeout değerlerini ayarlayın
* **Caching**: Cache mekanizmaları kullanın
* **Async Requests**: Asenkron istekler yapın

### Reliability

* **Error Handling**: Hata yönetimi yapın
* **Retry Logic**: Yeniden deneme mantığı ekleyin
* **Circuit Breaker**: Circuit breaker pattern kullanın
* **Monitoring**: İzleme yapın

## Örnek Senaryolar

### Senaryo 1: API'den Veri Çekme

```
Task: Get User Data
Method: GET
URL: https://api.example.com/users/${userId}
Headers:
  Authorization: Bearer ${token}
Output: ${userData}
```

### Senaryo 2: API'ye Veri Gönderme

```
Task: Create User
Method: POST
URL: https://api.example.com/users
Headers:
  Content-Type: application/json
  Authorization: Bearer ${token}
Body:
  {
    "name": "${userName}",
    "email": "${userEmail}"
  }
Output: ${createdUser}
```

### Senaryo 3: OAuth2 ile API Çağrısı

```
Task: Get Access Token
Method: POST
URL: https://auth.example.com/token
Body:
  grant_type=client_credentials
  client_id=${clientId}
  client_secret=${clientSecret}
Output: ${accessToken}

Task: Call API
Method: GET
URL: https://api.example.com/data
Headers:
  Authorization: Bearer ${accessToken}
```
