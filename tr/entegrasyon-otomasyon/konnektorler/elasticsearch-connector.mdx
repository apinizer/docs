Elasticsearch Connector, Task Flow'larda Elasticsearch ile etkileşim kurmanızı sağlayan konnektördür. Bu connector sayesinde veri indeksleyebilir, arama sorguları çalıştırabilir ve Elasticsearch cluster'ını yönetebilirsiniz.

## Genel Bakış

Elasticsearch Connector şu özellikleri sunar:

* Veri indeksleme
* Arama sorguları
* Aggregation'lar
* Index yönetimi
* Document operations
* Cluster monitoring

## Elasticsearch Connector Kullanımı

Elasticsearch Connector'ı Task Flow'da kullanmak için:

1. Task Flow tasarım ekranında **Elasticsearch Task** ekleyin
2. Elasticsearch connection'ı seçin
3. İşlem tipini belirleyin (Index, Search, Update, Delete)
4. Query veya document'ı yapılandırın

## Connection Yapılandırması

Elasticsearch connection yapılandırmak için:

1. **Connection Management** → **Elasticsearch Configuration** bölümüne gidin
2. Yeni connection oluşturun

**Elasticsearch Ayarları:**
* **Hosts**: Elasticsearch node'ları
* **Port**: Port (9200)
* **Username**: Kullanıcı adı (opsiyonel)
* **Password**: Şifre (opsiyonel)
* **SSL**: SSL ayarları
* **Connection Timeout**: Bağlantı timeout'u

## Veri İndeksleme

Document indekslemek için:

### Ayarlar

* **Index**: Index adı
* **Document ID**: Document ID (opsiyonel)
* **Document**: Document verisi

### Örnek

```
Index: users
Document ID: ${userId}
Document:
{
  "name": "${userName}",
  "email": "${userEmail}",
  "timestamp": "${timestamp}"
}
```

## Arama Sorguları

Arama sorguları çalıştırmak için:

### Basit Arama

```json
{
  "query": {
    "match": {
      "name": "${searchTerm}"
    }
  }
}
```

### Bool Query

```json
{
  "query": {
    "bool": {
      "must": [
        {"match": {"status": "active"}},
        {"range": {"age": {"gte": 18}}}
      ]
    }
  }
}
```

## Aggregation'lar

Veri toplama için:

```json
{
  "aggs": {
    "status_count": {
      "terms": {
        "field": "status"
      }
    }
  }
}
```

## Best Practices

### Performance

* **Bulk Operations**: Toplu işlemler yapın
* **Index Settings**: Index ayarlarını optimize edin
* **Query Optimization**: Sorguları optimize edin

### Reliability

* **Error Handling**: Hata yönetimi yapın
* **Retry Logic**: Yeniden deneme mantığı ekleyin
* **Monitoring**: İzleme yapın
