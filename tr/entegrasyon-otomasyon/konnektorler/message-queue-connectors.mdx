Message Queue Connectors, çeşitli mesaj kuyruğu sistemlerine bağlantı sağlar ve asenkron mesajlaşma desenlerini ve event-driven entegrasyonu etkinleştirir.

## Kafka Connector

Apache Kafka, gerçek zamanlı veri pipeline'ları ve streaming uygulamaları oluşturmak için dağıtılmış bir streaming platformudur.

### Özellikler

* **Producer**: Kafka topic'lerine mesaj gönderin
* **Consumer**: Kafka topic'lerinden mesaj alın
* **Topic Management**: Topic'leri oluşturun ve yönetin
* **Partition Support**: Kafka partition'larıyla çalışın
* **Offset Management**: Consumer offset'lerini yönetin

### Yapılandırma

**Bağlantı Yapılandırması:**

* **Bootstrap Servers**: Kafka broker adresleri
* **Security**: SASL/SSL yapılandırması
* **Client ID**: Benzersiz client tanımlayıcısı
* **Connection Timeout**: Bağlantı zaman aşımı ayarları

### Mesaj Gönderme (Producer)

Kafka topic'lerine mesaj gönderin:

**Yapılandırma:**

* **Topic**: Hedef Kafka topic'i
* **Key**: Mesaj anahtarı (opsiyonel)
* **Message**: Mesaj payload'ı
* **Partition**: Hedef partition (opsiyonel)
* **Headers**: Mesaj başlıkları (opsiyonel)

**Örnek:**

```json
{
  "topic": "orders",
  "key": "${orderId}",
  "message": "${orderJson}",
  "headers": {
    "content-type": "application/json",
    "source": "taskflow"
  }
}
```

### Mesaj Alma (Consumer)

Kafka topic'lerinden mesaj alın:

**Yapılandırma:**

* **Topic**: Kaynak Kafka topic'i
* **Consumer Group**: Consumer group ID
* **Partition**: Belirli partition (opsiyonel)
* **Offset**: Başlangıç offset'i
* **Max Messages**: Alınacak maksimum mesaj sayısı

**Örnek:**

```json
{
  "topic": "orders",
  "consumerGroup": "order-processor",
  "maxMessages": 100,
  "timeout": 5000
}
```

### Mesaj Formatı

Kafka mesajları genellikle JSON'dur:

**Mesaj Yapısı:**

```json
{
  "key": "order-123",
  "value": {
    "orderId": "123",
    "customerId": "456",
    "total": 99.99,
    "status": "pending"
  },
  "headers": {
    "content-type": "application/json"
  },
  "partition": 0,
  "offset": 12345,
  "timestamp": "2024-01-15T10:30:00Z"
}
```

## RabbitMQ Connector

RabbitMQ, Advanced Message Queuing Protocol (AMQP) uygulayan bir mesaj broker'ıdır.

### Özellikler

* **Publish**: Exchange'lere mesaj yayınlayın
* **Consume**: Queue'lerden mesaj tüketin
* **Exchange Types**: Çeşitli exchange türleri desteği
* **Queue Management**: Queue'leri oluşturun ve yönetin
* **Routing**: Routing key'lerle mesaj yönlendirme

### Yapılandırma

**Bağlantı Yapılandırması:**

* **Host**: RabbitMQ sunucu hostname'i
* **Port**: RabbitMQ sunucu portu
* **Virtual Host**: Virtual host adı
* **Username**: RabbitMQ kullanıcı adı
* **Password**: RabbitMQ şifresi
* **Connection Timeout**: Bağlantı zaman aşımı

### Mesaj Yayınlama

RabbitMQ exchange'lerine mesaj yayınlayın:

**Yapılandırma:**

* **Exchange**: Hedef exchange adı
* **Routing Key**: Routing key
* **Message**: Mesaj payload'ı
* **Properties**: Mesaj özellikleri
* **Exchange Type**: Exchange türü (direct, topic, fanout, headers)

**Örnek:**

```json
{
  "exchange": "orders",
  "routingKey": "order.created",
  "message": "${orderJson}",
  "properties": {
    "contentType": "application/json",
    "deliveryMode": 2,
    "priority": 5
  }
}
```

### Mesaj Tüketme

RabbitMQ queue'lerinden mesaj tüketin:

**Yapılandırma:**

* **Queue**: Kaynak queue adı
* **Auto Acknowledge**: Mesajları otomatik onaylayın
* **Prefetch Count**: Onaylanmamış mesaj sayısı
* **Timeout**: Alma zaman aşımı

**Örnek:**

```json
{
  "queue": "order-processing",
  "autoAcknowledge": false,
  "prefetchCount": 10,
  "timeout": 5000
}
```

### Mesaj Onaylama

İşlenen mesajları onaylayın:

**Onaylama:**

* **Auto Acknowledge**: Alma sırasında otomatik onaylama
* **Manual Acknowledge**: İşlemeden sonra manuel onaylama
* **Negative Acknowledge**: Mesajı reddet ve yeniden kuyruğa al

## ActiveMQ Connector

Apache ActiveMQ, birden fazla protokolü destekleyen açık kaynaklı bir mesaj broker'ıdır.

### Özellikler

* **Send**: Queue'lere veya topic'lere mesaj gönderin
* **Receive**: Queue'lerden veya topic'lerden mesaj alın
* **Queue/Topic Support**: Queue ve topic desteği
* **Message Selectors**: Selector'larla mesaj filtreleme
* **Durable Subscriptions**: Kalıcı topic abonelikleri desteği

### Yapılandırma

**Bağlantı Yapılandırması:**

* **Broker URL**: ActiveMQ broker URL'i
* **Username**: ActiveMQ kullanıcı adı
* **Password**: ActiveMQ şifresi
* **Client ID**: Benzersiz client tanımlayıcısı

### Mesaj Gönderme

ActiveMQ destination'larına mesaj gönderin:

**Yapılandırma:**

* **Destination**: Queue veya topic adı
* **Destination Type**: Queue veya Topic
* **Message**: Mesaj payload'ı
* **Properties**: Mesaj özellikleri

**Örnek:**

```json
{
  "destination": "orders.queue",
  "destinationType": "queue",
  "message": "${orderJson}",
  "properties": {
    "JMSType": "Order",
    "JMSPriority": 5
  }
}
```

### Mesaj Alma

ActiveMQ destination'larından mesaj alın:

**Yapılandırma:**

* **Destination**: Queue veya topic adı
* **Destination Type**: Queue veya Topic
* **Message Selector**: JMS mesaj seçicisi
* **Timeout**: Alma zaman aşımı

**Örnek:**

```json
{
  "destination": "orders.queue",
  "destinationType": "queue",
  "messageSelector": "JMSType = 'Order' AND status = 'pending'",
  "timeout": 5000
}
```

## Yaygın Desenler

### Producer Pattern

Mesaj kuyruğuna mesaj gönderin:

```
Task Flow:
  ├─ Process Data
  ├─ Format Message
  └─ Send to Queue (Message Queue Connector)
```

### Consumer Pattern

Mesajları alın ve işleyin:

```
Task Flow:
  ├─ Receive from Queue (Message Queue Connector)
  ├─ Process Message
  └─ Send Acknowledgment
```

### Request-Reply Pattern

İstek gönderin ve yanıt bekleyin:

```
Task Flow:
  ├─ Send Request Message
  ├─ Wait for Reply
  └─ Process Reply Message
```

## Hata Yönetimi

### Mesaj İşleme Hataları

Mesaj işleme sırasındaki hataları yönetin:

**Hata Yönetimi:**

* **Retry**: Başarısız mesaj işlemeyi tekrar deneyin
* **Dead Letter Queue**: Başarısız mesajları DLQ'ya gönderin
* **Error Notification**: İşleme hatalarında bildirim gönderin
* **Message Rejection**: Mesajları reddedin ve yeniden kuyruğa alın

### Bağlantı Hataları

Bağlantı hatalarını yönetin:

**Hata Yönetimi:**

* **Retry Connection**: Bağlantı başarısızlıklarında retry
* **Failover**: Yedek broker'lara failover
* **Error Logging**: Bağlantı hatalarını loglayın
