## Genel Bakış

Message Queue Connectors sayesinde:

* Mesaj kuyruklarına mesaj gönderebilirsiniz
* Mesaj kuyruklarından mesaj alabilirsiniz
* Event-driven entegrasyonlar oluşturabilirsiniz
* Asenkron işlemler gerçekleştirebilirsiniz
* Yüksek throughput ile mesaj işleyebilirsiniz

## Kafka Connector

Apache Kafka ile entegrasyon için Kafka Connector kullanılır.

### Kafka Bağlantı Yapılandırması

1. **Connection Management** menüsüne gidin
2. **Kafka Connections** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin:
   * **Name**: Bağlantı adı
   * **Bootstrap Servers**: Kafka broker adresleri (örn: localhost:9092)
   * **Security Protocol**: Güvenlik protokolü (PLAINTEXT, SSL, SASL_PLAINTEXT, SASL_SSL)
   * **Authentication**: Kimlik doğrulama ayarları (opsiyonel)
5. **Test Connection** butonuna tıklayın
6. **Save** butonuna tıklayın

### Mesaj Gönderme (Producer)

Kafka'ya mesaj göndermek için:

1. Task Flow'a **Kafka Connector** ekleyin
2. **Operation Type** olarak **Produce** seçin
3. Yapılandırma:
   * **Topic**: Mesaj gönderilecek topic adı
   * **Key**: Mesaj anahtarı (opsiyonel)
   * **Value**: Mesaj içeriği
   * **Partition**: Partition numarası (opsiyonel)
   * **Headers**: Mesaj başlıkları (opsiyonel)

**Örnek:**

```javascript
{
  "topic": "user-events",
  "key": "${input.userId}",
  "value": JSON.stringify({
    userId: "${input.userId}",
    event: "user.created",
    timestamp: new Date().toISOString()
  }),
  "headers": {
    "content-type": "application/json"
  }
}
```

### Mesaj Alma (Consumer)

Kafka'dan mesaj almak için:

1. Task Flow'a **Kafka Connector** ekleyin
2. **Operation Type** olarak **Consume** seçin
3. Yapılandırma:
   * **Topics**: Dinlenecek topic'ler
   * **Consumer Group**: Consumer group adı
   * **Auto Commit**: Otomatik commit (true/false)
   * **Offset Reset**: Offset reset stratejisi (earliest/latest)

**Örnek:**

```javascript
{
  "topics": ["user-events", "order-events"],
  "consumerGroup": "taskflow-consumer",
  "autoCommit": false,
  "offsetReset": "earliest"
}
```

### Kafka Özellikleri

* **High Throughput**: Yüksek throughput
* **Partitioning**: Partition desteği
* **Replication**: Replication desteği
* **Consumer Groups**: Consumer group desteği
* **Exactly-Once Semantics**: Exactly-once semantik desteği

## RabbitMQ Connector

RabbitMQ ile entegrasyon için RabbitMQ Connector kullanılır.

### RabbitMQ Bağlantı Yapılandırması

1. **Connection Management** menüsüne gidin
2. **RabbitMQ Connections** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin:
   * **Name**: Bağlantı adı
   * **Host**: RabbitMQ sunucu adresi
   * **Port**: Port numarası (varsayılan: 5672)
   * **Virtual Host**: Virtual host adı (varsayılan: /)
   * **Username**: Kullanıcı adı
   * **Password**: Şifre
   * **SSL**: SSL kullanımı (true/false)
5. **Test Connection** butonuna tıklayın
6. **Save** butonuna tıklayın

### Mesaj Gönderme (Publisher)

RabbitMQ'ya mesaj göndermek için:

1. Task Flow'a **RabbitMQ Connector** ekleyin
2. **Operation Type** olarak **Publish** seçin
3. Yapılandırma:
   * **Exchange**: Exchange adı
   * **Routing Key**: Routing key
   * **Message**: Mesaj içeriği
   * **Properties**: Mesaj özellikleri (durable, persistent, vb.)

**Örnek:**

```javascript
{
  "exchange": "user-events",
  "routingKey": "user.created",
  "message": JSON.stringify({
    userId: "${input.userId}",
    event: "user.created",
    timestamp: new Date().toISOString()
  }),
  "properties": {
    "contentType": "application/json",
    "deliveryMode": 2, // Persistent
    "priority": 0,
    "correlationId": "${variables.correlationId}",
    "replyTo": "response-queue"
  }
}
```

### Mesaj Alma (Consumer)

RabbitMQ'dan mesaj almak için:

1. Task Flow'a **RabbitMQ Connector** ekleyin
2. **Operation Type** olarak **Consume** seçin
3. Yapılandırma:
   * **Queue**: Dinlenecek queue adı
   * **Auto Acknowledge**: Otomatik acknowledge (true/false)
   * **Prefetch Count**: Prefetch count

**Örnek:**

```javascript
{
  "queue": "user-events-queue",
  "autoAcknowledge": false,
  "prefetchCount": 10
}
```

### RabbitMQ Özellikleri

* **Exchanges**: Exchange desteği (direct, topic, fanout, headers)
* **Queues**: Queue yönetimi
* **Routing**: Routing desteği
* **Message Persistence**: Mesaj kalıcılığı
* **Dead Letter Queue**: Dead letter queue desteği

## ActiveMQ Connector

Apache ActiveMQ ile entegrasyon için ActiveMQ Connector kullanılır.

### ActiveMQ Bağlantı Yapılandırması

1. **Connection Management** menüsüne gidin
2. **ActiveMQ Connections** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin:
   * **Name**: Bağlantı adı
   * **Broker URL**: Broker URL (örn: tcp://localhost:61616)
   * **Username**: Kullanıcı adı (opsiyonel)
   * **Password**: Şifre (opsiyonel)
5. **Test Connection** butonuna tıklayın
6. **Save** butonuna tıklayın

### Mesaj Gönderme (Producer)

ActiveMQ'ya mesaj göndermek için:

1. Task Flow'a **ActiveMQ Connector** ekleyin
2. **Operation Type** olarak **Send** seçin
3. Yapılandırma:
   * **Destination**: Destination adı (queue veya topic)
   * **Destination Type**: Destination türü (Queue/Topic)
   * **Message**: Mesaj içeriği
   * **Properties**: Mesaj özellikleri

**Örnek:**

```javascript
{
  "destination": "user-events-queue",
  "destinationType": "Queue",
  "message": JSON.stringify({
    userId: "${input.userId}",
    event: "user.created"
  }),
  "properties": {
    "JMSType": "user.created",
    "JMSCorrelationID": "${variables.correlationId}"
  }
}
```

### Mesaj Alma (Consumer)

ActiveMQ'dan mesaj almak için:

1. Task Flow'a **ActiveMQ Connector** ekleyin
2. **Operation Type** olarak **Receive** seçin
3. Yapılandırma:
   * **Destination**: Destination adı
   * **Destination Type**: Destination türü
   * **Message Selector**: Mesaj seçici (opsiyonel)
   * **Timeout**: Timeout süresi

**Örnek:**

```javascript
{
  "destination": "user-events-queue",
  "destinationType": "Queue",
  "messageSelector": "JMSType = 'user.created'",
  "timeout": 5000
}
```

### ActiveMQ Özellikleri

* **Queue ve Topic**: Queue ve Topic desteği
* **Message Selectors**: Mesaj seçiciler
* **Durable Subscriptions**: Kalıcı abonelikler
* **Virtual Destinations**: Sanal destination'lar

## Kullanım Senaryoları

### Senaryo 1: Event-Driven Integration

```
Kafka Consumer (Event alma)
├─→ Script Task (Event işleme)
├─→ Database Connector (Veri kaydetme)
└─→ HTTP Connector (Başka sisteme bildirim)
```

### Senaryo 2: Asenkron İşleme

```
HTTP Connector (API isteği)
├─→ RabbitMQ Publisher (Mesaj gönderme)
└─→ Response (Hemen yanıt dön)

(Arka planda)
RabbitMQ Consumer (Mesaj alma)
├─→ Database Connector (İşleme)
└─→ Email Connector (Bildirim)
```

### Senaryo 3: Workflow Orchestration

```
Kafka Consumer (Başlangıç event'i)
├─→ Task 1 (Paralel)
├─→ Task 2 (Paralel)
└─→ Task 3 (Paralel)
    ↓
Kafka Producer (Tamamlanma event'i)
```
