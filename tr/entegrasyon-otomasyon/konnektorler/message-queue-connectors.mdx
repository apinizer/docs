Message Queue Connectors, Task Flow'larda mesaj kuyruğu sistemleriyle entegrasyon sağlar. Kafka, RabbitMQ ve ActiveMQ desteği ile mesaj gönderebilir ve alabilirsiniz.

## Genel Bakış

Message Queue Connectors özellikleri:

* **Kafka Connector**: Apache Kafka entegrasyonu
* **RabbitMQ Connector**: RabbitMQ entegrasyonu
* **ActiveMQ Connector**: Apache ActiveMQ entegrasyonu
* **Message Publishing**: Mesaj yayınlama
* **Message Consumption**: Mesaj tüketme

## Kafka Connector

### Kafka Overview

Apache Kafka, yüksek performanslı, dağıtık mesaj kuyruğu sistemidir.

### Kafka Connection

Kafka Connector kullanmak için önce bir Kafka Connection oluşturmanız gerekir:

1. **Connection Management** sayfasına gidin
2. **Kafka Queue Connections** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin:
   * **Name**: Bağlantı adı
   * **Bootstrap Servers**: Kafka broker adresleri (örn: localhost:9092)
   * **Security Protocol**: Güvenlik protokolü (PLAINTEXT, SSL, SASL_PLAINTEXT, SASL_SSL)
   * **Authentication**: Kimlik doğrulama ayarları
   * **Consumer Group**: Consumer group ID

### Publishing Messages

Kafka'ya mesaj yayınlama:

* **Topic**: Kafka topic adı
* **Key**: Mesaj anahtarı (opsiyonel)
* **Value**: Mesaj değeri
* **Headers**: Mesaj header'ları (opsiyonel)
* **Partition**: Partition numarası (opsiyonel)

**Örnek:**
```
Kafka Connector:
- Connection: Kafka Connection
- Operation: Publish
- Topic: orders
- Key: ${orderId}
- Value: ${orderData}
- Headers:
    source: taskflow
    timestamp: ${currentTimestamp}
```

### Consuming Messages

Kafka'dan mesaj tüketme:

* **Topic**: Kafka topic adı
* **Consumer Group**: Consumer group ID
* **Auto Commit**: Otomatik commit
* **Max Poll Records**: Maksimum poll kayıt sayısı

**Örnek:**
```
Kafka Connector:
- Connection: Kafka Connection
- Operation: Consume
- Topic: orders
- Consumer Group: order-processors
- Max Poll Records: 10
- Output: messages (array)
```

## RabbitMQ Connector

### RabbitMQ Overview

RabbitMQ, açık kaynaklı mesaj broker'ıdır.

### RabbitMQ Connection

RabbitMQ Connector kullanmak için önce bir RabbitMQ Connection oluşturmanız gerekir:

1. **Connection Management** sayfasına gidin
2. **RabbitMQ Queue Connections** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin:
   * **Name**: Bağlantı adı
   * **Host**: RabbitMQ sunucu adresi
   * **Port**: Port numarası (varsayılan: 5672)
   * **Virtual Host**: Virtual host adı (varsayılan: /)
   * **Username**: Kullanıcı adı
   * **Password**: Şifre (Secret Manager'da saklanır)

Daha fazla bilgi için [RabbitMQ Queue Connections](/tr/entegrasyon-otomasyon/baglanti-yonetimi/message-queue-connections) sayfasına bakın.

### Publishing Messages

RabbitMQ'ya mesaj yayınlama:

* **Exchange**: Exchange adı
* **Routing Key**: Routing key
* **Queue**: Queue adı (direct exchange için)
* **Message**: Mesaj içeriği
* **Properties**: Mesaj özellikleri

**Örnek:**
```
RabbitMQ Connector:
- Connection: RabbitMQ Connection
- Operation: Publish
- Exchange: orders
- Routing Key: order.created
- Message: ${orderData}
- Properties:
    contentType: application/json
    deliveryMode: 2 (persistent)
```

### Consuming Messages

RabbitMQ'dan mesaj tüketme:

* **Queue**: Queue adı
* **Auto Acknowledge**: Otomatik onaylama
* **Prefetch Count**: Prefetch sayısı

**Örnek:**
```
RabbitMQ Connector:
- Connection: RabbitMQ Connection
- Operation: Consume
- Queue: order-queue
- Auto Acknowledge: false
- Prefetch Count: 10
- Output: message
```

## ActiveMQ Connector

### ActiveMQ Overview

Apache ActiveMQ, açık kaynaklı mesaj broker'ıdır.

### ActiveMQ Connection

ActiveMQ Connector kullanmak için önce bir ActiveMQ Connection oluşturmanız gerekir:

1. **Connection Management** sayfasına gidin
2. **ActiveMQ Queue Connections** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin:
   * **Name**: Bağlantı adı
   * **Broker URL**: ActiveMQ broker URL'i (örn: tcp://localhost:61616)
   * **Username**: Kullanıcı adı
   * **Password**: Şifre (Secret Manager'da saklanır)

Daha fazla bilgi için [ActiveMQ Queue Connections](/tr/entegrasyon-otomasyon/baglanti-yonetimi/message-queue-connections) sayfasına bakın.

### Publishing Messages

ActiveMQ'ya mesaj yayınlama:

* **Destination**: Destination adı (queue veya topic)
* **Destination Type**: Destination tipi (Queue veya Topic)
* **Message**: Mesaj içeriği
* **Properties**: Mesaj özellikleri

**Örnek:**
```
ActiveMQ Connector:
- Connection: ActiveMQ Connection
- Operation: Publish
- Destination: orders.queue
- Destination Type: Queue
- Message: ${orderData}
- Properties:
    JMSType: OrderCreated
    JMSDeliveryMode: PERSISTENT
```

### Consuming Messages

ActiveMQ'dan mesaj tüketme:

* **Destination**: Destination adı
* **Destination Type**: Destination tipi
* **Message Selector**: Mesaj seçici (opsiyonel)

**Örnek:**
```
ActiveMQ Connector:
- Connection: ActiveMQ Connection
- Operation: Consume
- Destination: orders.queue
- Destination Type: Queue
- Message Selector: JMSType = 'OrderCreated'
- Output: message
```

## Message Format

### JSON Messages

JSON formatında mesajlar:

```json
{
  "orderId": "ORD-12345",
  "customerId": "CUST-001",
  "items": [
    {"productId": "PROD-001", "quantity": 2},
    {"productId": "PROD-002", "quantity": 1}
  ],
  "totalAmount": 1500.00,
  "timestamp": "2024-01-15T10:30:00Z"
}
```

### XML Messages

XML formatında mesajlar:

```xml
<order>
  <orderId>ORD-12345</orderId>
  <customerId>CUST-001</customerId>
  <items>
    <item>
      <productId>PROD-001</productId>
      <quantity>2</quantity>
    </item>
  </items>
  <totalAmount>1500.00</totalAmount>
  <timestamp>2024-01-15T10:30:00Z</timestamp>
</order>
```

## Error Handling

### Message Publishing Errors

Mesaj yayınlama hataları:

* **Connection Errors**: Bağlantı hataları
* **Authentication Errors**: Kimlik doğrulama hataları
* **Serialization Errors**: Serileştirme hataları
* **Queue/Topic Errors**: Queue/Topic hataları

### Message Consumption Errors

Mesaj tüketme hataları:

* **Connection Errors**: Bağlantı hataları
* **Deserialization Errors**: Ters serileştirme hataları
* **Processing Errors**: İşleme hataları
* **Acknowledgment Errors**: Onaylama hataları

### Error Handling Strategies

Hata yönetimi stratejileri:

* **Retry on Failure**: Başarısızlıkta yeniden deneme
* **Dead Letter Queue**: Ölü mektup kuyruğu
* **Error Logging**: Hata loglama
* **Error Notification**: Hata bildirimi

## Usage Examples

### Example 1: Publish Order Event (Kafka)

```
Kafka Connector:
- Connection: Kafka Connection
- Operation: Publish
- Topic: order-events
- Key: ${orderId}
- Value: ${orderEventData}
- Headers:
    eventType: OrderCreated
    source: TaskFlow
```

### Example 2: Consume Order Events (RabbitMQ)

```
RabbitMQ Connector:
- Connection: RabbitMQ Connection
- Operation: Consume
- Queue: order-events-queue
- Auto Acknowledge: false
- On Message:
    Process Order
    Acknowledge Message
```

### Example 3: Publish Notification (ActiveMQ)

```
ActiveMQ Connector:
- Connection: ActiveMQ Connection
- Operation: Publish
- Destination: notifications.topic
- Destination Type: Topic
- Message: ${notificationData}
```

## Best Practices

### Message Design

* **Structured Messages**: Yapılandırılmış mesajlar kullanın
* **Message Versioning**: Mesaj versiyonlama
* **Idempotency**: İdempotent işlemler
* **Message Size**: Mesaj boyutunu optimize edin

### Performance

* **Batch Publishing**: Toplu yayınlama
* **Connection Pooling**: Bağlantı havuzlama
* **Async Processing**: Asenkron işleme
* **Consumer Concurrency**: Consumer eşzamanlılığı

### Reliability

* **Message Persistence**: Mesaj kalıcılığı
* **Acknowledgment**: Onaylama mekanizması
* **Dead Letter Queue**: Ölü mektup kuyruğu
* **Monitoring**: İzleme ve alerting

### Security

* **Authentication**: Kimlik doğrulama
* **Encryption**: Şifreleme
* **Access Control**: Erişim kontrolü
* **Secure Credentials**: Güvenli kimlik bilgileri
