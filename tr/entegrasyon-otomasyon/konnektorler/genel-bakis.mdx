<Info>
Konnektörler iki ana alanda kullanılır: **Task Flow'larda** (API Integrator) iş akışları oluşturmak ve **Ağ Geçidi Ortamları'nda** API trafik loglarını farklı sistemlere göndermek. Her konnektör, belirli bir sistem veya protokol ile iletişim kurmak için özelleştirilmiştir ve form tabanlı yapılandırma ile kod yazmaya gerek kalmadan kolayca kullanılabilir.
</Info>

## Modülün Temel Bileşenleri

Konnektörler modülü, Apinizer platformunda entegrasyon ve otomasyon işlemlerinin temelini oluşturur. Bu modül ile:

<CardGroup cols={2}>
  <Card title="Task Flow Entegrasyonları" icon="puzzle-piece">
    Task Flow'larda farklı sistemlere bağlantı kurarak veri alışverişi sağlayabilirsiniz.
  </Card>
  
  <Card title="API Trafik Log Yönetimi" icon="file-lines">
    API trafik loglarını farklı hedef sistemlere yönlendirebilirsiniz.
  </Card>
  
  <Card title="Veri İşlemleri" icon="database">
    Veritabanı işlemleri, mesaj kuyruğu operasyonları ve dosya transferleri gerçekleştirebilirsiniz.
  </Card>
  
  <Card title="İletişim ve Loglama" icon="envelope">
    E-posta gönderimi, bildirim ve loglama işlemlerini yapılandırabilirsiniz.
  </Card>
</CardGroup>

<Tip>
**Elasticsearch Konnektörü** özellikle önemlidir. Apinizer Yönetim Konsolunda API Trafiğinin sorgulanması ve analitik olarak görselleştirilmesi ancak ilgili ortama Elasticsearch Konnektörü eklenmesi ile mümkün olmaktadır. Elasticsearch Konnektörü olmadan analitik ekranlar, API Trafiği sekmeleri ve rapor oluşturucu menüleri kullanılamaz.
</Tip>

<Warning>
Ağ Geçidi Ortamları'nda konnektörler kullanılırken, API trafik loglarının yapısı ve gizlilik ayarları önemlidir. Hassas verilerin log kayıtlarında açık bir şekilde görünmemesi için API Proxy Trafik Gizlilik Ayarları yapılandırılmalıdır.
</Warning>
