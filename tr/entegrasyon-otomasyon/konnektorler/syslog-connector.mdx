Syslog Connector özellikleri:

* **Syslog Protocol**: Syslog protokolü desteği
* **UDP/TCP Support**: UDP ve TCP desteği
* **Multiple Formats**: Çoklu format desteği
* **Priority Levels**: Öncelik seviyeleri

## Syslog Connection

### Connection Configuration

Syslog Connector kullanmak için önce bir Syslog Connection oluşturmanız gerekir:

1. **Connection Management** sayfasına gidin
2. **Syslog Connections** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin:
   * **Name**: Bağlantı adı
   * **Syslog Host**: Syslog sunucu adresi
   * **Syslog Port**: Port numarası (varsayılan: UDP 514, TCP 514)
   * **Protocol**: UDP veya TCP
   * **Format**: Syslog formatı (RFC3164, RFC5424)

Daha fazla bilgi için [Syslog Connections](/tr/entegrasyon-otomasyon/baglanti-yonetimi/syslog-connections) sayfasına bakın.

## Syslog Formats

### RFC3164 Format

RFC3164 formatı (BSD Syslog):

```
<priority>timestamp hostname tag: message
```

**Örnek:**
```
<134>Jan 15 10:30:00 server01 taskflow: Order processed successfully
```

### RFC5424 Format

RFC5424 formatı (Structured Syslog):

```
<priority>version timestamp hostname app-name procid msgid structured-data message
```

**Örnek:**
```
<134>1 2024-01-15T10:30:00.000Z server01 taskflow 12345 - [order@12345 orderId="ORD-001"] Order processed successfully
```

## Priority Levels

### Facility Codes

Facility kodları:

* **0**: kernel messages
* **1**: user-level messages
* **2**: mail system
* **3**: system daemons
* **4**: security/authorization messages
* **5**: messages generated internally by syslogd
* **6**: line printer subsystem
* **7**: network news subsystem
* **8**: UUCP subsystem
* **9**: clock daemon
* **10**: security/authorization messages
* **11**: FTP daemon
* **12**: NTP subsystem
* **13**: log audit
* **14**: log alert
* **15**: clock daemon
* **16-23**: local use

### Severity Levels

Severity seviyeleri:

* **0**: Emergency
* **1**: Alert
* **2**: Critical
* **3**: Error
* **4**: Warning
* **5**: Notice
* **6**: Informational
* **7**: Debug

### Priority Calculation

Öncelik hesaplama:

```
Priority = (Facility * 8) + Severity
```

## Syslog Message Configuration

### Basic Message

Temel mesaj:

```json
{
  "facility": "local0",
  "severity": "info",
  "message": "${logMessage}",
  "hostname": "${hostname}",
  "appName": "TaskFlow"
}
```

### Detailed Message Configuration

Detaylı mesaj yapılandırması:

```json
{
  "operation": "sendSyslog",
  "host": "${syslogHost}",
  "port": 514,
  "protocol": "UDP",
  "format": "RFC3164",
  "facility": 16,
  "severity": 6,
  "hostname": "${hostname}",
  "tag": "taskflow",
  "message": "${logMessage}"
}
```

### Structured Message

Yapılandırılmış mesaj:

```json
{
  "operation": "sendSyslog",
  "host": "${syslogHost}",
  "port": 514,
  "protocol": "TCP",
  "format": "RFC5424",
  "facility": 16,
  "severity": 6,
  "hostname": "${hostname}",
  "appName": "taskflow",
  "procId": "${processId}",
  "msgId": "ORDER-001",
  "structuredData": {
    "order@12345": {
      "orderId": "${orderId}",
      "customerId": "${customerId}",
      "status": "${status}"
    }
  },
  "message": "Order processed successfully"
}
```

### Message with Priority

Öncelikli mesaj:

```json
{
  "operation": "sendSyslog",
  "host": "${syslogHost}",
  "port": 514,
  "protocol": "UDP",
  "format": "RFC3164",
  "priority": 134,
  "hostname": "${hostname}",
  "tag": "taskflow",
  "message": "${logMessage}"
}
```

## Usage Examples

### Example 1: Info Log

```
Task Completion
└─→ Syslog Connector (Bilgi logu)
    - Severity: Informational (6)
    - Facility: Local0 (16)
    - Message: "Task completed successfully"
```

### Example 2: Error Log

```
Error Handler
└─→ Syslog Connector (Hata logu)
    - Severity: Error (3)
    - Facility: Local0 (16)
    - Message: ${errorMessage}
    - Structured Data: ${errorDetails}
```

### Example 3: Audit Log

```
Audit Event
└─→ Syslog Connector (Denetim logu)
    - Severity: Notice (5)
    - Facility: Security (4)
    - Message: ${auditMessage}
    - Structured Data: ${auditData}
```

## Protocol Selection

### UDP Protocol

UDP protokolü:

* **Faster**: Daha hızlı
* **No Guaranteed Delivery**: Garantili teslimat yok
* **Lower Overhead**: Daha düşük overhead
* **Use Case**: Yüksek hacimli loglar için

### TCP Protocol

TCP protokolü:

* **Reliable**: Güvenilir teslimat
* **Guaranteed Delivery**: Garantili teslimat
* **Higher Overhead**: Daha yüksek overhead
* **Use Case**: Kritik loglar için

## Error Handling

### Syslog Errors

Syslog hataları:

* **Connection Errors**: Bağlantı hataları
* **Network Errors**: Ağ hataları
* **Format Errors**: Format hataları
* **Timeout Errors**: Zaman aşımı hataları

Hata yönetimi stratejileri:

### Message Design

* **Clear Messages**: Net mesajlar
* **Structured Data**: Yapılandırılmış veri
