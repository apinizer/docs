Notification Connector, Task Flow'larda çeşitli kanallar üzerinden bildirim göndermenizi sağlar. E-posta, SMS, push notification ve diğer bildirim kanallarını destekler.

## Genel Bakış

Notification Connector özellikleri:

* **Multiple Channels**: Çoklu kanal desteği
* **Template Support**: Şablon desteği
* **Priority Levels**: Öncelik seviyeleri
* **Delivery Tracking**: Teslimat takibi

## Notification Channels

### Email Channel

E-posta kanalı:

* **SMTP Configuration**: SMTP yapılandırması
* **HTML/Text Support**: HTML ve metin desteği
* **Attachments**: Ek desteği
* **Templates**: E-posta şablonları

### SMS Channel

SMS kanalı:

* **SMS Provider**: SMS sağlayıcı entegrasyonu
* **Character Limit**: Karakter limiti
* **Delivery Confirmation**: Teslimat onayı

### Push Notification Channel

Push bildirim kanalı:

* **Mobile Push**: Mobil push bildirimleri
* **Web Push**: Web push bildirimleri
* **Platform Support**: Platform desteği (iOS, Android, Web)

## Notification Configuration

### Basic Notification

Temel bildirim:

```json
{
  "channel": "email",
  "recipient": "${recipientEmail}",
  "subject": "Alert: ${alertType}",
  "message": "${alertMessage}",
  "priority": "high"
}
```

### Detailed Notification Configuration

Detaylı bildirim yapılandırması:

```json
{
  "channel": "email",
  "recipient": "${userEmail}",
  "subject": "Bildirim: ${eventType}",
  "message": "${notificationMessage}",
  "priority": "normal"
}
```

### Template-based Notification

Şablon tabanlı bildirim:

```json
{
  "channel": "email",
  "recipient": "${userEmail}",
  "template": "order-confirmation",
  "templateVariables": {
    "orderId": "${orderId}",
    "customerName": "${customerName}",
    "totalAmount": "${totalAmount}"
  },
  "priority": "high"
}
```

### Multi-channel Notification

Çoklu kanal bildirimi:

```json
{
  "channels": ["email", "sms", "push"],
  "recipient": {
    "email": "${userEmail}",
    "phone": "${userPhone}",
    "deviceToken": "${deviceToken}"
  },
  "message": "${notificationMessage}",
  "priority": "urgent"
}
```

## Priority Levels

### Priority Configuration

Öncelik seviyeleri:

* **Low**: Düşük öncelik
* **Normal**: Normal öncelik (varsayılan)
* **High**: Yüksek öncelik
* **Urgent**: Acil öncelik

### Priority-based Routing

Öncelik tabanlı yönlendirme:

* **Channel Selection**: Kanal seçimi
* **Retry Policy**: Yeniden deneme politikası
* **Delivery Time**: Teslimat zamanı

## Notification Templates

### Template Definition

Şablon tanımlama:

```json
{
  "templateId": "order-confirmation",
  "channels": ["email", "sms"],
  "subject": "Sipariş Onayı - {{orderId}}",
  "body": {
    "email": "<html>...</html>",
    "sms": "Siparişiniz {{orderId}} onaylandı. Tutar: {{totalAmount}} TL"
  }
}
```

### Template Variables

Şablon değişkenleri:

* **Variable Substitution**: Değişken yerine koyma
* **Conditional Content**: Koşullu içerik
* **Dynamic Content**: Dinamik içerik

## Delivery Tracking

### Delivery Status

Teslimat durumu:

* **Pending**: Beklemede
* **Sent**: Gönderildi
* **Delivered**: Teslim edildi
* **Failed**: Başarısız
* **Bounced**: Geri döndü

### Delivery Confirmation

Teslimat onayı:

* **Read Receipt**: Okundu bilgisi
* **Delivery Receipt**: Teslimat bilgisi
* **Status Updates**: Durum güncellemeleri

## Usage Examples

### Example 1: Order Confirmation

```
Database Connector (Sipariş oluşturma)
├─→ Notification Connector (E-posta bildirimi)
│   - Channel: email
│   - Template: order-confirmation
│   - Recipient: ${customerEmail}
└─→ Notification Connector (SMS bildirimi)
    - Channel: sms
    - Message: "Siparişiniz oluşturuldu"
    - Recipient: ${customerPhone}
```

### Example 2: Error Notification

```
Error Handler
└─→ Notification Connector (Hata bildirimi)
    - Channel: email
    - Priority: urgent
    - Recipient: ${adminEmail}
    - Subject: "Hata Bildirimi"
    - Message: ${errorDetails}
```

### Example 3: Multi-channel Alert

```
Alert Trigger
└─→ Notification Connector (Çoklu kanal)
    - Channels: [email, sms, push]
    - Priority: high
    - Recipient: ${userContactInfo}
    - Message: ${alertMessage}
```

## Error Handling

### Notification Errors

Bildirim hataları:

* **Channel Errors**: Kanal hataları
* **Delivery Failures**: Teslimat başarısızlıkları
* **Template Errors**: Şablon hataları
* **Recipient Errors**: Alıcı hataları

### Error Handling Strategies

Hata yönetimi stratejileri:

* **Retry on Failure**: Başarısızlıkta yeniden deneme
* **Fallback Channel**: Yedek kanal
* **Error Logging**: Hata loglama
* **Error Notification**: Hata bildirimi
