Email Connector, Task Flow'lar içinde SMTP sunucuları üzerinden e-posta göndermenizi sağlar. HTML ve düz metin e-postaları, ekler ve çeşitli e-posta özelliklerini destekler.

## Genel Bakış

Email Connector, bildirimler, uyarılar ve diğer e-posta iletişimlerini entegrasyon iş akışlarınızın bir parçası olarak göndermenize olanak tanır.

## Yapılandırma

### Bağlantı Seçimi

SMTP bağlantısı seçin veya oluşturun:

**SMTP Bağlantı Özellikleri:**

* **SMTP Host**: SMTP sunucu hostname'i
* **SMTP Port**: SMTP sunucu portu (25, 465, 587)
* **Username**: SMTP kullanıcı adı
* **Password**: SMTP şifresi
* **Security**: SSL/TLS yapılandırması
* **Authentication**: Kimlik doğrulama yöntemi

## E-posta Gönderme

### Temel Yapılandırma

**E-posta Yapılandırması:**

* **From**: Gönderen e-posta adresi
* **To**: Alıcı e-posta adresleri (virgülle ayrılmış)
* **CC**: CC alıcıları (opsiyonel)
* **BCC**: BCC alıcıları (opsiyonel)
* **Subject**: E-posta konusu
* **Body**: E-posta içerik gövdesi

**Örnek:**

```json
{
  "from": "noreply@example.com",
  "to": "customer@example.com",
  "subject": "Sipariş Onayı",
  "body": "Siparişiniz onaylandı."
}
```

### HTML E-posta

HTML formatlı e-postalar gönderin:

**HTML Gövdesi:**

```html
<html>
  <body>
    <h1>Sipariş Onayı</h1>
    <p>Sayın ${customerName},</p>
    <p>Siparişiniz #${orderId} onaylandı.</p>
    <p>Toplam: $${orderTotal}</p>
  </body>
</html>
```

**Yapılandırma:**

```json
{
  "from": "noreply@example.com",
  "to": "${customerEmail}",
  "subject": "Sipariş Onayı - Sipariş #${orderId}",
  "body": "${htmlBody}",
  "contentType": "text/html"
}
```

### Düz Metin E-posta

Düz metin e-postalar gönderin:

**Yapılandırma:**

```json
{
  "from": "noreply@example.com",
  "to": "${customerEmail}",
  "subject": "Sipariş Onayı",
  "body": "Siparişiniz onaylandı.",
  "contentType": "text/plain"
}
```

### Ekli E-posta

E-postalara dosya ekleyin:

**Ek Yapılandırması:**

* **File Path**: Eklenecek dosyanın yolu
* **File Name**: Ekli dosya için ad
* **Content Type**: Ekin MIME türü
* **Inline**: Ekin inline olup olmadığı

**Örnek:**

```json
{
  "from": "noreply@example.com",
  "to": "${customerEmail}",
  "subject": "Sipariş Faturası",
  "body": "Lütfen faturanızı ekte bulun.",
  "attachments": [
    {
      "filePath": "${invoiceFilePath}",
      "fileName": "invoice-${orderId}.pdf",
      "contentType": "application/pdf"
    }
  ]
}
```

### Çoklu Ekler

Birden fazla dosya ekleyin:

```json
{
  "attachments": [
    {
      "filePath": "${invoicePath}",
      "fileName": "invoice.pdf"
    },
    {
      "filePath": "${receiptPath}",
      "fileName": "receipt.pdf"
    }
  ]
}
```

## E-posta Şablonları

### Şablon Kullanımı

Tutarlı formatlama için e-posta şablonlarını kullanın:

**Şablon Değişkenleri:**

```html
<html>
  <body>
    <h1>Sipariş Onayı</h1>
    <p>Sayın ${customerName},</p>
    <p>Siparişiniz #${orderId} onaylandı.</p>
    <table>
      <tr>
        <th>Ürün</th>
        <th>Miktar</th>
        <th>Fiyat</th>
      </tr>
      ${orderItems}
    </table>
    <p>Toplam: $${orderTotal}</p>
  </body>
</html>
```

**Şablon Kullanımı:**

```json
{
  "template": "order-confirmation",
  "templateVariables": {
    "customerName": "${customerName}",
    "orderId": "${orderId}",
    "orderItems": "${orderItemsHtml}",
    "orderTotal": "${orderTotal}"
  }
}
```

## Dinamik İçerik

### Değişken Yerine Koyma

E-posta içeriğinde Task Flow değişkenlerini kullanın:

**Değişkenli Konu:**

```
Sipariş Onayı - Sipariş #${orderId}
```

**Değişkenli Gövde:**

```
Sayın ${customerName},

Siparişiniz #${orderId} onaylandı.
Toplam tutar: $${orderTotal}

Satın aldığınız için teşekkürler!
```

### Koşullu İçerik

Koşullu içerik ekleyin:

```html
${#if isPremium}
  <p>Premium müşteri olarak %10 indirim alıyorsunuz!</p>
${/if}
```

## Hata Yönetimi

### E-posta Gönderme Hataları

E-posta gönderme hatalarını yönetin:

**Hata Türleri:**

* **SMTP Bağlantı Hatası**: SMTP sunucusuna bağlanılamadı
* **Kimlik Doğrulama Hatası**: Geçersiz kimlik bilgileri
* **Geçersiz Adres**: Geçersiz e-posta adresi
* **Zaman Aşımı Hatası**: SMTP işlemi zaman aşımı

**Hata Yönetimi:**

* **Retry Logic**: Geçici hatalarda retry
* **Error Logging**: E-posta gönderme hatalarını loglayın
* **Fallback**: Alternatif bildirim yöntemi kullanın

## Yaygın Desenler

### Bildirim Deseni

Bildirim e-postaları gönderin:

```
Task Flow:
  ├─ Process Order
  ├─ Format Email
  └─ Send Email (Email Connector)
```

### Uyarı Deseni

Uyarı e-postaları gönderin:

```
Task Flow:
  ├─ Detect Error
  ├─ Format Alert
  └─ Send Alert Email (Email Connector)
```

### Rapor Deseni

Ekli rapor e-postaları gönderin:

```
Task Flow:
  ├─ Generate Report
  ├─ Create PDF
  └─ Send Report Email (Email Connector)
```
