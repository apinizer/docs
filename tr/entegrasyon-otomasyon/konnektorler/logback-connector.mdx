Logback Connector, Task Flow'larda Logback logging framework'ü ile log kayıtları göndermenizi sağlar. Merkezi log yönetimi sistemlerine log kayıtları gönderebilirsiniz.

## Genel Bakış

Logback Connector özellikleri:

* **Logback Integration**: Logback entegrasyonu
* **Multiple Appenders**: Çoklu appender desteği
* **Log Levels**: Log seviyeleri
* **Structured Logging**: Yapılandırılmış loglama

## Logback Connection

### Connection Configuration

Logback Connector kullanmak için önce bir Logback Connection oluşturmanız gerekir:

1. **Connection Management** sayfasına gidin
2. **Logback Connections** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin:
   * **Name**: Bağlantı adı
   * **Appender Type**: Appender tipi (File, Console, Socket, SMTP, vb.)
   * **Configuration**: Logback yapılandırması

Daha fazla bilgi için [Logback Connections](/tr/entegrasyon-otomasyon/baglanti-yonetimi/logback-connections) sayfasına bakın.

## Log Levels

### Supported Levels

Desteklenen log seviyeleri:

* **TRACE**: En detaylı log seviyesi
* **DEBUG**: Debug bilgileri
* **INFO**: Bilgilendirme mesajları
* **WARN**: Uyarı mesajları
* **ERROR**: Hata mesajları

## Logback Configuration

### Basic Logging

Temel loglama:

```json
{
  "level": "INFO",
  "message": "${logMessage}",
  "logger": "${loggerName}",
  "properties": {
    "orderId": "${orderId}",
    "customerId": "${customerId}"
  }
}
```

### Detailed Logging Configuration

Detaylı loglama yapılandırması:

```json
{
  "operation": "log",
  "level": "INFO",
  "message": "${logMessage}",
  "logger": "taskflow"
}
```

### Logging with Context

Bağlam ile loglama:

```json
{
  "operation": "log",
  "level": "INFO",
  "message": "${logMessage}",
  "logger": "taskflow",
  "context": {
    "taskId": "${taskId}",
    "flowId": "${flowId}",
    "correlationId": "${correlationId}"
  }
}
```

### Structured Logging

Yapılandırılmış loglama:

```json
{
  "operation": "log",
  "level": "INFO",
  "message": "Order processed",
  "logger": "taskflow",
  "structuredData": {
    "orderId": "${orderId}",
    "customerId": "${customerId}",
    "totalAmount": ${totalAmount},
    "status": "${status}"
  }
}
```

### Exception Logging

Hata loglama:

```json
{
  "operation": "log",
  "level": "ERROR",
  "message": "${errorMessage}",
  "logger": "taskflow",
  "exception": {
    "type": "${exceptionType}",
    "message": "${exceptionMessage}",
    "stackTrace": "${stackTrace}"
  }
}
```

## Appender Types

### File Appender

Dosya appender:

* **File Path**: Log dosyası yolu
* **Rolling Policy**: Dosya döndürme politikası
* **Pattern**: Log formatı

### Console Appender

Konsol appender:

* **Output Stream**: Çıktı akışı (stdout/stderr)
* **Pattern**: Log formatı

### Socket Appender

Soket appender:

* **Host**: Sunucu adresi
* **Port**: Port numarası
* **Protocol**: Protokol

### SMTP Appender

SMTP appender:

* **SMTP Configuration**: SMTP yapılandırması
* **Recipients**: Alıcılar
* **Subject**: E-posta konusu

## Usage Examples

### Example 1: Info Log

```
Task Completion
└─→ Logback Connector (Bilgi logu)
    - Level: INFO
    - Message: "Task completed successfully"
    - Logger: taskflow
```

### Example 2: Error Log

```
Error Handler
└─→ Logback Connector (Hata logu)
    - Level: ERROR
    - Message: ${errorMessage}
    - Logger: taskflow
    - Exception: ${exception}
```

### Example 3: Debug Log

```
Debug Point
└─→ Logback Connector (Debug logu)
    - Level: DEBUG
    - Message: ${debugMessage}
    - Logger: taskflow
    - Context: ${debugContext}
```

## Log Patterns

### Pattern Configuration

Pattern yapılandırması:

```
%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n
```

### Pattern Variables

Pattern değişkenleri:

* **%d**: Tarih/saat
* **%thread**: Thread adı
* **%level**: Log seviyesi
* **%logger**: Logger adı
* **%msg**: Mesaj
* **%n**: Yeni satır
* **%mdc**: MDC (Mapped Diagnostic Context)

## MDC (Mapped Diagnostic Context)

### MDC Usage

MDC kullanımı:

```json
{
  "operation": "log",
  "level": "INFO",
  "message": "${logMessage}",
  "logger": "taskflow",
  "mdc": {
    "taskId": "${taskId}",
    "flowId": "${flowId}",
    "correlationId": "${correlationId}",
    "userId": "${userId}"
  }
}
```
