Database Connector, Task Flow'larda veritabanı işlemleri gerçekleştirmenizi sağlar. MySQL, PostgreSQL, Oracle, MSSQL, DB2 gibi popüler veritabanlarına bağlanabilir ve SQL sorguları çalıştırabilirsiniz.

## Genel Bakış

Database Connector sayesinde:

* Veritabanı sorguları çalıştırabilirsiniz
* INSERT, UPDATE, DELETE işlemleri yapabilirsiniz
* Stored procedure'ları çağırabilirsiniz
* Transaction yönetimi yapabilirsiniz
* Batch işlemler gerçekleştirebilirsiniz

## Desteklenen Veritabanları

Database Connector şu veritabanlarını destekler:

* **MySQL**: MySQL 5.7+
* **PostgreSQL**: PostgreSQL 9.6+
* **Oracle**: Oracle 11g+
* **MSSQL**: Microsoft SQL Server 2012+
* **DB2**: IBM DB2 10.5+

## Bağlantı Yapılandırması

Database Connector kullanmak için önce bir Database Connection oluşturmanız gerekir.

### Connection Oluşturma

1. **Connection Management** menüsüne gidin
2. **Database Connections** sekmesine tıklayın
3. **New Connection** butonuna tıklayın
4. Bağlantı bilgilerini girin:
   * **Name**: Bağlantı adı
   * **Database Type**: Veritabanı türü
   * **Host**: Sunucu adresi
   * **Port**: Port numarası
   * **Database Name**: Veritabanı adı
   * **Username**: Kullanıcı adı
   * **Password**: Şifre
5. **Test Connection** butonuna tıklayın
6. **Save** butonuna tıklayın

## Database Connector Yapılandırması

### Connector Ekleme

1. Task Flow tasarımında **Database Connector**'ı ekleyin
2. Connector özelliklerinde **Connection** seçin
3. **Operation Type** seçin:
   * **Select**: SELECT sorgusu
   * **Insert**: INSERT işlemi
   * **Update**: UPDATE işlemi
   * **Delete**: DELETE işlemi
   * **Stored Procedure**: Stored procedure çağrısı
   * **Custom SQL**: Özel SQL sorgusu

### SELECT İşlemi

SELECT sorgusu çalıştırmak için:

1. **Operation Type** olarak **Select** seçin
2. **SQL Query** alanına sorguyu yazın:

```sql
SELECT id, name, email FROM users WHERE status = ?
```

3. **Parameters** sekmesinde parametreleri tanımlayın:

```javascript
{
  "status": "active"
}
```

4. **Output Mapping** ile sonuçları map edin

### INSERT İşlemi

INSERT işlemi yapmak için:

1. **Operation Type** olarak **Insert** seçin
2. **Table Name** alanına tablo adını girin
3. **Columns** alanında kolonları tanımlayın:

```javascript
{
  "name": "${input.name}",
  "email": "${input.email}",
  "status": "active"
}
```

4. **Return Generated Keys** seçeneğini açın (opsiyonel)

### UPDATE İşlemi

UPDATE işlemi yapmak için:

1. **Operation Type** olarak **Update** seçin
2. **Table Name** alanına tablo adını girin
3. **Set Clause** alanında güncellenecek kolonları tanımlayın:

```javascript
{
  "name": "${input.name}",
  "email": "${input.email}"
}
```

4. **Where Clause** alanında koşulu tanımlayın:

```sql
id = ?
```

### DELETE İşlemi

DELETE işlemi yapmak için:

1. **Operation Type** olarak **Delete** seçin
2. **Table Name** alanına tablo adını girin
3. **Where Clause** alanında koşulu tanımlayın:

```sql
id = ? AND status = ?
```

### Stored Procedure Çağrısı

Stored procedure çağırmak için:

1. **Operation Type** olarak **Stored Procedure** seçin
2. **Procedure Name** alanına procedure adını girin
3. **Parameters** sekmesinde parametreleri tanımlayın:

```javascript
{
  "param1": "${input.value1}",
  "param2": "${input.value2}"
}
```

4. **Output Parameters** tanımlayın (opsiyonel)

### Custom SQL

Özel SQL sorgusu çalıştırmak için:

1. **Operation Type** olarak **Custom SQL** seçin
2. **SQL Query** alanına sorguyu yazın:

```sql
SELECT u.*, p.name as profile_name 
FROM users u 
LEFT JOIN profiles p ON u.profile_id = p.id 
WHERE u.status = ?
```

3. **Parameters** sekmesinde parametreleri tanımlayın

## Parametreler ve Mapping

### Input Parameters

Girdi parametrelerini tanımlamak için:

```javascript
{
  "userId": "${variables.userId}",
  "status": "active",
  "limit": 100
}
```

### Output Mapping

Çıktı mapping'i yapmak için:

```javascript
{
  "id": "${result.id}",
  "name": "${result.name}",
  "email": "${result.email}"
}
```

### Array Results

Çoklu sonuçları işlemek için:

```javascript
// ForEach ile işleme
forEach(result in results) {
    // Her sonuç için işlem
}
```

## Transaction Yönetimi

### Transaction Başlatma

Transaction başlatmak için:

1. **Transaction Mode** seçeneğini açın
2. **Begin Transaction** task'ını ekleyin
3. Database işlemlerini yapın
4. **Commit Transaction** veya **Rollback Transaction** task'ını ekleyin

### Transaction Örneği

```
Begin Transaction
├─→ INSERT users
├─→ INSERT user_profiles
└─→ Decision: Success?
    ├─→ [True] → Commit Transaction
    └─→ [False] → Rollback Transaction
```

## Batch İşlemler

Toplu işlemler için:

### Batch Insert

```javascript
// Batch insert
var batch = [
    {name: "User1", email: "user1@example.com"},
    {name: "User2", email: "user2@example.com"},
    {name: "User3", email: "user3@example.com"}
];

// Batch size ayarlayın
batchSize: 100
```

### Batch Update

```javascript
// Batch update
var updates = [
    {id: 1, status: "active"},
    {id: 2, status: "inactive"},
    {id: 3, status: "active"}
];
```

## Hata Yönetimi

### Connection Errors

Bağlantı hatalarını yönetmek için:

* **Retry Mechanism**: Retry mekanizması kullanın
* **Connection Pooling**: Connection pooling kullanın
* **Timeout Settings**: Timeout ayarlarını yapın

### SQL Errors

SQL hatalarını yönetmek için:

* **Error Handling**: Try-Catch blokları kullanın
* **Validation**: Girdi doğrulama yapın
* **Error Logging**: Hata loglama yapın

## Performans Optimizasyonu

### Connection Pooling

Connection pooling kullanın:

```yaml
Connection Pool:
  Min Size: 5
  Max Size: 20
  Idle Timeout: 300 seconds
```

### Query Optimization

Sorgu optimizasyonu:

* **Indexes**: İndeksleri kullanın
* **Limit Clauses**: LIMIT kullanın
* **Efficient Queries**: Verimli sorgular yazın
* **Batch Operations**: Toplu işlemler kullanın

### Caching

Sık kullanılan sorguları cache'leyin:

* **Query Result Cache**: Sorgu sonuç cache'i
* **TTL**: Cache süresi
* **Cache Invalidation**: Cache geçersiz kılma

## Kullanım Senaryoları

### Senaryo 1: Veri Çekme

```
Database Connector (SELECT)
├─→ Script Task (Veri işleme)
└─→ HTTP Connector (API'ye gönderme)
```

### Senaryo 2: Veri Kaydetme

```
HTTP Connector (API'den veri alma)
├─→ Script Task (Veri doğrulama)
└─→ Database Connector (INSERT)
```

### Senaryo 3: Veri Güncelleme

```
Database Connector (SELECT)
├─→ Script Task (Veri güncelleme)
└─→ Database Connector (UPDATE)
```
