Webhook Connector, Task Flow'larda webhook'lar göndermenizi sağlar. Event-driven mimarilerde, sistemler arası bildirimlerde ve entegrasyon senaryolarında webhook'lar kullanabilirsiniz.

## Genel Bakış

Webhook Connector sayesinde:

* HTTP POST istekleri ile webhook gönderebilirsiniz
* Event-driven entegrasyonlar oluşturabilirsiniz
* Sistemler arası bildirimler yapabilirsiniz
* Retry ve error handling yapabilirsiniz
* Authentication yapılandırabilirsiniz

## Webhook Connector Yapılandırması

### Connector Ekleme

1. Task Flow tasarımında **Webhook Connector**'ı ekleyin
2. Connector özelliklerinde yapılandırma yapın:
   * **URL**: Webhook URL'i
   * **Method**: HTTP metodu (genellikle POST)
   * **Headers**: HTTP header'ları
   * **Body**: Webhook payload'ı
   * **Authentication**: Kimlik doğrulama

### Basit Webhook

```javascript
{
  "url": "https://api.example.com/webhooks/events",
  "method": "POST",
  "headers": {
    "Content-Type": "application/json"
  },
  "body": JSON.stringify({
    event: "task.completed",
    taskId: "${variables.taskId}",
    timestamp: new Date().toISOString()
  })
}
```

### Authentication ile Webhook

```javascript
{
  "url": "https://api.example.com/webhooks/events",
  "method": "POST",
  "headers": {
    "Content-Type": "application/json",
    "X-Webhook-Signature": "${variables.signature}",
    "Authorization": "Bearer ${variables.token}"
  },
  "body": JSON.stringify({
    event: "task.completed",
    data: "${variables.taskData}"
  })
}
```

## Webhook Payload Yapısı

### Event Payload

```javascript
{
  "event": "user.created",
  "timestamp": "2024-01-15T10:30:00Z",
  "data": {
    "userId": "12345",
    "email": "user@example.com",
    "name": "John Doe"
  },
  "metadata": {
    "source": "taskflow",
    "correlationId": "${variables.correlationId}"
  }
}
```

### Custom Payload

```javascript
{
  "webhookType": "notification",
  "recipient": "${input.email}",
  "message": "${input.message}",
  "priority": "${input.priority}",
  "attachments": "${input.attachments}"
}
```

## Retry Mekanizması

Webhook gönderme başarısız olduğunda:

```yaml
Retry Configuration:
  Enabled: true
  Max Retries: 3
  Retry Interval: 5 seconds
  Retry Strategy: Exponential Backoff
  Retry On:
    - 5xx Server Errors
    - Connection Errors
    - Timeout Errors
```

## Hata Yönetimi

### Error Handling

```javascript
try {
    var response = webhookConnector.execute();
    if (response.statusCode >= 400) {
        logger.warn("Webhook failed", {
            statusCode: response.statusCode,
            body: response.body
        });
        // Fallback logic
    }
} catch (error) {
    logger.error("Webhook execution failed", error);
    // Error handling
}
```

## Kullanım Senaryoları

### Senaryo 1: Event Notification

```
Task Completion
├─→ Webhook Connector (Event bildirimi)
└─→ Logback Connector (Log kaydı)
```

### Senaryo 2: System Integration

```
Database Connector (Veri değişikliği)
├─→ Script Task (Event oluşturma)
└─→ Webhook Connector (Sistem bildirimi)
```

### Senaryo 3: Multi-Webhook

```
Event Occurred
├─→ Parallel Start
│   ├─→ Webhook Connector (Webhook 1)
│   ├─→ Webhook Connector (Webhook 2)
│   └─→ Webhook Connector (Webhook 3)
└─→ Parallel End
```
