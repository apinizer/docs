Cron expressions, zamanlanmış işlerin ne zaman çalıştırılacağını tanımlamak için kullanılan bir zamanlama formatıdır. Bu sayfada cron expression syntax'ını ve kullanım örneklerini öğreneceksiniz.

## Genel Bakış

Cron expressions sayesinde:

* Esnek zamanlama yapabilirsiniz
* Periyodik işlemler tanımlayabilirsiniz
* Karmaşık zamanlama senaryoları oluşturabilirsiniz
* Zaman dilimi desteği kullanabilirsiniz

## Cron Expression Formatı

Cron expression şu formatta yazılır:

```
┌───────────── dakika (0 - 59)
│ ┌───────────── saat (0 - 23)
│ │ ┌───────────── gün (1 - 31)
│ │ │ ┌───────────── ay (1 - 12)
│ │ │ │ ┌───────────── hafta günü (0 - 6) (0 = Pazar)
│ │ │ │ │
* * * * *
```

### Alanlar

* **Dakika (0-59)**: Dakika
* **Saat (0-23)**: Saat
* **Gün (1-31)**: Ayın günü
* **Ay (1-12)**: Ay
* **Hafta Günü (0-6)**: Haftanın günü (0 = Pazar, 6 = Cumartesi)

## Özel Karakterler

### Yıldız (*)

Tüm değerleri temsil eder:

```
* * * * *  → Her dakika
0 * * * *  → Her saat başı
0 0 * * *  → Her gün saat 00:00
```

### Virgül (,)

Değer listesi:

```
0 9,17 * * *  → Her gün saat 09:00 ve 17:00
0 0 1,15 * *  → Her ayın 1. ve 15. günü saat 00:00
```

### Tire (-)

Değer aralığı:

```
0 9-17 * * *  → Her gün saat 09:00-17:00 arası her saat başı
0 0 1-5 * *   → Her ayın 1-5. günleri saat 00:00
```

### Eğik Çizgi (/)

Adım değeri:

```
*/5 * * * *   → Her 5 dakikada bir
0 */2 * * *   → Her 2 saatte bir
0 0 */3 * *   → Her 3 günde bir
```

### Soru İşareti (?)

Sadece gün veya hafta günü için "belirtilmemiş":

```
0 0 ? * *     → Her gün saat 00:00
0 0 ? * MON   → Her Pazartesi saat 00:00
```

## Yaygın Cron Expression Örnekleri

### Her Dakika

```
* * * * *
```

### Her Saat Başı

```
0 * * * *
```

### Her Gün Saat 00:00

```
0 0 * * *
```

### Her Gün Saat 12:00

```
0 12 * * *
```

### Her Pazartesi Saat 09:00

```
0 9 * * MON
```

veya

```
0 9 * * 1
```

### Her Ayın 1. Günü Saat 00:00

```
0 0 1 * *
```

### Her 5 Dakikada Bir

```
*/5 * * * *
```

### Her 15 Dakikada Bir

```
*/15 * * * *
```

### Her 30 Dakikada Bir

```
*/30 * * * *
```

### Her 2 Saatte Bir

```
0 */2 * * *
```

### Her Gün Çalışma Saatleri (09:00-17:00)

```
0 9-17 * * *
```

### Her Gün Çalışma Saatleri Her 30 Dakikada Bir

```
*/30 9-17 * * *
```

### Hafta İçi Her Gün Saat 09:00

```
0 9 * * MON-FRI
```

veya

```
0 9 * * 1-5
```

### Hafta Sonu Her Gün Saat 10:00

```
0 10 * * SAT,SUN
```

veya

```
0 10 * * 0,6
```

### Her Ayın Son Günü Saat 23:59

```
59 23 L * *
```

### Her Ayın İlk Pazartesi Saat 09:00

```
0 9 ? * MON#1
```

## Karmaşık Senaryolar

### İş Günleri (Pazartesi-Cuma) Saat 09:00-17:00 Her Saat Başı

```
0 9-17 * * MON-FRI
```

### Her Ayın 1. ve 15. Günü Saat 00:00

```
0 0 1,15 * *
```

### Her 3 Ayda Bir (Ocak, Nisan, Temmuz, Ekim) 1. Günü Saat 00:00

```
0 0 1 1,4,7,10 *
```

### Her Gün Gece Yarısı ve Öğlen

```
0 0,12 * * *
```

## Zaman Dilimi (Timezone)

Cron expression'da zaman dilimi belirtmek için:

```yaml
Cron Expression: 0 9 * * *
Time Zone: Europe/Istanbul
```

### Zaman Dilimi Örnekleri

* **UTC**: `UTC`
* **Türkiye**: `Europe/Istanbul`
* **ABD Doğu**: `America/New_York`
* **ABD Batı**: `America/Los_Angeles`

## Cron Expression Doğrulama

Cron expression'ı doğrulamak için:

1. **Validate** butonuna tıklayın
2. Geçerli olup olmadığını kontrol edin
3. Sonraki çalıştırma zamanlarını görüntüleyin

## En İyi Uygulamalar

### Zamanlama Stratejisi

* **Peak Hours**: Yoğun saatlerden kaçının
* **Resource Availability**: Kaynak kullanılabilirliğini göz önünde bulundurun
* **Dependencies**: Bağımlılıkları dikkate alın
* **Maintenance Windows**: Bakım pencerelerini planlayın

### Cron Expression Tasarımı

* **Readability**: Okunabilirlik
* **Simplicity**: Basitlik
* **Documentation**: Dokümantasyon
* **Testing**: Test etme

### Performans

* **Job Distribution**: Job dağılımı
* **Resource Management**: Kaynak yönetimi
* **Concurrency**: Eşzamanlılık kontrolü
* **Monitoring**: İzleme
