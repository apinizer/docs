Cron Expressions, zamanlanmış işlerin ne zaman çalışacağını belirlemek için kullanılan zamanlama ifadeleridir. Bu rehber, cron expression syntax'ını ve kullanımını detaylı olarak açıklar.

## Genel Bakış

Cron expression'lar şu formatı kullanır:

```
┌───────────── saniye (0-59)
│ ┌─────────── dakika (0-59)
│ │ ┌───────── saat (0-23)
│ │ │ ┌─────── gün (1-31)
│ │ │ │ ┌───── ay (1-12 veya JAN-DEC)
│ │ │ │ │ ┌─── haftanın günü (0-7 veya SUN-SAT)
│ │ │ │ │ │
* * * * * * ?
```

## Cron Expression Formatı

### Standart Format

```
Saniye Dakika Saat Gün Ay Haftanın Günü [Yıl]
```

### Örnekler

```
Her saniye: * * * * * * ?
Her dakika: 0 * * * * * ?
Her saat: 0 0 * * * * ?
Her gün saat 00:00: 0 0 0 * * * ?
Her Pazartesi saat 09:00: 0 0 9 ? * MON *
Her ayın 1'i saat 00:00: 0 0 0 1 * * ?
```

## Alanlar ve Değerler

### Saniye (0-59)

```
* : Her saniye
0-59 : Belirli saniye
0/5 : Her 5 saniyede bir
15,30,45 : 15, 30 ve 45. saniyelerde
```

### Dakika (0-59)

```
* : Her dakika
0-59 : Belirli dakika
0/15 : Her 15 dakikada bir
0,30 : 0 ve 30. dakikalarda
```

### Saat (0-23)

```
* : Her saat
0-23 : Belirli saat
0/2 : Her 2 saatte bir
9-17 : 9 ile 17 arası
```

### Gün (1-31)

```
* : Her gün
1-31 : Belirli gün
1/5 : Her 5 günde bir
L : Ayın son günü
```

### Ay (1-12 veya JAN-DEC)

```
* : Her ay
1-12 : Belirli ay
JAN-DEC : Ay isimleri
1/3 : Her 3 ayda bir
```

### Haftanın Günü (0-7 veya SUN-SAT)

```
* veya ? : Her gün
0-7 : Belirli gün (0 ve 7 = Pazar)
SUN-SAT : Gün isimleri
MON-FRI : Hafta içi
```

### Yıl (opsiyonel, 1970-2099)

```
* : Her yıl
2024-2030 : Belirli yıl aralığı
```

## Özel Karakterler

### * (Yıldız)

Tüm değerleri temsil eder.

```
* * * * * * ? → Her saniye
```

### ? (Soru İşareti)

Sadece gün ve haftanın günü alanlarında kullanılır. "Belirsiz" anlamına gelir.

```
0 0 9 ? * MON → Her Pazartesi saat 09:00
```

### - (Tire)

Aralık belirtir.

```
0 0 9-17 * * ? → Her gün 09:00-17:00 arası
```

### , (Virgül)

Birden fazla değer belirtir.

```
0 0 9,12,15 * * ? → Her gün 09:00, 12:00 ve 15:00'te
```

### / (Slash)

Artış değeri belirtir.

```
0 0/15 * * * ? → Her 15 dakikada bir
0 0 0/2 * * ? → Her 2 saatte bir
```

### L (Last)

Son değeri temsil eder.

```
0 0 0 L * ? → Her ayın son günü saat 00:00
0 0 0 ? * L → Her haftanın son günü (Pazar) saat 00:00
```

### W (Weekday)

Hafta içi günü temsil eder.

```
0 0 0 15W * ? → Ayın 15'i veya en yakın hafta içi günü
```

### # (Hash)

Haftanın belirli bir gününü temsil eder.

```
0 0 9 ? * MON#2 → Her ayın 2. Pazartesi günü saat 09:00
```

## Yaygın Kullanım Senaryoları

### Günlük İşler

```
Her gün saat 02:00: 0 0 2 * * ?
Her gün saat 00:00 ve 12:00: 0 0 0,12 * * ?
Her gün saat 09:00-17:00 arası her saat: 0 0 9-17 * * ?
```

### Haftalık İşler

```
Her Pazartesi saat 09:00: 0 0 9 ? * MON
Her Cuma saat 17:00: 0 0 17 ? * FRI
Her hafta sonu saat 00:00: 0 0 0 ? * SAT,SUN
```

### Aylık İşler

```
Her ayın 1'i saat 00:00: 0 0 0 1 * ?
Her ayın son günü saat 00:00: 0 0 0 L * ?
Her ayın 15'i saat 12:00: 0 0 12 15 * ?
```

### Periyodik İşler

```
Her 5 dakikada bir: 0 0/5 * * * ?
Her 30 dakikada bir: 0 0/30 * * * ?
Her 2 saatte bir: 0 0 0/2 * * ?
Her 6 saatte bir: 0 0 0/6 * * ?
```

### İş Saatleri

```
Hafta içi saat 09:00-17:00 arası her saat: 0 0 9-17 ? * MON-FRI
Hafta içi saat 09:00-17:00 arası her 30 dakika: 0 0/30 9-17 ? * MON-FRI
```

## Time Zone

Time zone belirtmek için:

```
Cron Expression: 0 0 2 * * ?
Time Zone: Europe/Istanbul
→ Türkiye saati ile her gün saat 02:00
```

## Best Practices

### Açıklayıcı İsimler

Job adlarında zamanlamayı belirtin:

```
Job Name: Daily Backup at 02:00
Cron: 0 0 2 * * ?
```

### Test

Cron expression'ları test edin:

1. **Test** butonuna tıklayın
2. Sonraki çalıştırma zamanlarını kontrol edin
3. Beklenen zamanları doğrulayın

### Dokümantasyon

Cron expression'ları dokümante edin:

```
Job Name: Weekly Report
Description: Her Pazartesi saat 09:00'de haftalık rapor oluşturur
Cron: 0 0 9 ? * MON
```

## Örnek Senaryolar

### Senaryo 1: Günlük Veri Yedekleme

```
Job Name: Daily Backup
Cron: 0 0 2 * * ?
Time Zone: Europe/Istanbul
Description: Her gün saat 02:00'de veri yedekleme yapar
```

### Senaryo 2: Haftalık Rapor

```
Job Name: Weekly Report
Cron: 0 0 9 ? * MON
Time Zone: Europe/Istanbul
Description: Her Pazartesi saat 09:00'de haftalık rapor oluşturur
```

### Senaryo 3: Periyodik Veri Senkronizasyonu

```
Job Name: Data Sync
Cron: 0 0/30 * * * ?
Time Zone: Europe/Istanbul
Description: Her 30 dakikada bir veri senkronizasyonu yapar
```
