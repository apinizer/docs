Job Definition, zamanlanmış işleri (scheduled jobs) tanımlamanızı ve yapılandırmanızı sağlayan özelliktir. Bu özellik sayesinde belirli zamanlarda otomatik olarak çalışacak işleri oluşturabilir ve yönetebilirsiniz.

## Genel Bakış

Job Definition şu özellikleri sunar:

* Zamanlanmış iş tanımlama
* Cron expression desteği
* Farklı job tipleri
* Parametre yönetimi
* Execution history
* Job yönetimi

## Job Oluşturma

Yeni bir job oluşturmak için:

1. **Scheduled Jobs** sayfasına gidin
2. **New Job** butonuna tıklayın
3. Job bilgilerini girin
4. Zamanlamayı yapılandırın
5. Job tipini seçin
6. Parametreleri ayarlayın

## Job Ayarları

### Temel Ayarlar

* **Job Name**: Job adı (unique)
* **Description**: Açıklama
* **Enabled**: Aktif/Pasif
* **Priority**: Öncelik

### Zamanlama Ayarları

* **Schedule Type**: Zamanlama tipi (Cron, Fixed Rate, Fixed Delay)
* **Cron Expression**: Cron ifadesi
* **Time Zone**: Zaman dilimi
* **Start Date**: Başlangıç tarihi
* **End Date**: Bitiş tarihi (opsiyonel)

### Execution Ayarları

* **Max Retries**: Maksimum deneme sayısı
* **Retry Interval**: Deneme aralığı
* **Timeout**: Timeout süresi
* **Concurrent Execution**: Eşzamanlı çalıştırma izni

## Job Tipleri

### API Call Job

API çağrısı yapan job:

**Ayarlar:**
* **URL**: API endpoint URL'i
* **Method**: HTTP metodu
* **Headers**: Request header'ları
* **Body**: Request body

**Örnek:**
```
Job Name: Daily Data Sync
Schedule: 0 0 2 * * ? (Her gün saat 02:00)
Type: API Call
URL: https://api.example.com/sync
Method: POST
```

### Script Job

Script çalıştıran job:

**Ayarlar:**
* **Language**: Script dili (JavaScript, Groovy)
* **Script**: Script kodu

**Örnek:**
```
Job Name: Data Cleanup
Schedule: 0 0 3 * * ? (Her gün saat 03:00)
Type: Script
Script: cleanupOldData()
```

### Task Flow Job

Task Flow çalıştıran job:

**Ayarlar:**
* **Task Flow**: Çalıştırılacak Task Flow
* **Parameters**: Task Flow parametreleri

**Örnek:**
```
Job Name: Weekly Report
Schedule: 0 0 9 ? * MON (Her Pazartesi saat 09:00)
Type: Task Flow
Task Flow: Generate Weekly Report
```

## Cron Expressions

Cron expression kullanımı için detaylı bilgi: [Cron Expressions](/tr/entegrasyon-otomasyon/zamanlanmis-isler/cron-expressions)

### Basit Örnekler

```
Her dakika: * * * * * ?
Her saat: 0 * * * * ?
Her gün saat 02:00: 0 0 2 * * ?
Her Pazartesi saat 09:00: 0 0 9 ? * MON
Her ayın 1'i saat 00:00: 0 0 0 1 * ?
```

## Parametreler

Job parametreleri:

### Parametre Tanımlama

```
Parameters:
  - name: userId
    type: Number
    defaultValue: 0
  - name: userName
    type: String
    defaultValue: ""
```

### Parametre Kullanımı

Parametreleri job içinde kullanın:

```
API Call:
  URL: https://api.example.com/users/${userId}
  Body: {"name": "${userName}"}
```

## Job Yönetimi

### Job Listesi

Tüm job'ları görüntülemek için:

1. **Scheduled Jobs** sayfasına gidin
2. Job listesi görüntülenir

**Görüntülenen Bilgiler:**
* Job adı
* Durum (Enabled/Disabled)
* Son çalıştırma zamanı
* Sonraki çalıştırma zamanı
* Durum (Success/Failed)

### Job Düzenleme

Job düzenlemek için:

1. Job listesinden bir job seçin
2. **Edit** butonuna tıklayın
3. Ayarları değiştirin
4. **Save** butonuna tıklayın

### Job Silme

Job silmek için:

1. Job listesinden bir job seçin
2. **Delete** butonuna tıklayın
3. Onaylayın

### Job Durdurma/Başlatma

Job'u durdurmak/başlatmak için:

1. Job listesinden bir job seçin
2. **Disable/Enable** butonuna tıklayın

## Execution History

Job çalıştırma geçmişi:

### Geçmiş Görüntüleme

1. Job listesinden bir job seçin
2. **History** sekmesine gidin
3. Çalıştırma geçmişi görüntülenir

**Görüntülenen Bilgiler:**
* Çalıştırma zamanı
* Durum (Success/Failed)
* Süre
* Hata mesajı (varsa)
* Log'lar

### Execution Details

Detaylı bilgi için:

1. Geçmişten bir execution seçin
2. **View Details** butonuna tıklayın
3. Detaylı bilgiler görüntülenir

## Best Practices

### Job Tasarımı

* **Idempotent**: Job'lar idempotent olmalı
* **Error Handling**: Hata yönetimi yapın
* **Logging**: Detaylı loglama yapın
* **Monitoring**: İzleme yapın

### Performance

* **Efficient**: Verimli job'lar yazın
* **Resource Management**: Kaynak yönetimi yapın
* **Concurrent Execution**: Eşzamanlı çalıştırmayı kontrol edin

### Reliability

* **Retry Logic**: Yeniden deneme mantığı ekleyin
* **Timeout**: Timeout değerlerini ayarlayın
* **Alerting**: Alert mekanizmaları kurun
