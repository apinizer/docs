Zamanlanmış işler (Scheduled Jobs), belirli zamanlarda veya periyodik olarak çalıştırılan görevlerdir. Job Definition sayfasında zamanlanmış işlerin nasıl tanımlandığını ve yapılandırıldığını öğreneceksiniz.

## Genel Bakış

Zamanlanmış işler sayesinde:

* Belirli zamanlarda görevler çalıştırabilirsiniz
* Periyodik işlemler tanımlayabilirsiniz
* Otomatik senkronizasyon yapabilirsiniz
* Raporlama ve bakım işlemlerini otomatikleştirebilirsiniz

## Job Oluşturma

### Yeni Job Oluşturma

1. **Scheduled Jobs** menüsüne gidin
2. **New Job** butonuna tıklayın
3. Job bilgilerini girin:
   * **Name**: Job adı
   * **Description**: Açıklama
   * **Enabled**: Aktif/Pasif durumu
   * **Job Type**: Job türü (API Call, Script, Task Flow)

### Temel Yapılandırma

#### Genel Ayarlar

* **Name**: Job'un benzersiz adı
* **Description**: Job açıklaması
* **Enabled**: Job'un aktif olup olmadığı
* **Priority**: Job önceliği (Low, Normal, High)

#### Zamanlama Ayarları

* **Schedule Type**: Zamanlama türü
  * **Cron Expression**: Cron expression kullanımı
  * **Fixed Rate**: Sabit aralık
  * **Fixed Delay**: Sabit gecikme
* **Cron Expression**: Cron expression (örn: `0 0 * * *`)
* **Time Zone**: Zaman dilimi

## Job Türleri

### API Call Job

API endpoint'ine çağrı yapan job:

```yaml
Job Type: API Call
Configuration:
  URL: https://api.example.com/sync
  Method: POST
  Headers:
    Authorization: Bearer ${token}
  Body:
    action: sync
    timestamp: ${currentTimestamp}
```

### Script Job

Script çalıştıran job:

```yaml
Job Type: Script
Configuration:
  Language: JavaScript
  Script: |
    // Script kodu
    var data = fetchData();
    processData(data);
    saveResults(data);
```

### Task Flow Job

Task Flow çalıştıran job:

```yaml
Job Type: Task Flow
Configuration:
  Task Flow: data-sync-taskflow
  Input Parameters:
    source: database
    target: api
```

## Job Parametreleri

### Input Parameters

Job'a parametre geçirmek için:

```yaml
Input Parameters:
  source: database
  target: api
  batchSize: 100
  timeout: 30000
```

### Environment Variables

Ortam değişkenleri kullanımı:

```yaml
Environment Variables:
  API_URL: https://api.example.com
  DB_HOST: localhost
  LOG_LEVEL: INFO
```

## Job Yönetimi

### Job Durumları

* **Scheduled**: Zamanlanmış
* **Running**: Çalışıyor
* **Completed**: Tamamlandı
* **Failed**: Başarısız
* **Paused**: Duraklatıldı
* **Disabled**: Devre dışı

### Job İşlemleri

* **Run Now**: Hemen çalıştır
* **Pause**: Duraklat
* **Resume**: Devam ettir
* **Disable**: Devre dışı bırak
* **Delete**: Sil
* **Duplicate**: Kopyala

## Job Geçmişi

### Execution History

Job çalıştırma geçmişi:

* **Execution Time**: Çalıştırma zamanı
* **Duration**: Süre
* **Status**: Durum
* **Output**: Çıktı
* **Error**: Hata (varsa)

### History Filtreleme

* **Time Range**: Zaman aralığı
* **Status**: Durum filtresi
* **Search**: Arama

## En İyi Uygulamalar

### Job Tasarımı

* **Idempotency**: İdempotent job'lar tasarlayın
* **Error Handling**: Kapsamlı hata yönetimi
* **Logging**: Detaylı loglama
* **Monitoring**: İzleme ve alerting

### Performans

* **Resource Management**: Kaynak yönetimi
* **Concurrency**: Eşzamanlılık kontrolü
* **Timeout**: Timeout ayarları
* **Batch Processing**: Toplu işleme

### Güvenlik

* **Authentication**: Kimlik doğrulama
* **Authorization**: Yetkilendirme
* **Credential Management**: Kimlik bilgisi yönetimi
* **Audit Logging**: Denetim loglama
