Job Definition, belirli zamanlarda otomatik olarak çalışacak işleri tanımlamanızı sağlar. API çağrıları, script çalıştırma veya Task Flow çalıştırma gibi işlemleri zamanlayabilirsiniz.

## Genel Bakış

Job Definition özellikleri:

* **Scheduled Execution**: Zamanlanmış çalıştırma
* **Multiple Job Types**: Çoklu iş tipi desteği
* **Cron Expressions**: Cron ifadeleri
* **Job Management**: İş yönetimi

## Job Types

### API Call Job

API çağrısı yapan iş:

* **HTTP Method**: GET, POST, PUT, DELETE vb.
* **URL**: API endpoint URL'i
* **Headers**: HTTP header'ları
* **Body**: İstek gövdesi
* **Authentication**: Kimlik doğrulama

### Script Job

Script çalıştıran iş:

* **Script Language**: JavaScript, Groovy
* **Script Content**: Script içeriği
* **Input Variables**: Girdi değişkenleri
* **Output Variables**: Çıktı değişkenleri

### Task Flow Job

Task Flow çalıştıran iş:

* **Task Flow**: Çalıştırılacak Task Flow
* **Input Variables**: Girdi değişkenleri
* **Output Variables**: Çıktı değişkenleri

## Job Configuration

### Basic Configuration

Temel yapılandırma:

* **Job Name**: İş adı
* **Description**: Açıklama
* **Job Type**: İş tipi (API Call, Script, Task Flow)
* **Enabled**: Aktif/Pasif durumu

### Schedule Configuration

Zamanlama yapılandırması:

* **Schedule Type**: Zamanlama tipi
  * **Cron Expression**: Cron ifadesi
  * **Fixed Interval**: Sabit aralık
  * **One Time**: Tek seferlik
* **Cron Expression**: Cron ifadesi (örn: `0 0 * * *`)
* **Time Zone**: Zaman dilimi

Daha fazla bilgi için [Cron Expressions](/tr/entegrasyon-otomasyon/zamanlanmis-isler/cron-expressions) sayfasına bakın.

### Execution Configuration

Çalıştırma yapılandırması:

* **Timeout**: Maksimum çalıştırma süresi
* **Retry Policy**: Yeniden deneme politikası
* **Error Handling**: Hata yönetimi stratejisi
* **Resource Limits**: Kaynak limitleri

## Job Definition Example

### Example 1: Daily Report Job

```
Job Definition:
- Name: Daily Sales Report
- Type: Task Flow
- Task Flow: Generate Sales Report
- Schedule: Cron Expression
- Cron: 0 0 9 * * ? (Her gün saat 09:00)
- Time Zone: Europe/Istanbul
- Enabled: true
- Input Variables:
    reportDate: ${currentDate}
    reportType: daily
```

### Example 2: API Health Check Job

```
Job Definition:
- Name: API Health Check
- Type: API Call
- Method: GET
- URL: https://api.example.com/health
- Schedule: Fixed Interval
- Interval: 5 minutes
- Enabled: true
- On Error:
    Send Notification
    Log Error
```

### Example 3: Data Sync Job

```
Job Definition:
- Name: Customer Data Sync
- Type: Task Flow
- Task Flow: Sync Customer Data
- Schedule: Cron Expression
- Cron: 0 */30 * * * ? (Her 30 dakikada bir)
- Enabled: true
- Retry Policy:
    Max Retries: 3
    Retry Interval: 5 minutes
```

## Job Management

### Job States

İş durumları:

* **Scheduled**: Zamanlanmış
* **Running**: Çalışıyor
* **Completed**: Tamamlandı
* **Failed**: Başarısız
* **Paused**: Duraklatıldı
* **Disabled**: Devre dışı

### Job Operations

İş işlemleri:

* **Enable/Disable**: Aktif/Pasif yapma
* **Pause/Resume**: Duraklatma/Devam ettirme
* **Run Now**: Hemen çalıştırma
* **View History**: Geçmişi görüntüleme
* **Edit**: Düzenleme
* **Delete**: Silme

### Job Monitoring

İş izleme:

* **Execution History**: Çalıştırma geçmişi
* **Success Rate**: Başarı oranı
* **Average Duration**: Ortalama süre
* **Error Logs**: Hata logları

Daha fazla bilgi için [Monitoring & Logging](/tr/entegrasyon-otomasyon/zamanlanmis-isler/monitoring-logging) sayfasına bakın.

## Error Handling

### Error Notifications

Hata bildirimleri:

* **Email Notification**: E-posta bildirimi
* **Webhook Notification**: Webhook bildirimi
* **SNMP Notification**: SNMP bildirimi

Daha fazla bilgi için [Error Notifications](/tr/entegrasyon-otomasyon/zamanlanmis-isler/error-notifications) sayfasına bakın.

### Retry Policy

Yeniden deneme politikası:

* **Max Retries**: Maksimum yeniden deneme sayısı
* **Retry Interval**: Yeniden deneme aralığı
* **Retry Strategy**: Yeniden deneme stratejisi

## Resource Management

### Resource Limits

Kaynak limitleri:

* **CPU Limit**: CPU limiti

Daha fazla bilgi için [Resource Management](/tr/entegrasyon-otomasyon/zamanlanmis-isler/resource-management) sayfasına bakın.

## Best Practices

### Job Design

* **Idempotent Jobs**: İdempotent işler tasarlayın
* **Error Handling**: Hata yönetimi ekleyin
* **Logging**: Loglama yapın
* **Monitoring**: İzleme kurun

### Schedule Design

* **Avoid Overlap**: Çakışmalardan kaçının
* **Consider Load**: Yükü göz önünde bulundurun
* **Time Zone**: Zaman dilimini doğru ayarlayın
* **Maintenance Windows**: Bakım pencerelerini dikkate alın

### Performance

* **Resource Limits**: Kaynak limitlerini ayarlayın
* **Concurrent Execution**: Eşzamanlı çalıştırmayı yönetin
* **Timeout**: Zaman aşımı ayarlayın
* **Optimization**: İşleri optimize edin
