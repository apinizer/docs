Zamanlanmış işler (Scheduled Jobs), belirli zamanlarda veya periyodik olarak çalıştırılan görevlerdir. Job Definition sayfasında zamanlanmış işlerin nasıl tanımlandığını ve yapılandırıldığını öğreneceksiniz.

## Genel Bakış

Zamanlanmış işler sayesinde:

* Belirli zamanlarda görevler çalıştırabilirsiniz
* Periyodik işlemler tanımlayabilirsiniz
* Otomatik senkronizasyon yapabilirsiniz
* Raporlama ve bakım işlemlerini otomatikleştirebilirsiniz

## Job Oluşturma

### Yeni Job Oluşturma

1. **Scheduled Jobs** menüsüne gidin
2. **New Job** butonuna tıklayın
3. Job bilgilerini girin:
   * **Name**: Job adı
   * **Description**: Açıklama
   * **Enabled**: Aktif/Pasif durumu
   * **Job Type**: Job türü (API Call, Script, Task Flow)

### Temel Yapılandırma

#### Genel Ayarlar

* **Name**: Job'un benzersiz adı
* **Description**: Job açıklaması
* **Enabled**: Job'un aktif olup olmadığı
* **Priority**: Job önceliği (Low, Normal, High)

#### Zamanlama Ayarları

* **Schedule Type**: Zamanlama türü
  * **Cron Expression**: Cron expression kullanımı
  * **Fixed Rate**: Sabit aralık
  * **Fixed Delay**: Sabit gecikme
* **Cron Expression**: Cron expression (örn: `0 0 * * *`)
* **Time Zone**: Zaman dilimi

## Job Türleri

### API Call Job

API endpoint'ine çağrı yapan job:

```yaml
Job Type: API Call
Configuration:
  URL: https://api.example.com/sync
  Method: POST
  Headers:
    Authorization: Bearer ${token}
  Body:
    action: sync
    timestamp: ${currentTimestamp}
```

### Script Job

Script çalıştıran job:

```yaml
Job Type: Script
Configuration:
  Language: JavaScript
  Script: |
    // Script kodu
    var data = fetchData();
    processData(data);
    saveResults(data);
```

### Task Flow Job

Zamanlanmış olarak Task Flow çalıştırma:

**Yapılandırma:**

* **Task Flow**: Çalıştırılacak Task Flow'u seçin
* **Input Variables**: Girdi değişkenlerini sağlayın
* **Timeout**: Çalıştırma timeout'u

**Örnek:**

```json
{
  "jobType": "taskflow",
  "taskFlow": "ProcessOrders",
  "inputVariables": {
    "batchSize": 100
  },
  "timeout": 300000
}
```

### Script Job

Zamanlanmış olarak script çalıştırma:

**Yapılandırma:**

* **Script**: Script içeriği veya script dosyası
* **Script Type**: JavaScript veya Groovy
* **Variables**: Script değişkenleri

**Örnek:**

```json
{
  "jobType": "script",
  "script": "console.log('Job executed');",
  "scriptType": "javascript"
}
```

### API Call Job

Zamanlanmış olarak API çağrısı yapma:

**Yapılandırma:**

* **URL**: API endpoint URL'i
* **Method**: HTTP metodu
* **Headers**: İstek başlıkları
* **Body**: İstek gövdesi

**Örnek:**

```json
{
  "jobType": "apicall",
  "url": "https://api.example.com/sync",
  "method": "POST",
  "headers": {
    "Authorization": "Bearer ${token}"
  }
}
```

## Job Parametreleri

### Input Parameters

Job'a parametre geçirmek için:

```yaml
Input Parameters:
  source: database
  target: api
  batchSize: 100
  timeout: 30000
```

### Environment Variables

Ortam değişkenleri kullanımı:

```yaml
Environment Variables:
  API_URL: https://api.example.com
  DB_HOST: localhost
  LOG_LEVEL: INFO
```

## Job Yönetimi

### Job Durumları

* **Scheduled**: Zamanlanmış
* **Running**: Çalışıyor
* **Completed**: Tamamlandı
* **Failed**: Başarısız
* **Paused**: Duraklatıldı
* **Disabled**: Devre dışı

### Job İşlemleri

* **Run Now**: Hemen çalıştır
* **Pause**: Duraklat
* **Resume**: Devam ettir
* **Disable**: Devre dışı bırak
* **Delete**: Sil
* **Duplicate**: Kopyala

## Job Geçmişi

### Execution History

Job çalıştırma geçmişi:

* **Execution Time**: Çalıştırma zamanı
* **Duration**: Süre
* **Status**: Durum
* **Output**: Çıktı
* **Error**: Hata (varsa)

### History Filtreleme

* **Time Range**: Zaman aralığı
* **Status**: Durum filtresi
* **Search**: Arama
