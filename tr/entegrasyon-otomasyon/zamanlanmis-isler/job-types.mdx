Job Types, zamanlanmış işlerin farklı türlerini tanımlar. Her job tipi farklı bir işlem türünü temsil eder ve kendine özgü yapılandırma seçenekleri sunar.

## Genel Bakış

API Integrator şu job tiplerini destekler:

* **API Call Job**: HTTP/HTTPS API çağrıları
* **Script Job**: JavaScript veya Groovy script çalıştırma
* **Task Flow Job**: Task Flow çalıştırma

## API Call Job

API Call Job, HTTP/HTTPS API çağrıları yapan job tipidir.

### Yapılandırma

**Ayarlar:**
* **URL**: API endpoint URL'i
* **Method**: HTTP metodu (GET, POST, PUT, DELETE, PATCH)
* **Headers**: Request header'ları
* **Body**: Request body
* **Authentication**: Kimlik doğrulama
* **Timeout**: Timeout süresi

### Örnek

```
Job Name: Daily Data Sync
Type: API Call
Schedule: 0 0 2 * * ?
URL: https://api.example.com/sync
Method: POST
Headers:
  Content-Type: application/json
  Authorization: Bearer ${token}
Body:
{
  "date": "${currentDate}",
  "type": "daily"
}
```

## Script Job

Script Job, JavaScript veya Groovy script çalıştıran job tipidir.

### Yapılandırma

**Ayarlar:**
* **Language**: Script dili (JavaScript, Groovy)
* **Script**: Script kodu
* **Parameters**: Script parametreleri

### Örnek

```
Job Name: Data Cleanup
Type: Script
Schedule: 0 0 3 * * ?
Language: JavaScript
Script:
  function cleanupOldData() {
    // Cleanup logic
    return { deleted: 100 };
  }
  cleanupOldData();
```

## Task Flow Job

Task Flow Job, Task Flow çalıştıran job tipidir.

### Yapılandırma

**Ayarlar:**
* **Task Flow**: Çalıştırılacak Task Flow
* **Parameters**: Task Flow parametreleri
* **Variables**: Task Flow variable'ları

### Örnek

```
Job Name: Weekly Report
Type: Task Flow
Schedule: 0 0 9 ? * MON
Task Flow: Generate Weekly Report
Parameters:
  reportType: weekly
  format: pdf
```

## Job Tipi Seçimi

### Ne Zaman API Call Job?

* Basit API çağrıları için
* Dış sistemlerle entegrasyon için
* RESTful API'ler için

### Ne Zaman Script Job?

* Veri işleme için
* Hesaplamalar için
* Basit dönüşümler için

### Ne Zaman Task Flow Job?

* Karmaşık iş akışları için
* Birden fazla adım gerektiğinde
* Hata yönetimi gerektiğinde

## Best Practices

### Job Tasarımı

* **Idempotent**: Job'lar idempotent olmalı
* **Error Handling**: Hata yönetimi yapın
* **Logging**: Detaylı loglama yapın

### Performance

* **Efficient**: Verimli job'lar yazın
* **Timeout**: Timeout değerlerini ayarlayın
* **Resource Management**: Kaynak yönetimi yapın
