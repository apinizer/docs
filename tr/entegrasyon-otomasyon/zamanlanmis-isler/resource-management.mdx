Zamanlanmış işlerde kaynak yönetimi, job'ların sistem kaynaklarını verimli kullanmasını sağlar. Bu sayfa, kaynak yönetimi yapılandırmasını detaylı olarak açıklar.

## Genel Bakış

Resource management sayesinde:

* CPU ve memory kullanımını kontrol edebilirsiniz
* Eşzamanlılık (concurrency) ayarlayabilirsiniz
* Timeout ayarları yapabilirsiniz
* Kaynak limitleri belirleyebilirsiniz

## CPU Management

### CPU Limit

CPU limiti:

* **CPU Limit**: CPU limiti (cores veya %)
* **CPU Priority**: CPU önceliği
* **CPU Affinity**: CPU affinity

**Örnek:**
```yaml
Resource Management:
  CPU:
    Limit: 2 cores
    Priority: normal
    Affinity: [0, 1]
```

### CPU Monitoring

CPU izleme:

* **CPU Usage**: CPU kullanımı
* **CPU Peak**: CPU zirve kullanımı
* **CPU Average**: CPU ortalama kullanımı

## Memory Management

### Memory Limit

Bellek limiti:

* **Memory Limit**: Bellek limiti (MB veya GB)
* **Memory Allocation**: Bellek tahsisi
* **Memory Monitoring**: Bellek izleme

**Örnek:**
```yaml
Resource Management:
  Memory:
    Limit: 2GB
    Allocation: dynamic
    Monitoring: enabled
```

### Memory Optimization

Bellek optimizasyonu:

* **Garbage Collection**: Çöp toplama
* **Memory Cleanup**: Bellek temizliği
* **Memory Pooling**: Bellek havuzlama

## Concurrency

### Max Concurrent Jobs

Maksimum eşzamanlı job sayısı:

* **Global Limit**: Global limit
* **Per Job Type**: Job tipi başına limit
* **Per User**: Kullanıcı başına limit

**Örnek:**
```yaml
Resource Management:
  Concurrency:
    Max Concurrent Jobs: 10
    Per Job Type:
      API Call: 5
      Script: 3
      Task Flow: 2
```

### Job Queue

Job kuyruğu:

* **Queue Size**: Kuyruk boyutu
* **Queue Priority**: Kuyruk önceliği
* **Queue Management**: Kuyruk yönetimi

### Priority Queue

Öncelik kuyruğu:

* **High Priority**: Yüksek öncelik
* **Normal Priority**: Normal öncelik
* **Low Priority**: Düşük öncelik

## Timeout

### Job Timeout

Job zaman aşımı:

* **Default Timeout**: Varsayılan zaman aşımı
* **Per Job Type**: Job tipi başına zaman aşımı
* **Per Job**: Job başına zaman aşımı

**Örnek:**
```yaml
Resource Management:
  Timeout:
    Default: 3600000ms (1 hour)
    Per Job Type:
      API Call: 300000ms (5 minutes)
      Script: 600000ms (10 minutes)
      Task Flow: 3600000ms (1 hour)
```

### Task Timeout

Task zaman aşımı:

* **Task Timeout**: Task zaman aşımı
* **Nested Task Timeout**: İç içe task zaman aşımı

### Connection Timeout

Bağlantı zaman aşımı:

* **Connection Timeout**: Bağlantı zaman aşımı
* **Read Timeout**: Okuma zaman aşımı
* **Write Timeout**: Yazma zaman aşımı

### Resource Usage

Kaynak kullanımı:

* **CPU Usage**: CPU kullanımı
* **Memory Usage**: Bellek kullanımı
* **Network Usage**: Ağ kullanımı
* **Disk Usage**: Disk kullanımı

### Resource Alerts

Kaynak alert'leri:

* **CPU Alert**: CPU alert'i
* **Memory Alert**: Bellek alert'i
* **Resource Exhaustion**: Kaynak tükenmesi

## Resource Limits

### Setting Limits

Limit ayarlama:

1. **Job Definition** sayfasında ilgili job'ı seçin
2. **Resource Management** sekmesine gidin
3. Kaynak limitlerini ayarlayın
4. **Save** ile kaydedin

### Limit Enforcement

Limit uygulama:

* **Hard Limits**: Sert limitler
* **Soft Limits**: Yumuşak limitler
* **Limit Exceeded Actions**: Limit aşıldığında eylemler
