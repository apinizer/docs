Error Notifications, zamanlanmış işlerde hata oluştuğunda bildirim almanızı sağlar. Bu özellik sayesinde hataları hızlıca tespit edebilir ve gerekli aksiyonları alabilirsiniz.

## Genel Bakış

Error Notifications şu özellikleri sunar:

* Hata bildirimleri
* Çoklu notification kanalları
* Alert yapılandırması
* Hata kategorizasyonu
* Bildirim şablonları

## Notification Yapılandırması

Notification yapılandırmak için:

1. Job'u seçin
2. **Settings** → **Notifications** sekmesine gidin
3. Notification kanallarını yapılandırın

## Notification Kanalları

### Email

E-posta bildirimi:

```
Channel: Email
Recipients: admin@example.com, team@example.com
On: Failure, Retry Exhausted
Template: job-error-template
```

### Webhook

Webhook bildirimi:

```
Channel: Webhook
URL: https://api.example.com/webhooks/job-error
Method: POST
Headers:
  Authorization: Bearer ${token}
Body:
{
  "jobName": "${jobName}",
  "error": "${errorMessage}",
  "timestamp": "${timestamp}"
}
```

### Slack

Slack bildirimi:

```
Channel: Slack
Webhook URL: https://hooks.slack.com/services/...
On: Failure
```

## Alert Yapılandırması

Alert yapılandırmak için:

1. **Alerts** sekmesine gidin
2. **Add Alert** butonuna tıklayın
3. Alert kriterlerini belirleyin

### Alert Kriterleri

**Error Count:**
```
Condition: Error count > 3 in 5 minutes
Action: Send notification
```

**Consecutive Failures:**
```
Condition: 3 consecutive failures
Action: Send notification
```

**Error Rate:**
```
Condition: Error rate > 10%
Action: Send notification
```

## Notification Templates

Bildirim şablonları:

### Email Template

```
Subject: Job Failed: ${jobName}
Body:
Job Name: ${jobName}
Error: ${errorMessage}
Timestamp: ${timestamp}
Execution ID: ${executionId}
```

### Webhook Template

```json
{
  "event": "job.failed",
  "jobName": "${jobName}",
  "error": "${errorMessage}",
  "timestamp": "${timestamp}",
  "executionId": "${executionId}"
}
```

## Best Practices

### Notification Strategy

* **Critical Errors**: Kritik hatalar için anında bildirim
* **Warning Errors**: Uyarılar için toplu bildirim
* **Error Aggregation**: Hata toplama

### Alert Tuning

* **Appropriate Thresholds**: Uygun eşikler
* **Avoid Noise**: Gürültüden kaçının
* **Actionable Alerts**: Aksiyon alınabilir alert'ler
