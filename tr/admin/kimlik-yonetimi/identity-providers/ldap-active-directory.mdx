# LDAP

Kimlik Sağlayıcı, API'ye istek gönderecek kullanıcı havuzunu belirtir. Bu ön tanımlı Kimlik Sağlayıcıları, Kimlik Doğrulama Politikası oluşturulurken kullanılır.

"Project Owner" gibi "Kimlik Doğrulama Hizmetlerini Yönet" yetkisine sahip roller tarafından erişilebilir ve yönetilebilir.

LDAP ile kullanıcı doğrulama bağlantı ayarlarında kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan | Açıklama |
| --- | --- |
| Ad *(Name)* | Oluşturulan LDAP Kimlik Sağlayıcı adı bilgisidir. |
| Açıklama *(Description)* | Oluşturulan Kimlik Sağlayıcı ile ilgili yönetimi kolaylaştırmak için açıklama yazılabilir. |
| LDAP Bağlantısı *(LDAP Connection)* | LDAP bağlantısının alınacağı havuz seçilir ya da oluşturulur. |
| Kullanıcı Arama Filtresi *(User Search Filter)* | Kullanıcıları filtrelemek için kullanılacak filtre değeridir. |
| Kullanıcı DN Şablonu *(User DN Template)* | Kullanıcı DN'ini oluşturmak için kullanılacak şablon değeridir. |
| Kullanıcı Adı Niteliği *(Username Attribute)* | Kullanıcının kullanıcı adını belirten niteliğin adı girilir. Varsayılan değeri: uid. |
| Ad Niteliği *(First Name Attribute)* | Kullanıcının adını belirten niteliğin adı girilir. Varsayılan değeri: givenName. |
| Soyad Niteliği *(Last Name Attribute)* | Kullanıcının soyadını belirten niteliğin adı girilir. Varsayılan değeri: sn. |
| E-Posta Niteliği *(E-mail Attribute)* | Kullanıcının e-posta adresini belirten niteliğin adı girilir. Varsayılan değeri: mail. |
| Üyelik Niteliği *(Membership Attribute)* | Kullanıcının grup üyeliklerini belirten niteliğin adı girilir. Varsayılan değeri: isMemberOf. |
| Alınacak Nitelikler *(Attributes To Fetch)* | LDAP ile kimlik doğrulama yapıldığında, kimlik doğrulamanın yanı sıra kullanıcı hakkında hangi bilgilerin alınacağını belirtir.<br/><br/>Ldap Authentication Provider sayfasında, Advanced Settings (Gelişmiş Ayarlar) seçeneği etkinleştirildiğinde, "Attributes to Fetch" (Alınacak Attributeler) alanı belirir. Bu alana girilen attributeler, LDAP kullanıcısında mevcutsa, değerleriyle birlikte alınır ve;<br/>- Eğer LDAP provider JWT üretiminde kullanılıyorsa, bu attributeler ve değerleri (null olanlar hariç) JWT'nin içine claim olarak eklenir.<br/>- LDAP provider, plaintext, basic veya digest yetkilendirme yöntemlerinde kullanılıyorsa, bu attributeler ve değerleri, message context'ine custom variable olarak eklenir. Bu sırada key değerinin başına #clientLDAPAttribute#. prefix'i eklenir. Örneğin, LDAP üzerinden mail attribute'u alınmışsa, key değeri #clientLDAPAttribute#mail, value değeri ise "dummy@dummy.com" şeklinde olur. |

**LDAP ile kullanıcı doğrulama bağlantı ayarlarında gelişmiş ayarlardan Grup Sınıfı Tanımı ayarlarını içeren görsele aşağıda yer verilmiştir:**

![LDAP Grup Sınıfı Tanımı](https://docs.apinizer.com/apinizer-dokumantasyonu/files/1179929/130515563/1/1747811622229/image-2025-5-21_10-13-42.png)

LDAP ile kullanıcı doğrulama bağlantı ayarlarında gelişmiş ayarlardan Grup Sınıfı Tanımı için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan | Açıklama |
| --- | --- |
| Grup Sınıfı Adı *(Group Object Class Name)* | Grupları filtrelemek için kullanılacak sınıf adı girilir. Varsayılan değeri: groupOfUniqueNames. |
| Custom Filter Niteliği *(Custom Filter Attribute)* | Gruplar alınırken bağlantıdaki filtreye ek olarak kullanılabilecek filtre değeridir. |
| Group Base DN Niteliği *(Group Base DN Attribute)* | Bu alanda değer varsa bağlantıda Base DN yerine grup aranırken ve yüklenirken bu Base DN kullanılır. Hiçbir değer sağlanmazsa bağlantıdaki Base DN geçerli olur. |
| Arama Kapsamı *(Search Scope)* | Arama işleminin base dn üzerinde hangi seviyede yapılacağını belirtir. |
| Grup Adı Niteliği *(Group Name Attribute)* | Grup adını tutan niteliğin adını girilir. |
| Grup Üyeliği Niteliği *(Member Attribute)* | Grup üyelerini tutan niteliğin adını girilir. |
| Grup Üyeliği Belirleme Niteliği *(Member Strategy)* | Grup üyelerini belirlemek için kullanılacak yöntem seçilir. Varsayılan değeri: USER DN. Alabileceği değerler:<br/>- USER DN<br/>- USER LOGIN |
