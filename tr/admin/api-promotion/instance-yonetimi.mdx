## Instance Listesi

Instance yönetim ekranında tanımlı tüm hedef ortamları görebilirsiniz. Liste aşağıdaki bilgileri sunar:

- **Ad ve Açıklama**: Instance’ın tanımlayıcı bilgileri
- **URL**: Hedef ortamın adresi (Base URL)
- **Onay Durumu**: Onay gerekli mi ve (varsa) onaylayıcılar
- **Alıcılar**: (Varsa) bildirim alıcıları
- **Bağlantı Durumu**: Hedef ortama erişilebilirlik durumu (ör. Reachable / Not Reachable)

## Yeni Instance Oluşturma

Yeni bir hedef ortam eklemek için **New Instance** butonunu kullanabilirsiniz.

### Temel Bilgiler

| Alan | Açıklama |
|------|----------|
| **İsim** | Instance için benzersiz bir ad. Sadece İngilizce harf ve rakam kabul eder, boşluk kullanılamaz |
| **Açıklama** | Instance hakkında açıklayıcı bilgi |
| **Base URL** | Hedef Apinizer ortamının adresi |
| **API Token** | Hedef ortama erişim için kullanılacak API anahtarı (hedef ortamın ilgili ayarlarından temin edilir) |
| **Renk** | Instance’ı görsel olarak ayırt etmek için renk seçimi |

<img src="/images/promotion/promotion2.png" alt="Yeni instance oluşturma" width="800" />

## Bağlantı Testi

Instance kaydedildikten sonra hedef ortama erişimi doğrulamak için bağlantı testi yapabilirsiniz.

- **Liste ekranından**: İlgili instance satırındaki **(… menü)** → **Test**
- **Detay ekranından**: **Test** butonu

Test sonucunda bağlantı durumu ekranda güncellenir:

- **Başarılı**: Connection Status alanında **Reachable** görünür.
- **Başarısız**: Connection Status alanında **Not Reachable** görünür ve (varsa) hata mesajı görüntülenir.

## Onay ve Bildirim Ayarları

Kritik ortamlara (özellikle Production) yapılan aktarımlar için onay mekanizması yapılandırabilir, ayrıca bildirim alıcılarını belirleyebilirsiniz.

### Onay Mekanizması

**Approval required for promoting to this instance** seçeneğini aktifleştirdiğinizde, bu instance’a yapılacak promotion işlemleri çalıştırılmadan önce belirlenen onaylayıcılardan onay bekler.

| Ayar | Açıklama |
|------|----------|
| **Onay Gerekli** | Aktif olduğunda, çalıştırma öncesi belirlenen kişilerden onay beklenir |
| **Onaylayıcılar** | Çalıştırma isteğini onaylayabilecek kullanıcılar |

<img src="/images/promotion/promotion4.png" alt="Onay ayarları" width="800" />

### Bildirim Alıcıları

Promotion işlemlerinin durumu hakkında bilgilendirilecek alıcıları tanımlayabilirsiniz. Bildirimler şu durumlarda gönderilir:

- Çalıştırma başladığında
- Çalıştırma başarıyla tamamlandığında
- Çalıştırma hata aldığında

Bildirim alıcıları olarak sistemde tanımlı connector’ları (ör. e-posta connector’ı) ekleyebilirsiniz.

<img src="/images/promotion/promotion5.png" alt="Bildirim alıcıları" width="800" />

## Instance Detay Ekranı

Instance detay ekranında temel bilgiler ile birlikte bağlantı durumunu görüntüleyebilir ve **Test** butonu ile bağlantı testini tetikleyebilirsiniz. Bağlantı başarısız olduğunda, (varsa) hata mesajı da bu ekranda görülebilir.

<img src="/images/promotion/promotion6.png" alt="Instance detay ekranı" width="800" />

## Instance Silme

Bir instance’ı silmek istediğinizde onay dialogu görüntülenir.

<Warning>
    Bir instance silindiğinde, bu instance ile ilişkili mapping’ler ve execution akışları etkilenebilir. Silmeden önce ilişkili kayıtları kontrol etmeniz önerilir.
</Warning>
