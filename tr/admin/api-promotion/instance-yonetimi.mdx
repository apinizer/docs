Instance, API Promotion işlemlerinde hedef olarak kullanılan bir Apinizer ortamını temsil eder. Her instance, bir Apinizer kurulumunun bağlantı bilgilerini, sağlık durumunu, onay gereksinimlerini ve bildirim ayarlarını içerir.

## Instance Listesi

Instance yönetim ekranında tanımlı tüm hedef ortamları görebilirsiniz. Liste aşağıdaki bilgileri sunar:

- **Ad ve Açıklama**: Instance'ın tanımlayıcı bilgileri
- **Tip**: Ortam tipi (Development, Test, Staging, Production)
- **URL**: Hedef ortamın adresi
- **Sağlık Durumu**: Bağlantı sağlığı (Sağlıklı, Sorunlu, Bilinmiyor)
- **Onay Durumu**: Onay gerekli mi, kaç onaylayıcı tanımlı
- **Bildirim Alıcıları**: Kaç bildirim alıcısı tanımlı

## Yeni Instance Oluşturma

Yeni bir hedef ortam eklemek için **Yeni Instance** butonunu kullanabilirsiniz.

### Temel Bilgiler

| Alan | Açıklama |
|------|----------|
| **İsim** | Instance için benzersiz bir ad. Sadece İngilizce harf ve rakam kabul eder, boşluk kullanılamaz |
| **Açıklama** | Instance hakkında açıklayıcı bilgi |
| **Tip** | Ortam tipi: Development, Test, Staging veya Production |
| **Base URL** | Hedef Apinizer ortamının adresi |
| **API Token** | Hedef ortama erişim için kullanılacak API anahtarı. Bu token, hedef Apinizer ortamının APIops ayarlarından elde edilir |
| **Renk** | Instance'ı görsel olarak ayırt etmek için renk seçimi |
| **Aktif** | Instance'ın aktif/pasif durumu |

<!-- SCREENSHOT REQUIRED: Yeni instance oluşturma formu — temel bilgiler alanları dolu şekilde -->

### Bağlantı Testi

Instance bilgilerini girdikten sonra **Bağlantı Testi** butonuyla hedef ortama erişimi doğrulayabilirsiniz. Test sonucunda:

- **Başarılı**: Hedef ortama bağlantı kurulabildi
- **Başarısız**: Bağlantı kurulamadı — URL veya API Token bilgilerini kontrol edin

<!-- SCREENSHOT REQUIRED: Bağlantı testi başarılı sonucu -->

## Onay ve Bildirim Ayarları

Kritik ortamlara (özellikle Production) yapılan aktarımlar için onay mekanizması yapılandırabilirsiniz.

### Onay Mekanizması

**Onay Gerekli** seçeneğini aktifleştirdiğinizde, bu instance'a yapılacak tüm promotion işlemleri çalıştırılmadan önce belirlenen onaylayıcılardan onay bekler.

| Ayar | Açıklama |
|------|----------|
| **Onay Gerekli** | Aktif olduğunda, çalıştırma öncesi belirlenen kişilerden onay beklenir |
| **Onaylayıcılar** | Çalıştırma isteğini onaylayabilecek kullanıcılar |

<!-- SCREENSHOT REQUIRED: Onay ayarları bölümü — onay toggle aktif, onaylayıcı listesi dolu -->

### Bildirim Alıcıları

Promotion işlemlerinin durumu hakkında bilgilendirilecek alıcıları tanımlayabilirsiniz. Bildirimler şu durumlarda gönderilir:

- Çalıştırma başladığında
- Çalıştırma başarıyla tamamlandığında
- Çalıştırma hata aldığında

Bildirim alıcıları olarak sistemde tanımlı connector'ları (e-posta vb.) seçebilirsiniz.

<!-- SCREENSHOT REQUIRED: Bildirim alıcıları bölümü — en az bir alıcı eklenmiş -->

## Sağlık Kontrolü

Tanımlı instance'ların sağlık durumu otomatik olarak izlenir. Sağlık durumları:

| Durum | Anlamı |
|-------|--------|
| **Sağlıklı** | Hedef ortam ulaşılabilir ve yanıt veriyor |
| **Sorunlu** | Hedef ortam ulaşılabilir değil veya hata veriyor |
| **Bilinmiyor** | Henüz kontrol yapılmamış |

Instance detay ekranından sağlık durumu ve yanıt süresi bilgilerini görebilirsiniz.

<!-- SCREENSHOT REQUIRED: Instance detay ekranı — sağlık durumu bilgileri ve yanıt süresi -->

## Instance Silme

Bir instance'ı silmek istediğinizde onay dialogu görüntülenir.

<Warning>
Bir instance silindiğinde, bu instance'a ait mapping'ler ve execution geçmişi de etkilenebilir. Silmeden önce ilişkili mapping'leri kontrol edin.
</Warning>
