:::info

Erişim Kontrolü, kimlik bilgilerinin API Proxy'lere ve API Proxy Gruplarına erişim izinlerini; bu izinlere bağlı geçerlilik tarihi, ortam bazlı kota/daraltma ve izin verilmeyen metot/proxy ayarlarını **tek bir ekrandan** yönetmenizi sağlar. Bu sayfa, Kimlik Yönetimi altındaki Erişim Kontrolü ile ilgili sayfaları bir arada listeler.

:::

## Görünüm Modları

Erişim Kontrolü ekranı, aynı erişim ilişkisine üç farklı bakış açısıyla bakmanızı sağlayan üç moda sahiptir:

- **API Proxy modu**: Soldaki listeden bir API Proxy seçilir; sağda o API Proxy'e erişim izni verilmiş kimlik bilgileri ile API Proxy'nin metot/endpoint bazlı erişim kısıtları yönetilir.
- **Proxy Grubu modu**: Aynı mantık bir API Proxy Grubu ve grup üyesi API Proxy'ler için uygulanır.
- **Kimlik Bilgisi modu**: Soldaki listeden bir kimlik bilgisi seçilir; sağda bu kimlik bilgisinin erişim izni olduğu tüm API Proxy ve API Proxy Gruplarının listesi görüntülenir ve yönetilir (ters yönden bakış).

Seçilen kimlik bilgisi/API Proxy/Proxy Grubu ilişkisi için geçerlilik tarihi, ortam bazlı kota ve daraltma ayarları ile izin verilmeyen metot/proxy tanımları yapılandırılabilir. Değişiklikler **Kaydet ve Yükle** ile etkinleştirilir.

Alan bazında detaylı konfigürasyon açıklamaları için [API Proxy ACL](/tr/admin/identity-management/api-proxy-acl) ve [API Proxy Grup ACL](/tr/admin/identity-management/api-proxy-grup-acl) sayfalarını inceleyebilirsiniz.

:::warning

Kimlik bilgisi erişim izni, API proxy'e erişim için tek başına yeterli değildir. Kimlik bilgisindeki ve erişim iznindeki ayarların geçerli olması için API Proxy üzerinde kimlik doğrulama politikalarından birisi eklenmiş olmalı ve bu politikada kimlik doğrulama şekli olarak **Security Manager** seçeneği seçili olmalıdır.

:::
