## Filtreleme ve Arama

Uygulama Logları üzerinde yapılan işlemler ve filtreleme için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan | Açıklama |
| --- | --- |
| **Başlat**  | Uygulama loglarının API Proxy bazında kaydının tutulmasını sağlar. |
| **Ortam Filtresi** | Uygulama loglarının belirtilen ortam için filtrelenmesini sağlar. |
| **Zaman Filtresi**  | Uygulama loglarının belirtilen tarih aralığı için filtrelenmesini sağlar. |
| **Log Seviyesi**  | Uygulama loglarında belirtilen log seviyesindeki kayıtların filtrelenmesini sağlar. |
| **Correlation ID'ye göre ara**  | API Proxy'ye gelen isteklere Apinizer tarafından tanımlanan korelasyon ID'si ile uygulama loglarında detaylı arama yapılmasını sağlar. API Proxy Trafik loglarından alınan korelasyon ID ile sadece ilgili isteğin uygulama loglarına erişilebilir. |
| **Detaya göre ara**  | Uygulama loglarının metin alanlarında arama yapılmasını sağlar. |
| **Stack Trace'e göre ara**  | Uygulama loglarında exception oluşmuş ise exception metni üzerinde arama yapılmasını sağlar. |
| **İndir**  | Uygulama loglarının tek bir dosya olarak indirilmesini sağlar. "Hepsi" seçeneği ile API Proxy'ye ait tüm loglar, "Arama Sonuçları" seçeneği ile filtrelenen loglar indirilebilir. |
| **Sil** | Bu API Proxy'e ait uygulama log kayıtlarının silinmesini sağlar. |

<Note>
Bu sayfa yazılımsal bir hata olduğundan şüphe edildiğinde veya uygulamanın arka planda nasıl davrandığı araştırılmak istendiğinde kullanılabilir.
</Note>
