---
title: "SSL/TLS ile Apinizer Modüllerini Başlatma"
description: "Apinizer modüllerini (Manager, Gateway, Portal) SSL/TLS sertifikaları ile güvenli şekilde başlatır ve HTTPS bağlantıları sağlar."
---

## Yapılandırma Adımları

<Steps>
    <Step title="Sertifika Hazırlığı">
        Sertifikaları hazırlayın:

        <ul>
            <li>SSL/TLS sertifikası oluşturun veya temin edin</li>
            <li>Private key'i hazırlayın</li>
            <li>Certificate chain'i hazırlayın (gerekirse)</li>
        </ul>

        <Note>
            Sertifikaların geçerli olması ve güvenilir bir sertifika otoritesi tarafından imzalanmış olması önerilir.
        </Note>
    </Step>

    <Step title="Sertifika Formatı Dönüşümü">
        Gerekirse sertifika formatını JKS formatına dönüştürün:

        <ul>
            <li>PFX'ten JKS'e dönüştürme</li>
            <li>PEM'den JKS'e dönüştürme</li>
        </ul>

        <Tip>
            Sertifika dönüşüm işlemleri için <a href="/tr/operations/yonetici-kilavuzlari/pfx-jks-donusturme">PFX JKS Dönüştürme</a> sayfasına bakabilirsiniz.
        </Tip>
    </Step>

    <Step title="Kubernetes Secret Oluşturma">
        Sertifikaları Kubernetes secret olarak oluşturun:

        ```bash
        kubectl create secret tls apinizer-tls \
          --cert=certificate.crt \
          --key=private.key \
          -n apinizer
        ```

        <Warning>
            Secret oluştururken sertifika ve private key dosyalarının doğru yollarını belirttiğinizden emin olun.
        </Warning>
    </Step>

    <Step title="Deployment Yapılandırması">
        Deployment yapılandırmasında SSL ayarlarını yapın:

        ```yaml
        env:
          - name: SSL_ENABLED
            value: "true"
          - name: SSL_KEY_STORE
            value: "/etc/ssl/certs/keystore.jks"
          - name: SSL_KEY_STORE_PASSWORD
            valueFrom:
              secretKeyRef:
                name: apinizer-tls
                key: password
          - name: SERVER_PORT
            value: "8443"
        ```

        <Info>
            SSL ayarları environment variable'lar üzerinden yapılandırılır. Keystore dosyasının pod içinde erişilebilir bir konumda olması gerekir.
        </Info>
    </Step>
</Steps>
