## Apinizer Container İmajları

Kubernetes deployment ve Helm yapılandırmalarında aşağıdaki imaj depoları kullanılır:

| Bileşen | Varsayılan imaj deposu |
|---------|------------------------|
| API Manager | `apinizercloud/apimanager` |
| Worker | `apinizercloud/worker` |
| Cache | `apinizercloud/cache` |
| Integration | `apinizercloud/integration` |
| API Portal | `apinizercloud/apiportal` |

`<APINIZER_VERSION>` ifadesini kurduğunuz veya yükselttiğiniz sürümle değiştirin (örn. `2026.04.4`).

## Deployment Dosyalarında İmaj Alanı

Kurumsal registry kullanıldığında `image` alanı, kurumunuzun verdiği registry host ve path yapısına göre güncellenir:

```yaml
image: <REGISTRY_HOST>/<REGISTRY_PATH>/apinizercloud/cache:<APINIZER_VERSION>
```

**Nexus docker-proxy örneği:**

```yaml
image: repo.xxx.gov.tr/docker-proxy/apinizercloud/cache:2025.07.6
```

**Private registry örneği:**

```yaml
image: myregistry.local:5000/apinizercloud/manager:<APINIZER_VERSION>
```

:::info

Tüm Kubernetes worker node'larında registry yapılandırması tutarlı olmalıdır. Yapılandırma değişikliğinden sonra imaj çekimini node üzerinde test edin.

:::
