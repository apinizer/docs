Kurumunuz Nexus üzerinde docker-proxy repository tanımladıysa, Kubernetes worker node'larında containerd registry mirror yapılandırması yapılır ve Apinizer deployment dosyalarındaki imaj path'leri Nexus adresine göre güncellenir.

:::info

Bu sayfadaki `<NEXUS_HOST>` placeholder'ını kurumunuzun registry adresiyle değiştirin (örn. `repo.xxx.gov.tr`). Path segmenti (`docker-proxy` vb.) kurumunuzun Nexus yapılandırmasına göre değişebilir.

:::

## Containerd Yapılandırması (Önerilen)

Modern Kubernetes kurulumları containerd kullanır. Yapılandırmayı **cluster'daki tüm worker node'larda** uygulayın.

<Steps>
  <Step title="containerd config.toml dosyasını düzenleyin">
    ```bash
    sudo su
    sudo vi /etc/containerd/config.toml
    ```

    `[plugins."io.containerd.grpc.v1.cri".registry]` bölümüne aşağıdaki mirror tanımını ekleyin (`<NEXUS_HOST>` değerini kurum adresinizle değiştirin):

    ```toml
    [plugins."io.containerd.grpc.v1.cri".registry.mirrors."<NEXUS_HOST>"]
      endpoint = ["https://<NEXUS_HOST>"]
    ```

    **Örnek:**

    ```toml
    [plugins."io.containerd.grpc.v1.cri".registry.mirrors."repo.xxx.gov.tr"]
      endpoint = ["https://repo.xxx.gov.tr"]
    ```
  </Step>

  <Step title="containerd servisini yeniden başlatın">
    ```bash
    sudo systemctl restart containerd
    sudo systemctl status containerd
    ```
  </Step>

  <Step title="İmaj çekimini doğrulayın">
    Node üzerinde imajın çekilebildiğini test edin:

    ```bash
    sudo ctr images pull <NEXUS_HOST>/docker-proxy/apinizercloud/cache:<APINIZER_VERSION>
    ```

    **Örnek:**

    ```bash
    sudo ctr images pull repo.xxx.gov.tr/docker-proxy/apinizercloud/cache:2025.07.6
    ```
  </Step>
</Steps>

## Kubernetes Deployment Yapılandırması

Deployment, StatefulSet veya Helm values dosyalarında `image` alanını Nexus path yapınıza göre güncelleyin:

```yaml
image: <NEXUS_HOST>/docker-proxy/apinizercloud/cache:<APINIZER_VERSION>
```

**Örnek:**

```yaml
image: repo.xxx.gov.tr/docker-proxy/apinizercloud/cache:2025.07.6
```

Diğer Apinizer bileşenleri için aynı path kalıbını kullanın:

```yaml
# API Manager
image: repo.xxx.gov.tr/docker-proxy/apinizercloud/apimanager:<APINIZER_VERSION>

# Worker
image: repo.xxx.gov.tr/docker-proxy/apinizercloud/worker:<APINIZER_VERSION>

# Integration
image: repo.xxx.gov.tr/docker-proxy/apinizercloud/integration:<APINIZER_VERSION>

# API Portal
image: repo.xxx.gov.tr/docker-proxy/apinizercloud/apiportal:<APINIZER_VERSION>
```

## Docker Runtime Kullanıyorsanız

<Tabs>
  <TabItem value="docker-nexus" label="Docker daemon.json">
    Eski ortamlarda veya Docker runtime kullanan node'larda `/etc/docker/daemon.json` dosyasını düzenleyin. **Docker kullanan tüm node'larda** aynı yapılandırmayı uygulayın.

    ```bash
    sudo vi /etc/docker/daemon.json
    ```

    ```json
    {
      "data-root": "/docker-data",
      "insecure-registries": ["<NEXUS_HOST>"],
      "registry-mirrors": ["https://<NEXUS_HOST>"],
      "exec-opts": ["native.cgroupdriver=systemd"],
      "log-driver": "json-file",
      "log-opts": {
        "max-size": "100m"
      },
      "storage-driver": "overlay2"
    }
    ```

    ```bash
    sudo systemctl daemon-reload
    sudo systemctl restart docker
    docker info
    ```
  </TabItem>
</Tabs>
