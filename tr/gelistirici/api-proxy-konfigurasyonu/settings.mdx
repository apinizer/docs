## CORS Ayarları

CORS (Cross-Origin Resource Sharing) ayarları, farklı origin'lerden gelen isteklere izin vermek için kullanılır.

### CORS Etkinleştirme

CORS'u etkinleştirmek için **Enable CORS** seçeneğini işaretleyin.

### Allowed Origins

İzin verilen origin'lerin listesi. Wildcard (`*`) kullanarak tüm origin'lere izin verebilirsiniz.

**Örnekler:**
```
https://example.com
https://*.example.com
*
```

### Allowed Methods

CORS isteklerinde izin verilen HTTP metodları.

**Varsayılan Metodlar:**
* GET
* POST
* PUT
* DELETE
* OPTIONS

### Allowed Headers

CORS isteklerinde izin verilen header'lar.

**Yaygın Header'lar:**
* `Content-Type`
* `Authorization`
* `X-Requested-With`

### Exposed Headers

İstemciye expose edilecek header'lar.

### Max Age

Preflight isteklerinin cache süresi (saniye cinsinden).

**Varsayılan**: 3600 saniye (1 saat)

### Credentials

Credentials (cookies, authorization headers) gönderilip gönderilmeyeceği.

## Timeout Ayarları

Timeout ayarları, API Proxy'nin istek ve yanıt timeout değerlerini kontrol eder.

### Request Timeout

İstemciden gelen isteğin timeout süresi (milisaniye cinsinden).

**Varsayılan**: 30000 ms (30 saniye)

**Öneriler:**
* Kısa süreli işlemler için: 10000-30000 ms
* Uzun süreli işlemler için: 60000-300000 ms

### Backend Timeout

Backend API'ye gönderilen isteğin timeout süresi (milisaniye cinsinden).

**Varsayılan**: 30000 ms (30 saniye)

**Öneriler:**
* Hızlı backend'ler için: 10000-20000 ms
* Yavaş backend'ler için: 60000-120000 ms

### Connection Timeout

Backend API'ye bağlantı kurma timeout süresi (milisaniye cinsinden).

**Varsayılan**: 5000 ms (5 saniye)

## Retry Politikaları

Retry politikaları, başarısız istekler için yeniden deneme davranışını kontrol eder.

### Retry Etkinleştirme

Retry mekanizmasını etkinleştirmek için **Enable Retry** seçeneğini işaretleyin.

### Max Retry Count

Maksimum yeniden deneme sayısı.

**Varsayılan**: 3

**Öneriler:**
* Kritik olmayan işlemler için: 1-2
* Kritik işlemler için: 3-5

### Retry Conditions

Yeniden deneme koşulları. Hangi durumlarda retry yapılacağını belirler.

**Koşullar:**
* **Connection Error**: Bağlantı hatası
* **Timeout**: Timeout hatası
* **5xx Errors**: Sunucu hataları (500-599)
* **4xx Errors**: İstemci hataları (400-499) - genellikle retry edilmez

### Retry Interval

Yeniden denemeler arasındaki bekleme süresi (milisaniye cinsinden).

**Stratejiler:**
* **Fixed**: Sabit bekleme süresi
* **Exponential Backoff**: Üstel artan bekleme süresi

**Örnek Exponential Backoff:**
* 1. retry: 1000 ms
* 2. retry: 2000 ms
* 3. retry: 4000 ms

## Hata Mesajı Şablonları

Hata mesajı şablonları, API Proxy'nin döndüreceği hata mesajlarını özelleştirmenizi sağlar.

### Hata Mesajı Formatı

Hata mesajlarının formatını belirleyin.

**Varsayılan Format:**
```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Error message",
    "timestamp": "2024-01-01T00:00:00Z"
  }
}
```

### Özelleştirme

Hata mesajı şablonlarını özelleştirebilirsiniz:

* **Error Code**: Hata kodu
* **Error Message**: Hata mesajı
* **Additional Fields**: Ek alanlar

### Hata Türleri

Farklı hata türleri için farklı şablonlar tanımlanabilir:

* **400 Bad Request**: Geçersiz istek
* **401 Unauthorized**: Yetkilendirme hatası
* **403 Forbidden**: Erişim reddedildi
* **404 Not Found**: Kaynak bulunamadı
* **500 Internal Server Error**: Sunucu hatası
* **502 Bad Gateway**: Gateway hatası
* **503 Service Unavailable**: Servis kullanılamıyor
* **504 Gateway Timeout**: Gateway timeout

## Backend API Ayarları

Backend API ayarları, API Proxy'nin backend API ile nasıl iletişim kuracağını kontrol eder.

### Backend URL

Backend API'nin base URL'i.

**Örnek:**
```
https://api.example.com
http://localhost:8080
```

### Connection Pooling

Backend bağlantı havuzu ayarları.

**Ayarlar:**
* **Max Connections**: Maksimum bağlantı sayısı
* **Max Connections Per Route**: Route başına maksimum bağlantı sayısı
* **Connection Timeout**: Bağlantı timeout süresi

### SSL/TLS Ayarları

Backend API ile SSL/TLS bağlantısı ayarları.

**Ayarlar:**
* **SSL Verification**: SSL sertifika doğrulama
* **Trust All Certificates**: Tüm sertifikalara güven (sadece development için)
* **Custom Truststore**: Özel truststore kullanımı

### Authentication

Backend API kimlik doğrulama ayarları.

**Desteklenen Yöntemler:**
* **Basic Authentication**: Kullanıcı adı ve şifre
* **Bearer Token**: Token tabanlı kimlik doğrulama
* **API Key**: API anahtarı
* **Custom Header**: Özel header tabanlı kimlik doğrulama

## Güvenlik Ayarları

Güvenlik ayarları, API Proxy'nin güvenlik davranışını kontrol eder.

### Request Validation

İstek doğrulama ayarları.

**Seçenekler:**
* **Validate Request Size**: İstek boyutu doğrulama
* **Validate Content-Type**: Content-Type doğrulama
* **Validate Required Headers**: Zorunlu header doğrulama

### Response Validation

Yanıt doğrulama ayarları.

**Seçenekler:**
* **Validate Response Size**: Yanıt boyutu doğrulama
* **Validate Content-Type**: Content-Type doğrulama

### Rate Limiting

Rate limiting ayarları (global policies ile yönetilir).

## Kullanım Senaryoları

### Senaryo 1: Public API

Public API için CORS ayarları:

* **Enable CORS**: ✅
* **Allowed Origins**: `*`
* **Allowed Methods**: GET, POST, PUT, DELETE
* **Credentials**: ❌

### Senaryo 2: High Availability Backend

Yüksek kullanılabilirlik için retry ayarları:

* **Enable Retry**: ✅
* **Max Retry Count**: 5
* **Retry Conditions**: Connection Error, Timeout, 5xx Errors
* **Retry Interval**: Exponential Backoff

### Senaryo 3: Slow Backend

Yavaş backend için timeout ayarları:

* **Request Timeout**: 120000 ms (2 dakika)
* **Backend Timeout**: 60000 ms (1 dakika)
* **Connection Timeout**: 10000 ms (10 saniye)

## En İyi Uygulamalar

### Timeout Ayarları

* **Request Timeout**: İstemci beklentilerine göre ayarlayın
* **Backend Timeout**: Backend performansına göre ayarlayın
* **Connection Timeout**: Ağ koşullarına göre ayarlayın

### Retry Politikaları

* **Idempotent Operations**: Sadece idempotent işlemler için retry kullanın
* **Exponential Backoff**: Üstel backoff kullanın
* **Max Retry Count**: Makul bir değer seçin (3-5)

### CORS Ayarları

* **Specific Origins**: Mümkünse wildcard yerine spesifik origin'ler kullanın
* **Credentials**: Gerekli olmadıkça credentials göndermeyin
* **Max Age**: Preflight cache süresini optimize edin
