## Genel Bilgiler

### Name (Ad)

API Proxy'nin adı. Bu alan zorunludur ve proje bazında benzersiz olmalıdır.

**Özellikler:**
* Proje bazında benzersiz olmalıdır
* Boşluk ve özel karakterler kullanılabilir, ancak önerilmez
* Anlamlı ve açıklayıcı bir isim seçilmelidir

### Description (Açıklama)

API Proxy'nin amacını ve kullanım senaryosunu açıklayan opsiyonel bir metin alanı.

**Kullanım:**
* API Proxy'nin ne için kullanıldığını açıklar
* Yönetim ve bakım süreçlerinde referans olarak kullanılır
* API dokümantasyonunda görüntülenebilir

## Client Route

Client Route, istemcilerin API Proxy'ye erişmek için kullanacağı yönlendirme bilgisidir. Client Route yapılandırması için en az bir seçenek (Path, Method veya Header bazlı) seçilmelidir.

### Relative Paths

API Proxy'ye yönlendirilecek relative URL path'leri. Birden fazla path tanımlanabilir.

**Özellikler:**
* Her path "/" ile başlamalıdır
* Uygulama bazında tekil olmalıdır
* Başka bir path'in başlangıcı olamaz

**Örnek:**
```
/petstoreProxy
/api/v1/users
/products
```

### Methods

Bu API Proxy için izin verilen HTTP metodları.

**Desteklenen Metodlar:**
* **GET**: Veri okuma işlemleri
* **POST**: Yeni kayıt oluşturma
* **PUT**: Kayıt güncelleme (tam güncelleme)
* **DELETE**: Kayıt silme
* **PATCH**: Kısmi güncelleme
* **OPTIONS**: CORS preflight istekleri
* **HEAD**: Header bilgilerini alma

Varsayılan olarak tüm metodlar seçilebilir.

### Hosts

Backend servislerin host adresleri. Birden fazla host eklenebilir.

**Kullanım Senaryoları:**
* **Load Balancing**: Birden fazla backend arasında yük dağılımı
* **Failover**: Bir backend başarısız olursa diğerine geçiş
* **Environment Management**: Farklı ortamlar için farklı host'lar

**Örnek:**
```
https://api.example.com
https://api-backup.example.com
```

### Headers

API Proxy seviyesinde eklenecek veya değiştirilecek HTTP header'ları. Key-value çiftleri olarak tanımlanır.

**Kullanım Senaryoları:**
* **Custom Headers**: Özel header'lar ekleme
* **Header Transformation**: Header değerlerini dönüştürme
* **Security Headers**: Güvenlik header'ları ekleme

**Örnek:**
```
X-API-Version: v1
X-Request-ID: ${request.id}
Authorization: Bearer ${token}
```

## API Proxy Key

API Proxy Key bölümü, JWT veya OAuth2 yetkilendirmesi için gerekli Public ve Secret Key'lerin yönetildiği bölümdür.

### Key Yönetimi

API Proxy Key'ler şu amaçlarla kullanılır:

* **JWT Token Verification**: JWT token'ların doğrulanması
* **OAuth2 Authorization**: OAuth2 yetkilendirme işlemleri
* **API Authentication**: API kimlik doğrulama

### Public Key

Public Key, token doğrulama işlemlerinde kullanılan açık anahtardır.

**Özellikler:**
* Güvenli bir şekilde paylaşılabilir
* Token doğrulama için kullanılır
* PEM formatında saklanır

### Secret Key

Secret Key, token imzalama ve şifreleme işlemlerinde kullanılan gizli anahtardır.

**Özellikler:**
* Güvenli bir şekilde saklanmalıdır
* Token imzalama için kullanılır
* Asla paylaşılmamalıdır

### Key Oluşturma

Yeni bir key çifti oluşturmak için:

1. **Generate Key Pair** butonuna tıklayın
2. Key formatını seçin (RSA, ECDSA, vb.)
3. Key boyutunu belirleyin
4. **Generate** butonuna tıklayın

### Key Import

Mevcut bir key çiftini import etmek için:

1. **Import Key** butonuna tıklayın
2. Public Key'i yapıştırın veya dosyadan yükleyin
3. Secret Key'i yapıştırın veya dosyadan yükleyin
4. **Import** butonuna tıklayın

## Kullanım Senaryoları

### Senaryo 1: Basit REST API Proxy

Basit bir REST API Proxy için minimal konfigürasyon:

* **Name**: `UserServiceAPI`
* **Relative Path**: `/users`
* **Methods**: GET, POST, PUT, DELETE
* **Host**: `https://api.example.com`

### Senaryo 2: Load Balanced API Proxy

Yük dengeleme için birden fazla host:

* **Name**: `ProductServiceAPI`
* **Relative Path**: `/products`
* **Hosts**: 
  * `https://api1.example.com`
  * `https://api2.example.com`
  * `https://api3.example.com`

### Senaryo 3: JWT Protected API Proxy

JWT token doğrulama ile korumalı API Proxy:

* **Name**: `SecureAPI`
* **Relative Path**: `/secure`
* **API Proxy Key**: JWT Public Key tanımlı
* **Methods**: GET, POST

## En İyi Uygulamalar

### İsimlendirme

* **Anlamlı İsimler**: API Proxy isimlerini açıklayıcı tutun
* **Tutarlılık**: İsimlendirme konvansiyonunu tutarlı tutun
* **Kısa ve Öz**: İsimleri kısa ve öz tutun

### Client Route

* **Tekil Path'ler**: Path'lerin uygulama bazında tekil olduğundan emin olun
* **RESTful Convention**: RESTful konvansiyonlarına uyun
* **Versioning**: Versiyonlama için path'leri kullanın (örn: `/api/v1/users`)

### Key Yönetimi

* **Güvenlik**: Secret Key'leri güvenli bir şekilde saklayın
* **Rotation**: Key'leri düzenli olarak rotate edin
* **Backup**: Key'lerin yedeğini alın
