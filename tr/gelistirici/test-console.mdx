Test Console, API Proxy'lerinizi test etmek için kapsamlı bir test aracıdır. Bu araç sayesinde API endpoint'lerinize istek gönderebilir, yanıtları inceleyebilir, test senaryolarını kaydedebilir ve assertion'lar tanımlayabilirsiniz.

## Genel Bakış

Test Console, API Proxy'lerinizi test etmek ve doğrulamak için kullanılan bir araçtır. Bu araç özellikle:

* API endpoint'lerini test etmek
* İstek/yanıt akışını incelemek
* Test senaryolarını kaydetmek ve yeniden kullanmak
* Assertion'lar ile otomatik doğrulama yapmak

için kullanılır.

## Test Console Kullanımı

### Test Console'a Erişim

Test Console'a erişmek için:

1. API Proxy detay sayfasına gidin
2. Sol taraftan bir endpoint seçin
3. **Test Endpoint** butonuna tıklayın
4. Test Console açılır

### İstek Oluşturma

Test Console'da bir istek oluşturmak için:

#### HTTP Metodu Seçimi

İstek için HTTP metodunu seçin:

* **GET**: Veri okuma
* **POST**: Yeni kayıt oluşturma
* **PUT**: Kayıt güncelleme
* **DELETE**: Kayıt silme
* **PATCH**: Kısmi güncelleme
* **OPTIONS**: CORS preflight
* **HEAD**: Header bilgileri

#### URL Yapılandırması

İstek URL'ini yapılandırın:

* **Base URL**: API Proxy'nin base URL'i (otomatik doldurulur)
* **Path**: Endpoint path'i (otomatik doldurulur)
* **Query Parameters**: Query parametreleri

**Örnek:**
```
GET https://demo.apinizer.com/apigateway/petstoreProxy/pet/findByStatus?status=pending
```

#### Headers

İstek header'larını ekleyin:

**Varsayılan Header'lar:**
* `Content-Type`: `application/json`
* `Accept`: `application/json`

**Özel Header'lar:**
* `Authorization`: `Bearer token`
* `X-API-Key`: `your-api-key`
* `X-Request-ID`: `unique-id`

#### Request Body

İstek gövdesini yapılandırın:

**Body Formatları:**
* **JSON**: JSON formatında gövde
* **XML**: XML formatında gövde
* **Form Data**: Form data formatında gövde
* **Raw**: Ham metin formatında gövde

**JSON Örneği:**
```json
{
  "name": "Test Pet",
  "status": "available"
}
```

#### Path Parameters

Path parametrelerini doldurun:

**Örnek:**
```
Path: /pet/{petId}
Parameter: petId = 123
Result: /pet/123
```

#### Query Parameters

Query parametrelerini ekleyin:

**Örnek:**
```
status=pending
limit=10
offset=0
```

### İsteği Gönderme

İsteği göndermek için:

1. Tüm gerekli alanları doldurun
2. **Send** butonuna tıklayın
3. Yanıt bekleyin

### Yanıt İnceleme

Yanıt geldiğinde şu bilgileri görüntüleyebilirsiniz:

#### Status Code

HTTP durum kodu:

* **2xx**: Başarılı
* **4xx**: İstemci hatası
* **5xx**: Sunucu hatası

#### Response Headers

Yanıt header'ları:

* `Content-Type`
* `Content-Length`
* `Cache-Control`
* Özel header'lar

#### Response Body

Yanıt gövdesi:

* **JSON**: JSON formatında görüntüleme
* **XML**: XML formatında görüntüleme
* **Raw**: Ham metin formatında görüntüleme
* **Preview**: Önizleme modu

#### Response Time

Yanıt süresi:

* **Total Time**: Toplam süre
* **Connection Time**: Bağlantı süresi
* **Response Time**: Yanıt süresi

## Test Senaryoları

Test Console, test senaryolarını kaydetmenizi ve yeniden kullanmanızı sağlar.

### Test Senaryosu Kaydetme

Bir test senaryosunu kaydetmek için:

1. İsteği yapılandırın
2. **Save Test Case** butonuna tıklayın
3. Test senaryosu adını girin
4. Açıklama ekleyin (opsiyonel)
5. **Save** butonuna tıklayın

### Test Senaryosu Yükleme

Kaydedilmiş bir test senaryosunu yüklemek için:

1. **Load Test Case** butonuna tıklayın
2. Test senaryosu listesinden birini seçin
3. Senaryo yüklenir

### Test Senaryosu Yönetimi

Test senaryolarını yönetmek için:

* **List**: Tüm test senaryolarını listele
* **Edit**: Test senaryosunu düzenle
* **Delete**: Test senaryosunu sil
* **Duplicate**: Test senaryosunu kopyala
* **Export**: Test senaryosunu export et
* **Import**: Test senaryosunu import et

## Assertions

Assertions, yanıtın doğru olup olmadığını otomatik olarak kontrol eder.

### Assertion Türleri

#### Status Code Assertion

Durum kodu assertion'ı:

```
Expected: 200
Actual: 200
Result: ✅ Pass
```

#### Header Assertion

Header assertion'ı:

```
Header: Content-Type
Expected: application/json
Actual: application/json
Result: ✅ Pass
```

#### Body Assertion

Gövde assertion'ı:

**JSON Path Assertion:**
```
Path: $.status
Expected: available
Actual: available
Result: ✅ Pass
```

**Contains Assertion:**
```
Expected: "name"
Actual: {"name": "Test Pet", ...}
Result: ✅ Pass
```

**Regex Assertion:**
```
Pattern: ^[0-9]+$
Value: 123
Result: ✅ Pass
```

### Assertion Ekleme

Assertion eklemek için:

1. **Add Assertion** butonuna tıklayın
2. Assertion türünü seçin
3. Beklenen değeri girin
4. **Save** butonuna tıklayın

### Assertion Sonuçları

Assertion sonuçları:

* **Pass**: Assertion başarılı
* **Fail**: Assertion başarısız
* **Error**: Assertion hatası

## Environment Variables

Test Console, environment variable'ları kullanmanızı sağlar.

### Variable Kullanımı

Variable'ları istek içinde kullanabilirsiniz:

**Örnek:**
```
URL: ${baseUrl}/api/users
Header: Authorization: Bearer ${token}
Body: {"userId": "${userId}"}
```

### Variable Tanımlama

Variable tanımlamak için:

1. **Variables** sekmesine gidin
2. **Add Variable** butonuna tıklayın
3. Variable adı ve değerini girin
4. **Save** butonuna tıklayın

### Predefined Variables

Önceden tanımlı variable'lar:

* `${baseUrl}`: API Proxy base URL'i
* `${timestamp}`: Mevcut timestamp
* `${randomString}`: Rastgele string
* `${randomNumber}`: Rastgele sayı

### Request History

İstek geçmişi:

* Son gönderilen istekler
* İstek detayları
* Yanıt bilgileri
* Yeniden gönderme

### Request Collection

İstek koleksiyonu:

* Birden fazla isteği gruplandırma
* Sıralı istek gönderme
* Koleksiyon çalıştırma

### Script Execution

Script çalıştırma:

* Pre-request script'leri
* Post-response script'leri
* Dinamik değer üretme

### Authentication

Kimlik doğrulama:

* **Basic Auth**: Kullanıcı adı ve şifre
* **Bearer Token**: Token tabanlı
* **API Key**: API anahtarı
* **OAuth2**: OAuth2 flow

## Kullanım Senaryoları

### Senaryo 1: Basit GET İsteği

Basit bir GET isteği test etme:

1. Metod: GET
2. Path: `/api/users`
3. **Send** butonuna tıklayın
4. Yanıtı inceleyin

### Senaryo 2: POST İsteği ile Veri Oluşturma

POST isteği ile veri oluşturma:

1. Metod: POST
2. Path: `/api/users`
3. Body: `{"name": "John", "email": "john@example.com"}`
4. **Send** butonuna tıklayın
5. Yanıtı inceleyin

### Senaryo 3: Assertion ile Doğrulama

Assertion ile doğrulama:

1. İsteği yapılandırın
2. Assertion ekleyin:
   * Status Code: 200
   * Body contains: "success"
3. **Send** butonuna tıklayın
4. Assertion sonuçlarını kontrol edin
