Global Policies (Global Politikalar), birden fazla API Proxy'de ortak olarak kullanılan politikaları merkezi bir noktadan yönetmenizi sağlar. Bu politikalar, cross-cutting concerns (çapraz kesen endişeler) olarak adlandırılır ve tüm API Proxy'lerde tutarlı bir şekilde uygulanır.

## Global Policies Nedir?

Global Policies, proje seviyesinde tanımlanan ve birden fazla API Proxy'de kullanılabilen politikalardır. Bu politikalar, API Proxy seviyesinde tanımlanan politikalardan önce çalışır ve merkezi yönetim sağlar.

## Cross-Cutting Concerns

Cross-cutting concerns, birden fazla API Proxy'de ortak olarak uygulanması gereken endişelerdir. Bu endişeler genellikle şunları içerir:

<AccordionGroup>
  <Accordion title="Güvenlik">

    * Authorization (Yetkilendirme)
    * IP Whitelist/Blacklist
    * mTLS
    * Rate Limiting
  </Accordion>
  
  <Accordion title="Logging ve İzleme">
    * Request/Response Logging
    * Audit Logging
    * Performance Monitoring
    * Error Tracking

  <Accordion title="Dönüştürme">

  <Accordion title="Uyumluluk">

## Global Policy Oluşturma

### Adım 1: Global Policies Sayfasına Gidin

1. Sol taraftaki ana menüde **Geliştirme → Global Policies** ögesini seçin.
2. Açılan arayüzde sağ üstteki **+Create** butonuna tıklayın.

### Adım 2: Policy Tipini Seçin

Kullanmak istediğiniz policy tipini seçin:

* **Authentication Policies**: OAuth2, OIDC, JWT, Basic, Digest, mTLS
* **Authorization Policies**: IP Whitelist/Blacklist, Time Restriction
* **Traffic Management**: Rate Limiting, Quota Management
* **Validation**: JSON Schema Validation, XML Schema Validation
* **Transformation**: JSON Transformation, XML Transformation
* **Logging**: Request/Response Logging, Audit Logging

### Adım 3: Policy Yapılandırması

Seçtiğiniz policy tipine göre yapılandırma yapın:

* **Policy Adı**: Policy'yi tanımlayan benzersiz isim
* **Açıklama**: Policy'nin amacı ve kapsamı
* **Yapılandırma Parametreleri**: Policy tipine özel parametreler
* **Koşullar**: Policy'nin ne zaman çalışacağını belirleyen koşullar (opsiyonel)

### Adım 4: Priority Ayarlama

Global Policies, priority (öncelik) sırasına göre çalışır. Priority değeri düşük olan policy, yüksek olan policy'den önce çalışır.

**Priority Yönetimi:**
* Priority değeri 1-1000 arasında olabilir
* Düşük priority değeri = Yüksek öncelik
* Aynı priority değerine sahip policy'ler tanımlandıkları sırayla çalışır

### Adım 5: Policy'yi Kaydedin

1. Tüm bilgileri kontrol edin.
2. Priority değerini ayarlayın.
3. Sağ üstteki **Kaydet (Save)** butonuna tıklayın.

**Global Policy başarıyla oluşturuldu!**

## Global Policy Kullanımı

### API Proxy'ye Global Policy Ekleme

1. API Proxy düzenleme sayfasına gidin.
2. **Design** sekmesine tıklayın.
3. **Global Policies** bölümüne gidin.
4. Kullanmak istediğiniz Global Policy'leri seçin.
5. Policy'lerin sırasını ayarlayın (drag & drop ile).
6. Kaydedin.

### Global Policy Önceliği

Global Policies, API Proxy seviyesinde tanımlanan politikalardan önce çalışır:

```
1. Global Policies (Pre-flow)
2. API Proxy Policies (Pre-flow)
3. Conditional Policies
4. Route Step
5. API Proxy Policies (Post-flow)
6. Global Policies (Post-flow)
```

## Global Policy Import/Export

### Global Policy Import

Mevcut bir Global Policy'yi import edebilirsiniz:

1. **Global Policies** sayfasına gidin.
2. **Import** butonuna tıklayın.
3. Import edilecek policy dosyasını seçin (JSON formatında).
4. Import ayarlarını yapılandırın:
   * **Conflict Resolution**: Mevcut policy'lerle çakışma durumunda ne yapılacağını seçin
     * **Skip**: Mevcut policy'yi koru
     * **Replace**: Mevcut policy'yi değiştir
     * **Rename**: Yeni bir isimle ekle
5. **Import** butonuna tıklayın.

:::info Import Formatı
Global Policy import dosyası JSON formatında olmalıdır ve policy yapılandırmasını içermelidir.
:::

### Global Policy Export

Mevcut Global Policy'leri export edebilirsiniz:

1. **Global Policies** sayfasına gidin.
2. Export etmek istediğiniz policy'leri seçin.
3. **Export** butonuna tıklayın.
4. Export formatını seçin (JSON).
5. **Export** butonuna tıklayın.
6. Dosya indirilir.

## Priority Handling

### Priority Sıralaması

Global Policies, priority değerine göre sıralanır:

* **Priority 1-100**: Yüksek öncelikli politikalar (örn: Authentication)
* **Priority 101-500**: Orta öncelikli politikalar (örn: Rate Limiting)
* **Priority 501-1000**: Düşük öncelikli politikalar (örn: Logging)

### Priority Çakışmaları

Aynı priority değerine sahip policy'ler varsa:

* Policy'ler tanımlandıkları sırayla çalışır
* Sıralama, policy oluşturulma tarihine göre belirlenir
* Manuel olarak sıralama değiştirilebilir (drag & drop)

## Global Policy Yönetimi

### Policy Güncelleme

1. **Global Policies** sayfasına gidin.
2. Güncellemek istediğiniz policy'ye tıklayın.
3. Yapılandırmayı düzenleyin.
4. **Kaydet (Save)** butonuna tıklayın.

:::warning Güncelleme Etkisi
Global Policy güncellendiğinde, bu policy'yi kullanan tüm API Proxy'ler otomatik olarak güncellenir. Bu, tüm API Proxy'leri etkileyebilir.
:::

### Policy Silme

1. **Global Policies** sayfasına gidin.
2. Silmek istediğiniz policy'yi seçin.
3. **Sil (Delete)** butonuna tıklayın.
4. Silme işlemini onaylayın.

:::warning Silme Etkisi
Global Policy silindiğinde, bu policy'yi kullanan tüm API Proxy'lerden kaldırılır. Bu, API Proxy'lerin davranışını değiştirebilir.
:::

### Policy Kullanımını Görüntüleme

Hangi API Proxy'lerin bir Global Policy'yi kullandığını görebilirsiniz:

1. **Global Policies** sayfasına gidin.
2. Policy'ye tıklayın.
3. **Kullanım (Usage)** sekmesine gidin.
4. Policy'yi kullanan API Proxy'lerin listesi görüntülenir.

## Yaygın Senaryolar

### Senaryo 1: Merkezi Authentication

Tüm API Proxy'lerde aynı authentication mekanizmasını kullanmak:

1. OAuth2 Global Policy oluşturun.
2. Tüm API Proxy'lere bu policy'yi ekleyin.
3. Authentication yapılandırmasını tek bir yerden yönetin.

### Senaryo 2: Merkezi Rate Limiting

Tüm API Proxy'lerde aynı rate limiting kurallarını uygulamak:

1. Rate Limiting Global Policy oluşturun.
2. Rate limit değerlerini yapılandırın.
3. Tüm API Proxy'lere bu policy'yi ekleyin.

### Senaryo 3: Merkezi Logging

Tüm API Proxy'lerde aynı logging stratejisini uygulamak:

1. Request/Response Logging Global Policy oluşturun.
2. Log formatını ve hedefini yapılandırın.
3. Tüm API Proxy'lere bu policy'yi ekleyin.

## Sorun Giderme

### Policy Çalışmıyor

**Sorun:** Global Policy çalışmıyor.

**Çözüm:**
* Policy'nin API Proxy'ye eklendiğinden emin olun.
* Priority değerini kontrol edin.
* Policy yapılandırmasını kontrol edin.
* Policy loglarını inceleyin.

### Priority Çakışması

**Sorun:** Policy'ler beklenen sırada çalışmıyor.

**Çözüm:**
* Priority değerlerini kontrol edin.
* Aynı priority değerine sahip policy'lerin sırasını kontrol edin.
* Policy sıralamasını manuel olarak ayarlayın.

### Import Hatası

**Sorun:** Global Policy import edilemiyor.

**Çözüm:**
* Import dosyasının geçerli JSON formatında olduğundan emin olun.
* Policy yapılandırmasının doğru olduğunu kontrol edin.
* Conflict resolution ayarlarını kontrol edin.
