## Yeni Mock API Oluşturma

<Steps>
  <Step title="Menüden Mock API Sayfasına Gidin">
    Ana menüde **Geliştirme → API Oluşturucu → Mock API** (*Development → API Creator → Mock API*) ögesi seçilir.
  </Step>
  
  <Step title="Yeni API Oluşturun">
    Açılan arayüzde sağ üstteki **+Oluştur** tuşuna tıklanır.
  </Step>
  
  <Step title="API Bilgilerini Doldurun">
    Yeni oluşturulacak *API* için gerekli alanlar doldurularak sağ üstteki **→Kaydet ve Devam** tuşuna tıklanmasıyla *API* oluşturma işi başlamış olur.
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Blank tipinde Mock API Oluşturma">
    Bu tipte API oluşturmak, API endpoint'lerinin kullanıcı tarafından tasarlanacağı anlamına gelmektedir.

    <Info>
      Eğer **Generate CRUD Operations** seçeneği aktifleştirilirse, API'ye GET, POST, PUT, DELETE endpoint'leri otomatik olarak eklenmiş olur.
    </Info>

    **Blank tipinde API oluşturma ayarlarını içeren görsele aşağıda yer verilmiştir:**
  </Accordion>
  
  <Accordion title="Open API veya Swagger'dan Mock API Oluşturma">
    Bu tipte API oluşturmak, API'nın tanım belgesi üzerinden oluşturulacağı anlamına gelmektedir.

    İçe aktarılacak ya da linki verilecek API dokümanın tipi Swagger ya da Open API olmalıdır.

### Mock API Tanım Alanları

Mock API'nın tanımı için kullanılan alanlar aşağıda kategorilere ayrılarak açıklanmıştır:

<CardGroup cols={2}>
  <Card title="Tip (Type)" icon="list">
    Mock API Oluşturma tipidir. Blank ve Spec dosyası yükleyerek yapılır.
  </Card>
  
  <Card title="Ad (Name)" icon="tag">
    API'ye kullanıcı tarafından verilen addır. Arayüzlerde bu ad görünür.
  </Card>
  
  <Card title="Açıklama (Description)" icon="file-lines">
    API'nin tanım bilgisidir.
  </Card>
  
  <Card title="Generate CRUD Operations" icon="gear">
    Mock API için GET, POST, PUT ve DELETE tiplerinde hazır metot/endpoint'ler oluşturulması istenirse seçilir.
    
    <Tip>
      Bu seçenek aktifleştirildiğinde varsayılan CRUD endpoint'leri otomatik olarak oluşturulur.
    </Tip>
  </Card>
  
  <Card title="API Spec Tipi" icon="file-code">
    **Alan:** API Spec Tipi (API Spec Type)
    
    Yüklenecek Spec dosyasının tür bilgisidir. Desteklenen türler: OpenAPI 3.0.x, Swagger 2.x
  </Card>
  
  <Card title="İçe Aktarma Tipi" icon="upload">
    **Alan:** İçe Aktarma Tipi (Import Type)
    
    Yüklenecek Spec dosyasının tür bilgisidir. Desteklenen tipler: URL, Upload File
  </Card>
  
  <Card title="URL" icon="link">
    Yüklenecek Spec dosyasının adresi.
  </Card>
</CardGroup>

## Endpoint Oluşturma

<Info>
  Eğer *Generate CRUD Operations* seçeneği seçilmişse varsayılan olarak GET, POST, PUT, DELETE tiplerinde metot/endpoint'ler hazır gelecektir.
</Info>

<Steps>
  <Step title="Endpoint Ekle">
    **Ekle (Add)** butonuna tıklayarak yeni metot/endpoint eklenir.
  </Step>
  
  <Step title="Endpoint Bilgilerini Doldurun">
    Gelen ekranda **Ekle** tuşuna tıklayarak API'ye metot/endpoint eklenir. *Method/Endpoint* tanımlama ekranındaki alanlar doldurularak **Kaydet (Save)** tuşuna tıklanır.
  </Step>
</Steps>

### Yanıt Ekleme ya da Düzenleme

Yanıt Bilgisi (Response Information) bölümünden metot/endpoint'e ait yanıtlar yönetilir.

<Info>
  Yanıt tanımları, Durum Kodu (Status Code) bilgisine göre gruplandırılarak oluşturulur. Her yanıt tanımı içinde koşul bazlı birden fazla örnek yanıt oluşturulabilir.
</Info>

**Yanıt tanım ayarlarını içeren dialoga ait görsele aşağıda yer verilmiştir:**

Yanıt ekleme/düzenleme konfigürasyonu için kullanılan alanlar:

<Card title="Durum Kodu" icon="code">
  **Alan:** Durum Kodu (Status Code)
  
  HTTP durum kodu bilgisidir.
</Card>
