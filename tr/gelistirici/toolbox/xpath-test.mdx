## XPath Nedir?

XPath (XML Path Language), XML belgelerinde veri seçmek ve çıkarmak için kullanılan bir sorgu dilidir. XPath expression'ları, XML yapısında belirli bir yolu tanımlar ve o yoldaki verileri döndürür.

### XPath Syntax Temelleri

XPath expression'ları şu temel yapılardan oluşur:

* **`/`**: Root'tan başlayan mutlak yol
* **`//`**: Herhangi bir seviyede arama
* **`.`**: Mevcut node
* **`..`**: Parent node
* **`@`**: Attribute
* **`[]`**: Predicate (filtre)
* **`*`**: Wildcard

## XPath Test Kullanımı

### Arayüz Bileşenleri

XPath Test aracı üç ana bölümden oluşur:

1. **Input XML**: Üzerinde XPath expression'ı çalıştırılacak XML verisi
2. **XPath Expression**: Test edilecek XPath expression'ı
3. **Result**: Expression'ın döndürdüğü sonuç

### Adım 1: Input XML Girişi

Input XML bölümüne test edilecek XML verisini girin:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<bookstore>
  <book category="reference">
    <title lang="en">XML Guide</title>
    <author>John Doe</author>
    <price>29.99</price>
  </book>
  <book category="fiction">
    <title lang="en">Novel Story</title>
    <author>Jane Smith</author>
    <price>39.99</price>
  </book>
</bookstore>
```

#### Input Seçenekleri

* **Manuel Giriş**: XML'i doğrudan metin alanına yazın
* **Dosya Yükleme**: XML dosyasını yükleyin
* **Örnek Yükleme**: Hazır örneklerden birini seçin
* **Format**: XML formatını otomatik olarak düzenleme

### Adım 2: XPath Expression Yazma

XPath Expression bölümüne test edilecek expression'ı yazın:

```
/bookstore/book[@category='reference']/title
```

#### Expression Özellikleri

* **Syntax Highlighting**: XPath expression için syntax highlighting desteği
* **Auto-completion**: Otomatik tamamlama önerileri
* **Error Detection**: Syntax hatalarını anında tespit etme
* **Syntax Help**: XPath syntax yardımı

### Adım 3: Sonucu Görüntüleme

Result bölümünde expression'ın döndürdüğü sonuç görüntülenir:

```xml
<title lang="en">XML Guide</title>
```

#### Sonuç Özellikleri

* **XML Format**: Sonuçlar XML formatında gösterilir
* **Highlighting**: Eşleşen node'lar vurgulanır
* **Copy**: Sonucu kopyalama
* **Export**: Sonucu dosya olarak export etme

## Namespace-Aware Queries

XPath Test aracı, namespace-aware sorgular yazmanızı sağlar:

### Namespace Tanımlama

Input XML'de namespace tanımlayabilirsiniz:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<books xmlns="http://example.com/books"
       xmlns:author="http://example.com/author">
  <book id="1">
    <title>XML Guide</title>
    <author:name>John Doe</author:name>
    <price>29.99</price>
  </book>
</books>
```

### Namespace ile XPath Kullanımı

XPath expression'ında namespace'leri kullanabilirsiniz:

```
//*[local-name()='book']/author:name
```

veya namespace prefix ile:

```
//book:book/author:name
```

### Namespace Özellikleri

* **Namespace Prefix Mapping**: Prefix'leri namespace'lere eşleme
* **Default Namespace**: Varsayılan namespace desteği
* **Local Name**: Local name kullanımı
* **Namespace URI**: Namespace URI kullanımı

## XPath Expression Örnekleri

### Temel Expression'lar

#### Root'tan Başlayan Yol

```
/bookstore/book
```

Root'tan başlayarak `bookstore` altındaki tüm `book` elementlerini döndürür.

#### Herhangi Bir Seviyede Arama

```
//book
```

Herhangi bir seviyede bulunan tüm `book` elementlerini döndürür.

#### Attribute Seçimi

```
//book[@category='reference']
```

`category` attribute'u `reference` olan tüm `book` elementlerini döndürür.

#### Child Element Seçimi

```
//book/title
```

Tüm `book` elementlerinin `title` child elementlerini döndürür.

### Gelişmiş Expression'lar

#### Predicate Kullanımı

```
//book[price > 30]
```

Fiyatı 30'dan büyük kitapları döndürür.

#### Multiple Conditions

```
//book[@category='fiction' and price < 40]
```

Kategorisi `fiction` ve fiyatı 40'tan küçük kitapları döndürür.

#### Position-Based Selection

```
//book[1]
```

İlk kitabı döndürür.

```
//book[last()]
```

Son kitabı döndürür.

#### Text Content

```
//book[title='XML Guide']
```

Başlığı `XML Guide` olan kitabı döndürür.

## Result Extraction

XPath Test aracı, sonuçları çıkarmak için çeşitli özellikler sunar:

### Extraction Seçenekleri

* **Node Set**: Tüm eşleşen node'ları döndürür
* **String Value**: Node'un string değerini döndürür
* **Number Value**: Node'un sayısal değerini döndürür
* **Boolean Value**: Node'un boolean değerini döndürür

### Extraction Örnekleri

**Expression**: `//book/title/text()`

**Result**: 
```
XML Guide
Novel Story
```

**Expression**: `sum(//book/price)`

**Result**: 
```
69.98
```

**Expression**: `count(//book)`

**Result**: 
```
2
```

## XPath Syntax Referansı

### XPath Axes

* **`child`**: Child node'lar
* **`parent`**: Parent node
* **`ancestor`**: Tüm ancestor'lar
* **`descendant`**: Tüm descendant'lar
* **`following`**: Sonraki node'lar
* **`preceding`**: Önceki node'lar
* **`following-sibling`**: Sonraki sibling'ler
* **`preceding-sibling`**: Önceki sibling'ler

### XPath Functions

#### String Functions

* **`string()`**: Node'u string'e dönüştürür
* **`concat()`**: String'leri birleştirir
* **`substring()`**: Substring alır
* **`contains()`**: İçerik kontrolü yapar
* **`starts-with()`**: Başlangıç kontrolü yapar
* **`ends-with()`**: Bitiş kontrolü yapar

#### Number Functions

* **`number()`**: Node'u sayıya dönüştürür
* **`sum()`**: Toplam hesaplar
* **`round()`**: Yuvarlar
* **`floor()`**: Aşağı yuvarlar
* **`ceiling()`**: Yukarı yuvarlar

#### Boolean Functions

* **`boolean()`**: Node'u boolean'a dönüştürür
* **`not()`**: Mantıksal NOT
* **`true()`**: True döndürür
* **`false()`**: False döndürür

#### Node Functions

* **`count()`**: Node sayısını döndürür
* **`last()`**: Son node'un pozisyonunu döndürür
* **`position()`**: Mevcut pozisyonu döndürür
* **`name()`**: Node'un adını döndürür
* **`local-name()`**: Local name'i döndürür

## Kullanım Senaryoları

### Senaryo 1: Belirli Attribute'a Sahip Elementleri Bulma

**Input XML:**
```xml
<products>
  <product id="1" status="active">Product A</product>
  <product id="2" status="inactive">Product B</product>
</products>
```

**Expression**: `//product[@status='active']`

**Result**: 
```xml
<product id="1" status="active">Product A</product>
```

### Senaryo 2: Text Content'e Göre Filtreleme

**Input XML:**
```xml
<users>
  <user>John Doe</user>
  <user>Jane Smith</user>
</users>
```

**Expression**: `//user[contains(text(), 'John')]`

**Result**: 
```xml
<user>John Doe</user>
```

### Senaryo 3: Nested Element Erişimi

**Input XML:**
```xml
<company>
  <department>
    <name>IT</name>
    <employees>
      <employee>John</employee>
      <employee>Jane</employee>
    </employees>
  </department>
</company>
```

**Expression**: `//department[name='IT']/employees/employee`

**Result**: 
```xml
<employee>John</employee>
<employee>Jane</employee>
```
