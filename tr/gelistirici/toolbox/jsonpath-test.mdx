## JSONPath Nedir?

JSONPath, JSON verilerinden veri çıkarmak için kullanılan bir sorgu dilidir. XPath'in JSON için karşılığıdır. JSONPath expression'ları, JSON yapısında belirli bir yolu tanımlar ve o yoldaki verileri döndürür.

### JSONPath Syntax Temelleri

JSONPath expression'ları şu temel yapılardan oluşur:

* **`$`**: Root element
* **`.`**: Child operator
* **`..`**: Recursive descent
* **`*`**: Wildcard
* **`[]`**: Array index veya filter
* **`[start:end:step]`**: Array slice

## JSONPath Test Kullanımı

### Arayüz Bileşenleri

JSONPath Test aracı üç ana bölümden oluşur:

1. **Input JSON**: Üzerinde JSONPath expression'ı çalıştırılacak JSON verisi
2. **JSONPath Expression**: Test edilecek JSONPath expression'ı
3. **Result**: Expression'ın döndürdüğü sonuç

### Adım 1: Input JSON Girişi

Input JSON bölümüne test edilecek JSON verisini girin:

```json
{
  "store": {
    "book": [
      {
        "category": "reference",
        "author": "Nigel Rees",
        "title": "Sayings of the Century",
        "price": 8.95
      },
      {
        "category": "fiction",
        "author": "Evelyn Waugh",
        "title": "Sword of Honour",
        "price": 12.99
      }
    ],
    "bicycle": {
      "color": "red",
      "price": 19.95
    }
  }
}
```

#### Input Seçenekleri

* **Manuel Giriş**: JSON'u doğrudan metin alanına yazın
* **Dosya Yükleme**: JSON dosyasını yükleyin
* **Örnek Yükleme**: Hazır örneklerden birini seçin
* **Format**: JSON formatını otomatik olarak düzenleme

### Adım 2: JSONPath Expression Yazma

JSONPath Expression bölümüne test edilecek expression'ı yazın:

```
$.store.book[*].author
```

#### Expression Özellikleri

* **Syntax Highlighting**: JSONPath expression için syntax highlighting desteği
* **Auto-completion**: Otomatik tamamlama önerileri
* **Error Detection**: Syntax hatalarını anında tespit etme
* **Syntax Help**: JSONPath syntax yardımı

### Adım 3: Sonucu Görüntüleme

Result bölümünde expression'ın döndürdüğü sonuç görüntülenir:

```json
[
  "Nigel Rees",
  "Evelyn Waugh"
]
```

#### Sonuç Özellikleri

* **Highlighting**: Sonuçların vurgulanması
* **Formatting**: JSON formatında güzel görünüm
* **Copy**: Sonucu kopyalama
* **Export**: Sonucu dosya olarak export etme

## Birden Fazla Expression Test Etme

JSONPath Test aracı, birden fazla expression'ı aynı anda test etmenizi sağlar:

### Multiple Expression Girişi

Her satıra bir expression yazın:

```
$.store.book[*].author
$.store.book[*].title
$.store.bicycle.color
$.store.book[?(@.price < 10)]
```

### Sonuç Görüntüleme

Her expression için ayrı sonuç gösterilir:

* **Expression 1**: `$.store.book[*].author`
  * Sonuç: `["Nigel Rees", "Evelyn Waugh"]`
* **Expression 2**: `$.store.book[*].title`
  * Sonuç: `["Sayings of the Century", "Sword of Honour"]`
* **Expression 3**: `$.store.bicycle.color`
  * Sonuç: `"red"`
* **Expression 4**: `$.store.book[?(@.price < 10)]`
  * Sonuç: `[{"category": "reference", ...}]`

## JSONPath Expression Örnekleri

### Temel Expression'lar

#### Root Element

```
$
```

Tüm JSON dokümanını döndürür.

#### Child Access

```
$.store
```

`store` objesini döndürür.

#### Array Access

```
$.store.book[0]
```

İlk kitabı döndürür.

#### Wildcard

```
$.store.book[*].author
```

Tüm kitapların yazarlarını döndürür.

### Gelişmiş Expression'lar

#### Recursive Descent

```
$..author
```

Tüm seviyelerdeki `author` alanlarını döndürür.

#### Array Slice

```
$.store.book[0:2]
```

İlk iki kitabı döndürür.

#### Filter Expression

```
$.store.book[?(@.price < 10)]
```

Fiyatı 10'dan küçük kitapları döndürür.

#### Multiple Conditions

```
$.store.book[?(@.price < 10 && @.category == 'reference')]
```

Fiyatı 10'dan küçük ve kategorisi 'reference' olan kitapları döndürür.

#### Nested Access

```
$.store.book[*].author
```

Tüm kitapların yazarlarını döndürür.

## JSONPath Syntax Yardımı

JSONPath Test aracı, JSONPath syntax'ı hakkında yardım sağlar:

### Syntax Referansı

#### Operatörler

* **`$`**: Root element
* **`.`**: Child operator
* **`..`**: Recursive descent
* **`*`**: Wildcard
* **`[]`**: Array index veya filter

#### Array İşlemleri

* **`[0]`**: İlk eleman
* **`[-1]`**: Son eleman
* **`[0:2]`**: İlk iki eleman
* **`[*]`**: Tüm elemanlar

#### Filter Expression'ları

* **`[?(@.field == 'value')]`**: Eşitlik kontrolü
* **`[?(@.field < 10)]`**: Karşılaştırma
* **`[?(@.field && @.other)]`**: Mantıksal operatörler

### Syntax Yardımı Kullanımı

1. **Syntax Help Butonu**: Syntax yardımı butonuna tıklayın
2. **Örnekler**: Hazır örnekleri inceleyin
3. **Referans**: Syntax referansını görüntüleyin

## Kullanım Senaryoları

### Senaryo 1: Belirli Alanları Çıkarma

**Input JSON:**
```json
{
  "users": [
    {"id": 1, "name": "John", "email": "john@example.com"},
    {"id": 2, "name": "Jane", "email": "jane@example.com"}
  ]
}
```

**Expression**: `$.users[*].email`

**Result**: `["john@example.com", "jane@example.com"]`

### Senaryo 2: Filtreleme

**Input JSON:**
```json
{
  "products": [
    {"id": 1, "name": "Product A", "price": 100, "stock": 10},
    {"id": 2, "name": "Product B", "price": 200, "stock": 0}
  ]
}
```

**Expression**: `$.products[?(@.stock > 0)]`

**Result**: `[{"id": 1, "name": "Product A", "price": 100, "stock": 10}]`

### Senaryo 3: Nested Data Access

**Input JSON:**
```json
{
  "company": {
    "departments": [
      {
        "name": "IT",
        "employees": [
          {"name": "John", "role": "Developer"},
          {"name": "Jane", "role": "Manager"}
        ]
      }
    ]
  }
}
```

**Expression**: `$..employees[*].name`

**Result**: `["John", "Jane"]`
