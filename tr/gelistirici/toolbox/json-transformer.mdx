## Jolt Nedir?

Jolt (JSON to JSON Transformation Language), JSON verilerini dönüştürmek için kullanılan bir dildir. Jolt spec'leri, kaynak JSON yapısını hedef JSON yapısına dönüştürmek için kullanılan kuralları tanımlar.

### Jolt Operasyonları

Jolt, çeşitli dönüşüm operasyonları sağlar:

* **shift**: Veri taşıma ve yeniden yapılandırma
* **default**: Varsayılan değer atama
* **remove**: Alan kaldırma
* **sort**: Sıralama
* **cardinality**: Dizi eleman sayısı kontrolü
* **modify**: Değer değiştirme

## JSON Transformer Kullanımı

### Arayüz Bileşenleri

JSON Transformer aracı üç ana bölümden oluşur:

1. **Input JSON**: Dönüştürülecek kaynak JSON verisi
2. **Jolt Spec**: Dönüşüm kurallarını içeren Jolt spec
3. **Output JSON**: Dönüşüm sonucu

### Adım 1: Input JSON Girişi

Input JSON bölümüne dönüştürülecek kaynak JSON verisini girin:

```json
{
  "id": 12345,
  "name": "John Doe",
  "email": "john.doe@example.com",
  "address": {
    "street": "123 Main St",
    "city": "New York",
    "zip": "10001"
  }
}
```

#### Input Seçenekleri

* **Manuel Giriş**: JSON'u doğrudan metin alanına yazın
* **Dosya Yükleme**: JSON dosyasını yükleyin
* **Örnek Yükleme**: Hazır örneklerden birini seçin

### Adım 2: Jolt Spec Yazma

Jolt Spec bölümüne dönüşüm kurallarını yazın:

```json
[
  {
    "operation": "shift",
    "spec": {
      "id": "userId",
      "name": "fullName",
      "email": "contact.email",
      "address": {
        "*": "contact.address.&"
      }
    }
  }
]
```

#### Jolt Spec Özellikleri

* **Syntax Highlighting**: Jolt spec için syntax highlighting desteği
* **Auto-completion**: Otomatik tamamlama önerileri
* **Error Detection**: Syntax hatalarını anında tespit etme

### Adım 3: Dönüşüm Sonucunu Görüntüleme

Output JSON bölümünde dönüşüm sonucu görüntülenir:

```json
{
  "userId": 12345,
  "fullName": "John Doe",
  "contact": {
    "email": "john.doe@example.com",
    "address": {
      "street": "123 Main St",
      "city": "New York",
      "zip": "10001"
    }
  }
}
```

## Syntax Validation

JSON Transformer aracı, hem Input JSON hem de Jolt Spec için syntax validation yapar:

### Input JSON Validation

* **JSON Format Kontrolü**: Geçerli JSON formatı kontrolü
* **Syntax Hataları**: Syntax hatalarını vurgulama
* **Hata Mesajları**: Detaylı hata mesajları

### Jolt Spec Validation

* **Jolt Syntax Kontrolü**: Jolt spec syntax kontrolü
* **Operasyon Doğrulama**: Geçerli operasyon kontrolü
* **Spec Yapısı Kontrolü**: Spec yapısının doğruluğu kontrolü

### Hata Gösterimi

Syntax hataları şu şekilde gösterilir:

* **Satır Numarası**: Hatanın bulunduğu satır
* **Hata Mesajı**: Açıklayıcı hata mesajı
* **Vurgulama**: Hatalı bölümün vurgulanması

## Kullanım Senaryoları

### Senaryo 1: Basit Alan Dönüşümü

**Input:**
```json
{
  "firstName": "John",
  "lastName": "Doe"
}
```

**Jolt Spec:**
```json
[
  {
    "operation": "shift",
    "spec": {
      "firstName": "name.first",
      "lastName": "name.last"
    }
  }
]
```

**Output:**
```json
{
  "name": {
    "first": "John",
    "last": "Doe"
  }
}
```

### Senaryo 2: Dizi İşleme

**Input:**
```json
{
  "items": [
    {"id": 1, "name": "Item 1"},
    {"id": 2, "name": "Item 2"}
  ]
}
```

**Jolt Spec:**
```json
[
  {
    "operation": "shift",
    "spec": {
      "items": {
        "*": {
          "id": "products[&1].productId",
          "name": "products[&1].productName"
        }
      }
    }
  }
]
```

**Output:**
```json
{
  "products": [
    {
      "productId": 1,
      "productName": "Item 1"
    },
    {
      "productId": 2,
      "productName": "Item 2"
    }
  ]
}
```

### Senaryo 3: Varsayılan Değer Atama

**Input:**
```json
{
  "status": "active"
}
```

**Jolt Spec:**
```json
[
  {
    "operation": "default",
    "spec": {
      "enabled": true,
      "priority": 0
    }
  },
  {
    "operation": "shift",
    "spec": {
      "status": "currentStatus",
      "enabled": "isEnabled",
      "priority": "priorityLevel"
    }
  }
]
```

**Output:**
```json
{
  "currentStatus": "active",
  "isEnabled": true,
  "priorityLevel": 0
}
```

* **Adım Adım Kontrol**: Her operasyonun sonucunu kontrol edin

## JSON Transformer ve Policy Kullanımı

JSON Transformer aracında test ettiğiniz Jolt spec'leri, Apinizer'da JSON Transformation Policy'de kullanılabilir:

1. JSON Transformer'da spec'inizi test edin
2. Test edilmiş spec'i kopyalayın
3. JSON Transformation Policy'de spec'i yapıştırın
4. Policy'yi API Proxy'nize ekleyin
