## API Proxy Listeleme ve Arama

### API Proxy Listesi

API Proxy listesi sayfasında, tüm API Proxy'lerinizi görüntüleyebilirsiniz. Liste görünümü aşağıdaki bilgileri içerir:

* **API Proxy Adı**: API Proxy'nin adı
* **Durum**: Deployment durumu (Deployed, Undeployed)
* **Versiyon**: Mevcut versiyon bilgisi
* **Son Değişiklik**: Son güncelleme tarihi
* **Tags**: API Proxy'ye atanan tag'ler
* **Group**: İlişkili API Proxy Group

### Arama ve Filtreleme

API Proxy listesi gelişmiş arama ve filtreleme özellikleri içerir:

#### Metin Arama

* **İsme Göre Arama**: API Proxy adına göre arama
* **Açıklamaya Göre Arama**: API Proxy açıklamaları içinde arama
* **Tam Metin Arama**: Tüm alanlarda arama

#### Filtreleme Seçenekleri

<AccordionGroup>
  <Accordion title="Durum Filtresi">
    * **Tümü**: Tüm API Proxy'ler
    * **Deployed**: Deploy edilmiş API Proxy'ler
    * **Undeployed**: Deploy edilmemiş API Proxy'ler
  </Accordion>
  
  <Accordion title="Tip Filtresi">
    * **REST API Proxy**
    * **SOAP API Proxy**
    * **gRPC API Proxy**
    * **WebSocket API Proxy**
    * **DB-2-API**
    * **Script-2-API**
    * **Mock API**
    * **Connector API Proxy**
  </Accordion>
  
  <Accordion title="Tag Filtresi">
    Tag'lere göre filtreleme. Kesişim kümesini görüntülemek için birden fazla tag seçebilirsiniz.
  </Accordion>
  
  <Accordion title="Group Filtresi">
    Belirli bir API Proxy Group'a ait API Proxy'leri filtreleme.
  </Accordion>
  
  <Accordion title="Tarih Filtresi">
    * **Son 7 Gün**: Son 7 gün içinde değiştirilenler
    * **Son 30 Gün**: Son 30 gün içinde değiştirilenler
    * **Özel Tarih Aralığı**: Belirli bir tarih aralığı seçme
  </Accordion>
</AccordionGroup>

### Sıralama

API Proxy listesini aşağıdaki kriterlere göre sıralayabilirsiniz:

* **İsme Göre**: Alfabetik sıralama (A-Z, Z-A)
* **Tarihe Göre**: Son değişiklik tarihine göre (Yeni-Eski, Eski-Yeni)
* **Duruma Göre**: Deployment durumuna göre
* **Versiyona Göre**: Versiyon numarasına göre

## API Proxy Groups

API Proxy Groups, ilgili API Proxy'leri mantıksal olarak gruplamanıza olanak tanır. Group bazlı yönetim şunları yapmanızı sağlar:

* Group seviyesinde ortak politikalar tanımlama
* Group üyelerine toplu işlemler uygulama
* Group bazlı erişim kontrolü gerçekleştirme
* Group istatistiklerini görüntüleme

### Group Görünümü

API Proxy listesinde group görünümünü etkinleştirerek API Proxy'leri gruplarına göre organize edebilirsiniz:

* **Group Bazlı Görünüm**: API Proxy'leri gruplarına göre görüntüleme
* **Group Detayları**: Group bilgilerini ve üyelerini görüntüleme
* **Group İstatistikleri**: Group bazlı kullanım istatistiklerini görüntüleme

:::info Daha Fazla Bilgi
API Proxy Group oluşturma ve yönetimi hakkında detaylı bilgi için [API Proxy Group Oluşturma](/tr/gelistirici/api-proxy-group-creation) sayfasına bakın.
:::

## Tagging ve Kategorizasyon

Tag'ler, API Proxy'lerinizi kategorize etmenin esnek bir yoludur. Bir API Proxy'ye birden fazla tag atayabilirsiniz.

### Tag Oluşturma ve Yönetimi

#### Tag Ekleme

1. API Proxy detay sayfasına gidin
2. **Tags** bölümüne tıklayın
3. Mevcut tag'lerden seçin veya yeni tag oluşturun
4. Tag'i kaydedin

#### Tag Önerileri

Tag'leme için önerilen kategoriler:

  <Card title="Fonksiyonel Kategoriler" icon="puzzle-piece">
    * authentication
    * payment
    * reporting
    * integration
    * notification
  </Card>
  <Card title="Müşteri Kategorileri" icon="building">
    * customer-a
    * customer-b
    * internal
    * external
  </Card>
  <Card title="Versiyon Kategorileri" icon="code-branch">
    * v1
    * v2
    * legacy
    * beta
  </Card>
  <Card title="Ortam Kategorileri" icon="server">
    * production

    * development
    * testing
  </Card>
</CardGroup>

### Tag Bazlı Filtreleme

Tag'lere göre filtreleme yaparak şunları yapabilirsiniz:

* Belirli bir kategoriye ait API Proxy'leri görüntüleme
* Birden fazla tag kombinasyonu kullanma
* Tag bazlı raporlar oluşturma

## Toplu İşlemler

Birden fazla API Proxy üzerinde aynı anda işlem gerçekleştirmek için toplu işlem özelliklerini kullanabilirsiniz.

### Toplu Seçim

API Proxy listesinde:

1. **Çoklu Seçim Modu**: Listede çoklu seçim modunu etkinleştirin
2. **Seçim**: İstenen API Proxy'leri seçin
3. **Toplu İşlem Menüsü**: Seçilen API Proxy'ler için toplu işlem menüsü görünür

### Desteklenen Toplu İşlemler

<AccordionGroup>
  <Accordion title="Toplu Deploy/Undeploy">
    Seçilen API Proxy'leri aynı anda deploy veya undeploy edebilirsiniz.
    
    :::warning Uyarı
    Toplu deploy işlemi, seçilen tüm API Proxy'leri hedef ortama deploy eder. Bu işlem geri alınamaz.
    :::
  </Accordion>
  
  <Accordion title="Toplu Tag Ekleme/Çıkarma">
    Seçilen API Proxy'lerden tag'leri aynı anda ekleyebilir veya çıkarabilirsiniz.
  </Accordion>
  
  <Accordion title="Toplu Group Atama">
    Seçilen API Proxy'leri bir API Proxy Group'a aynı anda ekleyebilirsiniz.
  </Accordion>
  
  <Accordion title="Toplu Silme">
    Seçilen API Proxy'leri aynı anda silebilirsiniz.
    
    :::danger Önemli
    Silme işlemi geri alınamaz. Silmeden önce emin olun.
    :::
  </Accordion>
  
  <Accordion title="Toplu Export">
    Seçilen API Proxy'leri aynı anda export edebilirsiniz. Export edilen dosyalar ZIP formatında indirilir.
  </Accordion>
  
  <Accordion title="Toplu ACL Yönetimi">
    Seçilen API Proxy'ler için toplu erişim kontrolü gerçekleştirebilirsiniz.
  </Accordion>
</AccordionGroup>

### Toplu İşlem Senaryoları

#### Senaryo 1: Yeni Ortama Toplu Deploy

1. Production ortamına deploy edilecek API Proxy'leri filtreleyin
2. Çoklu seçim modunu etkinleştirin
3. Deploy edilecek API Proxy'leri seçin
4. **Toplu Deploy** seçeneğini seçin
5. Hedef ortamı seçin ve onaylayın

#### Senaryo 2: Tag Bazlı Gruplama

1. Belirli bir tag'e sahip API Proxy'leri filtreleyin
2. Tümünü seçin
3. **Toplu Group Atama** seçeneğini seçin
4. Hedef group'u seçin ve onaylayın

#### Senaryo 3: Toplu Export ve Yedekleme

1. Yedeklenecek API Proxy'leri seçin
2. **Toplu Export** seçeneğini seçin
3. Export ayarlarını yapılandırın
4. Export işlemini başlatın

## Görünüm Seçenekleri

API Proxy listesi farklı görünüm seçenekleri sunar:

### Liste Görünümü

Klasik liste görünümü. Detaylı bilgileri tablo formatında gösterir.

### Kart Görünümü

API Proxy'leri kart formatında gösterir. Her kart şunları içerir:

* API Proxy adı ve ikonu
* Durum göstergesi
* Tag'ler
* Hızlı erişim butonları

### Kompakt Görünüm

Kompakt görünüm, aynı anda daha fazla API Proxy görüntülemek için kullanılabilir.

## İstatistikler ve Raporlama

API Proxy listesi sayfasında aşağıdaki istatistikler gösterilir:

* **Toplam API Proxy Sayısı**: Tüm API Proxy'lerin toplam sayısı
* **Deployed Sayısı**: Deploy edilmiş API Proxy sayısı
* **Undeployed Sayısı**: Deploy edilmemiş API Proxy sayısı
* **Group Sayısı**: API Proxy Group'ların toplam sayısı
* **Tag Dağılımı**: Tag'lere göre API Proxy dağılımı

## En İyi Uygulamalar

### Tagging Stratejisi

* **Tutarlılık**: Tag isimlerinde tutarlılık sağlayın
* **Açıklayıcılık**: Tag'lerin anlamının net olduğundan emin olun
* **Hiyerarşi**: Hiyerarşik tag yapısı kullanın (örn: `env:production`, `env:staging`)
* **Dokümantasyon**: Tag kullanım kurallarını dokümante edin

### Group Yönetimi

* **Mantıksal Gruplama**: İlgili API Proxy'leri mantıksal olarak gruplayın
* **Group Boyutu**: Group'ları çok büyük yapmayın (önerilen: 5-15 API Proxy)
* **Group İsimlendirme**: Group isimlerini açıklayıcı ve tutarlı tutun

### Toplu İşlemler

* **Test Ortamında Test**: Toplu işlemleri önce test ortamında deneyin
* **Yedekleme**: Önemli değişikliklerden önce yedekleme yapın
* **İşlem Öncesi Kontrol**: Toplu işlemlerden önce seçilen API Proxy'leri kontrol edin
