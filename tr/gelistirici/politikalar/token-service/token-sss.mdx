Bu sayfa, Token Service ile ilgili sık sorulan soruları ve cevaplarını içerir.

## Token Expire Süresi Nasıl Ayarlanır?

Token expire süresi, Token Service yapılandırmasında ayarlanır:

### Access Token Expire Süresi

Access token expire süresi genellikle şu şekilde ayarlanır:

* **Varsayılan**: 3600 saniye (1 saat)
* **Özelleştirilebilir**: Token Service yapılandırmasından değiştirilebilir
* **Güvenlik**: Kısa expire süreleri güvenlik açısından önerilir

### Refresh Token Expire Süresi

Refresh token expire süresi:

* **Varsayılan**: 2592000 saniye (30 gün)
* **Yenilenebilir**: Refresh token kullanılarak yenilenebilir
* **Uzun Ömürlü**: Access token'dan daha uzun süre geçerlidir

### Yapılandırma Örneği

```json
{
  "tokenService": {
    "accessTokenExpire": 3600,
    "refreshTokenExpire": 2592000
  }
}
```

## Token Revoke Nasıl Yapılır?

Token'ları iptal etmek (revoke) için Token Revocation endpoint'i kullanılır:

### Revocation Endpoint

```http
POST /oauth2/revoke HTTP/1.1
Content-Type: application/x-www-form-urlencoded

token=token_to_revoke&
token_type_hint=access_token
```

### Revocation Parametreleri

* **token**: İptal edilecek token
* **token_type_hint**: Token türü (access_token veya refresh_token)

### Revocation Sonrası

Token iptal edildikten sonra:

* **Cache'den Kaldırılır**: Token cache'den kaldırılır
* **Geçersiz Olur**: Token artık geçersizdir
* **Yeni Token Gerekir**: Yeni token alınması gerekir

## Multiple Token Desteği

Token Service, aynı client için birden fazla token desteği sağlar:

### Senaryolar

* **Farklı Scope'lar**: Farklı scope'lara sahip token'lar
* **Farklı Ortamlar**: Farklı ortamlar için token'lar
* **Paralel İstekler**: Paralel işlemler için token'lar

### Token Yönetimi

* **Token Listesi**: Client'a ait tüm token'lar listelenebilir
* **Token Filtreleme**: Scope, expire date gibi kriterlere göre filtreleme
* **Toplu İptal**: Birden fazla token'ı aynı anda iptal etme

## Token Debugging

Token ile ilgili sorunları debug etmek için:

### Token İçeriğini İnceleme

JWT token'ların içeriğini inceleyebilirsiniz:

```javascript
// JWT token decode
const tokenParts = token.split('.');
const payload = JSON.parse(atob(tokenParts[1]));
console.log(payload);
```

### Token Logları

Token işlemleri log sunucusunda kaydedilir:

* **Token Üretimi**: Token üretim logları
* **Token Doğrulama**: Token doğrulama logları
* **Token İptal**: Token iptal logları

### Debug Endpoint'leri

Token Service debug endpoint'leri:

* **Token Info**: Token bilgilerini görüntüleme
* **Token Validation**: Token doğrulama testi
* **Token List**: Client'a ait token listesi

## Sık Karşılaşılan Sorunlar

### Token Expire Hatası

**Sorun**: Token expire oldu hatası alıyorum.

**Çözüm**:
* Refresh token kullanarak yeni token alın
* Token expire süresini kontrol edin
* Token'ı yeniden alın

### Token Doğrulama Hatası

**Sorun**: Token doğrulama başarısız oluyor.

**Çözüm**:
* Token formatını kontrol edin
* Token'ın geçerli olduğundan emin olun
* Token'ın iptal edilmediğini kontrol edin

### Refresh Token Hatası

**Sorun**: Refresh token çalışmıyor.

**Çözüm**:
* Refresh token'ın geçerli olduğundan emin olun
* Refresh token expire süresini kontrol edin
* Client credentials'ları kontrol edin

## Token Güvenliği

### Güvenlik Önerileri

* **HTTPS Kullanın**: Token isteklerini HTTPS üzerinden yapın
* **Güvenli Saklama**: Token'ları güvenli saklayın
* **Kısa Expire Süreleri**: Kısa expire süreleri kullanın
* **Token Rotation**: Token rotation uygulayın

### Token Maskeleme

Token'lar log'larda maskeleme yapılmalıdır:

* **Log Maskeleme**: Token'lar log'larda maskelenir
* **Response Maskeleme**: Response'larda token'lar maskelenir
* **Debug Maskeleme**: Debug çıktılarında token'lar maskelenir
