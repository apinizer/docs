# XML Dönüştürme (XML Transformation) - XSLT

İstemciden gelen veya istemciye dönecek olan XML verisini farklı formata dönüştürmek için kullanılır.

Bu politika istek hattında (İstemci → API Proxy → API) kullanılmış ise istemciden gelen mesaj gövdesi üzerinde dönüşüm uygulanarak Backend API'ye iletilir.

Eğer yanıt hattında (API → API Proxy → İstemci) kullanılmış ise Backend API'den dönen mesaj gövdesi üzerinde dönüşüm uygulanarak sonuç istemciye iletilir.

Bu politikanın kullanımı istemciden gelen mesaj gövdesi XML ise anlamlıdır.

Apinizer XML dönüşüm işlemleri için XSLT 1.0 ve XSLT 2.0 dillerini destekler.

**Politika ayarlarını içeren görsele aşağıda yer verilmiştir:**

![XML Dönüştürme Politikası](https://docs.apinizer.com/apinizer-dokumantasyonu/files/1180216/120521276/1/1733914323255/image-2024-12-11_13-52-2.png)

Politika konfigürasyonu için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan | Açıklama |
| --- | --- |
| İsim *(Name)* | Politikanın kullanımını ve yönetimini kolaylaştırmak için isim yazılabilir. Politikaların yönetimi ve seçimi sırasında bu isme ihtiyacınız olacaktır. |
| Açıklama *(Description)* | Politikanın kullanımını ve yönetimini kolaylaştırmak için açıklama yazılabilir. |
| Mesaj Tipi *(Message Type)* | Mesaj tipi **XML to XML** veya **XML to JSON** olarak seçilebilir. |
| XSLT Değeri *(XSLT Value)* | Dönüşüm için gerekli olan XSLT scripti girilir. (XSLT kullanım örnekleri için [XSLT Örnekleri](/tr/tutorials/nasil-yapilir-kilavuzlari/gelistirici-kilavuzlari/xslt-kullanimi) sayfasını ziyaret edebilirsiniz.) |

Mesaj dönüşümü için yazılan XSLT scriptlerinin nasıl test edilebileceği ile ilgili detaylı bilgi için [XML Dönüştürücü](/tr/gelistirici/test-debug-araclari/xml-transformer) sayfasını ziyaret edebilirsiniz.

**Koşullar (Conditions)** ve **Hata Mesajı Özelleştirme (Error Message Customization)** panellerinin detayı için [Politikalar (Policies)](/tr/gelistirici/politika-yonetimi) sayfasını ziyaret edebilirsiniz.
