# Çoklu API Yönetimi

API ekosisteminiz büyüdükçe, birden fazla API Proxy'yi etkili bir şekilde yönetmek zorunlu hale gelir. Apinizer, API Proxy'lerinizi ölçekte organize etmek, aramak ve yönetmek için kapsamlı araçlar sunar.

## API Proxy Listeleme ve Arama

API Proxy liste görünümü, tüm API Proxy'lerinizi görüntülemek ve yönetmek için merkezi bir arayüz sağlar.

### API Proxy Listesini Görüntüleme

API Proxy listesine erişmek için ana menüden **Geliştirme → API Proxy'ler** seçeneğine gidin. Liste şunları gösterir:

- **Proxy Adı**: Her API Proxy için benzersiz tanımlayıcı
- **Tip**: API Proxy tipi (REST, SOAP, WebSocket, gRPC, vb.)
- **Durum**: Deployment durumu (Deployed, Not Deployed)
- **Ortam**: Proxy'nin deploy edildiği ortam
- **Son Değişiklik**: Son değişiklik zaman damgası
- **İşlemler**: Ortak işlemler için hızlı işlem butonları

### API Proxy Arama

Belirli API Proxy'leri hızlıca bulmak için arama işlevini kullanın:

- **İsim Arama**: API Proxy adına göre arama
- **Tip Filtresi**: API Proxy tipine göre filtreleme
- **Durum Filtresi**: Deployment durumuna göre filtreleme
- **Ortam Filtresi**: Deployment ortamına göre filtreleme
- **Etiket Arama**: Proxy'lere atanmış etiketlere göre arama

Arama, yazdıkça gerçek zamanlı olarak gerçekleştirilir, büyük listelerde belirli proxy'leri bulmayı kolaylaştırır.

### Sıralama ve Filtreleme

API Proxy listesi birden fazla sıralama ve filtreleme seçeneğini destekler:

- **Sıralama**: İsim, Tip, Durum, Son Değişiklik, Ortam
- **Filtreleme**: Tip, Durum, Ortam, Etiketler
- **Görünüm Seçenekleri**: Liste görünümü, Grid görünümü, Kompakt görünüm

## API Proxy Grupları

API Proxy Grupları, ilgili API Proxy'leri daha kolay yönetim ve paylaşılan konfigürasyon için birlikte organize etmenizi sağlar.

### API Proxy Gruplarının Avantajları

- **Merkezi Yönetim**: Birden fazla proxy'yi tek bir arayüzden yönetin
- **Paylaşılan Politikalar**: Gruptaki tüm proxy'lere ortak politikalar uygulayın
- **Birleşik Yönlendirme**: İstekleri tek bir endpoint üzerinden birden fazla proxy'ye yönlendirin
- **Basitleştirilmiş Deployment**: Birden fazla proxy'yi birlikte deploy edin

### Grup Oluşturma ve Yönetimi

1. **Geliştirme → API Proxy Grupları** menüsüne gidin
2. Yeni bir grup oluşturmak için **+Create** butonuna tıklayın
3. Gruba API Proxy'leri ekleyin
4. Grup seviyesi politikalar ve ayarları yapılandırın

Detaylı bilgi için [API Proxy Grupları](/en/developer/api-proxy-group-creation) sayfasına bakın.

### Grup Üyeliği

- Bir API Proxy birden fazla gruba ait olabilir
- Grup üyeliği, bireysel proxy konfigürasyonlarını etkilemez
- Grup seviyesi politikalar, proxy seviyesi politikalara ek olarak uygulanır

## Etiketleme ve Kategorizasyon

Etiketler, geleneksel gruplamaların ötesinde API Proxy'lerinizi kategorize etmek ve organize etmek için esnek bir yol sağlar.

### Etiket Kullanımı

Etiketler şunları yapmanıza olanak tanır:

- **Fonksiyona Göre Kategorize Etme**: Proxy'leri iş fonksiyonlarına göre etiketleme (örn: `payment`, `authentication`, `inventory`)
- **Takıma Göre Organize Etme**: Proxy'leri sorumlu takıma göre etiketleme (örn: `backend-team`, `frontend-team`)
- **Duruma Göre İşaretleme**: Proxy'leri geliştirme durumuna göre etiketleme (örn: `deprecated`, `beta`, `production-ready`)
- **Domain'e Göre Gruplama**: Proxy'leri iş domain'ine göre etiketleme (örn: `e-commerce`, `analytics`, `reporting`)

### Etiket Ekleme

1. Listeden bir API Proxy seçin
2. **Overview** sekmesine gidin
3. **Etiketler** bölümünü bulun
4. Etiket yazarak Enter'a basarak ekleyin
5. Etiketler otomatik olarak kaydedilir

### Etiket Tabanlı Filtreleme

API Proxy listenizi filtrelemek için etiketleri kullanın:

- Etiket bulutunda bir etikete tıklayarak o etikete sahip proxy'leri filtreleyin
- Daha spesifik filtreleme için birden fazla etiketi birleştirin
- Belirli etiket kombinasyonlarına sahip proxy'leri bulmak için etiket araması kullanın

### Etiketleme En İyi Uygulamaları

- **Tutarlı İsimlendirme Kullanın**: Bir etiketleme kuralı oluşturun ve buna bağlı kalın
- **Etiket Sayısını Sınırlayın**: Daha iyi organizasyon için proxy başına 3-5 etiket kullanın
- **Hiyerarşik Etiketler Kullanın**: Hiyerarşik etiketler için nokta notasyonu kullanın (örn: `payment.credit-card`, `payment.wallet`)
- **Etiket Anlamlarını Dokümante Edin**: Ekibiniz için bir etiket sözlüğü tutun

## Toplu İşlemler

Toplu işlemler, birden fazla API Proxy üzerinde aynı anda işlem yapmanıza olanak tanır, zaman kazandırır ve tutarlılık sağlar.

### Birden Fazla Proxy Seçme

- **Tekil Seçim**: Her proxy'nin yanındaki checkbox'ı tıklayın
- **Tümünü Seç**: Başlıktaki "Tümünü Seç" checkbox'ını kullanın
- **Filtreye Göre Seç**: Filtreleri uygulayın ve ardından filtrelenmiş sonuçların tümünü seçin
- **Etikete Göre Seç**: Etikete göre filtreleyin ve eşleşen tüm proxy'leri seçin

### Mevcut Toplu İşlemler

#### Deployment İşlemleri

- **Seçilenleri Deploy Et**: Birden fazla proxy'yi seçilen bir ortama deploy edin
- **Seçilenleri Undeploy Et**: Birden fazla proxy'yi bir ortamdan kaldırın
- **Seçilenleri Yeniden Deploy Et**: Birden fazla proxy'yi yeniden deploy edin

#### Konfigürasyon İşlemleri

- **Etiket Ekle**: Birden fazla proxy'ye aynı anda etiket ekleyin
- **Etiket Kaldır**: Birden fazla proxy'den etiket kaldırın
- **Ayarları Güncelle**: Birden fazla proxy'ye ortak ayarlar uygulayın
- **Gruba Ata**: Birden fazla proxy'yi bir API Proxy Grubuna ekleyin

#### Yönetim İşlemleri

- **Seçilenleri Export Et**: Yedekleme veya migrasyon için birden fazla proxy'yi export edin
- **Seçilenleri Sil**: Birden fazla proxy'yi silin (onay ile)
- **Seçilenleri Çoğalt**: Birden fazla proxy'nin kopyalarını oluşturun
- **Sahibi Değiştir**: Birden fazla proxy'nin sahipliğini transfer edin

### Toplu İşlem Gerçekleştirme

1. İşlem yapmak istediğiniz proxy'leri seçin
2. Araç çubuğundaki **Toplu İşlemler** butonuna tıklayın
3. Açılır menüden istediğiniz işlemi seçin
4. İşleme özel ayarları yapılandırın
5. Gerçekleştirilecek işlemlerin özetini gözden geçirin
6. İşlemi onaylayın

### Toplu İşlem En İyi Uygulamaları

- **Önce Test Edin**: Önce küçük bir alt kümede top lu işlemleri test edin
- **Silmeden Önce Yedekleyin**: Yıkıcı işlemler yapmadan önce her zaman yedekleyin
- **Özeti Gözden Geçirin**: Onaylamadan önce işlem özetini dikkatlice gözden geçirin
- **Filtreleri Kullanın**: Doğru proxy'ler üzerinde işlem yaptığınızdan emin olmak için filtreleri kullanın
- **Yoğun Olmayan Saatlerde Planlayın**: Toplu deployment'ları yoğun olmayan saatlerde gerçekleştirin

## Gelişmiş Yönetim Özellikleri

### API Proxy Şablonları

Benzer proxy'leri hızlıca oluşturmak için mevcut proxy'lerden şablonlar oluşturun:

1. Şablon olarak kullanılacak bir proxy seçin
2. **Şablon Olarak Kaydet** butonuna tıklayın
3. Şablonu adlandırın ve açıklayın
4. Yeni proxy'ler oluştururken şablonu kullanın

### Import/Export

- **Export**: Yedekleme veya migrasyon için proxy'leri tek tek veya toplu olarak export edin
- **Import**: Export edilmiş dosyalardan proxy'leri import edin
- **Format**: Kolay versiyon kontrolü için JSON formatını destekler

### Versiyon Geçmişi

API Proxy'lerinizdeki değişiklikleri takip edin:

- Her proxy için değişiklik geçmişini görüntüleyin
- Farklı versiyonları karşılaştırın
- Gerekirse önceki versiyonları geri yükleyin

### Erişim Kontrolü

API Proxy'lerinizi kimlerin görüntüleyebileceğini, değiştirebileceğini veya deploy edebileceğini yönetin:

- Takım üyelerine roller atayın
- Proxy seviyesinde izinler ayarlayın
- Takım tabanlı erişim kontrolü için grupları kullanın

## Etkili Yönetim İpuçları

1. **Tutarlı İsimlendirme Kullanın**: Proxy'leriniz için isimlendirme kuralları oluşturun
2. **Etiketleri Kullanın**: Esnek organizasyon için etiketleri kapsamlı şekilde kullanın
3. **Gruplar Oluşturun**: İlgili proxy'leri daha kolay yönetim için gruplayın
4. **Düzenli Temizlik**: Periyodik olarak kullanılmayan proxy'leri gözden geçirin ve kaldırın
5. **Dokümantasyon**: Her proxy için dokümantasyonu güncel tutun
6. **Kullanımı İzleyin**: Kullanılmayan veya kullanımdan kaldırılmış proxy'leri belirlemek için analitikleri kullanın
