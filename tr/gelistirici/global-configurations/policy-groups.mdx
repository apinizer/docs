# Politika Grupları (Policy Groups)

Politika grubu ekranında, birden fazla politika gruplandırılarak kullanılması için kullanılır. Gruplanan politikalar API Proxy/API Proxy Grup sayfasında politika eklenir gibi grup halinde eklenebilir ya da oluşturulurken belirli bir işlem sırasına göre API Proxy ve API Proxy Gruplarına bağlanabilir. Kullanıcılar, 'Add Policy' butonu ile grup içine istediği yerel veya global politikaları ekleyebilir ve sırasını düzenleyebilir, isterse politikanın durumunu aktif/pasif hale getirilebilir.

<Info>
Politikalar detayı için [Politikalar (Policies)](/tr/gelistirici/politika-yonetimi) sayfasını ziyaret edebilirsiniz.
</Info>

Politika Grup konfigürasyonu için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan | Açıklama |
| --- | --- |
| İsim <em>(Name)</em> | Politika Grup için kullanılan isim. |
| Açıklama <em>(Description)</em> | Policy Group'a dair açıklayıcı bilgi eklenebilir. |
| Add Policy | - Yeni bir politika oluşturmak ve listeye eklemek için kullanılır.<br />- Eklenen her politika, Proxy üzerinde belirlenen işlem sırasına göre uygulanır.<br />- Sıralama, ekran üzerindeki taşıma simgeleri aracılığıyla değiştirilebilir. |

## Politika Grubu Deploy İşlemi

Politika grubu üzerinde herhangi bir değişiklik yapıldığında, bu grubu kullanan API Proxy'lerin ve API Proxy Grupların durumu "Redeploy Required" olarak değişir. Bunları tek tek deploy etmek yerine, "Politika Grup Deploy" özelliği ile bu politika grubunu kullanan tüm API Proxy'lerin ve API Proxy Grupların belirli bir çalışma ortamına *(environment)* toplu olarak deploy edilmesini sağlar.

Deploy işlemi tamamlandığında, kullanıcıya deploy işleminin sonucunu gösteren **Deployment Results** adında bir sonuç ekranı sunulur.
