Global politikalar API Proxy'lerden bağımsız olarak Global Politikalar arayüzlerinden ya da bir API Proxy'nin [Geliştirme Sekmesi içinden](/tr/gelistirici/api-proxy/gelistirme-sekmesi#global-politika-olusturma) oluşturulabilir.

Bir API Proxy silindiğinde, o API Proxy tarafından kullanılan global politikalar bundan etkilenmez. Bununla birlikte, global politikada yapılan herhangi bir güncelleme, o politikayı kullanan bütün API Proxy'leri etkiler.

Global Politikalarla yapılabilecek işlemler:

- [Global Politikaları İçe Aktar](/tr/admin/export-import/import-global-policies)
- [Global Politikaları Dışa Aktar](/tr/admin/export-import/export-global-policies)

## Global Politikalar Deploy İşlemi

Global politika üzerinde herhangi bir değişiklik yapıldığında, bu politikayı kullanan API Proxy'lerin ve API Proxy Grupların durumu "Redeploy Required" olarak değişir. Bunları tek tek deploy etmek yerine, bu sayfadaki deploy özelliği ile bu politikayı kullanan tüm API Proxy'lerin ve API Proxy Grupların belirli bir çalışma ortamına *(environment)* toplu olarak deploy edilmesini sağlanır.

Deploy işlemi tamamlandığında, kullanıcıya deploy işleminin sonucunu gösteren **Deployment Results** adında bir sonuç ekranı sunulur.

## Yerel Politikayı Globalleştirme

API Proxy'nin Geliştirme sekmesinde iken oluşturacağınız herhangi bir politikada sağ üst alanda bulunan "Politikayı, Global Politika Olarak Ekle *(Add The Policy As a Global Policy)*" butonuna basıp açılan "İsim *(Name)*" alanına bir isim girerek burada oluşturduğunuz politikanın global bir politika olmasını sağlayabilirsiniz.

**Yerel olarak ayarlanmış Plain-Text Kimlik Doğrulama *(Plain-Text Authentication)* politikası üzerinden politikayı global forma dönüştürme ayarlarını içeren görsele aşağıda yer verilmiştir:**
