XML to JSON konfigürasyonu için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| **Alan** | **Açıklama** |
|----------|--------------|
| **NULL değerleri görmezden gel** <em>(Ignore fields with NULL values)</em> | Değeri NULL olan elemanların JSON mesajının içine koyulmaması için işaretlenir. |
| **Boş değerleri görmezden gel** <em>(Ignore EMPTY fields)</em> | Değeri boş olan elemanların JSON mesajının içine koyulmaması için işaretlenir. |
| **Numerik değerleri String olarak yaz** <em>(Write Numbers as Strings)</em> | Numerik değer içeren elemanları JSON mesajının içine String olarak yazılması için işaretlenir. |
| **NULL değerler için xsi:nil="true" kullan** <em>(Use xsi:nil="true" for NULL values)</em> | Değeri xsi:nil="true" olan XML elemanlar için JSON mesajın içinde "null" kullanmak için işaretlenir. |
| **Elemanı Çöz** <em>(Unwrap Element)</em> | İşaretlenirse, Kaynak XPath ile verilen elemanın içini (çocuklarını), işaretlenmezse elemanın kendisini (içindekilerle/çocuklarıyla beraber) dönüştürür. |
| **Dizi olan elemanlar** <em>(Paths for Arrays within message)</em> | JSON mesajın içinde hangi bölümlerin Array olarak yorumlanması gerektiğini belirtir. Her bir Path, Kaynak XPath'in seviyesinden (Kaynak XPath verilmemişse en üst seviyeden) başlayarak ve seviyeler arasına # işareti koyularak yazılır. Örnek: envelope#body#abc ifadesi, envelope içindeki, body'nin içindeki abc elementinin array olarak işleme alınacağını belirtir. |
