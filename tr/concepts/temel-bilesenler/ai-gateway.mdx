## AI Gateway Kavramı

AI Gateway, Apinizer'ın Veri Düzlemi (Data Plane) katmanına yapay zeka trafiği için eklenen bileşendir. Klasik [API Gateway](/tr/concepts/temel-bilesenler/api-gateway)'in üstlendiği görevi (istek karşılama, politika uygulama, yönlendirme) LLM istekleri için de üstlenir: istemcileriniz tek bir OpenAI-uyumlu uç noktaya bağlanır, AI Gateway isteği doğru LLM sağlayıcısına yönlendirir ve gateway'in zaten uyguladığı yönetişimi (güvenlik, kota, gözlemlenebilirlik) AI trafiğine de genişletir.

## Diğer Bileşenlerle İlişki

AI Gateway tek başına çalışmaz — Apinizer'ın diğer bileşenleriyle aynı temel altyapıyı paylaşır:

- **API Manager** (Kontrol Düzlemi) — LLM sağlayıcı bağlantıları, bilgi tabanları ve politikalar API Manager üzerinden yapılandırılır ve AI Gateway'e deploy edilir; bkz. [API Manager](/tr/concepts/temel-bilesenler/api-manager).
- **Kimlik Bilgisi Sistemi** — LLM sağlayıcı ve vektör veritabanı kimlik bilgileri, platformun paylaşılan şifreli kimlik bilgisi altyapısında saklanır.
- **Cache** — Semantik önbellek, platformun dağıtık cache altyapısı üzerine kurulur.
- **Analitik / Gözlemlenebilirlik** — AI kullanım metrikleri, platformun genel izleme ve raporlama altyapısıyla aynı temel üzerinde toplanır.

## Nasıl Konumlanır?

```
API Manager (Control Plane)
   │
   │ LLM sağlayıcı / vektör veritabanı / bilgi tabanı / politika konfigürasyonu
   │ Deployment
   ▼
AI Gateway (Data Plane — AI katmanı)
   │
   │ OpenAI-uyumlu istek karşılama
   │ Korumalar → RAG → Semantik Önbellek → Kota → Yönlendirme
   │ Metrik gönderme
   ▼
LLM Sağlayıcı / İstemci
```

AI Gateway, klasik API Gateway'in yanında **ek bir katman** olarak çalışır; aynı deploy modelini, aynı proje/ortam yapısını ve aynı yetkilendirme modelini kullanır.
