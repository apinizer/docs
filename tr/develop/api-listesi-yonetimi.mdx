### Gelişmiş Filtreleme

Daha detaylı aramalar yapabilmek için **Gelişmiş Filtreleme** özelliğini kullanın. Liste üzerinde bulunan filtre alanlarının hemen yanındaki ilgili ikona tıklayarak gelişmiş filtreleme seçeneklerine erişebilirsiniz.

**Gelişmiş Filtreleme Kriterleri:**

| Kriter | Açıklama |
|--------|----------|
| **API Proxy Adı** | Belirli bir adı içeren API Proxy kayıtlarını getirir. Büyük/küçük harf duyarlılığı yoktur. |
| **API Proxy Açıklaması** | Belirli bir açıklamayı içeren API Proxy kayıtlarını getirir. Büyük/küçük harf duyarlılığı yoktur. |
| **API Tipi** | Protokol tipine göre filtreleme yapar. Seçenekler: SOAP, REST |
| **API Proxy Tipi** | Oluşturulurken seçilen tipe göre filtreleme yapar. Seçenekler: Swagger, WSDL, Open API/Swagger 3.0.x, WADL, Fake API, Reverse Proxy, Mirror API, No-Spec API, DB-2-API, Script-2-API, Mock API, Websocket, gRPC, gRPC Proto |
| **Kullanım Şekli** | Kullanım şekline göre filtreleme yapar. Seçenekler: Hepsi, Üretici, Tüketici, Üretici ve Tüketici |
| **Paylaşım Tipi** | API Proxy'nin paylaşım tipine göre filtreleme yapar. |
| **Kategori** | Kategorilere göre filtreleme yapar. |
| **API Proxy Adresi** | Belirli bir adresi içeren API Proxy kayıtlarını getirir. Büyük/küçük harf duyarlılığı yoktur. |
| **Metot/Uç Nokta Adı** | Belirli bir metot veya uç nokta adını içeren API Proxy kayıtlarını getirir. Büyük/küçük harf duyarlılığı yoktur. |
| **Metot/Uç Nokta Açıklaması** | Belirli bir metot veya uç nokta açıklamasını içeren API Proxy kayıtlarını getirir. Büyük/küçük harf duyarlılığı yoktur. |
| **Parametre Adı** | Belirli bir parametre adını içeren API Proxy kayıtlarını getirir. Büyük/küçük harf duyarlılığı yoktur. |
| **Parametre Açıklaması** | Belirli bir parametre açıklamasını içeren API Proxy kayıtlarını getirir. Büyük/küçük harf duyarlılığı yoktur. |
| **Router Adresi** | Belirli bir yönlendirme adresini içeren API Proxy kayıtlarını getirir. Büyük/küçük harf duyarlılığı yoktur. |
| **Yükleme Durumu** | Yükleme durumuna göre filtreleme yapar. Seçenekler: Hepsi, Yüklenmiş, Yüklenmemiş |
| **Ortam** | Ortam bilgisine göre filtreleme yapar. |
| **Trace Modu** | İzleme moduna göre filtreleme yapar. Seçenekler: Aktif, Pasif |
| **Uygulama Log Durumu** | Uygulama log durumuna göre filtreleme yapar. Seçenekler: Aktif, Pasif |

### Sıralama ve Görünüm Seçenekleri

API Proxy listesi birden fazla sıralama ve görünüm seçeneğini destekler:

- **Sıralama**: İsim, Tip, Durum, Son Değişiklik, Ortam
- **Görünüm Seçenekleri**: Liste görünümü, Grid görünümü, Kompakt görünüm

### API Proxy Grup Bilgisi

API Proxy listesinde, her bir API Proxy'nin ait olduğu API Proxy grupları görüntülenir. Bir API Proxy birden fazla gruba ait olabilir ve grup bilgileri liste görünümünde gösterilir.

<Info>
API Proxy grupları hakkında detaylı bilgi için [API Proxy Grupları](/tr/develop/api-proxy-group-olusturma) sayfasına bakabilirsiniz.
</Info>

## Etiketleme ve Kategorizasyon

### Etiket Kullanımı

<CardGroup cols={2}>
  <Card title="Fonksiyona Göre" icon="folder">
    Proxy'leri iş fonksiyonlarına göre etiketleme
    
    `payment` `authentication` `inventory`
  </Card>
  <Card title="Takıma Göre" icon="users">
    Proxy'leri sorumlu takıma göre etiketleme
    
    `backend-team` `frontend-team`
  </Card>
  <Card title="Duruma Göre" icon="signal">
    Proxy'leri geliştirme durumuna göre etiketleme
    
    `deprecated` `beta` `production-ready`
  </Card>
  <Card title="Domain'e Göre" icon="sitemap">
    Proxy'leri iş domain'ine göre etiketleme
    
    `e-commerce` `analytics` `reporting`
  </Card>
</CardGroup>

### Etiket Ekleme

<Steps>
  <Step title="Proxy Seç">
    Listeden bir API Proxy seçin
  </Step>
  <Step title="Overview Sekmesi">
    **Overview** sekmesine gidin
  </Step>
  <Step title="Etiketler Bölümü">
    **Etiketler** bölümünü bulun
  </Step>
  <Step title="Etiket Ekle">
    Etiket yazarak Enter'a basın - etiketler otomatik olarak kaydedilir
  </Step>
</Steps>

### Etiket Tabanlı Filtreleme

<AccordionGroup>
  <Accordion title="Filtreleme Yöntemleri" icon="filter">
    - Etiket bulutunda bir etikete tıklayarak o etikete sahip proxy'leri filtreleyin
    - Daha spesifik filtreleme için birden fazla etiketi birleştirin
    - Belirli etiket kombinasyonlarına sahip proxy'leri bulmak için etiket araması kullanın
  </Accordion>
</AccordionGroup>

### Etiketleme En İyi Uygulamaları

<Warning>
**Dikkat Edilmesi Gerekenler:**
- **Tutarlı İsimlendirme**: Bir etiketleme kuralı oluşturun ve buna bağlı kalın
- **Etiket Sayısı**: Proxy başına 3-5 etiket kullanın
- **Hiyerarşik Etiketler**: Nokta notasyonu kullanın (örn: `payment.credit-card`)
- **Dokümantasyon**: Ekibiniz için bir etiket sözlüğü tutun
</Warning>


### Liste Üzerinden Yapılabilecek İşlemler

API Proxy listesinde, her bir API Proxy için aşağıdaki işlemler gerçekleştirilebilir:

<CardGroup cols={2}>
  <Card title="Export" icon="file-export">
    API Proxy'yi yedekleme veya migrasyon için export edebilirsiniz. Export işlemi JSON formatında dosya oluşturur.
  </Card>
  <Card title="Sil" icon="trash">
    API Proxy'yi listeden silebilirsiniz. Silme işlemi öncesinde onay istenir.
  </Card>
  <Card title="Duplicate" icon="copy">
    Mevcut API Proxy'nin bir kopyasını oluşturabilirsiniz. Kopyalama işlemi tüm konfigürasyonları içerir.
  </Card>
  <Card title="Reparse" icon="refresh">
    API Proxy tipine göre yeniden ayrıştırma (reparse) işlemi yapabilirsiniz. Bu işlem, API Proxy'nin spec dosyasını yeniden okur ve günceller.
  </Card>
</CardGroup>

<Info>
Bu işlemler tek tek API Proxy'ler üzerinde gerçekleştirilir. Toplu işlem desteği bulunmamaktadır.
</Info>
