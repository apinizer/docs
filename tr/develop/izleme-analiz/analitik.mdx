API Analizi sekmesi, API Proxy'nin performansına, uç nokta kullanım yoğunluğuna ve istemci aktivitesine ilişkin özet bilgi sunar.

Yenileme Zamanını Seçininiz (Choose your Refresh Time) girdisinden yenilenme aralığı seçilerek, veriler sayfa değiştirilene kadar otomatik olarak yenilenir.

## Yönlendirme Teşhisi

Analitik sekmesinde, bu API Proxy üzerinden backend'e giden isteklerde yaşanan yönlendirme sorunlarını özetleyen bir **Yönlendirme Teşhisi** bölümü yer alır. Bu bölüm, bağlantı kurma, okuma ve bağlantı havuzundan bağlantı alma aşamalarında oluşan gecikme ve hataları toplu halde gösterir.

:::info

Bu veriler, ilgili konnektörün Log Ayarları'ndaki **Performans Metrikleri** seçeneği açıkken (varsayılan durum) otomatik olarak toplanır; bu seçenek kapalı olsa bile [İzleme (Tracing)](/tr/analytic/tracing) açıkken veya istek hata/blokla sonuçlandığında yine toplanır. Veriler yalnızca backend'e giden istekler için oluşur; önbellekten dönen veya backend'e hiç gitmeyen istekler bu bölüme dahil değildir.

:::

### Hata Nedeni Dağılımı

Seçilen zaman aralığında yaşanan yönlendirme hatalarının nedene göre dağılımını gösteren bir çubuk grafiktir. Bir çubuğa tıklandığında, o hata nedenine ait kayıtlar [API Trafiği](/tr/develop/izleme-analiz/api-trafigi) tablosunda filtrelenerek listelenir.

Olası hata nedenleri:

| Hata Nedeni | Açıklama |
|-------------|----------|
| **Havuz Zaman Aşımı** | Yapılandırılan süre içinde bağlantı havuzundan bağlantı kiralanamadı — havuz tükenmiş olabilir |
| **Bağlanma Zaman Aşımı** | Backend'e TCP bağlantısı kurulamadı — backend kapalı, yanlış host/port ya da ağ/güvenlik duvarı engeli |
| **DNS Hatası** | Backend host adı çözümlenemedi |
| **TLS El Sıkışma Hatası** | Backend ile TLS el sıkışması başarısız — sertifika, protokol veya cipher uyumsuzluğu |
| **Okuma Zaman Aşımı (yanıt yok)** | Yeniden kullanılan bir bağlantıda okuma zaman aşımına kadar ilk bayt alınamadı — bayat/kopuk bağlantı olasılığı |
| **Okuma Zaman Aşımı (yavaş backend)** | İlk bayt alındı ama backend yanıtı zamanında tamamlayamadı |
| **Backend Bağlantıyı Kapattı** | Backend bağlantıyı beklenmedik şekilde kapattı |
| **İstemci Bağlantıyı Kapattı** | İstemci, yanıt tamamlanmadan bağlantıyı sonlandırdı |
| **Sağlıklı Upstream Yok** | Yönlendirilecek sağlıklı bir backend adresi yoktu |
| **Devre Açık** | Bu rota için devre kesici açıktı, istek backend'e gönderilmedi |
| **Tüm Denemeler Tükendi** | Tüm yeniden deneme/yük devretme denemeleri başarısız oldu |
| **Upstream HTTP Hatası** | Backend isteği aldı ve bir HTTP hata durum kodu döndürdü |
| **Bilinmiyor** | Hata mevcut sinyallerden sınıflandırılamadı |

### Faz Gecikmesi (p50/p95/p99)

Backend'e giden isteklerin hangi aşamada ne kadar zaman harcadığını gösteren bir grafiktir. Aşağıdaki fazlar için p50/p95/p99 gecikme değerleri (milisaniye) hesaplanır:

| Faz | Açıklama |
|-----|----------|
| **Seçim** | Yük dengeleme ve devre kesici kontrolü ile hedef adresin seçilmesi |
| **DNS** | Backend host adının çözümlenmesi |
| **TCP Bağlanma** | Backend ile TCP bağlantısının kurulması |
| **TLS El Sıkışma** | HTTPS bağlantılarında TLS el sıkışması |
| **İlk Yanıt Baytı (TTFB)** | İsteğin gönderilmesinden backend'in ilk yanıt baytını dönmesine kadar geçen süre |
| **Gövde Okuma** | Yanıt gövdesinin tamamının okunması |
| **Havuzdan Bağlantı Bekleme** | Bağlantı havuzundan uygun bir bağlantının kiralanması için beklenen süre |

Bu dağılım, yavaşlığın bağlantı kurulumunda mı (DNS/TCP/TLS), backend yanıt süresinde mi (TTFB/gövde okuma) yoksa bağlantı havuzunda mı (havuzdan bağlantı bekleme) yaşandığını ayırt etmeyi sağlar.

### Diğer Özetler

- **En Çok Hata Üreten Proxy'ler**: Yönlendirme hatası kaydı düşen isteklerde en çok hata üreten API Proxy'lerin listesi
- **Yavaş İstemci vs Yavaş Backend (ortalama)**: İstemciye yanıt yazma süresi ile backend işlem süresinin ortalama karşılaştırması — yavaşlığın istemci/ağ tarafında mı yoksa backend tarafında mı olduğunu gösterir
- **En Çok Upstream**: En sık hata alınan upstream adresleri (IP:Port)

:::tip

Tek bir isteğe ait Yönlendirme Teşhisi detayları (hata nedeni, güven düzeyi, faz süreleri, önerilen aksiyon vb.) için [API Trafiği](/tr/develop/izleme-analiz/api-trafigi) sekmesindeki Detaylı Görüntüleme penceresine bakabilirsiniz.

:::
