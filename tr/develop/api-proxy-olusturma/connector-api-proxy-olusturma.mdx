## Yeni Connector API Proxy Oluşturma

<Steps>
  <Step title="Menüden API Proxy Sayfasına Gidin">
    Ana menüde **Geliştirme → API Proxy'ler** (*Development → API Proxies*) ögesi seçilir.
  </Step>

  <Step title="Yeni API Proxy Oluşturun">
    Açılan arayüzde sağ üstteki **+Add API Proxy** tuşuna tıklanır.

    <img
      src="/images/develop/api-proxy-olusturma/connector1.png"
      alt="API Proxy oluştur butonu"
      width="1000"
      className="mr-auto"
    />
  </Step>

  <Step title="Connector Tipini Seçin">
    API Proxy tip seçenekleri arasından **Connector** kartındaki **Create an API Proxy** bağlantısına tıklanır.

    <img
      src="/images/develop/api-proxy-olusturma/connector2.png"
      alt="Connector tip seçimi"
      width="1000"
      className="mr-auto"
    />
  </Step>

  <Step title="Konnektör ve Proxy Bilgilerini Doldurun">
    **Name** alanına API Proxy adı girilir. **Connector** alanının yanındaki **+** tuşuna tıklanarak açılan **Connectors** penceresinden kullanılacak konnektör tipi seçilir. Ardından Client Route alanları doldurulup **Kaydet (Save)** tuşuna tıklanır.

    <img
      src="/images/develop/api-proxy-olusturma/connector3.png"
      alt="Connector API Proxy bilgilerini doldurma"
      width="1000"
      className="mr-auto"
    />
  </Step>
</Steps>

### Connector API Proxy Tanım Alanları

Connector API Proxy tanımı için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan | Açıklama |
|------|----------|
| **Ad** (Name) | API Proxy'ye kullanıcı tarafından verilen addır. Arayüzlerde bu ad görünür. |
| **Connector** | Kullanılacak konnektör tipidir. **+** tuşu ile açılan listeden seçilir: ActiveMQ, Elasticsearch, FTP List, FTP Read, Graylog, Kafka, Logback, Email, RabbitMQ, SNMP, Syslog. |
| **Relative Paths** | API Proxy'nin erişime açılan adresinin bir parçasıdır. Proje bazında tekil olmalıdır. Örnek: `/order-to-kafka` |
| **Hosts** | İsteğe bağlı host tanımlarıdır. |
| **Methods** | Bu API Proxy için izin verilen HTTP metodlarıdır (GET, POST, PUT, DELETE, PATCH vb.). |
| **Headers** | API Proxy seviyesinde eklenecek veya değiştirilecek HTTP header'larıdır. |

:::info

**Client Route** alanlarının detayı için [Client Route Ayarları](/tr/develop/api-proxy-olusturma/client-route-ayarlari) sayfasına bakabilirsiniz.

:::

## Konnektör Yapılandırması

Konnektör tipi seçildikten sonra ekranda ilgili konnektöre özel yapılandırma alanları görünür. Bu alanlar, seçilen konnektöre göre değişir.

  <Accordion title="Mesaj Kuyruğu Konnektörleri (Kafka / RabbitMQ / ActiveMQ)">
    HTTP isteğinin gövdesi seçilen mesaj kuyruğuna iletilir. Tipik kullanım: event üretme, asenkron iş kuyruğuna mesaj bırakma.

    | Alan | Açıklama |
    |------|----------|
    | **Bağlantı Tanımı** | Kafka, RabbitMQ veya ActiveMQ bağlantı havuzundan seçilir. |
    | **Hedef / Topic / Queue** | Mesajın yazılacağı topic veya queue bilgisidir. Konnektör tipine göre adlandırma değişebilir. |
  </Accordion>

  <Accordion title="Email Connector">
    HTTP isteği alındığında yapılandırılan bağlantı üzerinden e-posta gönderimi yapılır.

    | Alan | Açıklama |
    |------|----------|
    | **Bağlantı Tanımı** | SMTP / e-posta bağlantı tanımı seçilir. |
    | **Alıcı / Konu / İçerik** | E-posta gönderimi için kullanılan alanlardır. Konnektör ekranında görünen alanlara göre doldurulur. |
  </Accordion>

  <Accordion title="Elasticsearch / FTP / Graylog / Logback / SNMP / Syslog">
    Seçilen konnektör tipine göre indeks, dosya yolu, log hedefi veya trap ayarları gibi alanlar görünür. Alanların detayı için ilgili [konnektör](/tr/integrations/konnektorler/genel-bakis) sayfasına bakınız.
  </Accordion>

:::warning

Konnektörde değişken kavramı kullanılmaz. İstemciden gelen veri, seçilen hedefe **ham** olarak iletilir. Mapping veya dönüşüm gerekiyorsa API Proxy üzerinde ilgili politikalar ayrıca tanımlanmalıdır.

:::

## Oluşturma Sonrası

Kaydetme adımından sonra yapılabilecekler diğer API Proxy türleriyle aynıdır:

- Politika ekleme ve yönetim
- API trafik log ayarları
- Deploy / Redeploy / Undeploy

Detay için [API Proxy Oluşturma](/tr/develop/api-proxy-olusturma) ve [Yükleme ve Sürüm Yönetimi](/tr/develop/yukleme-ve-surum-yonetimi) sayfalarına bakabilirsiniz.

## Yükle (Deploy) / Yeniden Yükle (Redeploy)

API Proxy oluşturulduktan sonra istemcilerin erişebilmesi için ilgili ortama yüklenmesi gerekir. Detay için [Yükleme ve Sürüm Yönetimi](/tr/develop/yukleme-ve-surum-yonetimi) sayfasına bakabilirsiniz.

:::warning

Konnektör tipi veya Client Route alanında değişiklik yapıldığında değişikliklerin geçerli olabilmesi için API Proxy'nin **yeniden yüklenmesi (Redeploy)** gerekir.

:::

## Connector API Proxy Silme

Bir Connector API Proxy silindiğinde önce ortamdan kaldırılır (undeploy), ardından silinir.

:::info

Bağlantı tanımı (Connection) silinmez; yalnızca bu proxy ile ilişkisi sonlanır. Aynı bağlantı başka proxy, Task Flow veya log konnektörlerinde kullanılmaya devam edebilir.

:::
