Global politikalar API Proxy'lerden bağımsız olarak Global Politikalar arayüzlerinden ya da bir API Proxy'nin [Geliştirme Sekmesi içinden](/tr/develop/api-proxy-konfigurasyonu/politika-yonetimi) oluşturulabilir.

## Global Politikaların Özellikleri

- **Bağımsızlık**: Global politikalar, API Proxy'lerden bağımsız olarak oluşturulabilir ve yönetilebilir.
- **Yeniden Kullanılabilirlik**: Bir global politika, birden fazla API Proxy tarafından kullanılabilir.
- **Merkezi Güncelleme**: Bir global politikada yapılan güncellemeler, bu politikayı kullanan tüm API Proxy'leri etkiler.
- **Etkileşim**: Bir API Proxy silindiğinde, o API Proxy tarafından kullanılan global politikalar bundan etkilenmez. Bununla birlikte, global politikada yapılan herhangi bir güncelleme, o politikayı kullanan bütün API Proxy'leri etkiler.

Global Politikalarla yapılabilecek işlemler:

- [Global Politikaları İçe Aktar](/tr/admin/secrets-management/export-import)
- [Global Politikaları Dışa Aktar](/tr/admin/secrets-management/export-import)

## Global Politikalar Deploy İşlemi

Global politika üzerinde herhangi bir değişiklik yapıldığında, bu politikayı kullanan API Proxy'lerin ve API Proxy Grupların durumu "Redeploy Required" olarak değişir. Bunları tek tek deploy etmek yerine, bu sayfadaki deploy özelliği ile bu politikayı kullanan tüm API Proxy'lerin ve API Proxy Grupların belirli bir çalışma ortamına *(environment)* toplu olarak deploy edilmesini sağlanır.

Deploy işlemi başlatıldığında, bir onay dialogu açılır. Bu dialog'da aşağıdaki bilgiler ve seçenekler bulunur:

- **Politika/Ayar Grubu Adı**: Deploy edilecek global politikanın adı gösterilir. Bu bilgi, toplu işlem olduğundan onay mekanizmasında görüntülenir.
- **API Proxy ve API Proxy Group Seçimi**: Deploy işleminin API Proxy'lere mi, API Proxy Gruplarına mı uygulanacağını seçebilirsiniz. Her iki seçenek de işaretlenebilir.
- **Filtreleme Seçeneği**: Deploy edilecek API Proxy ve API Proxy Gruplarını filtrelemek için aşağıdaki seçeneklerden birini seçebilirsiniz:
  - **Hepsine**: Bu global politikayı kullanan tüm API Proxy ve API Proxy Grupları deploy edilir.
  - **Sadece deployed durumunda olanlara**: Sadece seçili ortamda deployed durumunda olan ve redeploy gerektirmeyen API Proxy ve API Proxy Grupları deploy edilir.
  - **Sadece deployed ve redeploy required durumda olanlara**: Sadece seçili ortamda deployed durumunda olan ve redeploy gerektiren API Proxy ve API Proxy Grupları deploy edilir.
- **Deploy Edilecek Nesne Sayıları**: Seçilen filtreleme seçeneğine göre deploy edilecek API Proxy ve API Proxy Group sayıları gösterilir.

Filtreleme seçeneği değiştirildiğinde, deploy edilecek nesne sayıları otomatik olarak güncellenir. Deploy işlemi başlatıldığında, seçilen filtreleme seçeneğine göre belirlenen API Proxy ve API Proxy Grupları deploy edilir.

Deploy işlemi tamamlandığında, kullanıcıya deploy işleminin sonucunu gösteren **Deployment Results** adında bir sonuç ekranı sunulur.

## Yerel Politikayı Globalleştirme

API Proxy'nin Geliştirme sekmesinde iken oluşturacağınız herhangi bir politikada sağ üst alanda bulunan "Politikayı, Global Politika Olarak Ekle *(Add The Policy As a Global Policy)*" butonuna basıp açılan "İsim " alanına bir isim girerek burada oluşturduğunuz politikanın global bir politika olmasını sağlayabilirsiniz.

Yerel olarak ayarlanmış Plain-Text Kimlik Doğrulama *(Plain-Text Authentication)* politikası üzerinden politikayı global forma dönüştürme ayarları aşağıdaki görselde gösterilmektedir:
