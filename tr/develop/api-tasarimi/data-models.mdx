## Ortak Veri Modeli Kavramı

Ortak Veri Modeli, API Designer'ın spec first yaklaşımında yeniden kullanılabilir şema tanımları oluşturmanıza olanak sağlar. Mesaj içeriğindeki veri tiplerini tanımlayan ortak şema modelleri oluşturarak, model kalıtımı ve paylaşılan bileşenler ile JSON Schema referansları kullanabilirsiniz. Oluşturulan modeller Spec Tasarım Editörü içerisinde içe aktarılarak kullanılabilir.

## Ortak Veri Modeli Oluşturma

Liste arayüzünün sağ üst kısmındaki **Yeni (Create)** butonuna tıklanarak yeni bir Veri Modeli oluşturulabilir.

<img src="/images/develop/data-models/data-model-1.png" alt="Liste arayüzü" style={{maxWidth: '1000px', width: '100%', height: 'auto'}} />

Ortak Veri Modeli oluşturma ayarları aşağıdaki görselde gösterilmektedir:

<img src="/images/develop/data-models/data-model-2.png" alt="Ortak Veri Modeli oluşturma ayarları" style={{maxWidth: '1000px', width: '100%', height: 'auto'}} />

Ortak veri modeli oluşturma konfigürasyonu için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan | Açıklama |
|------|----------|
| **Ad**  | Veri modelinin ad bilgisidir. |
| **Açıklama**  | Veri modeline ilişkin açıklamadır. |
| **Şema** <em>(Schema)</em> | Veride yer bulunacak özellikler ve alanların tipi, pattern, min-max gibi özellikleri yönetilir. |
| **Örnek** <em>(Example)</em> | Veri tipinin bir örneği oluşturabilir. |

## Şema Özelliği Ekleme

Şema içerisinde özellikler (properties) ekleyerek veri modelinizin yapısını tanımlayabilirsiniz.

Şema Özelliği ekleme ayarları aşağıdaki görselde gösterilmektedir:

<img src="/images/develop/data-models/data-model-3.png" alt="Şema Özelliği ekleme ayarları" style={{maxWidth: '755px', width: '100%', height: 'auto'}} />

Şema Özelliği oluşturma konfigürasyonu için kullanılan alanlar aşağıdaki tabloda görülmektedir.

| Alan | Açıklama |
|------|----------|
| **Ad**  | Şemanın ad bilgisidir. |
| **Açıklama**  | Şema hakkında açıklama yapılabilir. |
| **Tip**  | Şemanın tipinin belirtildiği alandır. |
| **Format** <em>(Format)</em> | Seçilen tipe göre aktif olan bir alandır. Bazı tipler için seçilebilecek formatlar bulunur:<br/><br/>**integer**<br/>- int32: İşaretli 32 bit tam sayılar (yaygın olarak kullanılan tam sayı türü).<br/>- int64: İşaretli 64 bit tam sayılar (uzun tip).<br/><br/>**number**<br/>- float: Ondalıklı sayılar (6 basamak).<br/>- double: Ondalıklı sayılar (14 basamak).<br/><br/>**string**<br/>- byte: Base64 ile kodlanmış karakterler. Örneğin, **U3dhZ2dlciByb2Nrcw==**<br/>- binary: Dosyaları tanımlamak için kullanılan ikili veri<br/>- date: Tam tarih gösterimi. Örneğin, **2017-07-21**<br/>- date-time: Tarih-saat gösterimi. Örneğin, **2017-07-21T17:32:28Z**<br/>- password: Girişi maskelemek için kullanılır. |
| **$Ref Target** | Eğer tip alanından $ref seçilirse bu alan aktifleşir.<br/><br/>Başka bir veri modelinin referansı verilebilir. |
| **Özellik** <em>(Property is)</em> | Modelin kullanımının zorunlu olup olmadığı belirtilir. |
| **Davranım** <em>(Behaviour)</em> | Şemanın yalnızca okunur/yazılır (Read Only, Write Only) bilgisi belirlenir. |
| **Integer Özellikleri** <em>(Integer Properties)</em> | Tip olarak Integer seçilmiş ise bu alanlar aktif olur:<br/>- Minimum: Girilebilecek değerlerin en küçüğünü belirlemek içindir.<br/>- Maximum: Girilebilecek değerlerin en büyüğünü belirlemek içindir.<br/>- MultipleOf: Girilecek değerlerin bir sayının katı olması isteniyorsa bunu belirlemek içindir. |
| **String Özellikleri** <em>(String Properties)</em> | Tip olarak String seçilmiş ise bu alanlar aktif olur:<br/>- Desen <em>(Pattern)</em>: String değeri için bir normal ifade şablonu tanımlamanıza olanak tanır. Yalnızca bu şablonla eşleşen değerler kabul edilecektir.<br/>- Minimum Uzunluğu <em>(Min Length)</em>: Girilebilecek değerlerin minimum uzunluğunu belirlemek içindir.<br/>- Maksimum Uzunluğu <em>(Max Length)</em>: Girilebilecek değerlerin maksimum uzunluğunu belirlemek içindir. |
| **Enum Değerleri** <em>(Enum Values)</em> | Tip olarak Enum seçilmiş ise bu alan aktif olur.<br/><br/>Modelin olası değerlerininin girildiği alandır. |
| **Örnek** <em>(Example)</em> | Şemayı tarif eden bir örnek verilebilir. |

## Ortak Veri Modeli ve Spec Tasarım Editörü İlişkisi

Ortak Veri Modelleri, Spec Tasarım Editörü ile entegre çalışır. İşleyiş akışı:

```
Ortak Veri Modelleri
   │
   │ Şema Tanımları
   │ Model Referansları
   │
   ▼
Spec Tasarım Editörü
   │
   │ Model İçe Aktarma
   │ $ref Referansları
   │
   ▼
API Spec
   │
   │ Paylaşılan Bileşenler
   │
   ▼
API Proxy
```

<CardGroup cols={4}>
  <Card title="1. Model Oluşturma" icon="plus-circle">
    Ortak Veri Modelleri sayfasında model oluşturulur
    
    Şema özellikleri ve yapısı tanımlanır.
  </Card>
  
  <Card title="2. Model Tanımlama" icon="pencil">
    Şema özellikleri ve validation kuralları belirlenir
    
    Property'ler, tipler ve formatlar tanımlanır.
  </Card>
  
  <Card title="3. Spec'te Kullanım" icon="file-code">
    Spec Tasarım Editörü'nde model içe aktarılır
    
    $ref referansları ile model kullanılır.
  </Card>
  
  <Card title="4. Yeniden Kullanım" icon="repeat">
    Model birden çok API Spec'te kullanılabilir
    
    Merkezi yönetim ve tutarlılık sağlanır.
  </Card>
</CardGroup>

<Info>
Ortak Veri Modelleri ve Spec Tasarım Editörü arasındaki bu entegrasyon, spec first yaklaşımında yeniden kullanılabilir şema tanımları ile tutarlı ve merkezi yönetilen veri yapıları oluşturmanıza olanak sağlar. Modeller bir kez oluşturulur ve birden çok API Spec'te kullanılabilir.
</Info>
