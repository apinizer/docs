Bu sayfa, Apinizer platformunun farklı deployment senaryoları için detaylı donanım gereksinimlerini içerir. Gereksinimler, trafik hacmi ve kullanım senaryosuna göre **Tier 1, Tier 2 ve Tier 3** olarak kategorize edilmiştir.

## Donanım Kategorileri

  <Card title="Tier 1" icon="circle-1">

  <Card title="Tier 2" icon="circle-2">
    **Orta Trafik**<br/>

    Production ortamları
  </Card>
  <Card title="Tier 3" icon="circle-3">
    **Yüksek Trafik**<br/>
    > 1M istek/gün<br/>
    Enterprise production
  </Card>
</CardGroup>

## Worker Nodes (API Gateway)

Worker node'ları API trafiğini işleyen ana bileşenlerdir.

| Bileşen | Tier 1 | Tier 2 | Tier 3 |
|---------|--------|--------|--------|
| **CPU** | 4 core | 8 core | 16+ core |
| **RAM** | 8 GB | 16 GB | 32+ GB |
| **Disk** | 50 GB SSD | 100 GB SSD | 200+ GB SSD |
| **Network** | 1 Gbps | 1-10 Gbps | 10+ Gbps |
| **Node Sayısı** | 2 (min) | 3-5 | 5+ |
| **IOPS** | 1,000 | 3,000 | 5,000+ |

<Info>
  **Not**: Worker node sayısı, trafik hacmi ve yüksek erişilebilirlik gereksinimlerine göre artırılabilir. Her node, yaklaşık 1000-5000 istek/saniye işleyebilir (politika karmaşıklığına bağlı).
</Info>

## Manager Nodes

Manager node'ları konfigürasyon ve yönetim işlevlerini sağlar.

| **Disk** | 100 GB SSD | 100-200 GB SSD (her node) |
| **Network** | 1 Gbps | 1-10 Gbps |
| **Node Sayısı** | 1 | 2 (minimum) |
| **IOPS** | 1,000 | 2,000 |

## MongoDB Replica Set

MongoDB, konfigürasyon ve metadata verilerini saklar.

| Bileşen | Tier 1 | Tier 2 | Tier 3 |
|---------|--------|--------|--------|
| **CPU** | 4 core | 8 core | 16 core |
| **RAM** | 16 GB | 32 GB | 64 GB |
| **Disk** | 200 GB SSD | 500 GB SSD | 1-2 TB SSD |
| **Network** | 1 Gbps | 1-10 Gbps | 10 Gbps |
| **Node Sayısı** | 3 (1P+2S) | 3-5 | 5+ |
| **IOPS** | 2,000 | 5,000 | 10,000+ |
| **Veri Tahmini** | 100-500 GB | 500 GB - 2 TB | 2 TB+ |

<Warning>
  **Kritik**: MongoDB mutlaka **Replica Set** olarak yapılandırılmalıdır. Tek node dahi olsa replica set olarak kurulmalıdır. Standalone instance kullanılmamalıdır.
</Warning>

## Elasticsearch Cluster

Elasticsearch, log ve analitik verilerini saklar.

| Bileşen | Tier 1 | Tier 2 | Tier 3 |
|---------|--------|--------|--------|
| **CPU** | 4 core | 8 core | 16 core |
| **RAM** | 16 GB | 32 GB | 64 GB |
| **Disk (Hot)** | 500 GB SSD | 2 TB SSD | 5+ TB SSD |
| **Disk (Warm)** | 1 TB HDD | 3 TB HDD | 10+ TB HDD |
| **Network** | 1 Gbps | 1-10 Gbps | 10+ Gbps |
| **Node Sayısı** | 3 (min) | 3-5 | 5+ |
| **Shard/Index** | 1-2 | 3-5 | 5+ |
| **Günlük Log** | < 10 GB | 10-100 GB | > 100 GB |

<Info>
  **Disk Stratejisi**: Hot tier için SSD, Warm/Cold tier için HDD kullanılması önerilir. Bu, maliyet optimizasyonu sağlar.
</Info>

## Donanım Özellikleri Detayları

### CPU Gereksinimleri

<AccordionGroup>
  <Accordion title="CPU Türü">
    * **Önerilen**: x86_64 (Intel/AMD)
    * **Minimum**: 64-bit işlemci
    * **Özellikler**: 
      * Virtualization desteği (Kubernetes için)
      * AES-NI desteği (şifreleme için)
      * AVX desteği (performans için)
  </Accordion>
  
  <Accordion title="CPU Performansı">
    * **Clock Speed**: 2.0+ GHz
    * **Core Sayısı**: Trafik hacmine göre (yukarıdaki tablolara bakın)
    * **Hyperthreading**: Aktif olmalı
    * **NUMA**: Büyük sistemlerde önemli
  </Accordion>
</AccordionGroup>

### RAM Gereksinimleri

<AccordionGroup>
  <Accordion title="RAM Türü">
    * **Tür**: DDR4 veya daha yeni
    * **ECC**: Production için önerilir
    * **Speed**: 2400+ MHz
  </Accordion>
  
  <Accordion title="RAM Dağılımı">
    * **JVM Heap**: RAM'in %50-70'i
    * **OS ve Diğer**: RAM'in %20-30'u
    * **Buffer**: RAM'in %10-20'si
  </Accordion>
</AccordionGroup>

### Disk Gereksinimleri

<AccordionGroup>
  <Accordion title="Disk Türü">
    * **SSD**: Tüm bileşenler için önerilir
    * **NVMe SSD**: Yüksek performans gereksinimleri için
    * **HDD**: Sadece Elasticsearch Warm/Cold tier için
  </Accordion>
  
  <Accordion title="Disk Performansı">
    * **IOPS**: 
      * Worker: 1,000-5,000 IOPS
      * Database: 2,000-10,000 IOPS
      * Log Storage: 1,000-3,000 IOPS
    * **Latency**: < 1ms (SSD)
    * **Throughput**: 200+ MB/s
  </Accordion>
  
  <Accordion title="Disk Yapılandırması">
    * **RAID**: 
      * Database için RAID 10 önerilir
      * Log storage için RAID 5 yeterli
    * **LVM**: Disk genişletme için önerilir
    * **Partitioning**: OS ve data ayrımı
  </Accordion>
</AccordionGroup>

### Network Gereksinimleri

<AccordionGroup>
  <Accordion title="Network Bandwidth">
    * **Tier 1**: 1 Gbps
    * **Tier 2**: 1-10 Gbps
    * **Tier 3**: 10+ Gbps
  </Accordion>
  
  <Accordion title="Network Latency">
    * **Intra-cluster**: < 1ms
    * **DMZ-LAN**: < 5ms
    * **Internet**: < 50ms (coğrafi konuma göre)
  </Accordion>
  
  <Accordion title="Network Yapılandırması">
    * **Jumbo Frames**: 9000 MTU (cluster içi iletişim için)
    * **Bonding**: Network redundancy için
    * **VLAN**: Network segmentation için
  </Accordion>
</AccordionGroup>

## Donanım Seçim Kılavuzu

### Fiziksel Sunucu vs. Sanal Sunucu

<CardGroup cols={2}>
  <Card title="Fiziksel Sunucu" icon="server">
    **Avantajlar:**
    * Yüksek performans
    * Öngörülebilir kaynaklar
    * Düşük latency
    
    **Dezavantajlar:**
    * Yüksek maliyet
    * Esneklik eksikliği
    * Uzun sağlama süresi
    
    **Kullanım**: Yüksek performans gereksinimleri
  </Card>
  <Card title="Sanal Sunucu" icon="cloud">
    **Avantajlar:**
    * Düşük maliyet
    * Hızlı sağlama
    * Esnek ölçeklendirme
    
    **Dezavantajlar:**
    * Performans varyasyonu
    * Noisy neighbor problemi
    
    **Kullanım**: Çoğu senaryo için uygun
  </Card>
</CardGroup>

### Cloud vs. On-Premise

<CardGroup cols={2}>
  <Card title="Cloud (AWS/Azure/GCP)" icon="cloud">
    **Avantajlar:**
    * Hızlı başlangıç
    * Otomatik ölçeklendirme
    * Yönetilen servisler
    * Düşük başlangıç maliyeti
    
    **Dezavantajlar:**
    * Uzun vadede yüksek maliyet
    * Vendor lock-in riski
    * Veri lokasyonu kontrolü
    
    **Kullanım**: Hızlı başlangıç, değişken trafik
  </Card>
  <Card title="On-Premise" icon="building">
    **Avantajlar:**
    * Veri kontrolü
    * Uzun vadede düşük maliyet
    * Compliance kolaylığı
    * Özelleştirme imkanı
    
    **Dezavantajlar:**
    * Yüksek başlangıç maliyeti
    * Yönetim yükü
    * Uzun sağlama süresi
    
    **Kullanım**: Büyük kurumlar, compliance gereksinimleri
  </Card>
</CardGroup>
