## 1. All-in-One (Standalone) Deployment

All-in-One deployment modeli, tüm Apinizer bileşenlerinin tek bir sunucu üzerinde çalıştığı basit kurulum modelidir.

### Mimari Yapı

```
┌─────────────────────────────────────┐
│         Single Server               │
│  ┌───────────────────────────────┐  │
│  │     Manager Module            │  │
│  └───────────────────────────────┘  │
│  ┌───────────────────────────────┐  │
│  │     Worker Module             │  │
│  └───────────────────────────────┘  │
│  ┌───────────────────────────────┐  │
│  │     Cache Module              │  │
│  └───────────────────────────────┘  │
│  ┌───────────────────────────────┐  │
│  │     MongoDB                  │  │
│  └───────────────────────────────┘  │
│  ┌───────────────────────────────┐  │
│  │     Elasticsearch            │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

### Kullanım Senaryoları

* **Test Ortamları**: Geliştirme ve test için ideal
* **Proof of Concept (POC)**: Hızlı prototip geliştirme
* **Düşük Trafikli Uygulamalar**: Sınırlı kullanıcı sayısı
* **Eğitim Ortamları**: Öğrenme ve deneme amaçlı

### Avantajlar

* Basit kurulum ve yönetim
* Düşük kaynak gereksinimi
* Hızlı başlangıç
* Tek sunucu yönetimi
* Düşük maliyet

### Dezavantajlar

* Tek nokta arıza riski (SPOF - Single Point of Failure)
* Sınırlı ölçeklenebilirlik
* Production için uygun değil
* Yüksek erişilebilirlik yok
* Performans sınırlamaları

### Sistem Gereksinimleri

* **CPU**: 4-8 core
* **RAM**: 16-32 GB
* **Disk**: 200-500 GB SSD
* **Network**: 1 Gbps

<Warning>
  **Önemli**: All-in-One modeli production ortamları için önerilmez. Yüksek erişilebilirlik ve güvenlik gereksinimleri için Distributed veya Clustered model kullanılmalıdır.
</Warning>

## 2. Distributed Deployment

Distributed deployment modeli, Apinizer bileşenlerinin farklı sunucularda çalıştığı ve birbirleri ile ağ üzerinden iletişim kurduğu modeldir.

### Mimari Yapı

```
┌─────────────────┐     ┌─────────────────┐
│  Manager Server │────▶│  Worker Server  │
│                 │     │   (Gateway)     │
│  - Web Manager │     │  - Proxy Handler│
│  - Analytics   │     │  - Local Cache  │
│  - Monitoring  │     └─────────────────┘
└─────────────────┘              │
         │                        │
         ▼                        ▼
┌─────────────────┐     ┌─────────────────┐
│  Database Server│     │  Cache Server   │
│                 │     │                 │
│  - MongoDB     │     │  - Hazelcast    │
│  - Elasticsearch│     │  - Distributed  │
└─────────────────┘     └─────────────────┘
```

### Kullanım Senaryoları

* **Orta-Büyük Ölçekli Kurulumlar**: Kurumsal ortamlar
* **Bileşen Bazlı Ölçeklendirme**: Her bileşenin bağımsız ölçeklendirilmesi
* **Kaynak Optimizasyonu**: Farklı kaynak gereksinimleri
* **Coğrafi Dağılım**: Farklı lokasyonlarda bileşenler

### Avantajlar

* Bileşen bazlı ölçeklendirme
* Kaynak optimizasyonu
* Esnek mimari
* Bileşen bağımsız yönetim
* Yüksek performans potansiyeli

### Dezavantajlar

* Daha karmaşık yönetim
* Ağ bağımlılığı
* Yüksek kaynak gereksinimi
* Network latency riski
* Daha fazla sunucu yönetimi

### Sistem Gereksinimleri

**Manager Server:**
* CPU: 4-8 core
* RAM: 16-32 GB
* Disk: 100-200 GB SSD

**Worker Server (her biri):**
* CPU: 4-8 core
* RAM: 8-16 GB
* Disk: 50-100 GB SSD

**Database Server:**
* CPU: 8-16 core
* RAM: 32-64 GB
* Disk: 500 GB - 2 TB SSD

**Cache Server:**
* CPU: 4-8 core
* RAM: 16-32 GB
* Disk: 100-200 GB SSD

## 3. Clustered (HA) Deployment

Clustered deployment modeli, yüksek erişilebilirlik ve yüksek performans için tasarlanmış, birden fazla sunucunun cluster yapısında çalıştığı modeldir.

### Mimari Yapı

```
                    ┌──────────────┐
                    │ Load Balancer│
                    └──────┬───────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ Worker Node 1│  │ Worker Node 2│  │ Worker Node N│
│              │  │              │  │              │
│ - Proxy      │  │ - Proxy      │  │ - Proxy      │
│ - Local Cache│  │ - Local Cache│  │ - Local Cache│
└──────┬───────┘  └──────┬───────┘  └──────┬───────┘
       │                 │                 │
       └─────────────────┼─────────────────┘
                         │
        ┌────────────────┼────────────────┐
        │                │                │
        ▼                ▼                ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ Manager Node1│  │ Manager Node2│  │ Cache Cluster │
│              │  │              │  │              │
│ - Web Mgr    │  │ - Web Mgr    │  │ - Hazelcast  │
│ - Analytics  │  │ - Analytics  │  │   (3+ nodes)  │
└──────┬───────┘  └──────┬───────┘  └──────┬───────┘
       │                 │                 │
       └─────────────────┼─────────────────┘
                         │
        ┌────────────────┼────────────────┐
        │                │                │
        ▼                ▼                ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ MongoDB      │  │ MongoDB      │  │ Elasticsearch │
│ Replica Set  │  │ Replica Set  │  │ Cluster      │
│ (Primary)    │  │ (Secondary)  │  │ (3+ nodes)   │
└──────────────┘  └──────────────┘  └──────────────┘
```

### Kullanım Senaryoları

* **Production Ortamları**: Kritik iş uygulamaları
* **Yüksek Trafikli Sistemler**: Milyonlarca istek/gün
* **Yüksek Erişilebilirlik Gereksinimleri**: %99.9+ uptime
* **Kritik İş Süreçleri**: Kesintisiz hizmet
* **Global Dağılım**: Farklı coğrafyalarda cluster'lar

### Avantajlar

* Yüksek erişilebilirlik (%99.9+)
* Otomatik failover
* Yatay ölçeklendirme
* Load balancing
* Veri replikasyonu
* Performans optimizasyonu

### Dezavantajlar

* Karmaşık kurulum
* Yüksek kaynak gereksinimi
* Cluster yönetimi gereksinimi
* Yüksek maliyet
* Network gereksinimleri

### Sistem Gereksinimleri

**Worker Cluster (her node):**
* CPU: 8-16 core
* RAM: 16-32 GB
* Disk: 100-200 GB SSD
* Node sayısı: Minimum 2, önerilen 3+

**Manager Cluster (her node):**
* CPU: 4-8 core
* RAM: 16-32 GB
* Disk: 100-200 GB SSD
* Node sayısı: Minimum 2

**MongoDB Replica Set:**
* CPU: 8-16 core (her node)
* RAM: 32-64 GB (her node)
* Disk: 500 GB - 2 TB SSD (her node)
* Node sayısı: Minimum 3 (1 Primary + 2 Secondary)

**Elasticsearch Cluster:**
* CPU: 8-16 core (her node)
* RAM: 32-64 GB (her node)
* Disk: 1-5 TB (her node, trafiğe göre)
* Node sayısı: Minimum 3

**Cache Cluster (Hazelcast):**
* CPU: 4-8 core (her node)
* RAM: 16-32 GB (her node)
* Disk: 100-200 GB SSD (her node)
* Node sayısı: Minimum 3

### Yüksek Erişilebilirlik Özellikleri

* **Automatic Failover**: Bir node arızalandığında otomatik yedek devreye girer
* **Load Balancing**: Trafik tüm node'lar arasında dağıtılır
* **Data Replication**: Veriler birden fazla node'da saklanır
* **Health Monitoring**: Sürekli sağlık kontrolü ve otomatik recovery
* **Zero-Downtime Updates**: Güncellemeler kesintisiz yapılabilir

## Kubernetes Kurulum Topolojileri

Kubernetes üzerinde deployment yapılırken aşağıdaki gereksinimler tavsiye edilen **minimum** konfigürasyon için belirtilmiştir. Servis yüklerinize göre artırılmalıdır.

<Warning>
  **Apinizer, üretim ortamı için tek sunucuya kurulum yapılmasını tavsiye etmez. Böyle bir kurulum konfigürasyonunu sadece PoC ortamları için değerlendiriniz.**
</Warning>

<Warning>
  **Test/PoC kurulumlarını yük testi amacıyla kullanmayınız!** Yük testi yapmak için doğru konfigürasyonu değerlendirmek isterseniz lütfen [Benchmark Sonuçları](/tr/apinizer-anlama/benchmark-sonuclari) sayfamıza bakınız veya bizimle iletişime geçiniz.
</Warning>

### Topoloji Karşılaştırması

| Özellik | PoC / Test | Professional | High Available |
|---------|------------|--------------|----------------|
| **Toplam Sunucu Sayısı** | 2 | 3 | 12 |
| **Kubernetes Control-Plane** | 1 | 1 | 3 (HA) |
| **Kubernetes Worker** | 1 | 2 | 3 |
| **MongoDB** | Single Instance | Single Instance | Replica Set (3 nodes) |
| **Elasticsearch** | Single Node | Single Node | Cluster (3 nodes) |
| **Kullanım Amacı** | PoC, Test | Production | Production (HA) |
| **Yüksek Erişilebilirlik** | ❌ | ❌ | ✅ |

### PoC / Test / Minimum Deployment

Bu topoloji, Proof of Concept (PoC), test ve minimum deployment senaryoları için tasarlanmıştır.

| Sunucu | İşletim Sistemi | CPU | RAM | Disk | Kurulumlar |
|--------|----------------|-----|-----|------|------------|
| **Sunucu 1** | Ubuntu Server 24.04 LTS / RHEL 9.x | 8 Core | 16 GB | 200 GB | Kubernetes Control-Plane, Elasticsearch (Master+Data), Replica set MongoDB Single Instance |
| **Sunucu 2** | Ubuntu Server 24.04 LTS / RHEL 9.x | 8 Core | 12 GB | 80 GB | Kubernetes Worker |
