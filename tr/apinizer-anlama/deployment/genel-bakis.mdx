## Kurulum Ön Koşulları

Apinizer Platformu'nu kurmadan önce uygun ön koşulların mevcut olduğundan emin olunmalıdır. Kuruluma başlamadan işletim sisteminin desteklendiğini ve gerekli geliştirme platformlarına sahip olduğunu doğrulayınız.

### Desteklenen İşletim Sistemleri

Apinizer Kubernetes üzerinden çalıştığından, Kubernetes kurulu olan bütün işletim sistemlerini desteklemektedir. Ancak dokümantasyondaki sıfırdan kurulumlar Linux tabanlı işletim sistemlerine yöneliktir.

### Apinizer'ın İhtiyaç Duyduğu Bileşenler

| Adı | Versiyon/Gereksinim |
|-----|---------------------|
| Kubernetes / Tanzu / OpenShift / RKE2 | Versiyon bağımlılığı bulunmamaktadır. Apinizer imajları DockerHub üzerinden rootless sağlandığından, herhangi bir container orchestration tabanlı platformda çalıştırılabilir. |
| Mongodb | Herhangi bir versiyon bağımlılığı yoktur. Ancak kurulacak MongoDB replica set olarak yapılandırılmalıdır (Tek node dahi olsa replica set olarak yapılandırılmalıdır. Standalone Instance olarak yapılandırılmamalıdır). |
| Elasticsearch | Şu anda resmi olarak 7.9.2 versiyonu desteklenmektedir. |

## Hedef Kitle

### Ağ Uzmanları (Network Engineers)

Bu bölüm, ağ uzmanlarının şu konularda bilgi sahibi olmasını sağlar:

* **Ağ Mimarisi**: Apinizer bileşenlerinin ağ üzerindeki yerleşimi
* **Port Gereksinimleri**: Hangi portların açılması gerektiği
* **Trafik Akışı**: İstemci, Gateway ve Backend arasındaki trafik akışı
* **Load Balancing**: Yük dengeleme stratejileri ve konfigürasyonu
* **Network Segmentation**: DMZ ve LAN ayrımı
* **Firewall Kuralları**: Güvenlik duvarı konfigürasyon gereksinimleri

### Güvenlik Ekipleri (SecOps)

Güvenlik ekipleri için kritik bilgiler:

* **Güvenli Deployment Desenleri**: Best practice mimari yaklaşımları
* **DMZ Yerleşimi**: Hangi bileşenlerin DMZ'de olması gerektiği
* **LAN Koruması**: İç ağdaki hassas bileşenlerin korunması
* **TLS/SSL Sonlandırması**: Şifreleme ve sertifika yönetimi
* **Erişim Kontrolü**: Bileşenler arası iletişim güvenliği
* **Compliance**: Güvenlik standartlarına uyum

## Temel Kavramlar

### DMZ (Demilitarized Zone)

DMZ, güvenlik duvarları ile korunan, dış dünyaya açık olan ancak iç ağdan izole edilmiş ağ bölgesidir.

**DMZ'de Çalışan Bileşenler:**
* Worker modülleri (API Gateway)
* Load Balancer
* Public-facing servisler

### LAN (Local Area Network)

LAN, kurum içi güvenli ağ bölgesidir. Dış dünyadan erişilemez ve sıkı güvenlik kontrolleri altındadır.

**LAN'da Çalışan Bileşenler:**
* Manager modülü
* MongoDB veritabanı
* Elasticsearch
* Distributed Cache
* Yönetim araçları

### Deployment Modelleri

Apinizer platformu üç temel deployment modeli destekler:

1. **All-in-One (Standalone)**: Tüm bileşenlerin tek bir sunucuda çalıştığı model
2. **Distributed**: Bileşenlerin farklı sunucularda dağıtıldığı model
3. **Clustered (HA)**: Yüksek erişilebilirlik için cluster yapısında çalışan model

## Güvenlik Mimarisi

### Defense in Depth

Apinizer, katmanlı güvenlik yaklaşımı ile tasarlanmıştır:

```
Internet
   │
   ▼
┌─────────────────┐
│  Firewall (1)   │ ← İlk savunma hattı
└─────────────────┘
   │
   ▼
┌─────────────────┐
│   DMZ Zone      │ ← Worker modülleri
│  (Public API)   │
└─────────────────┘
   │
   ▼
┌─────────────────┐
│  Firewall (2)   │ ← İkinci savunma hattı
└─────────────────┘
   │
   ▼
┌─────────────────┐
│   LAN Zone      │ ← Manager, Database
│  (Internal)     │
└─────────────────┘
```

### Güvenlik Katmanları

1. **Network Layer**: Firewall kuralları ve ağ segmentasyonu
2. **Transport Layer**: TLS/SSL şifreleme
3. **Application Layer**: API güvenlik politikaları (OAuth2, JWT, mTLS)
4. **Data Layer**: Veri şifreleme ve maskeleme

## Karar Verme Kılavuzu

### Hangi Deployment Modeli Seçilmeli?

<AccordionGroup>
  <Accordion title="All-in-One (Standalone)">
    **Ne zaman kullanılır:**
    * Test ve geliştirme ortamları
    * Proof of Concept (POC) projeleri
    * Düşük trafikli uygulamalar
    * Hızlı kurulum gereksinimleri
    
    **Avantajlar:**
    * Basit kurulum ve yönetim
    * Düşük kaynak gereksinimi
    * Hızlı başlangıç
    
    **Dezavantajlar:**
    * Tek nokta arıza riski
    * Sınırlı ölçeklenebilirlik
    * Production için uygun değil
  </Accordion>
  
  <Accordion title="Distributed Deployment">
    **Ne zaman kullanılır:**
    * Orta-büyük ölçekli kurulumlar
    * Bileşenlerin ayrı yönetilmesi gereken durumlar
    * Farklı lokasyonlarda bileşenlerin çalışması
    
    **Avantajlar:**
    * Bileşen bazlı ölçeklendirme
    * Kaynak optimizasyonu
    * Esnek mimari
    
    **Dezavantajlar:**
    * Daha karmaşık yönetim
    * Ağ bağımlılığı
    * Yüksek kaynak gereksinimi
  </Accordion>
  
  <Accordion title="Clustered (HA) Deployment">
    **Ne zaman kullanılır:**
    * Production ortamları
    * Yüksek erişilebilirlik gereksinimleri
    * Yüksek trafikli uygulamalar
    * Kritik iş süreçleri
    
    **Avantajlar:**
    * Yüksek erişilebilirlik
    * Otomatik failover
    * Yatay ölçeklendirme
    * Load balancing
    
    **Dezavantajlar:**
    * Karmaşık kurulum
    * Yüksek kaynak gereksinimi
    * Cluster yönetimi gereksinimi
  </Accordion>
</AccordionGroup>

## Kurulum İşlemleri

Kurulum işlemleri farklı şekillerde yapılabilir. Mevcut bir Kubernetes Cluster'ınız varsa onu kullanabilir, Bulut ortamlarına kurabilir, sıfırdan Linux tabanlı bir işletim sistemine kurulum yapılabilir ya da Helm ile hızlıca kurulum yaptırılabilir.

Detaylar için size uygun olan yönteme göre ilgili sayfadaki yönergeleri takip edebilirsiniz:

- **Kurulum Öncesi Verilmesi Gereken Kararlar**: Kurulum öncesi alınması gereken kararlar ve hazırlıklar
- **Deployment Modelleri**: Farklı deployment modelleri ve gereksinimleri
- **Ağ Topolojisi**: DMZ ve LAN topolojisi, güvenli deployment desenleri
- **Kurulum için Erişim ve Port Gereksinimleri**: Ağ erişimleri ve port gereksinimleri
- **Apinizer Kurulumu ve Ayarları**: Apinizer'ın kurulumu ve yapılandırması
- **Kubernetes Kurulumu**: Kubernetes cluster kurulumu
- **MongoDB Kurulumu**: MongoDB kurulumu ve yapılandırması
- **Elasticsearch Kurulumu**: Elasticsearch kurulumu ve yapılandırması
- **Helm ile Kurulum**: Helm chart kullanarak hızlı kurulum
- **Offline Kurulum**: İnternet bağlantısı olmayan ortamlarda kurulum
- **Bulut (Cloud) Ortamlarına Kurulum**: AWS, Azure, GCP gibi bulut ortamlarına kurulum
