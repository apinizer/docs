Apinizer Yönetim Konsolu'nun tüm arayüzlerinde aynı yerleşim düzeni korunmaktadır. Bu tutarlılık, kullanıcıların platformu daha kolay öğrenmesini ve kullanmasını sağlar.

## Ekran Yerleşimi

Yönetim Konsolunun bütün arayüzlerinde aynı yerleşim düzeni korunmaktadır. Ekran üç ana bölümden oluşur:

```
┌─────────────────────────────────────────────────────────┐
│  [1] Hızlı Menü (Quick Menu)                          │
├──────────┬──────────────────────────────────────────────┤
│          │                                              │
│  [2]     │         [3] Çalışma Alanı                  │
│  Ana     │         (Working Area)                      │
│  Menü    │                                              │
│          │                                              │
│          │                                              │
└──────────┴──────────────────────────────────────────────┘
```

<CardGroup cols={3}>
  <Card title="1. Hızlı Menü" icon="bars">

  </Card>
  <Card title="2. Ana Menü" icon="list">

  </Card>

  <Accordion title="Proje Seçimi">
    * Seçili projenin adı görüntülenir
    * Proje değiştirme menüsü
    * Proje listesi
  </Accordion>
  
  <Accordion title="Test Araçları">
    * API test araçları
    * XPath/JSONPath test
    * Dönüşüm kodları test
  </Accordion>
  
  <Accordion title="Yönetim (Administration)">
    * Yönetim işlemlerine hızlı erişim
    * Yönetici rolü gerektirir
  </Accordion>
  
  <Accordion title="Bildirimler (Notifications)">

    * Okunmamış bildirim sayısı
    * Bildirim listesi
  </Accordion>
  
  <Accordion title="Yardım (Help)">

    * Apinizer Hakkında
  </Accordion>
  
  <Accordion title="Kişisel Ayarlar">
    * Profil ayarları

    * Dil seçimi

  </Accordion>
</AccordionGroup>

## Ana Menü (Main Menu)

Sol tarafta yer alan Ana Menü, aktif kullanıcının rolüne ve seçili projeye göre değişir:

### Geliştirici Menüsü

Bir proje seçildiğinde (Hızlı Menüde proje adı görünüyorsa) Geliştirici Menüsü görüntülenir:

<CardGroup cols={2}>
  <Card title="API Proxy'ler" icon="network-wired">
    * API Proxy oluşturma ve yönetimi
    * API Proxy listesi
    * API Proxy konfigürasyonu
  </Card>
  <Card title="Politikalar" icon="shield">
    * Policy oluşturma
    * Policy yönetimi
    * Policy grupları
  </Card>
  <Card title="Bağlantılar" icon="plug">
    * Connection yönetimi
    * Database connections
    * LDAP connections
  </Card>
  <Card title="Test" icon="flask">
    * API test araçları
    * Test senaryoları
  </Card>
</CardGroup>

### Yönetici Menüsü

Yönetici rolündeki kullanıcılar Hızlı Menüden bir öğe seçtiğinde (örneğin Genel Bakış) Yönetici Menüsü görüntülenir:

<AccordionGroup>
  <Accordion title="Genel Bakış (Overview)">
    * Platform genel bakış
    * Sistem durumu
    * İstatistikler
  </Accordion>
  
  <Accordion title="Kullanıcı Yönetimi">
    * Kullanıcı oluşturma ve yönetimi
    * Rol yönetimi
    * ACL yönetimi
  </Accordion>
  
  <Accordion title="Sistem Ayarları">
    * Sistem konfigürasyonu
    * Ortam yönetimi
    * Gateway yönetimi
  </Accordion>
  
  <Accordion title="Analitik">
    * API Analytics
    * Dashboard'lar
    * Raporlar
  </Accordion>
</AccordionGroup>

## Breadcrumb Navigasyonu

Breadcrumb (Ekmek Kırıntısı) navigasyonu, kullanıcının mevcut konumunu ve hiyerarşiyi gösterir:

```
Ana Sayfa > Proje > API Proxy > Endpoint
```

<Info>
  Breadcrumb navigasyonu, kullanıcıların sayfa hiyerarşisini anlamasına ve hızlıca üst seviyelere dönmesine yardımcı olur.
</Info>

## Menü Dinamikliği

Hızlı Menü ve Ana Menü, aktif kullanıcının sahip olduğu rol ve yetkilere göre hazırlanır:

<Warning>
  **Önemli**: Kullanıcının rolüne göre menü öğeleri görünür veya gizlenir. Yetkisi olmayan işlemlere erişim sağlanamaz.
</Warning>

### Rol Bazlı Menü Görünümü

* **Geliştirici**: Geliştirici Menüsü görüntülenir
* **Yönetici**: Yönetici Menüsü görüntülenir
* **Sadece Okuma**: Sınırlı menü öğeleri görüntülenir
