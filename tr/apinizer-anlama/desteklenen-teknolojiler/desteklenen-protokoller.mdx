Apinizer platformu, modern API mimarilerinde kullanılan çeşitli protokolleri destekler. Bu sayfa, platformun desteklediği protokoller ve özellikleri hakkında bilgi sağlar.

## HTTP/HTTPS

HTTP ve HTTPS protokolleri Apinizer'ın temel protokolleridir ve tam desteklenir.

### HTTP Versiyonları

<CardGroup cols={2}>
  <Card title="HTTP/1.1" icon="globe">
    * Tam destek
    * Keep-alive connections
    * Chunked transfer encoding
    * Compression (gzip, deflate)
  </Card>
  <Card title="HTTP/2" icon="globe">
    * Tam destek
    * Multiplexing
    * Server push
    * Header compression
  </Card>
  <Card title="HTTP/3" icon="globe">
    * Deneysel destek
    * QUIC protokolü
  </Card>
  <Card title="HTTPS" icon="shield-halved">
    * TLS/SSL sonlandırması
    * TLS 1.2+
    * TLS 1.3 desteği
    * SNI desteği
  </Card>
</CardGroup>

### HTTP Özellikleri

<AccordionGroup>
  <Accordion title="HTTP Methods">
    * GET, POST, PUT, DELETE
    * PATCH, HEAD, OPTIONS
    * Custom methods
    * Method override
  </Accordion>
  
  <Accordion title="Content Types">
    * application/json
    * application/xml
    * application/x-www-form-urlencoded
    * multipart/form-data
    * text/plain, text/html
    * Custom content types
  </Accordion>
  
  <Accordion title="HTTP Headers">
    * Standard HTTP headers
    * Custom headers
    * Header manipulation
    * CORS headers
  </Accordion>
</AccordionGroup>

## WebSocket

WebSocket protokolü gerçek zamanlı iletişim için desteklenir.

### WebSocket Özellikleri

<AccordionGroup>
  <Accordion title="WebSocket Protocol">
    * RFC 6455 uyumlu
    * Full-duplex communication
    * Low latency
    * Binary ve text frame desteği
  </Accordion>
  
  <Accordion title="WebSocket Features">
    * Connection management
    * Message routing
    * Subprotocol support
    * Ping/pong frames
  </Accordion>
  
  <Accordion title="Use Cases">
    * Real-time notifications
    * Chat applications
    * Live data streaming
    * Gaming applications
  </Accordion>
</AccordionGroup>

### WebSocket Security

<CardGroup cols={2}>

    * Secure WebSocket connections
  </Card>
  <Card title="Authentication" icon="key">

    * Header-based authentication
  </Card>
</CardGroup>

## SOAP

SOAP (Simple Object Access Protocol) web servisleri desteklenir.

### SOAP Özellikleri

<AccordionGroup>
  <Accordion title="SOAP Protocol">
    * SOAP 1.1 ve SOAP 1.2
    * XML-based messaging
    * WSDL support
  </Accordion>
  
  <Accordion title="WS-* Standards">
    * WS-Security
    * WS-Addressing
    * WS-ReliableMessaging
    * WS-Policy
  </Accordion>
  
  <Accordion title="SOAP Features">
    * SOAP to REST conversion
    * WSDL generation
    * SOAP fault handling
    * Attachment support (MTOM)
  </Accordion>
</AccordionGroup>

### SOAP Use Cases

* **Legacy System Integration**: Eski SOAP servisleriyle entegrasyon
* **Enterprise Services**: Kurumsal SOAP servisleri
* **Protocol Conversion**: SOAP'tan REST'e dönüştürme

## GraphQL

GraphQL desteği sınırlıdır ve gelecek sürümlerde genişletilecektir.

  **Not**: GraphQL desteği şu anda sınırlıdır. Tam destek için gelecek sürümleri takip edin.
</Info>

    * Field selection
  </Card>
  <Card title="Mutation Support" icon="code">
    * Basic mutation support
  </Card>
</CardGroup>

## Protokol Karşılaştırması

| Protokol | Destek Seviyesi | Kullanım Senaryosu |
|----------|----------------|-------------------|
| **HTTP/HTTPS** | ✅ Tam Destek | REST API'ler, Web servisleri |
| **REST** | ✅ Tam Destek | Modern API'ler, Microservices |
| **WebSocket** | ✅ Tam Destek | Real-time uygulamalar |
| **gRPC** | ✅ Tam Destek | Microservice communication |
| **SOAP** | ✅ Tam Destek | Legacy sistemler, Enterprise servisler |
| **GraphQL** | ⚠️ Sınırlı Destek | GraphQL API'ler (gelecek sürümlerde genişletilecek) |

## Protokol Dönüştürme

Apinizer, farklı protokoller arasında dönüştürme yapabilir:

<AccordionGroup>
  <Accordion title="SOAP to REST">
    * SOAP servislerini REST API olarak sunma
    * WSDL'den OpenAPI spec oluşturma
  </Accordion>
  
  <Accordion title="REST to gRPC">
    * REST API'leri gRPC servisleri olarak sunma
    * Protocol buffer tanımları
  </Accordion>
  
  <Accordion title="HTTP to WebSocket">
    * HTTP isteklerini WebSocket mesajlarına dönüştürme
    * Bidirectional communication
  </Accordion>
</AccordionGroup>
