#### **Veritabanı koleksiyon isimleri:**

Sayfa Adı| Veritabanındaki Koleksiyon İsmi| Açıklama  
---|---|---  
[Uygulama Logları (Application Logs)](/apinizer-dokumantasyonu/uygulama-loglari-1180211.html)| apinizer_log| apinizer_log koleksiyonunda, ilgili API Proxy'e gelen ve dönen mesajlar için akışın işletilmesi sırasında oluşan yazılımsal loglara ait kayıtlar bulunmaktadır.  
[Denetim Kayıtları (Audit Records)](/apinizer-dokumantasyonu/denetim-kayitlari-1180203.html)| audit_event | audit_event koleksiyonu, kullanıcıların sistem üzerinde yaptığı değişikliklerin kayıt edildiği koleksiyondur. Gerçekleştirilen işlemler tarih, operasyon tipi, kullanıcı, nesne ID, nesne adı ve denetim objesinin tipi parametreleriyle kayıt altına alınır.  
[ACL Denetim Kayıtları (ACL Audit Records)](/apinizer-dokumantasyonu/acl-denetim-kayitlari-39729919.html)| history_acl | history_acl koleksiyonunda, API Proxy'ye ya da API Proxy Grup'a erişim için yetki verilen ya da yetkisi alınan kullanıcı bilgileri (credential) kayıtlarının tutulduğu koleksiyondur. İşlem tarihi, işlem tipi, sebebi, hangi kullanıcı için geçerli olduğu, hangi API Proxy/API Proxy Grup ve metot için geçerli olduğu gibi bilgiler bu koleksiyonda tutulur.  
[Alarm (Alert)](/apinizer-dokumantasyonu/alarm-alert-39742481.html)| alert_history| Apinizer üzerinde belirli koşullarda action kullanılarak Alarm'lar oluşturulabilmektedir. Bu alarmların işlem saati, action içerisindeki mesaj bilgileri ve hangi obje üzerinden tetiklendiği gibi bilgiler alert_history koleksiyonunda tutulmaktadır.  
[Token İstekleri (Token Requests)](/apinizer-dokumantasyonu/token-istekleri-1179995.html)| log_tokentraffic| log_tokentraffic koleksiyonunda, Token alma istekleri için gelen ve dönen mesajların ilgili kısımlarına ait kayıtlar bulunmaktadır.  
[Anomali Dedektörü (Anomaly Detector)](/apinizer-dokumantasyonu/anomali-dedektoerue-1179985.html)| anomaly_detector_result| anomaly_detector_result koleksiyonunda, API trafiğinde Anomali bulmak için tutulan kayıtlar bulunmaktadır.  
[Çalışma Süresi Monitörü (Uptime Monitor)](/apinizer-dokumantasyonu/calisma-sueresi-monitoerue-1179987.html)| uptime_monitor_result| uptime_monitor_result koleksiyonunda, Uptime Monitoring işlemlerinin sonuçlarına ait log kayıtları bulunmaktadır.  
[İş Akışı (Task Flow)](/apinizer-dokumantasyonu/goerev-akisi-task-flow-26349543.html)| taskflow_execution_log| taskflow_execution_log koleksiyonunda, ilgili Integration Task Flow adımlarındaki eylemlerin işletilmesi sırasında oluşan loglara ait kayıtlar bulunmaktadır.  
[Rapor Üretici (Report Generator)](/apinizer-dokumantasyonu/rapor-ueretici-13272370.html)| report_configuration_result| report_configuration_result koleksiyonunda, ilgili Rapor tanımlarının çalışması sırasında oluşan yazılımsal loglara ait kayıtlar bulunmaktadır.  
[Trace (Development → API Proxies → Trace)](/apinizer-dokumantasyonu/adim-adim-izleme-sekmesi-trace-tab-13271586.html)| log_apiproxytraffic| API Proxy trafiğinde Trace açık olduğunda, log_apiproxytraffic, API trafiği ile ilgili bilgileri kaydeden bir koleksiyondur. Bu model, bir API'nin kullanımına ilişkin temel trafik bilgilerini ve hangi proxy üzerinden işlendiğine dair detayları barındırır.  
[Trace (Development → API Proxies → Trace)](/apinizer-dokumantasyonu/adim-adim-izleme-sekmesi-trace-tab-13271586.html)| log_apiproxytraffic_policy_execution| API Proxy trafiğinde Trace açık olduğunda, log_apiproxytraffic_policy_execution API trafiğinin Politika uygulama süreçleri ile ilgili bilgileri içeren bir yapıya sahiptir. Bu, politika(lar) uygulanmadan önce ve sonra ne durumda olduğuna dair bilgileri temsil eder.  
[Trace (Development → API Proxies → Trace)](/apinizer-dokumantasyonu/adim-adim-izleme-sekmesi-trace-tab-13271586.html)| log_apiproxytraffic_routing  
| API Proxy trafiğinde Trace açık olduğunda, log_apiproxytraffic_routing API trafiğinde meydana gelen yönlendirme, yeniden deneme ve failover gibi olayları temsil eden bir yapıya sahiptir. Bu yapı, bir API isteğinin hangi yönlendirmelerden geçtiğini, herhangi bir hata durumunda yeniden deneme veya failover süreçlerinin nasıl işlediği gibi bilgileri tutmaktadır.  
-| user_token_blackList| Kullanıcıların çoklu giriş yapmaları durumunda, her giriş işlemi user_token_blackList koleksiyonunda ayrı bir kayıt olarak tabloda tutulmaktadır.   
[Bildirimler (Notifications)](/apinizer-dokumantasyonu/bildirim-75808788.html)| notification_user| Kullanıcı Bildirimleri notification_user koleksiyonunda tutulur.
