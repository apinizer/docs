## Veritabanı Koleksiyon İsimleri

Aşağıdaki tablo, Apinizer'da kullanılan MongoDB koleksiyonlarını ve bunların içerdiği verileri gösterir:

| Sayfa Adı | Veritabanındaki Koleksiyon İsmi | Açıklama |
|---|---|---|
| [Uygulama Logları (Application Logs)](/tr/gelistirici/izleme-analiz/uygulama-loglari) | `apinizer_log` | API Proxy'e gelen ve dönen mesajlar için akışın işletilmesi sırasında oluşan yazılımsal loglara ait kayıtlar |
| [Denetim Kayıtları (Audit Records)](/tr/admin/denetim-uyumluluk/denetim-kayıtları) | `audit_event` | Kullanıcıların sistem üzerinde yaptığı değişikliklerin kayıt edildiği koleksiyon. Gerçekleştirilen işlemler tarih, operasyon tipi, kullanıcı, nesne ID, nesne adı ve denetim objesinin tipi parametreleriyle kayıt altına alınır |
| [ACL Denetim Kayıtları (ACL Audit Records)](/tr/admin/denetim-uyumluluk/denetim-kayıtları) | `history_acl` | API Proxy'ye ya da API Proxy Grup'a erişim için yetki verilen ya da yetkisi alınan kullanıcı bilgileri (credential) kayıtlarının tutulduğu koleksiyon. İşlem tarihi, işlem tipi, sebebi, hangi kullanıcı için geçerli olduğu, hangi API Proxy/API Proxy Grup ve metot için geçerli olduğu gibi bilgiler bu koleksiyonda tutulur |
| [Alarm (Alert)](/tr/izleme/alarm) | `alert_history` | Apinizer üzerinde belirli koşullarda action kullanılarak oluşturulan alarmların işlem saati, action içerisindeki mesaj bilgileri ve hangi obje üzerinden tetiklendiği gibi bilgiler |
| [Token İstekleri (Token Requests)](/tr/gelistirici/politikalar/token-service/genel-bakis) | `log_tokentraffic` | Token alma istekleri için gelen ve dönen mesajların ilgili kısımlarına ait kayıtlar |
| [Anomali Dedektörü (Anomaly Detector)](/tr/izleme/anomali-tespit-edici) | `anomaly_detector_result` | API trafiğinde anomali bulmak için tutulan kayıtlar |
| [Çalışma Süresi Monitörü (Uptime Monitor)](/tr/izleme/api-saglik-kontrolu) | `uptime_monitor_result` | Uptime Monitoring işlemlerinin sonuçlarına ait log kayıtları |
| [İş Akışı (Task Flow)](/tr/entegrasyon-otomasyon/api-integrator-task-flow-builder/taskflow-monitor) | `taskflow_execution_log` | Integration Task Flow adımlarındaki eylemlerin işletilmesi sırasında oluşan loglara ait kayıtlar |
| [Rapor Üretici (Report Generator)](/tr/analytic/analytics-reports/rapor-uretici) | `report_configuration_result` | Rapor tanımlarının çalışması sırasında oluşan yazılımsal loglara ait kayıtlar |
| [Trace](/tr/gelistirici/izleme-analiz/adim-adim-izleme) | `log_apiproxytraffic` | API Proxy trafiğinde Trace açık olduğunda, API trafiği ile ilgili bilgileri kaydeden koleksiyon. Bir API'nin kullanımına ilişkin temel trafik bilgilerini ve hangi proxy üzerinden işlendiğine dair detayları barındırır |
| [Trace](/tr/gelistirici/izleme-analiz/adim-adim-izleme) | `log_apiproxytraffic_policy_execution` | API Proxy trafiğinde Trace açık olduğunda, politika uygulama süreçleri ile ilgili bilgileri içeren yapı. Politika(lar) uygulanmadan önce ve sonra ne durumda olduğuna dair bilgileri temsil eder |
| [Trace](/tr/gelistirici/izleme-analiz/adim-adim-izleme) | `log_apiproxytraffic_routing` | API Proxy trafiğinde Trace açık olduğunda, yönlendirme, yeniden deneme ve failover gibi olayları temsil eden yapı. Bir API isteğinin hangi yönlendirmelerden geçtiğini, herhangi bir hata durumunda yeniden deneme veya failover süreçlerinin nasıl işlediği gibi bilgileri tutar |
| - | `user_token_blackList` | Kullanıcıların çoklu giriş yapmaları durumunda, her giriş işleminin ayrı bir kayıt olarak tutulduğu koleksiyon |
| [Bildirimler (Notifications)](/tr/entegrasyon-otomasyon/konnektorler/notification-konnektor) | `notification_user` | Kullanıcı bildirimlerinin tutulduğu koleksiyon |

<Warning>
  Bazı koleksiyonlar kritik veriler içerir. Özellikle `audit_event` ve `history_acl` koleksiyonlarının temizlenmesi önerilmez. Bu koleksiyonlara müdahale etmeden önce yedek alınması ve dikkatli değerlendirme yapılması gerekir.
</Warning>
