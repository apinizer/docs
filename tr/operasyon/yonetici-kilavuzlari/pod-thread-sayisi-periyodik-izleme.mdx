# Pod Thread Sayısının Periyodik İzlenmesi

Bu sayfa, Apinizer pod'larının thread sayılarının periyodik izlenmesi işlemini açıklamaktadır.

## Genel Bakış

Pod thread sayılarını periyodik olarak izleyerek performans sorunlarını erken tespit edebilirsiniz.

## İzleme Yöntemleri

### 1. kubectl ile İzleme

kubectl kullanarak thread sayısını izleyin:

```bash
# Pod içindeki thread sayısını görüntüleme
kubectl exec <pod-name> -- ps -eLf | wc -l

# Thread detaylarını görüntüleme
kubectl exec <pod-name> -- jstack <pid>
```

### 2. Prometheus Metrikleri

Prometheus metrikleri ile izleyin:

- `jvm_threads_current`: Mevcut thread sayısı
- `jvm_threads_peak`: Peak thread sayısı
- `jvm_threads_daemon`: Daemon thread sayısı

### 3. Otomatik İzleme Scripti

Periyodik izleme için script oluşturun:

```bash
#!/bin/bash
while true; do
  kubectl exec <pod-name> -- ps -eLf | wc -l
  sleep 60
done
```
