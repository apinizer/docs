---

## Yapılması Gerekenler

<Steps>
    <Step title="Environment Yapılandırması">
        Environment yapılandırmasını güncelleyin:

        <ul>
            <li>Gateway adreslerini yeni IP ile güncelleyin</li>
            <li>Manager adreslerini yeni IP ile güncelleyin</li>
            <li>Cache adreslerini yeni IP ile güncelleyin</li>
        </ul>

        <Warning>
            Environment yapılandırması güncellenmeden Apinizer modülleri birbirleriyle iletişim kuramaz.
        </Warning>
    </Step>

    <Step title="DNS Kayıtları">
        DNS kayıtlarını güncelleyin:

        <ul>
            <li>A kayıtlarını yeni IP adresi ile güncelleyin</li>
            <li>CNAME kayıtlarını kontrol edin ve gerekirse güncelleyin</li>
            <li>DNS cache'i temizleyin (TTL süresine bağlı olarak)</li>
        </ul>

        <Tip>
            DNS değişikliklerinin yayılması için TTL süresini bekleyin veya DNS cache'i manuel olarak temizleyin.
        </Tip>
    </Step>

    <Step title="Load Balancer Yapılandırması">
        Load balancer yapılandırmasını güncelleyin:

        <ul>
            <li>Backend server IP'lerini yeni IP adresi ile güncelleyin</li>
            <li>Health check endpoint'lerini kontrol edin</li>
            <li>Load balancer'ın yeni IP'yi tanıdığını doğrulayın</li>
        </ul>

        <Note>
            Load balancer kullanıyorsanız, backend server listesini mutlaka güncellemelisiniz.
        </Note>
    </Step>

    <Step title="Firewall Kuralları">
        Firewall kurallarını güncelleyin:

        <ul>
            <li>Yeni IP adreslerine gerekli portlar için izin verin</li>
            <li>Eski IP adreslerini firewall kurallarından kaldırın</li>
            <li>Güvenlik gruplarını kontrol edin</li>
        </ul>

        <Warning>
            Eski IP adreslerini kaldırmadan önce tüm trafiğin yeni IP'ye yönlendirildiğinden emin olun.
        </Warning>
    </Step>

    <Step title="Certificate Kontrolü">
        Certificate'leri kontrol edin:

        <ul>
            <li>Certificate'lerde IP adresi varsa güncelleyin</li>
            <li>SAN (Subject Alternative Name) kayıtlarını kontrol edin</li>
            <li>Gerekirse yeni sertifika oluşturun</li>
        </ul>

        <Info>
            SSL/TLS sertifikalarında IP adresi kullanılıyorsa, yeni IP için sertifika güncellemesi gerekebilir.
        </Info>
    </Step>
</Steps>
