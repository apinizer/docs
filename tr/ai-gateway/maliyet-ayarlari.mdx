## Genel Bakış

Apinizer AI Gateway, **model başına fiyatlandırma** ile LLM harcamalarını izler ve raporlar. Sistem:

- **Birim fiyat defteri** — Her model için USD/1 milyon token maliyeti tutar
- **Maliyet hesaplaması** — Her istek için token sayısı × birim fiyat = USD maliyeti
- **Çoklu para birimi** — Raporlarda USD → TRY (veya başka para birimlerine) dönüştürülmüş maliyetler gösterilebilir
- **Admin yönetimi** — Sistem yöneticileri fiyatlandırmayı ve para birimlerini kontrol eder

:::info

Tüm maliyetler dahili olarak **USD cinsinden** depolanır. Para birimi dönüşümü raporlama katmanında yapılır; saklanan veriler etkilenmez.

:::

## Model Fiyatlandırması

Her LLM modeli, **birim fiyat** (USD/1 milyon token) içeren bir fiyat tanımına sahiptir:

```
Model: "gpt-4o"
- Input: $2.50 per 1M tokens
- Output: $10.00 per 1M tokens

Model: "gpt-3.5-turbo"
- Input: $0.50 per 1M tokens
- Output: $1.50 per 1M tokens

Model: "claude-3-sonnet"
- Input: $3.00 per 1M tokens
- Output: $15.00 per 1M tokens
```

:::note

Giriş (input) ve çıkış (output) tokenleri ayrı ücretlendirilir. Apinizer sistem tarafından tanımlanan modellerle başlayıp, gerekirse özel modelleri ekleyebilirsiniz.

:::

### Fiyat Güncelleme

Yeni fiyatlar yayımlandığında:

<Steps>
  <Step title="Modeli Seçin">
    Tablodan düzenlemek istediğiniz modeli seçin veya arama kutusunu kullanın.
  </Step>
  
  <Step title="Giriş Fiyatını Güncelleyin">
    Yeni giriş fiyatını (USD/1 milyon token) girin.
  </Step>
  
  <Step title="Çıkış Fiyatını Güncelleyin">
    Yeni çıkış fiyatını (USD/1 milyon token) girin.
  </Step>
  
  <Step title="Kaydet">
    **Kaydet**'e tıklayın. Değişiklik hemen uygulanır ve yeni istekler yeni fiyatı kullanır.

:::warning

Fiyat değişiklikleri **geriye dönük değildir**. Önceki istekler eski fiyatla kaydedilir; gelecek istekler yeni fiyatı kullanır. Geçmiş raporlar otomatik olarak yeniden hesaplanmaz.

:::

## Para Birimselleştirme

Apinizer, raporlarda maliyetleri farklı para birimlerinde gösterebilir:

### Desteklenen Para Birimleri

- **USD** (varsayılan) — Tüm raporlar bu temelde hesaplanır
- **TRY** (Türk Lirası) — Statik kur kullanılır
- Gelecek sürümlerde diğer para birimleri eklenir

### Döviz Kurunu Ayarlama

**Yönetim** → **Sistem Ayarları** → **Para Birimi Ayarları**'nda:

| Para Birimi | Oran (USD başına) | Son Güncelleme |
|-------------|-------------------|----------------|
| USD | 1.00 | Sabit |
| TRY | 32.50 | 2026-06-15 |

:::note

Döviz kuru, sistem yöneticisi tarafından el ile (statik) girilir. İçe dönük raporlama için single-value kuru kullanılır. Gerçek zamanlı kur entegrasyonu (API) gelecek sürümde planlıdır.

:::

### Kurda Sıfır Ataması

Para birimi kuru **0 olarak ayarlanmışsa** (tanımsız):

- İlgili para birimi raporlarda gösterilmez
- Sistem **USD fallback**'a döner
- Örn. EUR kuru 0 ise, TRY raporlarında EUR sütunu boş ve USD sütunu gösterilir

### Para Biriminde Maliyet Görmek

**AI Raporları** → **Maliyet Dağılımı**'nda:

```
Raporlama Dönemi: Haziran 2026

Toplam Maliyet (USD): $12.345
Toplam Maliyet (TRY): ₺401.217  ← Otomatik dönüştürülmüş

Modeller:
- gpt-4o: $5.000 (₺162.500)
- claude-3-sonnet: $4.500 (₺146.250)
- gpt-3.5-turbo: $2.845 (₺92.467)
```

## Maliyet Raporlaması

Raporlamada maliyetler şu boyutlarda görüntülenir:

- **Model başına** — Hangi modellerin en pahalı olduğu
- **Sağlayıcı başına** — OpenAI vs. Anthropic vs. vb. maliyet karşılaştırması
- **Kişi başına** — Hangi kullanıcılar en fazla harcadığı
- **Ekip başına** — Ekip seviyesi harcama izleme
- **Dağıtım türüne göre** — Bulut vs. şirket içi maliyetlerin ayrılması

:::info

Tüm raporlar **sistem yöneticisine açık**. Proje ve ekip düzeyindeki raporlar gelen sürümde kontrol edilecektir.

:::

## Maliyet Kontrolü En İyi Uygulamaları

:::tip

1. **Fiyatlandırmayı güncel tutun**: Sağlayıcılar düzenli olarak fiyat değiştirdikçe tabloyu ayda bir güncelleyin
2. **Model başına kotalar ile birleştirin**: Pahalı modellere daha düşük kota sınırları ayarlayın
3. **Raporları düzenli kontrol edin**: Beklenmedik maliyet artışını erkenden tespit etmek için haftalık maliyet raporlarını izleyin
4. **Tahminleri yapın**: Projelendirilmiş token kullanımı × birim fiyat = beklenen aylık maliyet

:::

## Sorun Giderme

**Raporlarda Maliyet Çok Düşük veya Sıfır Gösteriyor**

- Modelin fiyatlandırma tablosunda tanımlı olup olmadığını kontrol edin
- Fiyat tanımlanmamış modeller maliyet hesaplamaz; bu modelleri ekleyin

**Para Birimi Dönüşümü Çalışmıyor**

- Para birimi kurunu kontrol edin; sıfıra ayarlanmışsa 0 döviz kuruna döner
- İstenen para biriminin kuru girin (örn. TRY → 32.50)

**Eski Maliyet Verileri Yanlış Görünüyor**

- Fiyat değişiklikleri geriye dönük uygulanmaz; yeni istekler yeni fiyata uyar
- Geçmiş maliyetler için tarihsel raporlar kontrol edin
