Elasticsearch, Apinizer'de API trafik loglarının saklandığı veritabanıdır. Apinizer'da tüm API Trafikleri varsayılan olarak Elasticsearch veritabanında tutulmaktadır.

<Info>
  İstenirse Apinizer'de API Traffic ve Analytics bölümü kapatılabilir ve Apinizer'in dahili eklentileri ile Kafka, RabbitMq, Syslog, Webhook gibi uygulamalarla yönetilebilir.
</Info>

## Elasticsearch'in Apinizer'da Kullanımı

Elasticsearch, Apinizer sisteminde şu verileri saklamak için kullanılır:

* **API Trafik Logları:** Tüm API istek ve yanıt logları
* **Audit Logları:** Sistem değişiklikleri ve kullanıcı aktiviteleri
* **Token Logları:** OAuth token kullanım logları
* **Application Logları:** Uygulama seviyesindeki log kayıtları
* **Analytics Verileri:** Performans metrikleri ve istatistikler

## Kurulum Öncesi Gereksinimler

Elasticsearch kurulumuna başlamadan önce aşağıdaki gereksinimleri kontrol edin:

* **İşletim Sistemi:** Ubuntu 2024.04 LTS veya Red Hat Enterprise Linux/Rocky Linux
* **Java:** Java 11 veya üzeri (Elasticsearch ile birlikte gelir)
* **Disk Alanı:** Yeterli disk alanı (log hacmine göre değişir)
* **Bellek:** En az 2GB RAM (4GB+ önerilir)
* **Ağ Erişimi:** Elasticsearch repository'lerine erişim
* **Lisans:** Elastic Basic License veya üzeri (X-Pack security için)

## Index Lifecycle Management (ILM)

Apinizer, Elasticsearch'te Index Lifecycle Management (ILM) kullanarak log verilerini otomatik olarak yönetir:

* **Hot Phase:** Aktif log yazma ve sorgulama
* **Warm Phase:** Daha az sıklıkla sorgulanan veriler
* **Cold Phase:** Arşivlenmiş veriler
* **Delete Phase:** Belirlenen süre sonunda otomatik silme
