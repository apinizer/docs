## Overview

### What is its Purpose?

- Creates dynamic message content from [variables](/en/concepts/core-concepts/variable) or templates and writes it to target variables.
- In Form mode, copies the source variable's value directly to the target variable; supports default value and required validation.
- In Template mode, creates complex message templates using `#{variableName}` syntax and JEXL expressions.
- Row-level conditions let you define when each row runs; offers continue-on-error option.
- Rows are processed in order; order can be changed via drag-and-drop.

### Working Principle

1. **Request Arrival**: For each request arriving at the API Gateway, if the Message Builder policy is active, processing starts.
2. **Policy Condition**: If a policy-level [condition](/en/concepts/core-concepts/conditional-policy-execution) is defined, it is checked first. If not met, the policy is not applied.
3. **Row Processing**: Rows are processed in the defined order. For each row:
   - If a row condition is defined, it is checked; if not met, the row is skipped.
   - **Form mode**: The source variable's value is written to the target variable. If empty, default value is used (if defined). If required and value is empty, an error is returned (when continueOnError is off).
   - **Template mode**: `#{variableName}` expressions in the template are resolved; JEXL is supported. The result is written to the target variable.
4. **Continue on Error**: If `continueOnError` is enabled for a row, processing continues to the next row even if that row fails. If disabled, the flow stops and an error message is returned.

## Features and Capabilities

### Basic Features

- **Two Build Modes**: Form (source→target copy) and Template (template-based creation).
- **Row-Level Conditions**: Define separate conditions for each row to control when it runs.
- **Variable Support**: [Variable](/en/concepts/core-concepts/variable) and [Environment Variable](/en/concepts/core-concepts/environment-variable) usage.
- **Drag-and-Drop Ordering**: Change row processing order via drag-and-drop.
- **Active/Passive Status Control**: Easily toggle the policy's active or passive status.
- **Condition-Based Application**: Use Query Builder to determine when the policy is applied.

### Advanced Features

- **JEXL Support**: Use JEXL expressions in Template mode for dynamic calculations.
- **Global Policy Support**: Define the policy globally and use it across multiple API Proxies.
- **Export/Import Feature**: Export and import policy configuration as ZIP.
- **Policy Group and Proxy Group Support**: Manage multiple policies in Policy Group, bulk assign to Proxy Groups.

## Usage Scenarios

| Scenario | Status | Solution (Policy Application) | Expected Behavior / Result |
|----------|--------|-------------------------------|---------------------------|
| **Header Aggregation** | Combine multiple headers from client into a single variable | Form mode: `request.header.X-User-Id` → `user.id`, `request.header.X-Tenant-Id` → `tenant.id`. Template mode: `#{user.id}_#{tenant.id}` → `correlation.id` | Correlation ID is created for use by subsequent policies. |
| **Dynamic Response Body** | Add extra info to backend response | Template mode: `{"status":"ok","requestId":"#{message.correlationId}"}` → `response.body` | Request ID is added to response body. |
| **Conditional Field Population** | Create variable only for specific paths | Row condition: `Path = /api/admin/*`. Form mode: `request.header.X-Admin-Key` → `admin.key` | Admin key variable is populated only for admin endpoints. |
| **Default Value** | Use default when source is empty | Form mode: `request.header.X-Locale` → `locale`, Default Value: `en-US`, Required: No | If header is missing, `en-US` is written. |
| **Error Tolerance** | Continue even if one row fails | Continue on Error: Yes. Define multiple rows. | Failed row is skipped, flow continues. |

## Configuring Policy Parameters

### Creating a New Message Builder Policy

<Frame caption="Message Builder Policy – Definition tab">
  <img
    src="/images/develop/politikalar/mesaj-olusturucu.png"

    width="1000"

| Step | Description / Operation |
|------|-------------------------|
| **Step 1: Go to Creation Page** | - Go to **Development → Global Settings → Global Policies → Message Builder** from the left menu.<br />- Click the **[+ Create]** button at the top right. |
| **Step 2: Enter Basic Information** | **Policy Status:** Shows Active/Passive status.<br /><br />**Name (Required):** Enter a unique name. System checks automatically.<br /><br />**Description:** Explain the purpose of the policy (max. 1000 characters). |
| **Step 3: Add Rows** | Click **Add** and select **Form** or **Template** mode.<br /><br />**Form mode:** Source Variable (required), Target Variable (required), Default Value (optional), Required (checkbox).<br /><br />**Template mode:** Template (required, `#{variable}` syntax), Target Variable (required). |
| **Step 4: Row Condition (Optional)** | Define a separate condition for each row. If not met, that row is skipped. |
| **Step 5: Continue on Error** | **Continue on Error** option per row. If enabled, processing continues to the next row even if the current row fails. |
| **Step 6: Define Condition (Optional)** | Define policy-level condition in the **Condition** tab. |
| **Step 7: Customize Error Message (Optional)** | Customize error message in the **Error Message Customization** tab. |
| **Step 8: Save** | Click the **[Save]** button. |

For descriptions of **Conditions** and **Error Message Customization** panels, you can refer to the relevant sections on the [What is Policy?](/en/concepts/core-concepts/what-is-policy) page.

### Row Parameters

| Parameter | Form Mode | Template Mode | Description |
|-----------|-----------|---------------|-------------|
| **Build Mode** | Form | Template | How the value is produced. |
| **Condition** | Optional | Optional | Condition for the row to run. |
| **Source Variable** | Required | - | Variable from which value is read. |
| **Template** | - | Required | Text with `#{variableName}` and JEXL expressions. |
| **Target Variable** | Required | Required | Variable to which result is written. |
| **Default Value** | Optional | - | Value used when source is empty. |
| **Required** | Checkbox | - | If yes, error when source is empty. |
| **Continue on Error** | Yes/No | Yes/No | Proceed to next row if this row fails. |
