## Home Page Settings

This area is the section where the content that users first encounter on the API Portal homepage is edited. Portal introduction texts, guiding descriptions, links, or visual content are defined in this tab.

During editing, a "Preview" section appears on the right side; this way, how the changes made will appear on the portal can be monitored instantly.

The content editing area is located on the left side, and HTML or plain text can be added here.

A welcome message, description, or guide text compatible with the portal's corporate identity can be created in this area.

Home Page offers two language support:

- **Portal Home (EN):** English content area
- **Portal Home (TR):** Turkish content area

If both language options are active on the Portal Settings screen, the system automatically shows the user the content in the appropriate language.

## How It Works

This area is used to manage the content of the "How It Works" page on the API Portal. It is the section where informative content such as the general operation of the system, API access process, or registration/step-by-step usage guide is presented to portal users.

The content editing area is on the left, and the **"Preview"** area showing how the changes made will appear on the portal is on the right.

Content can be prepared in HTML format; rich text elements such as headings, lists, or links are supported.

How It Works offers two language options:

- **How It Works (EN):** English content area
- **How It Works (TR):** Turkish content area

When both language options are active in the Portal Settings section, the "How It Works" page in the relevant language is automatically shown to the user.

This page is generally prepared to enhance user experience and explain the portal's usage.

## Footer Settings

This section is the area where the content to be displayed at the bottom (footer) of the API Portal is edited. Generally, contact information, legal texts, copyright statements, or corporate links are located here.

HTML or plain text can be added to the text area on the left; this way, links, logos, or short information texts can be displayed in formatted form.

The "Preview" area on the right enables instant viewing of how the changes made will appear at the bottom of the portal.

Footer offers two language support:

- **Footer (EN):** English content area
- **Footer (TR):** Turkish content area

If both TR and EN languages are active on the Portal Settings screen, the footer area in the user's preferred language is automatically shown to the user.

This way, the bottom of the portal can also be arranged in a multilingual, corporate structure.

The Footer HTML area is the HTML that appears in the portal's bottom section.

You can add social media icons, contact information, legal texts.

If embeds containing scripts (e.g., live chat) are needed, you can use `<script>` within HTML; the component is designed to execute these scripts.

Working logic:

- HTML content is made safe with DomSanitizer.
- If there is `<script>` in the content, the executeScripts() function runs and adds scripts to the DOM.
- When the setting changes, the footer is immediately updated (page refresh is sufficient).

## Saving and Preview

### Saving Process

**How to Save:**

- Click the "Save" button in the top right corner
- All tabs (Basic, Home Page, How It Works, Footer) are saved at the same time
- After successful save, changes are reflected on the portal instantly.

**Preview:**

- Real-time preview is available in Home Page, How It Works, and Footer tabs.
- While writing code on the left, you can see how it will appear live on the right.
- JavaScript codes also work in preview (thanks to apimRunScripts directive).

### Basic Fields

| Field | Description | Where It Appears on Portal |
|------|----------|-------------------------|
| Portal Home HTML | HTML content to be displayed on the homepage | catalog entry screen |
| Portal Home HTML (TR) | TR version of the same content if Turkish language is active | homepage when in TR language |
| How It Works HTML | HTML content of the "How It Works" page | entities/how-it-works/how-it-works.component |
| How It Works HTML (TR) | Turkish version | How It Works when in TR language |
| Footer HTML | Portal footer area | layout/footer/footer.component |
| Footer HTML (TR) | Turkish version | footer when in TR language |
| Logo / Logo Right / Favicon | Images used in the portal top bar | navbar and index.html |
| Primary Color | Theme primary color (e.g., plan cards, buttons) | Entire portal |

## Frequently Encountered Issues

<Accordion title="Why Are Changes Not Visible on Portal?">
- Refresh the page (Ctrl+F5 or Cmd+Shift+R)
- Clear browser cache
- Make sure you clicked the Save button
- Try in a different browser
</Accordion>

<Accordion title="Why is Favicon not updating?">
- Completely clear browser cache
- Close and reopen the browser
- Check favicon format and size
</Accordion>

<Accordion title="Why is HTML Preview Not Working?">
- There may be a syntax error in the HTML code
- Check browser console errors (F12)
- Check for unclosed tags
</Accordion>

<Accordion title="Why are Colors Different Than Expected?">
- Check hex code format (#RRGGBB)
- Clear browser cache
- There may be CSS specificity issues
</Accordion>

<Accordion title="Why are Multilingual Contents Not Visible?">
- Make sure both languages are active in Portal Settings
- Make sure you entered content for each language
- Make sure the user selected the correct language
</Accordion>
