JSON Transformer is a tool that allows you to test Jolt (JSON to JSON Transformation Language) specifications for transforming JSON data. With this tool, you can test your Jolt specs in real-time, preview inputs and outputs, and catch syntax errors.

## Overview

JSON Transformer is a testing tool located in Apinizer's Toolbox section, used for transforming JSON data. This tool is particularly used for:

* Testing JSON to JSON transformations
* Validating Jolt specs
* Previewing transformation results
* Catching syntax errors

## What is Jolt?

Jolt (JSON to JSON Transformation Language) is a language used to transform JSON data. Jolt specs define the rules used to transform source JSON structure into target JSON structure.

### Jolt Operations

Jolt provides various transformation operations:

* **shift**: Data movement and restructuring
* **default**: Default value assignment
* **remove**: Field removal
* **sort**: Sorting
* **cardinality**: Array element count control
* **modify**: Value modification

## Using JSON Transformer

### Interface Components

JSON Transformer tool consists of three main sections:

1. **Input JSON**: Source JSON data to be transformed
2. **Jolt Spec**: Jolt spec containing transformation rules
3. **Output JSON**: Transformation result

### Step 1: Input JSON Entry

Enter the source JSON data to be transformed in the Input JSON section:

```json
{
  "id": 12345,
  "name": "John Doe",
  "email": "john.doe@example.com",
  "address": {
    "street": "123 Main St",
    "city": "New York",
    "zip": "10001"
  }
}
```

#### Input Options

* **Manual Entry**: Write JSON directly into the text field
* **File Upload**: Upload a JSON file
* **Load Example**: Select from ready examples

### Step 2: Writing Jolt Spec

Write transformation rules in the Jolt Spec section:

```json
[
  {
    "operation": "shift",
    "spec": {
      "id": "userId",
      "name": "fullName",
      "email": "contact.email",
      "address": {
        "*": "contact.address.&"
      }
    }
  }
]
```

#### Jolt Spec Features

* **Syntax Highlighting**: Syntax highlighting support for Jolt spec
* **Auto-completion**: Auto-completion suggestions
* **Error Detection**: Instant syntax error detection

### Step 3: Viewing Transformation Result

The transformation result is displayed in the Output JSON section:

```json
{
  "userId": 12345,
  "fullName": "John Doe",
  "contact": {
    "email": "john.doe@example.com",
    "address": {
      "street": "123 Main St",
      "city": "New York",
      "zip": "10001"
    }
  }
}
```

## Syntax Validation

JSON Transformer tool performs syntax validation for both Input JSON and Jolt Spec:

### Input JSON Validation

* **JSON Format Check**: Valid JSON format check
* **Syntax Errors**: Syntax error highlighting
* **Error Messages**: Detailed error messages

### Jolt Spec Validation

* **Jolt Syntax Check**: Jolt spec syntax check
* **Operation Validation**: Valid operation check
* **Spec Structure Check**: Spec structure correctness check

### Error Display

Syntax errors are displayed as follows:

* **Line Number**: Line where error is located
* **Error Message**: Descriptive error message
* **Highlighting**: Highlighting of erroneous section

## Usage Scenarios

### Scenario 1: Simple Field Transformation

**Input:**
```json
{
  "firstName": "John",
  "lastName": "Doe"
}
```

**Jolt Spec:**
```json
[
  {
    "operation": "shift",
    "spec": {
      "firstName": "name.first",
      "lastName": "name.last"
    }
  }
]
```

**Output:**
```json
{
  "name": {
    "first": "John",
    "last": "Doe"
  }
}
```

### Scenario 2: Array Processing

**Input:**
```json
{
  "items": [
    {"id": 1, "name": "Item 1"},
    {"id": 2, "name": "Item 2"}
  ]
}
```

**Jolt Spec:**
```json
[
  {
    "operation": "shift",
    "spec": {
      "items": {
        "*": {
          "id": "products[&1].productId",
          "name": "products[&1].productName"
        }
      }
    }
  }
]
```

**Output:**
```json
{
  "products": [
    {
      "productId": 1,
      "productName": "Item 1"
    },
    {
      "productId": 2,
      "productName": "Item 2"
    }
  ]
}
```

### Scenario 3: Default Value Assignment

**Input:**
```json
{
  "status": "active"
}
```

**Jolt Spec:**
```json
[
  {
    "operation": "default",
    "spec": {
      "enabled": true,
      "priority": 0
    }
  },
  {
    "operation": "shift",
    "spec": {
      "status": "currentStatus",
      "enabled": "isEnabled",
      "priority": "priorityLevel"
    }
  }
]
```

**Output:**
```json
{
  "currentStatus": "active",
  "isEnabled": true,
  "priorityLevel": 0
}
```

## Tips and Best Practices

### Writing Jolt Specs

* **Step-by-Step Development**: Develop complex transformations step by step
* **Testing**: Test the result at each step
* **Documentation**: Document specs
* **Adding Comments**: Add comments for complex rules

### Performance

* **Large Data Sets**: Be careful with large JSON files
* **Complex Transformations**: Complex transformations may affect performance
* **Caching**: Cache frequently used specs

### Debugging

* **Small Examples**: Test with small examples first
* **Step-by-Step Check**: Check the result of each operation
* **Using Logs**: Use logs if necessary

## JSON Transformer and Policy Usage

Jolt specs tested in JSON Transformer can be used in JSON Transformation Policy in Apinizer:

1. Test your spec in JSON Transformer
2. Copy the tested spec
3. Paste the spec in JSON Transformation Policy
4. Add the policy to your API Proxy

:::info More Information
For detailed information about JSON Transformation Policy, see the [JSON Transformation Policy](/en/developer/policies/json-transformation) page.
:::
