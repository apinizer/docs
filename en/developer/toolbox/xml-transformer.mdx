XML Transformer is a tool that allows you to test XSLT (eXtensible Stylesheet Language Transformations) stylesheets for transforming XML data. With this tool, you can test your XSLT stylesheets in real-time, perform XML to XML transformations, manage namespaces, and debug XSLT errors.

## Overview

XML Transformer is a testing tool located in Apinizer's Toolbox section, used for transforming XML data. This tool is particularly used for:

* Testing XSLT stylesheets
* Performing XML to XML transformations
* Managing namespaces
* Debugging XSLT errors

## What is XSLT?

XSLT (eXtensible Stylesheet Language Transformations) is a language used to transform XML documents into other XML documents or other formats (HTML, text, etc.). XSLT stylesheets define the rules used to transform source XML structure into target structure.

### XSLT Basic Concepts

* **Template**: Templates that define transformation rules
* **XPath**: Expressions used to select data in XML
* **Namespace**: Managing XML namespaces
* **Output**: Format of transformation result

## Using XML Transformer

### Interface Components

XML Transformer tool consists of three main sections:

1. **Input XML**: Source XML data to be transformed
2. **XSLT Stylesheet**: XSLT stylesheet containing transformation rules
3. **Output XML**: Transformation result

### Step 1: Input XML Entry

Enter the source XML data to be transformed in the Input XML section:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<books>
  <book id="1">
    <title>XML Guide</title>
    <author>John Doe</author>
    <price>29.99</price>
  </book>
  <book id="2">
    <title>XSLT Tutorial</title>
    <author>Jane Smith</author>
    <price>39.99</price>
  </book>
</books>
```

#### Input Options

* **Manual Entry**: Write XML directly into the text field
* **File Upload**: Upload an XML file
* **Load Example**: Select from ready examples
* **Format**: Automatically format XML

### Step 2: Writing XSLT Stylesheet

Write transformation rules in the XSLT Stylesheet section:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:output method="xml" indent="yes"/>
  
  <xsl:template match="/">
    <catalog>
      <xsl:for-each select="books/book">
        <item>
          <id><xsl:value-of select="@id"/></id>
          <name><xsl:value-of select="title"/></name>
          <writer><xsl:value-of select="author"/></writer>
          <cost><xsl:value-of select="price"/></cost>
        </item>
      </xsl:for-each>
    </catalog>
  </xsl:template>
</xsl:stylesheet>
```

#### XSLT Stylesheet Features

* **Syntax Highlighting**: Syntax highlighting support for XSLT
* **Auto-completion**: Auto-completion suggestions
* **Error Detection**: Instant syntax error detection
* **Namespace Support**: Namespace support

### Step 3: Viewing Transformation Result

The transformation result is displayed in the Output XML section:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<catalog>
  <item>
    <id>1</id>
    <name>XML Guide</name>
    <writer>John Doe</writer>
    <cost>29.99</cost>
  </item>
  <item>
    <id>2</id>
    <name>XSLT Tutorial</name>
    <writer>Jane Smith</writer>
    <cost>39.99</cost>
  </item>
</catalog>
```

## Namespace Handling

XML Transformer tool offers advanced features for managing XML namespaces:

### Namespace Definition

You can define namespaces in Input XML:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<books xmlns="http://example.com/books"
       xmlns:author="http://example.com/author">
  <book id="1">
    <title>XML Guide</title>
    <author:name>John Doe</author:name>
    <price>29.99</price>
  </book>
</books>
```

### Using Namespaces in XSLT

You can use namespaces in XSLT stylesheet:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0" 
                xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
                xmlns:book="http://example.com/books"
                xmlns:author="http://example.com/author">
  
  <xsl:template match="book:books">
    <catalog>
      <xsl:for-each select="book:book">
        <item>
          <name><xsl:value-of select="book:title"/></name>
          <writer><xsl:value-of select="author:name"/></writer>
        </item>
      </xsl:for-each>
    </catalog>
  </xsl:template>
</xsl:stylesheet>
```

### Namespace Features

* **Namespace Prefix Mapping**: Mapping prefixes to namespaces
* **Default Namespace**: Defining default namespace
* **Namespace Preservation**: Preserving namespaces
* **Namespace Declaration**: Namespace declarations in output

## Usage Scenarios

### Scenario 1: Simple XML Transformation

**Input XML:**
```xml
<person>
  <firstName>John</firstName>
  <lastName>Doe</lastName>
</person>
```

**XSLT Stylesheet:**
```xml
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:template match="/">
    <user>
      <name>
        <first><xsl:value-of select="person/firstName"/></first>
        <last><xsl:value-of select="person/lastName"/></last>
      </name>
    </user>
  </xsl:template>
</xsl:stylesheet>
```

**Output XML:**
```xml
<user>
  <name>
    <first>John</first>
    <last>Doe</last>
  </name>
</user>
```

### Scenario 2: Array Processing

**Input XML:**
```xml
<products>
  <product id="1" name="Product A" price="100"/>
  <product id="2" name="Product B" price="200"/>
</products>
```

**XSLT Stylesheet:**
```xml
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:template match="/">
    <items>
      <xsl:for-each select="products/product">
        <item>
          <id><xsl:value-of select="@id"/></id>
          <title><xsl:value-of select="@name"/></title>
          <cost><xsl:value-of select="@price"/></cost>
        </item>
      </xsl:for-each>
    </items>
  </xsl:template>
</xsl:stylesheet>
```

**Output XML:**
```xml
<items>
  <item>
    <id>1</id>
    <title>Product A</title>
    <cost>100</cost>
  </item>
  <item>
    <id>2</id>
    <title>Product B</title>
    <cost>200</cost>
  </item>
</items>
```

### Scenario 3: Conditional Transformation

**Input XML:**
```xml
<orders>
  <order status="pending" amount="100"/>
  <order status="completed" amount="200"/>
</orders>
```

**XSLT Stylesheet:**
```xml
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:template match="/">
    <orderList>
      <xsl:for-each select="orders/order">
        <xsl:if test="@status='completed'">
          <completedOrder>
            <amount><xsl:value-of select="@amount"/></amount>
          </completedOrder>
        </xsl:if>
      </xsl:for-each>
    </orderList>
  </xsl:template>
</xsl:stylesheet>
```

**Output XML:**
```xml
<orderList>
  <completedOrder>
    <amount>200</amount>
  </completedOrder>
</orderList>
```

## Tips and Best Practices

### Writing XSLT

* **Template Usage**: Use templates modularly
* **XPath Optimization**: Optimize XPath expressions
* **Namespace Management**: Manage namespaces carefully
* **Output Format**: Specify output format

### Performance

* **Large XML Files**: Be careful with large XML files
* **Complex Transformations**: Complex transformations may affect performance
* **Caching**: Cache frequently used stylesheets

### Debugging

* **Small Examples**: Test with small examples first
* **Step-by-Step Check**: Check the result of each template
* **Using Logs**: Use logs if necessary
* **XPath Test**: Test XPath expressions in XPath Test tool

## XML Transformer and Policy Usage

XSLT stylesheets tested in XML Transformer can be used in XML Transformation Policy in Apinizer:

1. Test your stylesheet in XML Transformer
2. Copy the tested stylesheet
3. Paste the stylesheet in XML Transformation Policy
4. Add the policy to your API Proxy
