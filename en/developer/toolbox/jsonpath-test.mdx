## What is JSONPath?

JSONPath is a query language used to extract data from JSON data. It is the JSON equivalent of XPath. JSONPath expressions define a specific path in the JSON structure and return the data at that path.

### JSONPath Syntax Basics

JSONPath expressions consist of the following basic structures:

* **`$`**: Root element
* **`.`**: Child operator
* **`..`**: Recursive descent
* **`*`**: Wildcard
* **`[]`**: Array index or filter
* **`[start:end:step]`**: Array slice

## Using JSONPath Test

### Interface Components

JSONPath Test tool consists of three main sections:

1. **Input JSON**: JSON data on which JSONPath expression will be executed
2. **JSONPath Expression**: JSONPath expression to be tested
3. **Result**: Result returned by the expression

### Step 1: Input JSON Entry

Enter the JSON data to be tested in the Input JSON section:

```json
{
  "store": {
    "book": [
      {
        "category": "reference",
        "author": "Nigel Rees",
        "title": "Sayings of the Century",
        "price": 8.95
      },
      {

        "title": "Sword of Honour",
        "price": 12.99
      }
    ],
    "bicycle": {
      "color": "red",
      "price": 19.95
    }
  }
}
```

#### Input Options

* **Manual Entry**: Write JSON directly into the text field
* **File Upload**: Upload a JSON file
* **Load Example**: Select from ready examples
* **Format**: Automatically format JSON

### Step 2: Writing JSONPath Expression

Write the expression to be tested in the JSONPath Expression section:

```

```

#### Expression Features

* **Syntax Highlighting**: Syntax highlighting support for JSONPath expression
* **Auto-completion**: Auto-completion suggestions
* **Error Detection**: Instant syntax error detection
* **Syntax Help**: JSONPath syntax help

### Step 3: Viewing Result

The result returned by the expression is displayed in the Result section:

```json

  "Nigel Rees",
  "Evelyn Waugh"
]
```

#### Result Features

* **Highlighting**: Highlighting of results
* **Formatting**: Beautiful display in JSON format
* **Copy**: Copy result
* **Export**: Export result as file

## Testing Multiple Expressions

JSONPath Test tool allows you to test multiple expressions at once:

### Multiple Expression Entry

Write one expression per line:

```
$.store.book[*].author
$.store.book[*].title
$.store.bicycle.color
$.store.book[?(@.price < 10)]
```

### Result Display

Separate results are shown for each expression:

* **Expression 1**: `$.store.book[*].author`
  * Result: `["Nigel Rees", "Evelyn Waugh"]`
* **Expression 2**: `$.store.book[*].title`
  * Result: `["Sayings of the Century", "Sword of Honour"]`
* **Expression 3**: `$.store.bicycle.color`
  * Result: `"red"`
* **Expression 4**: `$.store.book[?(@.price < 10)]`
  * Result: `[{"category": "reference", ...}]`

## JSONPath Expression Examples

### Basic Expressions

#### Root Element

```
$
```

Returns the entire JSON document.

#### Child Access

```
$.store
```

Returns the `store` object.

#### Array Access

```
$.store.book[0]
```

Returns the first book.

#### Wildcard

```
$.store.book[*].author
```

Returns authors of all books.

### Advanced Expressions

#### Recursive Descent

```
$..author
```

Returns `author` fields at all levels.

#### Array Slice

```
$.store.book[0:2]
```

Returns the first two books.

#### Filter Expression

```
$.store.book[?(@.price < 10)]
```

Returns books with price less than 10.

#### Multiple Conditions

```
$.store.book[?(@.price < 10 && @.category == 'reference')]
```

Returns books with price less than 10 and category 'reference'.

#### Nested Access

```
$.store.book[*].author
```

Returns authors of all books.

## JSONPath Syntax Help

JSONPath Test tool provides help on JSONPath syntax:

### Syntax Reference

#### Operators

* **`$`**: Root element
* **`.`**: Child operator
* **`..`**: Recursive descent
* **`*`**: Wildcard
* **`[]`**: Array index or filter

#### Array Operations

* **`[0]`**: First element
* **`[-1]`**: Last element
* **`[0:2]`**: First two elements
* **`[*]`**: All elements

#### Filter Expressions

* **`[?(@.field == 'value')]`**: Equality check
* **`[?(@.field < 10)]`**: Comparison
* **`[?(@.field && @.other)]`**: Logical operators

1. **Syntax Help Button**: Click the syntax help button
2. **Examples**: Review ready examples
3. **Reference**: View syntax reference

## Usage Scenarios

### Scenario 1: Extracting Specific Fields

**Input JSON:**
```json
{
  "users": [
    {"id": 1, "name": "John", "email": "john@example.com"},
    {"id": 2, "name": "Jane", "email": "jane@example.com"}
  ]
}
```

**Expression**: `$.users[*].email`

**Result**: `["john@example.com", "jane@example.com"]`

### Scenario 2: Filtering

**Input JSON:**
```json
{
  "products": [
    {"id": 1, "name": "Product A", "price": 100, "stock": 10},
    {"id": 2, "name": "Product B", "price": 200, "stock": 0}
  ]
}
```

**Expression**: `$.products[?(@.stock > 0)]`

**Result**: `[{"id": 1, "name": "Product A", "price": 100, "stock": 10}]`

### Scenario 3: Nested Data Access

**Input JSON:**
```json
{
  "company": {
    "departments": [
      {
        "name": "IT",
        "employees": [
          {"name": "John", "role": "Developer"},
          {"name": "Jane", "role": "Manager"}
        ]
      }
    ]
  }
}
```

**Expression**: `$..employees[*].name`

**Result**: `["John", "Jane"]`

## Tips and Best Practices

### Writing Expressions

* **Start Simple**: Write simple expressions first
* **Develop Step by Step**: Develop complex expressions step by step
* **Test**: Test the result at each step
* **Document**: Document expressions

### Performance

* **Large Data Sets**: Be careful with large JSON files
* **Complex Expressions**: Complex expressions may affect performance
* **Indexing**: Use index as much as possible

### Debugging

* **Syntax Check**: Check expression syntax
* **Small Examples**: Test with small examples first
* **Step-by-Step Check**: Check the result at each step
