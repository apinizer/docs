## What is XPath?

XPath (XML Path Language) is a query language used to select and extract data from XML documents. XPath expressions define a specific path in the XML structure and return the data at that path.

### XPath Syntax Basics

XPath expressions consist of the following basic structures:

* **`/`**: Absolute path starting from root
* **`//`**: Search at any level
* **`.`**: Current node
* **`..`**: Parent node
* **`@`**: Attribute
* **`[]`**: Predicate (filter)
* **`*`**: Wildcard

## Using XPath Test

### Interface Components

XPath Test tool consists of three main sections:

1. **Input XML**: XML data on which XPath expression will be executed
2. **XPath Expression**: XPath expression to be tested
3. **Result**: Result returned by the expression

### Step 1: Input XML Entry

Enter the XML data to be tested in the Input XML section:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<bookstore>
  <book category="reference">
    <title lang="en">XML Guide</title>
    <author>John Doe</author>
    <price>29.99</price>
  </book>
  <book category="fiction">
    <title lang="en">Novel Story</title>
    <author>Jane Smith</author>
    <price>39.99</price>
  </book>
</bookstore>
```

#### Input Options

* **Manual Entry**: Write XML directly into the text field
* **File Upload**: Upload an XML file
* **Load Example**: Select from ready examples
* **Format**: Automatically format XML

### Step 2: Writing XPath Expression

Write the expression to be tested in the XPath Expression section:

```
/bookstore/book[@category='reference']/title
```

#### Expression Features

* **Syntax Highlighting**: Syntax highlighting support for XPath expression
* **Auto-completion**: Auto-completion suggestions
* **Error Detection**: Instant syntax error detection
* **Syntax Help**: XPath syntax help

### Step 3: Viewing Result

The result returned by the expression is displayed in the Result section:

```xml
<title lang="en">XML Guide</title>
```

#### Result Features

* **XML Format**: Results are displayed in XML format
* **Highlighting**: Matched nodes are highlighted
* **Copy**: Copy result
* **Export**: Export result as file

## Namespace-Aware Queries

XPath Test tool allows you to write namespace-aware queries:

### Namespace Definition

You can define namespaces in Input XML:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<books xmlns="http://example.com/books"
       xmlns:author="http://example.com/author">
  <book id="1">
    <title>XML Guide</title>
    <author:name>John Doe</author:name>
    <price>29.99</price>
  </book>
</books>
```

### Using Namespaces in XPath

You can use namespaces in XPath expression:

```
//*[local-name()='book']/author:name
```

or with namespace prefix:

```
//book:book/author:name
```

### Namespace Features

* **Namespace Prefix Mapping**: Mapping prefixes to namespaces
* **Default Namespace**: Default namespace support
* **Local Name**: Using local name
* **Namespace URI**: Using namespace URI

## XPath Expression Examples

### Basic Expressions

#### Path Starting from Root

```
/bookstore/book
```

Returns all `book` elements under `bookstore` starting from root.

#### Search at Any Level

```
//book
```

Returns all `book` elements at any level.

#### Attribute Selection

```
//book[@category='reference']
```

Returns all `book` elements with `category` attribute equal to `reference`.

#### Child Element Selection

```
//book/title
```

Returns `title` child elements of all `book` elements.

### Advanced Expressions

#### Using Predicates

```
//book[price > 30]
```

Returns books with price greater than 30.

#### Multiple Conditions

```
//book[@category='fiction' and price < 40]
```

Returns books with category `fiction` and price less than 40.

#### Position-Based Selection

```
//book[1]
```

Returns the first book.

```
//book[last()]
```

Returns the last book.

#### Text Content

```
//book[title='XML Guide']
```

Returns the book with title `XML Guide`.

## XPath Syntax Reference

### XPath Axes

* **`child`**: Child nodes
* **`parent`**: Parent node
* **`ancestor`**: All ancestors
* **`descendant`**: All descendants
* **`following`**: Following nodes
* **`preceding`**: Preceding nodes
* **`following-sibling`**: Following siblings
* **`preceding-sibling`**: Preceding siblings

### XPath Functions

#### String Functions

* **`string()`**: Converts node to string
* **`concat()`**: Concatenates strings
* **`substring()`**: Gets substring
* **`contains()`**: Checks content
* **`starts-with()`**: Checks start
* **`ends-with()`**: Checks end

#### Number Functions

* **`number()`**: Converts node to number
* **`sum()`**: Calculates sum
* **`round()`**: Rounds
* **`floor()`**: Rounds down
* **`ceiling()`**: Rounds up

#### Boolean Functions

* **`boolean()`**: Converts node to boolean
* **`not()`**: Logical NOT
* **`true()`**: Returns true
* **`false()`**: Returns false

#### Node Functions

* **`count()`**: Returns node count
* **`last()`**: Returns position of last node
* **`position()`**: Returns current position
* **`name()`**: Returns node name
* **`local-name()`**: Returns local name

## Usage Scenarios

### Scenario 1: Finding Elements with Specific Attribute

**Input XML:**
```xml
<products>
  <product id="1" status="active">Product A</product>
  <product id="2" status="inactive">Product B</product>
</products>
```

**Expression**: `//product[@status='active']`

**Result**: 
```xml
<product id="1" status="active">Product A</product>
```

### Scenario 2: Filtering by Text Content

**Input XML:**
```xml
<users>
  <user>John Doe</user>
  <user>Jane Smith</user>
</users>
```

**Expression**: `//user[contains(text(), 'John')]`

**Result**: 
```xml
<user>John Doe</user>
```

### Scenario 3: Nested Element Access

**Input XML:**
```xml
<company>
  <department>
    <name>IT</name>
    <employees>
      <employee>John</employee>
      <employee>Jane</employee>
    </employees>
  </department>
</company>
```

**Expression**: `//department[name='IT']/employees/employee`

**Result**: 
```xml
<employee>John</employee>
<employee>Jane</employee>
```

## Tips and Best Practices

### Writing Expressions

* **Start Simple**: Write simple expressions first
* **Develop Step by Step**: Develop complex expressions step by step
* **Test**: Test the result at each step
* **Performance**: Be careful with large XML files

### Namespace Management

* **Namespace Prefix**: Use namespace prefixes correctly
* **Local Name**: Use local name if necessary
* **Namespace URI**: Check namespace URIs

### Debugging

* **Syntax Check**: Check expression syntax
* **Small Examples**: Test with small examples first
* **Step-by-Step Check**: Check the result at each step
