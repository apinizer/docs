Test Console is a comprehensive testing tool for testing your API Proxies. With this tool, you can send requests to your API endpoints, inspect responses, save test scenarios, and define assertions.

## Overview

Test Console is a tool used to test and validate your API Proxies. This tool is particularly used for:

* Testing API endpoints
* Inspecting request/response flow
* Saving and reusing test scenarios
* Automatic validation with assertions

## Using Test Console

### Accessing Test Console

To access Test Console:

1. Go to the API Proxy detail page
2. Select an endpoint from the left side
3. Click the **Test Endpoint** button
4. Test Console opens

### Building Requests

To create a request in Test Console:

#### HTTP Method Selection

Select the HTTP method for the request:

* **GET**: Data reading
* **POST**: Creating new records
* **PUT**: Record update
* **DELETE**: Record deletion
* **PATCH**: Partial update
* **OPTIONS**: CORS preflight
* **HEAD**: Header information

#### URL Configuration

Configure the request URL:

* **Base URL**: API Proxy base URL (auto-filled)
* **Path**: Endpoint path (auto-filled)
* **Query Parameters**: Query parameters

**Example:**
```
GET https://demo.apinizer.com/apigateway/petstoreProxy/pet/findByStatus?status=pending
```

#### Headers

Add request headers:

**Default Headers:**
* `Content-Type`: `application/json`
* `Accept`: `application/json`

**Custom Headers:**
* `Authorization`: `Bearer token`
* `X-API-Key`: `your-api-key`
* `X-Request-ID`: `unique-id`

#### Request Body

Configure the request body:

**Body Formats:**
* **JSON**: Body in JSON format
* **XML**: Body in XML format
* **Form Data**: Body in form data format
* **Raw**: Body in raw text format

**JSON Example:**
```json
{
  "name": "Test Pet",
  "status": "available"
}
```

#### Path Parameters

Fill path parameters:

**Example:**
```
Path: /pet/{petId}
Parameter: petId = 123
Result: /pet/123
```

#### Query Parameters

Add query parameters:

**Example:**
```
status=pending
limit=10
offset=0
```

### Sending Request

To send the request:

1. Fill all required fields
2. Click the **Send** button
3. Wait for response

### Response Inspection

When the response arrives, you can view the following information:

#### Status Code

HTTP status code:

* **2xx**: Success
* **4xx**: Client error
* **5xx**: Server error

#### Response Headers

Response headers:

* `Content-Type`
* `Content-Length`
* `Cache-Control`
* Custom headers

#### Response Body

Response body:

* **JSON**: Display in JSON format
* **XML**: Display in XML format
* **Raw**: Display in raw text format
* **Preview**: Preview mode

#### Response Time

Response time:

* **Total Time**: Total time
* **Connection Time**: Connection time
* **Response Time**: Response time

## Test Scenarios

Test Console allows you to save and reuse test scenarios.

### Saving Test Scenario

To save a test scenario:

1. Configure the request
2. Click the **Save Test Case** button
3. Enter the test scenario name
4. Add description (optional)
5. Click the **Save** button

### Loading Test Scenario

To load a saved test scenario:

1. Click the **Load Test Case** button
2. Select one from the test scenario list
3. Scenario is loaded

### Test Scenario Management

To manage test scenarios:

* **List**: List all test scenarios
* **Edit**: Edit test scenario
* **Delete**: Delete test scenario
* **Duplicate**: Duplicate test scenario
* **Export**: Export test scenario
* **Import**: Import test scenario

## Assertions

Assertions automatically check if the response is correct.

### Assertion Types

#### Status Code Assertion

Status code assertion:

```
Expected: 200
Actual: 200
Result: ✅ Pass
```

#### Header Assertion

Header assertion:

```
Header: Content-Type
Expected: application/json
Actual: application/json
Result: ✅ Pass
```

#### Body Assertion

Body assertion:

**JSON Path Assertion:**
```
Path: $.status
Expected: available
Actual: available
Result: ✅ Pass
```

**Contains Assertion:**
```
Expected: "name"
Actual: {"name": "Test Pet", ...}
Result: ✅ Pass
```

**Regex Assertion:**
```
Pattern: ^[0-9]+$
Value: 123
Result: ✅ Pass
```

### Adding Assertion

To add an assertion:

1. Click the **Add Assertion** button
2. Select assertion type
3. Enter expected value
4. Click the **Save** button

### Assertion Results

Assertion results:

* **Pass**: Assertion successful
* **Fail**: Assertion failed
* **Error**: Assertion error

## Environment Variables

Test Console allows you to use environment variables.

### Variable Usage

You can use variables in requests:

**Example:**
```
URL: ${baseUrl}/api/users
Header: Authorization: Bearer ${token}
Body: {"userId": "${userId}"}
```

### Variable Definition

To define a variable:

1. Go to the **Variables** tab
2. Click the **Add Variable** button
3. Enter variable name and value
4. Click the **Save** button

### Predefined Variables

Predefined variables:

* `${baseUrl}`: API Proxy base URL
* `${timestamp}`: Current timestamp
* `${randomString}`: Random string
* `${randomNumber}`: Random number
