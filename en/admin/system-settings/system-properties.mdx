</Info>

## What is System Properties?

The **System Properties** screen is where you manage **platform-wide defaults** that underpin [routing and upstream](/en/concepts/core-concepts/routing-and-upstream) behavior for API Proxies in Apinizer. Starting values for connection and pool timing, retries, sticky sessions, health checks, circuit breaker, traffic mirroring, and gRPC/WebSocket channel settings—parameters that shape how traffic flows to backends—are maintained here.

**How it works:** Apinizer builds each API Proxy’s routing configuration from a starting point aligned with these system defaults. When you create a new API Proxy, fields are initialized or behave consistently with those reference values. If you change a field for a specific proxy, that value is stored on the proxy and **takes precedence for that proxy**; for fields you do not customize, the platform relies on the defaults you define in System Properties. Saving with **Apply Changes** updates the platform-wide reference defaults so new configuration and any behavior tied to those defaults stays aligned with your current values.

## System Properties configuration

The exact property set may grow by release. Major groups include:

- **Connection and pool:** Defaults for connect/read/request timeouts, pool size, idle connection handling, and validation intervals.
- **Routing and HTTP behavior:** Defaults for redirects, relative redirects, and content compression options.
- **Retry:** Defaults for fixed and exponential delay, caps, and multipliers.
- **Sticky session:** Cookie name, TTL, and security flags.
- **Health check and circuit breaker:** Intervals, timeouts, and failure/success thresholds.
- **Traffic mirroring:** Default mirrored traffic ratio.
- **General routing:** Default routing algorithm selection and the User-Agent string sent to backends.
- **gRPC and WebSocket:** Channel timeouts, size limits, retries, and reconnection-related defaults.

On-screen names and descriptions explain each property; units and allowed ranges are shown next to inputs or enforced by validation.
