---
title: "System Properties"
description: "You can centrally configure default values for API Gateway behaviors such as routing, connection pool, retry, circuit breaker, health check, sticky session, gRPC, and WebSocket settings."
---

<Info>
Access the System Properties screen by following **Management → System Settings → System Properties**. Click the **Apply Changes** button to save your changes.
</Info>

The System Properties screen allows you to manage the default values used when creating or configuring a new API Proxy on the API Gateway from a single central location. Instead of configuring each API Proxy individually, you can define standard values that will apply across the platform.

## Screen Features

| Feature | Description |
|---------|-------------|
| Categorized Structure | Settings are grouped by tab and category. You can quickly navigate to the relevant parameters. |
| Search | You can search across all properties by name or description. |
| Reset to Default | You can reset any changed property back to its factory default value with a single click. |
| Change Tracking | Unsaved changes are visually highlighted on the screen. |

## Configurable Categories

<AccordionGroup>
<Accordion title="Connection Settings">

Parameters for establishing and managing backend server connections.

| Field | Description | Default |
|-------|-------------|---------|
| Connect Timeout | Maximum time for establishing a connection to the backend server. | 30 sec |
| Read Timeout | Maximum time for waiting a response from the backend server. | 30 sec |
| Connection Request Timeout | Maximum waiting time for acquiring a connection from the connection pool. | 3 sec |
| Max Redirects | Maximum number of HTTP redirects to follow. | 50 |
| Validate After Inactivity | Time before inactive connections are validated before reuse. | 2000 ms |
| Idle Connection Timeout | Duration an idle connection is kept in the pool. | 30 sec |

</Accordion>

<Accordion title="Connection Pool">

Connection pool size configuration per API Proxy.

| Field | Description | Default |
|-------|-------------|---------|
| Custom Pool Size | Number of connections in the custom connection pool for each API Proxy. | 8 |

</Accordion>

<Accordion title="Redirect">

HTTP redirect behaviors.

| Field | Description | Default |
|-------|-------------|---------|
| Redirects Enabled | Whether HTTP redirects are automatically followed. | On |
| Relative Redirects Allowed | Whether relative redirect URLs are allowed. | On |
| Disable Content Compression | Disables automatic content decompression. | Off |

</Accordion>

<Accordion title="Retry">

Retry delay durations and exponential backoff parameters for failed requests.

| Field | Description | Default |
|-------|-------------|---------|
| Fixed Delay | Fixed waiting time between retry attempts. | 500 ms |
| Initial Delay | Initial waiting time for exponential backoff. | 500 ms |
| Max Delay | Maximum waiting time limit for exponential backoff. | 10000 ms |
| Backoff Multiplier | Exponential backoff multiplier (e.g., 2.0 means doubling). | 2.0 |

</Accordion>

<Accordion title="Sticky Session">

Sticky session cookie settings. Ensures clients are always routed to the same backend address.

| Field | Description | Default |
|-------|-------------|---------|
| Cookie Name | Name of the sticky session cookie. | APINIZER_BACKEND |
| Cookie TTL | Time-to-live of the sticky session cookie. | 1800 sec |
| Cookie Secure | Whether the sticky session cookie requires HTTPS. | Off |
| Cookie HttpOnly | Whether the sticky session cookie is accessible only via HTTP. | On |

</Accordion>

<Accordion title="Health Check">

Active health check parameters. Periodically monitors the health status of backend services.

| Field | Description | Default |
|-------|-------------|---------|
| Check Interval | Time between consecutive health check requests. | 30 sec |
| Check Timeout | Maximum waiting time for a health check response. | 5 sec |
| Fail Threshold | Number of consecutive failures required to mark a backend as unhealthy. | 3 |
| Pass Threshold | Number of consecutive successes required to mark a backend as healthy. | 2 |

</Accordion>

<Accordion title="Circuit Breaker">

Circuit breaker parameters. Temporarily stops requests to faulty backend services.

| Field | Description | Default |
|-------|-------------|---------|
| Error Window | Time window in which errors are counted to trigger the circuit breaker. | 10 sec |
| Error Threshold | Number of errors within the error window to trigger the circuit breaker. | 1 |
| Sleep Window | Duration the circuit stays open before a half-open test attempt. | 10 sec |

</Accordion>

<Accordion title="Traffic Mirror">

Traffic mirror parameters.

| Field | Description | Default |
|-------|-------------|---------|
| Mirror Percentage | Percentage of traffic to be mirrored to the mirror backend. | 10% |

</Accordion>

<Accordion title="General">

General routing parameters.

| Field | Description | Default |
|-------|-------------|---------|
| Routing Algorithm | Default load balancing algorithm used to distribute requests to backend servers. Options: Round Robin, LRU, Weighted, Random, Pick First. | Round Robin |
| User-Agent Value | Default User-Agent header value sent to backend servers. | Apinizer API Gateway |

</Accordion>

<Accordion title="gRPC Settings">

gRPC connection and channel management parameters.

| Field | Description | Default |
|-------|-------------|---------|
| Max Inbound Message Size | Maximum size of incoming gRPC messages in bytes. | 16 MB |
| Max Inbound Metadata Size | Maximum size for incoming gRPC metadata (headers) in bytes. | 8 KB |
| Keep-Alive Time | Interval for sending keep-alive pings to the backend. | 300 sec |
| Keep-Alive Timeout | Maximum waiting time for a keep-alive ping response. | 100 sec |
| Keep-Alive Without Calls | Whether keep-alive pings are sent even when there are no active RPCs. | On |
| Channel Idle Timeout | Maximum time a channel can remain idle before being closed. | 30 sec |
| Per-RPC Buffer Limit | Maximum buffer size in bytes per RPC for retry and hedging. | 1 MB |
| Max Retry Attempts | Maximum number of retry attempts for a failed gRPC call. | 5 |
| Max Hedged Attempts | Maximum number of hedged attempts for a gRPC call. | 5 |
| Max Trace Events | Maximum number of trace events kept per channel (0 to disable). | 0 |

</Accordion>

<Accordion title="WebSocket Settings">

WebSocket connection management and auto-reconnect parameters.

| Field | Description | Default |
|-------|-------------|---------|
| Connection Lost Timeout | Waiting time before a WebSocket connection is considered lost. | 60 sec |
| Reuse Address | Whether local address reuse is allowed for WebSocket connections. | On |
| Connect Timeout | Maximum time for establishing a WebSocket connection to the backend. | 10 sec |
| Max Retries | Maximum number of retry attempts for a failed WebSocket connection. | 3 |
| Retry Delay | Waiting time between WebSocket connection retry attempts. | 1 sec |
| Auto Reconnect | Whether to automatically reconnect when a WebSocket connection is lost. | Off |
| Auto Reconnect Max Retries | Maximum number of attempts for auto reconnect. | 3 |
| Auto Reconnect Retry Delay | Waiting time between auto reconnect attempts. | 1 sec |

</Accordion>
</AccordionGroup>

<Tip>
Changes made here are applied as defaults when new API Proxy configurations are created. Existing API Proxies with already customized settings are not affected by these changes.
</Tip>

<Warning>
Changes are automatically distributed to the relevant environments after saving. It is recommended to evaluate the impact of changes before modifying values in production environments.
</Warning>
