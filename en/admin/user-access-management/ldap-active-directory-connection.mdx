The fields used for LDAP connection configuration are shown in the table below.

| Field | Description |
|------|----------|
| Name<br/> | Name information of the created connection. |
| Description<br/> | A description can be written to facilitate management related to the created configuration. |
| Server Address<br/>*(Server Address)* | Connection address of the LDAP Server. Example: ldap://localhost:389 |
| Certificate Status<br/>*(Certificate Status)* | Certificate requirement for LDAP connection is specified, and if required, where user information will be retrieved from the certificate is specified.<br/>• Not Required<br/>• Trust All Certificates<br/>• Certificate Required |
| Certificate<br/>*(Certificate)* | If Certificate Status is set to "Certificate Required", the certificate required for LDAP connection is selected. See relevant documents for certificate..<br/>For certificate [see.](/en/admin/secrets-management/certificates)
||
| Username<br/>*(Username)* | Username information of the created client user. DN (Distinguished Name) must be entered for username. Environment Variable can be used. Optional field.<br/>DN must be entered for username. |
| Password<br/>*(Password)* | Password information of the created client user. Environment Variable can be used and must be marked as secret. Optional field. |
| Custom Filter<br/>*(Custom Filter)* | Additional restrictions/filters can be added to the search with this field. |
| Search Scope<br/>*(Search Scope)* | Specifies at what level the search operation will be performed on the base dn. |
| Base DN<br/>*(Base DN)* | Root where LDAP searches will be started is entered. |

<Info>
For detailed information on how Environment Variables are created and can be used, you can refer to the [Environment Variables (Environment Variables)](/en/concepts/core-concepts/environment-variable) section.
</Info>
