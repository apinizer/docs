## Instance List

In the instance management screen, you can view all defined target environments. The list provides the following information:

- **Name and Description**: Identifying information for the instance
- **Type**: Environment type (Development, Test, Staging, Production)
- **URL**: Address of the target environment
- **Health Status**: Connection health (Healthy, Unhealthy, Unknown)
- **Approval Status**: Whether approval is required, number of approvers defined
- **Notification Recipients**: Number of notification recipients defined

## Creating a New Instance

To add a new target environment, use the **New Instance** button.

### Basic Information

| Field | Description |
|-------|-------------|
| **Name** | A unique name for the instance. Only English letters and numbers are accepted, no spaces allowed |
| **Description** | Descriptive information about the instance |
| **Type** | Environment type: Development, Test, Staging, or Production |
| **Base URL** | Address of the target Apinizer environment |
| **API Token** | API key for accessing the target environment. This token is obtained from the APIops settings of the target Apinizer environment |
| **Color** | Color selection to visually distinguish the instance |
| **Active** | Active/inactive status of the instance |

{/* SCREENSHOT REQUIRED: New instance creation form — with basic information fields filled in */}

### Connection Test

After entering the instance details, you can verify access to the target environment with the **Connection Test** button. Test results:

- **Successful**: Connection to the target environment was established
- **Failed**: Connection could not be established — check the URL or API Token information

## Approval and Notification Settings

You can configure an approval mechanism for transfers to critical environments (especially Production).

### Approval Mechanism

When you activate the **Approval Required** option, all promotion operations to this instance will wait for approval from designated approvers before execution.

| Setting | Description |
|---------|-------------|
| **Approval Required** | When active, approval from designated people is required before execution |
| **Approvers** | Users who can approve execution requests |

{/* SCREENSHOT REQUIRED: Approval settings section — approval toggle active, approver list populated */}

### Notification Recipients

You can define recipients to be notified about the status of promotion operations. Notifications are sent when:

- Execution starts
- Execution completes successfully
- Execution encounters an error

You can select connectors (email, etc.) defined in the system as notification recipients.

{/* SCREENSHOT REQUIRED: Notification recipients section — at least one recipient added */}

## Health Check

The health status of defined instances is automatically monitored. Health statuses:

| Status | Meaning |
|--------|---------|
| **Healthy** | Target environment is reachable and responding |
| **Unhealthy** | Target environment is unreachable or returning errors |
| **Unknown** | Not yet checked |

You can view health status and response time information from the instance detail screen.

{/* SCREENSHOT REQUIRED: Instance detail screen — health status information and response time */}

## Deleting an Instance

When you want to delete an instance, a confirmation dialog is displayed.

<Warning>
When an instance is deleted, mappings and execution history associated with this instance may also be affected. Check related mappings before deleting.
</Warning>
