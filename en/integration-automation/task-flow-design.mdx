Task Flow Design allows you to create integration workflows visually without writing code. Using the visual designer, you can connect different connectors, configure data transformations, and build complex integration scenarios.

## Overview

Task Flow Design is the visual designer for creating Task Flows in Apinizer. It provides a drag-and-drop interface for building integration workflows using connectors, variables, and flow control elements.

## Accessing Task Flow Designer

To access Task Flow Designer:

1. Navigate to **Integration & Automation** in the main menu
2. Select **API Integrator (Task Flow Builder)**
3. Click **New Task Flow** or select an existing Task Flow to edit

## Task Flow Components

### Start Node

Every Task Flow begins with a Start node:

* **Input Variables**: Define input variables for the Task Flow
* **Input Validation**: Configure validation rules
* **Default Values**: Set default values for optional inputs

### Connector Nodes

Connector nodes represent actions to be performed:

* **Database Connector**: Execute SQL queries
* **HTTP/REST Connector**: Make HTTP requests
* **Message Queue Connectors**: Send/receive messages
* **Email Connector**: Send emails
* **FTP Connector**: File operations
* **Script Connector**: Custom logic execution
* And many more...

### Flow Control Nodes

Control the execution flow:

* **Decision Node**: Conditional branching (IF-THEN-ELSE)
* **Loop Node**: Iterate over collections
* **Parallel Node**: Execute tasks in parallel
* **Wait Node**: Delay execution
* **Error Handler**: Handle errors

### End Node

Task Flow ends with an End node:

* **Output Variables**: Define output variables
* **Response Format**: Configure response format
* **Status Codes**: Set HTTP status codes

## Creating a Task Flow

### Step 1: Create New Task Flow

1. Click **New Task Flow**
2. Enter Task Flow name and description
3. Select Task Flow type (API, Scheduled Job, or Sub-Flow)
4. Click **Create**

### Step 2: Configure Start Node

Configure the Start node:

**Input Variables:**

```json
{
  "customerId": {
    "type": "string",
    "required": true,
    "description": "Customer ID"
  },
  "orderId": {
    "type": "string",
    "required": false,
    "default": null
  }
}
```

**Input Validation:**

* Required field validation
* Type validation (string, number, boolean, object, array)
* Format validation (email, URL, regex)
* Custom validation rules

### Step 3: Add Connector Nodes

Add connector nodes to your flow:

1. Drag a connector from the connector palette
2. Drop it onto the canvas
3. Configure connector settings:
   * **Connection**: Select or create a connection
   * **Action**: Choose the action to perform
   * **Parameters**: Configure action parameters
   * **Input Mapping**: Map input variables
   * **Output Mapping**: Map output variables

**Example: Database Connector Configuration**

* **Connection**: Select database connection
* **Action**: Execute Query
* **Query**: `SELECT * FROM customers WHERE id = :customerId`
* **Input Mapping**: Map `customerId` variable
* **Output Mapping**: Store result in `customer` variable

### Step 4: Configure Flow Control

Add flow control nodes:

**Decision Node:**

* **Condition**: Define condition expression
* **True Path**: Tasks to execute if condition is true
* **False Path**: Tasks to execute if condition is false

**Example Condition:**

```javascript
customer.status === 'active' && customer.balance > 0
```

**Loop Node:**

* **Collection**: Variable containing array/collection
* **Item Variable**: Variable name for current item
* **Index Variable**: Variable name for current index
* **Loop Body**: Tasks to execute for each item

### Step 5: Configure Data Transformation

Transform data between connectors:

**Variable Mapping:**

* Map output from one connector to input of another
* Transform data format (JSON, XML, CSV)
* Extract nested values
* Combine multiple variables

**Transformation Examples:**

```javascript
// Extract nested value
fullName = customer.firstName + ' ' + customer.lastName

// Transform array
emailList = customers.map(c => c.email).join(',')

// Conditional transformation
status = order.total > 1000 ? 'premium' : 'standard'
```

### Step 6: Configure Error Handling

Add error handling:

**Error Handler Node:**

* **Error Types**: Types of errors to catch
* **Error Actions**: Actions to take on error
* **Retry Logic**: Configure retry attempts
* **Fallback**: Alternative flow on error

**Error Handling Strategies:**

* **Retry**: Retry failed task
* **Skip**: Skip failed task and continue
* **Fail**: Stop execution and return error
* **Fallback**: Execute alternative flow

### Step 7: Configure End Node

Configure the End node:

**Output Variables:**

```json
{
  "result": {
    "type": "object",
    "value": {
      "customer": "${customer}",
      "order": "${order}",
      "status": "success"
    }
  }
}
```

**Response Configuration:**

* **Status Code**: HTTP status code (200, 201, 400, etc.)
* **Response Format**: JSON, XML, or custom format
* **Headers**: Custom response headers

## Task Flow Types

### API Task Flow

Expose Task Flow as REST API:

* **HTTP Methods**: GET, POST, PUT, DELETE, PATCH
* **Path Parameters**: Define path parameters
* **Query Parameters**: Define query parameters
* **Request Body**: Configure request body schema
* **Response**: Configure response schema

### Scheduled Task Flow

Execute Task Flow on schedule:

* **Cron Expression**: Define schedule
* **Trigger Conditions**: Additional trigger conditions
* **Execution Context**: Execution environment

### Sub-Flow

Reusable Task Flow component:

* **Input Parameters**: Define input parameters
* **Output Parameters**: Define output parameters
* **Reusability**: Use in other Task Flows

## Advanced Features

### Variables and Expressions

Use variables and expressions throughout the Task Flow:

**Variable Types:**

* **String**: Text values
* **Number**: Numeric values
* **Boolean**: True/false values
* **Object**: JSON objects
* **Array**: Collections of values

**Expression Syntax:**

```javascript
// Variable reference
${variableName}

// Nested property access
${customer.address.city}

// Array access
${items[0].name}

// Expression evaluation
${customer.balance + order.total}
```

### Custom Scripts

Use Script Connector for custom logic:

* **JavaScript**: Write JavaScript code
* **Groovy**: Write Groovy scripts
* **Access Variables**: Read and write Task Flow variables
* **Return Values**: Return values to Task Flow

### Parallel Execution

Execute multiple tasks in parallel:

* **Parallel Node**: Execute multiple branches simultaneously
* **Synchronization**: Wait for all parallel tasks to complete
* **Error Handling**: Handle errors in parallel branches

## Testing Task Flows

### Test Execution

Test Task Flows before deployment:

1. Click **Test** button
2. Provide input values
3. Execute Task Flow
4. Review execution results
5. Check variable values at each step

### Debug Mode

Enable debug mode for detailed execution information:

* **Step-by-step execution**: Execute one step at a time
* **Variable inspection**: View variable values at each step
* **Execution logs**: Detailed execution logs
* **Error details**: Comprehensive error information
