Message Queue Connections allow you to connect to message queue systems including Apache Kafka, RabbitMQ, and Apache ActiveMQ.

## Overview

Message Queue Connections provide connectivity to various message queue systems for asynchronous messaging.

## Kafka Connection

### Configuration

**Kafka Properties:**

* **Bootstrap Servers**: Comma-separated list of Kafka brokers
* **Security**: SASL/SSL configuration
* **Client ID**: Unique client identifier

**Example:**

```json
{
  "name": "Kafka Production",
  "type": "kafka",
  "bootstrapServers": "kafka1.example.com:9092,kafka2.example.com:9092",
  "clientId": "apinizer-client"
}
```

## RabbitMQ Connection

### Configuration

**RabbitMQ Properties:**

* **Host**: RabbitMQ server hostname
* **Port**: RabbitMQ port (default: 5672)
* **Virtual Host**: Virtual host name
* **Username**: RabbitMQ username
* **Password**: RabbitMQ password

**Example:**

```json
{
  "name": "RabbitMQ Production",
  "type": "rabbitmq",
  "host": "rabbitmq.example.com",
  "port": 5672,
  "virtualHost": "/",
  "username": "app_user",
  "password": "secure_password"
}
```

## ActiveMQ Connection

### Configuration

**ActiveMQ Properties:**

* **Broker URL**: ActiveMQ broker URL
* **Username**: ActiveMQ username
* **Password**: ActiveMQ password

**Example:**

```json
{
  "name": "ActiveMQ Production",
  "type": "activemq",
  "brokerUrl": "tcp://activemq.example.com:61616",
  "username": "app_user",
  "password": "secure_password"
}
```
