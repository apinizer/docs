## Overview

HTTP/REST Connector provides comprehensive HTTP client capabilities, allowing you to integrate with REST APIs, web services, and HTTP-based systems.

## Configuration

### Connection Selection

Select or create an HTTP connection:

**HTTP Connection Properties:**

* **Base URL**: Base URL for API endpoints
* **Timeout**: Request timeout
* **Retry Configuration**: Retry settings
* **SSL/TLS**: SSL/TLS configuration

## HTTP Methods

### GET Request

Retrieve data from API:

**Configuration:**

* **URL**: Request URL
* **Headers**: Request headers
* **Query Parameters**: URL query parameters

**Example:**

```json
{
  "method": "GET",
  "url": "https://api.example.com/customers/${customerId}",
  "headers": {
    "Authorization": "Bearer ${token}",
    "Accept": "application/json"
  },
  "queryParameters": {
    "include": "orders",
    "limit": "10"
  }
}
```

### POST Request

Create new resources:

**Configuration:**

* **URL**: Request URL
* **Headers**: Request headers
* **Body**: Request body

**Example:**

```json
{
  "method": "POST",
  "url": "https://api.example.com/orders",
  "headers": {
    "Authorization": "Bearer ${token}",
    "Content-Type": "application/json"
  },
  "body": {
    "customerId": "${customerId}",
    "items": "${orderItems}",
    "total": "${orderTotal}"
  }
}
```

### PUT Request

Update existing resources:

**Configuration:**

* **URL**: Request URL
* **Headers**: Request headers
* **Body**: Request body

**Example:**

```json
{
  "method": "PUT",
  "url": "https://api.example.com/orders/${orderId}",
  "headers": {
    "Authorization": "Bearer ${token}",
    "Content-Type": "application/json"
  },
  "body": {
    "status": "completed"
  }
}
```

### DELETE Request

Delete resources:

**Configuration:**

* **URL**: Request URL
* **Headers**: Request headers

**Example:**

```json
{
  "method": "DELETE",
  "url": "https://api.example.com/orders/${orderId}",
  "headers": {
    "Authorization": "Bearer ${token}"
  }
}
```

## Request Configuration

### Headers

Configure request headers:

**Common Headers:**

* **Authorization**: Authentication token
* **Content-Type**: Request body content type
* **Accept**: Acceptable response content types
* **User-Agent**: Client user agent
* **Custom Headers**: Custom header values

**Example:**

```json
{
  "headers": {
    "Authorization": "Bearer ${token}",
    "Content-Type": "application/json",
    "Accept": "application/json",
    "X-Request-ID": "${requestId}",
    "X-API-Key": "${apiKey}"
  }
}
```

### Query Parameters

Add query parameters to URL:

**Example:**

```json
{
  "queryParameters": {
    "page": "1",
    "limit": "10",
    "sort": "created_date",
    "order": "desc"
  }
}
```

### Request Body

Configure request body:

**JSON Body:**

```json
{
  "body": {
    "name": "${customerName}",
    "email": "${customerEmail}",
    "status": "active"
  }
}
```

**Form Data:**

```json
{
  "body": {
    "name": "${customerName}",
    "email": "${customerEmail}"
  },
  "contentType": "application/x-www-form-urlencoded"
}
```

**XML Body:**

```json
{
  "body": "<customer><name>${customerName}</name></customer>",
  "contentType": "application/xml"
}
```

## Authentication

### Bearer Token

Use Bearer token authentication:

**Configuration:**

```json
{
  "authentication": {
    "type": "bearer",
    "token": "${accessToken}"
  }
}
```

### Basic Authentication

Use Basic authentication:

**Configuration:**

```json
{
  "authentication": {
    "type": "basic",
    "username": "${username}",
    "password": "${password}"
  }
}
```

### API Key

Use API key authentication:

**Configuration:**

```json
{
  "authentication": {
    "type": "apikey",
    "key": "${apiKey}",
    "location": "header",
    "name": "X-API-Key"
  }
}
```

### OAuth 2.0

Use OAuth 2.0 authentication:

**Configuration:**

```json
{
  "authentication": {
    "type": "oauth2",
    "tokenUrl": "https://auth.example.com/token",
    "clientId": "${clientId}",
    "clientSecret": "${clientSecret}",
    "grantType": "client_credentials"
  }
}
```

## Response Handling

### Response Mapping

Map response data to variables:

**Response Structure:**

```json
{
  "statusCode": "${response.statusCode}",
  "headers": "${response.headers}",
  "body": "${response.body}"
}
```

**Body Mapping:**

```json
{
  "customer": "${response.body.customer}",
  "orders": "${response.body.orders}",
  "total": "${response.body.total}"
}
```

### Error Handling

Handle HTTP errors:

**Error Status Codes:**

* **4xx**: Client errors (400, 401, 403, 404, etc.)
* **5xx**: Server errors (500, 502, 503, etc.)

**Error Handling:**

```javascript
if (response.statusCode >= 400) {
  // Handle error
  error = {
    "statusCode": response.statusCode,
    "message": response.body.message,
    "details": response.body
  };
}
```

## Retry Logic

### Retry Configuration

Configure retry logic:

**Retry Settings:**

* **Max Retries**: Maximum retry attempts
* **Retry Interval**: Time between retries
* **Retry on Status Codes**: Status codes to retry on
* **Retry Strategy**: Retry strategy (fixed, exponential backoff)

**Example:**

```json
{
  "retry": {
    "maxAttempts": 3,
    "interval": 1000,
    "retryOnStatusCodes": [500, 502, 503, 504],
    "strategy": "exponentialBackoff"
  }
}
```

## Common Patterns

### API Call Pattern

Call external API:

```
Task Flow:
  ├─ Prepare Request Data
  ├─ Call API (HTTP/REST Connector)
  └─ Process Response
```

### Chained API Calls

Make multiple API calls:

```
Task Flow:
  ├─ Call API 1 (HTTP/REST Connector)
  ├─ Process Response 1
  ├─ Call API 2 (HTTP/REST Connector)
  └─ Process Response 2
```

### Parallel API Calls

Make parallel API calls:

```
Task Flow:
  ├─ Parallel Branch
  │   ├─ Call API 1 (HTTP/REST Connector)
  │   └─ Call API 2 (HTTP/REST Connector)
  └─ Merge Results
```
