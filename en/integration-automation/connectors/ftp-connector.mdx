FTP Connector enables file operations on FTP and SFTP servers. It supports file upload, download, listing, and other file management operations.

## Overview

FTP Connector provides connectivity to FTP and SFTP servers, allowing you to transfer files, list directories, and perform file operations as part of Task Flows.

## Configuration

### Connection Selection

Select or create an FTP/SFTP connection:

**FTP Connection Properties:**

* **Host**: FTP server hostname or IP
* **Port**: FTP server port (21 for FTP, 22 for SFTP)
* **Username**: FTP username
* **Password**: FTP password
* **Protocol**: FTP or SFTP
* **Security**: SSL/TLS configuration (for FTPS)

## File Operations

### Upload File

Upload files to FTP/SFTP server:

**Configuration:**

* **Local File Path**: Path to local file
* **Remote File Path**: Destination path on server
* **Overwrite**: Whether to overwrite existing files
* **Mode**: Transfer mode (ASCII or Binary)

**Example:**

```json
{
  "operation": "upload",
  "localFilePath": "${localFilePath}",
  "remoteFilePath": "/uploads/${fileName}",
  "overwrite": true,
  "mode": "binary"
}
```

### Download File

Download files from FTP/SFTP server:

**Configuration:**

* **Remote File Path**: Path to file on server
* **Local File Path**: Destination path on local system
* **Overwrite**: Whether to overwrite existing files
* **Mode**: Transfer mode (ASCII or Binary)

**Example:**

```json
{
  "operation": "download",
  "remoteFilePath": "/data/${fileName}",
  "localFilePath": "${downloadPath}/${fileName}",
  "overwrite": true,
  "mode": "binary"
}
```

### List Files

List files in a directory:

**Configuration:**

* **Remote Directory**: Directory path on server
* **Pattern**: File pattern filter (optional)
* **Recursive**: Whether to list recursively

**Example:**

```json
{
  "operation": "list",
  "remoteDirectory": "/data",
  "pattern": "*.csv",
  "recursive": false
}
```

**Result Format:**

```json
{
  "files": [
    {
      "name": "file1.csv",
      "size": 1024,
      "modified": "2024-01-15T10:30:00Z",
      "type": "file"
    },
    {
      "name": "file2.csv",
      "size": 2048,
      "modified": "2024-01-15T11:00:00Z",
      "type": "file"
    }
  ]
}
```

### Delete File

Delete files from FTP/SFTP server:

**Configuration:**

* **Remote File Path**: Path to file to delete

**Example:**

```json
{
  "operation": "delete",
  "remoteFilePath": "/data/${fileName}"
}
```

### Create Directory

Create directories on FTP/SFTP server:

**Configuration:**

* **Remote Directory Path**: Path to directory to create

**Example:**

```json
{
  "operation": "createDirectory",
  "remoteDirectoryPath": "/data/${newDirectory}"
}
```

### Delete Directory

Delete directories from FTP/SFTP server:

**Configuration:**

* **Remote Directory Path**: Path to directory to delete
* **Recursive**: Whether to delete recursively

**Example:**

```json
{
  "operation": "deleteDirectory",
  "remoteDirectoryPath": "/data/${directory}",
  "recursive": true
}
```

## File Patterns

### Pattern Matching

Use patterns to filter files:

**Pattern Examples:**

* `*.csv` - All CSV files
* `data_*.txt` - Files starting with "data_"
* `*.log` - All log files
* `2024-*.csv` - CSV files starting with "2024-"

**Usage:**

```json
{
  "operation": "list",
  "remoteDirectory": "/data",
  "pattern": "*.csv"
}
```

## Transfer Modes

### Binary Mode

Use binary mode for binary files:

**Use Cases:**

* Images, PDFs, executables
* Compressed files
* Database files

**Configuration:**

```json
{
  "mode": "binary"
}
```

### ASCII Mode

Use ASCII mode for text files:

**Use Cases:**

* Text files
* CSV files
* Configuration files

**Configuration:**

```json
{
  "mode": "ascii"
}
```

## Error Handling

### File Operation Errors

Handle file operation errors:

**Error Types:**

* **Connection Error**: Failed to connect to FTP server
* **Authentication Error**: Invalid credentials
* **File Not Found**: File doesn't exist
* **Permission Error**: Insufficient permissions
* **Timeout Error**: Operation timeout

**Error Handling:**

* **Retry Logic**: Retry on transient errors
* **Error Logging**: Log file operation errors
* **Fallback**: Use alternative file source

## Common Patterns

### File Processing Pattern

Download, process, and upload files:

```
Task Flow:
  ├─ Download File (FTP Connector)
  ├─ Process File (Script Connector)
  └─ Upload Processed File (FTP Connector)
```

### File Monitoring Pattern

Monitor directory for new files:

```
Task Flow:
  ├─ List Files (FTP Connector)
  ├─ Filter New Files
  └─ Process Each File
```

### File Archive Pattern

Move files to archive directory:

```
Task Flow:
  ├─ List Files (FTP Connector)
  ├─ Process Files
  └─ Move to Archive (FTP Connector)
```
