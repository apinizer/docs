Email Connector enables sending emails through SMTP servers within Task Flows. It supports HTML and plain text emails, attachments, and various email features.

## Overview

Email Connector provides email sending capabilities, allowing you to send notifications, alerts, and other email communications as part of your integration workflows.

## Configuration

### Connection Selection

Select or create an SMTP connection:

**SMTP Connection Properties:**

* **SMTP Host**: SMTP server hostname
* **SMTP Port**: SMTP server port (25, 465, 587)
* **Username**: SMTP username
* **Password**: SMTP password
* **Security**: SSL/TLS configuration
* **Authentication**: Authentication method

## Send Email

### Basic Configuration

**Email Configuration:**

* **From**: Sender email address
* **To**: Recipient email addresses (comma-separated)
* **CC**: CC recipients (optional)
* **BCC**: BCC recipients (optional)
* **Subject**: Email subject
* **Body**: Email body content

**Example:**

```json
{
  "from": "noreply@example.com",
  "to": "customer@example.com",
  "subject": "Order Confirmation",
  "body": "Your order has been confirmed."
}
```

### HTML Email

Send HTML formatted emails:

**HTML Body:**

```html
<html>
  <body>
    <h1>Order Confirmation</h1>
    <p>Dear ${customerName},</p>
    <p>Your order #${orderId} has been confirmed.</p>
    <p>Total: $${orderTotal}</p>
  </body>
</html>
```

**Configuration:**

```json
{
  "from": "noreply@example.com",
  "to": "${customerEmail}",
  "subject": "Order Confirmation - Order #${orderId}",
  "body": "${htmlBody}",
  "contentType": "text/html"
}
```

### Plain Text Email

Send plain text emails:

**Configuration:**

```json
{
  "from": "noreply@example.com",
  "to": "${customerEmail}",
  "subject": "Order Confirmation",
  "body": "Your order has been confirmed.",
  "contentType": "text/plain"
}
```

### Email with Attachments

Attach files to emails:

**Attachment Configuration:**

* **File Path**: Path to file to attach
* **File Name**: Name for attached file
* **Content Type**: MIME type of attachment
* **Inline**: Whether attachment is inline

**Example:**

```json
{
  "from": "noreply@example.com",
  "to": "${customerEmail}",
  "subject": "Order Invoice",
  "body": "Please find your invoice attached.",
  "attachments": [
    {
      "filePath": "${invoiceFilePath}",
      "fileName": "invoice-${orderId}.pdf",
      "contentType": "application/pdf"
    }
  ]
}
```

### Multiple Attachments

Attach multiple files:

```json
{
  "attachments": [
    {
      "filePath": "${invoicePath}",
      "fileName": "invoice.pdf"
    },
    {
      "filePath": "${receiptPath}",
      "fileName": "receipt.pdf"
    }
  ]
}
```

## Email Templates

### Using Templates

Use email templates for consistent formatting:

**Template Variables:**

```html
<html>
  <body>
    <h1>Order Confirmation</h1>
    <p>Dear ${customerName},</p>
    <p>Your order #${orderId} has been confirmed.</p>
    <table>
      <tr>
        <th>Item</th>
        <th>Quantity</th>
        <th>Price</th>
      </tr>
      ${orderItems}
    </table>
    <p>Total: $${orderTotal}</p>
  </body>
</html>
```

**Template Usage:**

```json
{
  "template": "order-confirmation",
  "templateVariables": {
    "customerName": "${customerName}",
    "orderId": "${orderId}",
    "orderItems": "${orderItemsHtml}",
    "orderTotal": "${orderTotal}"
  }
}
```

## Dynamic Content

### Variable Substitution

Use Task Flow variables in email content:

**Subject with Variables:**

```
Order Confirmation - Order #${orderId}
```

**Body with Variables:**

```
Dear ${customerName},

Your order #${orderId} has been confirmed.
Total amount: $${orderTotal}

Thank you for your purchase!
```

### Conditional Content

Include conditional content:

```html
${#if isPremium}
  <p>As a premium customer, you receive 10% discount!</p>
${/if}
```

## Error Handling

### Email Sending Errors

Handle email sending errors:

**Error Types:**

* **SMTP Connection Error**: Failed to connect to SMTP server
* **Authentication Error**: Invalid credentials
* **Invalid Address**: Invalid email address
* **Timeout Error**: SMTP operation timeout

**Error Handling:**

* **Retry Logic**: Retry on transient errors
* **Error Logging**: Log email sending errors
* **Fallback**: Use alternative notification method

## Common Patterns

### Notification Pattern

Send notification emails:

```
Task Flow:
  ├─ Process Order
  ├─ Format Email
  └─ Send Email (Email Connector)
```

### Alert Pattern

Send alert emails:

```
Task Flow:
  ├─ Detect Error
  ├─ Format Alert
  └─ Send Alert Email (Email Connector)
```

### Report Pattern

Send report emails with attachments:

```
Task Flow:
  ├─ Generate Report
  ├─ Create PDF
  └─ Send Report Email (Email Connector)
```
