Job Definition allows you to create and configure scheduled jobs that execute Task Flows, scripts, or API calls on a schedule.

## Overview

Scheduled Jobs enable automated execution of Task Flows, scripts, or API calls based on time-based schedules or event triggers.

## Creating a Job

### Basic Configuration

**Job Properties:**

* **Job Name**: Unique name for the job
* **Description**: Job description
* **Enabled**: Whether job is enabled
* **Job Type**: Type of job (Task Flow, Script, API Call)

### Job Types

Select job type:

* **Task Flow**: Execute a Task Flow
* **Script**: Execute a script
* **API Call**: Make an API call

### Schedule Configuration

Configure execution schedule:

**Schedule Options:**

* **Cron Expression**: Use cron expression for scheduling
* **Fixed Rate**: Execute at fixed intervals
* **Fixed Delay**: Execute with fixed delay after completion
* **One-time**: Execute once at specified time

## Task Flow Job

Execute a Task Flow on schedule:

**Configuration:**

* **Task Flow**: Select Task Flow to execute
* **Input Variables**: Provide input variables
* **Timeout**: Execution timeout

**Example:**

```json
{
  "jobType": "taskflow",
  "taskFlow": "ProcessOrders",
  "inputVariables": {
    "batchSize": 100
  },
  "timeout": 300000
}
```

## Script Job

Execute a script on schedule:

**Configuration:**

* **Script**: Script content or script file
* **Script Type**: JavaScript or Groovy
* **Variables**: Script variables

**Example:**

```json
{
  "jobType": "script",
  "script": "console.log('Job executed');",
  "scriptType": "javascript"
}
```

## API Call Job

Make an API call on schedule:

**Configuration:**

* **URL**: API endpoint URL
* **Method**: HTTP method
* **Headers**: Request headers
* **Body**: Request body

**Example:**

```json
{
  "jobType": "apicall",
  "url": "https://api.example.com/sync",
  "method": "POST",
  "headers": {
    "Authorization": "Bearer ${token}"
  }
}
```

## Job Status

Monitor job status:

**Status Values:**

* **Scheduled**: Job is scheduled but not running
* **Running**: Job is currently executing
* **Completed**: Job completed successfully
* **Failed**: Job execution failed
* **Cancelled**: Job was cancelled

## Job History

View job execution history:

**History Information:**

* **Execution Time**: When job executed
* **Duration**: Execution duration
* **Status**: Execution status
* **Output**: Job output
* **Errors**: Error information if failed
