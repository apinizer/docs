Scheduled Jobs support different job types, each suited for different use cases. Understanding job types helps you choose the right one for your needs.

## Overview

Apinizer supports three main job types: Task Flow jobs, Script jobs, and API Call jobs.

## Task Flow Jobs

Execute Task Flows on a schedule.

### Use Cases

* **Data Synchronization**: Sync data between systems
* **Report Generation**: Generate and send reports
* **Data Processing**: Process batches of data
* **Integration Tasks**: Perform integration operations

### Configuration

**Task Flow Selection:**

* Select Task Flow to execute
* Provide input variables
* Set execution timeout

**Example:**

```json
{
  "jobType": "taskflow",
  "taskFlow": "SyncOrders",
  "inputVariables": {
    "batchSize": 1000,
    "sourceSystem": "ERP"
  }
}
```

### Advantages

* **Reusability**: Reuse existing Task Flows
* **Complex Logic**: Support for complex workflows
* **Visual Design**: Visual flow design
* **Monitoring**: Built-in monitoring capabilities

## Script Jobs

Execute scripts on a schedule.

### Use Cases

* **Simple Tasks**: Simple automation tasks
* **Data Transformation**: Transform data
* **Custom Logic**: Custom business logic
* **Quick Automation**: Quick automation scripts

### Configuration

**Script Configuration:**

* Script content or script file
* Script language (JavaScript/Groovy)
* Script variables

**Example:**

```json
{
  "jobType": "script",
  "script": "console.log('Processing data');",
  "scriptType": "javascript",
  "variables": {
    "data": "${data}"
  }
}
```

### Advantages

* **Simplicity**: Simple to create and maintain
* **Flexibility**: Flexible scripting capabilities
* **Quick Development**: Quick to develop

## API Call Jobs

Make API calls on a schedule.

### Use Cases

* **API Polling**: Poll APIs for updates
* **Webhook Triggers**: Trigger webhooks
* **Health Checks**: Perform health checks
* **Data Fetching**: Fetch data from APIs

### Configuration

**API Call Configuration:**

* API endpoint URL
* HTTP method
* Request headers and body
* Response handling

**Example:**

```json
{
  "jobType": "apicall",
  "url": "https://api.example.com/sync",
  "method": "POST",
  "headers": {
    "Authorization": "Bearer ${token}"
  },
  "body": {
    "action": "sync"
  }
}
```

### Advantages

* **Simplicity**: Simple API calls
* **Direct Integration**: Direct API integration
* **No Code**: No code required

## Choosing Job Type

### Use Task Flow Jobs When

* Complex workflows are needed
* Multiple connectors are required
* Visual design is preferred
* Reusability is important

### Use Script Jobs When

* Simple automation is needed
* Custom logic is required
* Quick development is needed
* No connectors are needed

### Use API Call Jobs When

* Simple API calls are needed
* No complex logic is required
* Direct API integration is sufficient
