title: "System Default Limits"
description: "Default system limits and thresholds for the Apinizer platform. These limits are set to ensure stable and secure operation. Most can be configured through the management panel or environment variables."
---

<Info>
The values listed on this page are the **default** limits of the Apinizer platform. Limits marked as **System Property** can be changed from the **Management Panel > Overview > System Properties** screen. Changes are automatically distributed to all running Worker servers. Other values can be managed via environment variables or YAML configuration.
</Info>

## API Traffic and Request Sizes

| Limit | Default | Configuration | Description |
|-------|---------|---------------|-------------|
| File upload (multipart) | 100 MB | Environment Variable | Maximum file and request size per upload |
| Pre-allocated request/response memory | 10 MB | Environment Variable | Body buffer pre-allocation upper limit |
| Maximum HTTP entity size | Unlimited | Environment Variable | Gateway can proxy requests of any size |
| WebSocket buffer size | 64 KB | Environment Variable | WebSocket message buffer |
| WebSocket idle timeout | 60 seconds | Environment Variable | Connection close time after inactivity |
