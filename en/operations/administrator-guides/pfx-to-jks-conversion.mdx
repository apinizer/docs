---
title: "Converting PFX File to JKS File"
description: "Converts certificates in PFX (PKCS#12) format to JKS (Java KeyStore) format and makes them ready for use in Apinizer modules."
---

When Apinizer Worker applications are desired to be opened with HTTPS, it supports pfx and jks certificates. However, organizations may still want to load pfx certificates in jks format.

This document explains how this conversion can be done with the help of "Keystore Explorer" program to help with this situation.

## **Converting Certificate Files to jks Format with Keystore Explorer (KSE)**

pfx file is opened with KSE.

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/01-pfx-open-kse.png"
  alt="Opening PFX File with KSE"
  width="800"
/>

Right-click **export key pair** is selected for the file inside.

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/02-export-keypair.jpg"
  alt="Export Key Pair"
  width="800"
/>

Keypair is exported in **pkcs12** format with **.p12** extension.

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/03-new-jks-file.png"
  alt="Export in PKCS12 Format"
  width="400"
/>

**JKS** type is selected with new file creation option.

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/04-import-keypair.jpg"
  alt="JKS Type Selection"
  width="800"
/>

The .p12 file exported just now is imported to the created file with import keypair option:

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/05-import-keypair-2.png"
  alt="Import Keypair"
  width="400"
/>

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/06-import-keypair-3.jpg"
  alt="Import Keypair 2"
  width="800"
/>

Optional alias can be entered. The point to pay attention to is that if there is another certificate with the same alias, it **should not be overwritten**.

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/07-alias-input.png"
  alt="Alias Entry"
  width="400"
/>

New password is given, the point to pay attention to is that the given password and jks password should be the same, if there is no password, jks should also have no password.

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/08-password-input.png"
  alt="Password Entry"
  width="400"
/>

Then right-click on the imported keypair and select export certificate chain option:

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/09-export-certificate-chain.jpg"
  alt="Export Certificate Chain"
  width="800"
/>

When exporting, it is necessary to select **Entire Chain** and **X.509** options

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/10-export-options.jpg"
  alt="Export Options"
  width="800"
/>

Then this exported certificate is imported to jks. Optional alias can be entered. The point to pay attention to is that if there is a certificate with the same alias, it **should not be overwritten.**

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/11-import-certificate.jpg"
  alt="Import Certificate"
  width="800"
/>

The final state should be as follows:

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/12-final-state.png"
  alt="Final State"
  width="800"
/>

File is saved and closed with the password given to keypair.

<img
  src="/images/tutorials/nasil-yapilir-kilavuzlari/yonetici-kilavuzlari/pfx-jks-donusturme/13-save-file.jpg"
  alt="Saving File"
  width="800"
/>
