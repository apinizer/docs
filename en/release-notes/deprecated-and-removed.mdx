The Apinizer platform is continuously evolving and new features are being added. During this process, some old features are being deprecated or completely removed. This page allows you to track these changes and perform necessary migration operations.

## Deprecated Features

Deprecated features are features that are still working but will be removed in future versions. You can continue to use these features, but it is recommended to switch to alternative solutions as soon as possible.

The following features are marked as deprecated:

#### API Endpoints

* **Old Endpoint**: `/api/v1/legacy/endpoint`
  * **Deprecated Version**: 2024.2.0
  * **Removal Version**: 2026.1.0
  * **Alternative**: `/api/v2/new/endpoint`
  * **Migration Guide**: [Migration Guide](#migration-guide)

#### Configuration Settings

* **Old Setting**: `legacy.config.enabled`
  * **Deprecated Version**: 2024.3.0
  * **Removal Version**: 2026.2.0
  * **Alternative**: `new.config.enabled`
  * **Migration Guide**: [Migration Guide](#migration-guide)

#### Policies

* **Old Policy**: Legacy Rate Limiting Policy
  * **Deprecated Version**: 2024.1.0
  * **Removal Version**: 2025.4.0
  * **Alternative**: Enhanced Rate Limiting Policy
  * **Migration Guide**: [Migration Guide](#migration-guide)

## Removal Timeline

The Removal Timeline below shows when Deprecated features will be removed:

### 2025

* **2025.2.0** (February 2025)
  * Legacy Authentication Method removed
  * Old API Gateway configuration removed

* **2025.4.0** (April 2025)
  * Legacy Rate Limiting Policy will be removed
  * Old cache configuration will be removed

### 2026

* **2026.1.0** (January 2026)
  * `/api/v1/legacy/*` endpoints will be removed
  * Old monitoring APIs will be removed

* **2026.2.0** (February 2026)
  * `legacy.config.*` settings will be removed
  * Old log formats will be removed

## Migration Guide

You can use the following guide to transition to alternative solutions for Deprecated features.

### API Endpoint Migration

**Old Usage:**
```bash
GET /api/v1/legacy/endpoint
```

**New Usage:**
```bash
GET /api/v2/new/endpoint
```

**Migration Steps:**
1. Test the new endpoint
2. Find old endpoint usages
3. Transition to new endpoint
4. Remove old endpoint

### Configuration Migration

**Old Configuration:**
```yaml
legacy:
  config:
    enabled: true
```

**New Configuration:**
```yaml
new:
  config:
    enabled: true
```

**Migration Steps:**
1. Use new configuration format
2. Remove old configuration
3. Test configuration

### Policy Migration

**Old Policy:**
* Legacy Rate Limiting Policy usage

**New Policy:**
* Enhanced Rate Limiting Policy usage

**Migration Steps:**
1. Create new policy
2. Disable old policy
3. Test new policy
4. Remove old policy

## Alternative Solutions

Alternative solutions for Deprecated features:

### Cache Management

**Old Solution**: Old cache configuration
**New Solution**: Hazelcast distributed cache usage

**Advantages:**
* Better performance
* Distributed architecture support
* Automatic failover
* Better scalability

### Monitoring

**Old Solution**: Old monitoring APIs
**New Solution**: New monitoring APIs and dashboard

**Advantages:**
* More detailed metrics
* Real-time monitoring
* Better visualization
* Alert management

### Authentication

**Old Solution**: Legacy Authentication Method
**New Solution**: OAuth 2.0 and JWT-based authentication

**Advantages:**
* More secure
* Standard protocols
* Better token management
* Refresh token support
