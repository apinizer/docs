---

import {Steps, Step} from '@site/src/components/Steps';
import {Card, CardGroup} from '@site/src/components/Card';

<Steps>
  <Step title="Add an LLM Provider Connection">
    Connect Apinizer to at least one LLM provider — cloud (OpenAI, Anthropic, Azure, Vertex, Bedrock) or self-hosted (vLLM, Ollama, Hugging Face TGI). The connection stores the provider's credentials encrypted, so your application never handles the real key.

    See [LLM Providers and Connections](/en/ai-gateway/llm-providers) for the full setup steps.
  </Step>

  <Step title="Create a Virtual API Key">
    Issue a **virtual API key** scoped to a user, role, project, or team. This is the key your application will use — it maps to your real provider connection(s) behind the scenes, so nothing provider-specific ever reaches client code.

    See [Virtual API Keys](/en/ai-gateway/virtual-api-keys) for scopes and rotation.
  </Step>

  <Step title="Point Your OpenAI SDK at Apinizer">
    Configure the OpenAI SDK (or any OpenAI-compatible client) with your gateway's `base_url` and the virtual key you just created:

    ```python
    from openai import OpenAI

    client = OpenAI(
        api_key="your-virtual-api-key",
        base_url="https://your-apinizer-gateway.com/api/ai/v1"
    )
    ```

    No other client-side changes are needed — the SDK talks to Apinizer exactly as it would talk to OpenAI directly.
  </Step>

  <Step title="Send Your First (Streaming) Chat Completion">
    ```python
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": "Hello, Apinizer!"}],
        stream=True
    )

    for chunk in response:
        if chunk.choices[0].delta.content:
            print(chunk.choices[0].delta.content, end="", flush=True)
    ```

    The response streams back chunk-by-chunk as it arrives from the provider — the same shape you'd get calling the provider directly, regardless of which provider actually served the request.
  </Step>
</Steps>
