## Overview

AI Cost Settings let you:
- Define **per-model pricing** in USD per 1M tokens
- Enable **multi-currency display** (USD or Turkish Lira)
- Control who can see and edit costs (system admins only)

## Per-Model Pricing

Each LLM model has a configurable price:

```
Model: gpt-4o
  Input price: $0.005 per 1M tokens
  Output price: $0.015 per 1M tokens

Model: gpt-4-turbo
  Input price: $0.01 per 1M tokens
  Output price: $0.03 per 1M tokens
```

Pricing is **stored in USD** regardless of display currency.

## Currency Display

By default, costs display in **USD** (e.g., `$0.05`).

Optionally, enable **Turkish Lira (TRY)** display:
1. Go to **AI Governance** → **AI Cost Settings**
2. Set **Exchange Rate**: USD to TRY (e.g., `38.5`)
3. Select **Display Currency**: USD or TRY
4. Click **Save**

When display currency is **TRY**:
- Internal cost: `$0.0012` (stored as USD)
- Display cost: `0.05 ₺` (calculated as `0.0012 × 38.5`)

## Configuring Per-Model Pricing

<Steps>
  <Step title="Access Admin Settings">
    **Admin** → **AI Gateway** → **AI Cost Settings** (system admin only).
  </Step>
  
  <Step title="View Model List">
    A table of all models in use is displayed.
    
    Click on a model row to expand or edit its pricing.
  </Step>
  
  <Step title="Update Price">
    Edit the **Input Price** and **Output Price** fields (USD per 1M tokens).
    
    Prices are **optional** — if left blank or zero, that model uses a fallback default (usually $0, meaning free).
  </Step>
  
  <Step title="Save Changes">
    Click **Save**. Pricing changes are **retroactive** — historical logs are not recalculated, but new requests use the updated price.
  </Step>
</Steps>

## Multi-Currency Flow

<Card title="Internal Storage">
  All costs are stored in **USD** in the database and logs:
  - `LogApiProxy.costTotalUsd`
  - `AiTokenUsageDaily.costUsd`
  - Monthly budget limits (always in USD)
</Card>

<Card title="Display Layer">
  UI and reports use the **display currency setting**:
  - `formatCost(value, "TRY", rate)` → "0.04 ₺"
  - `formatCost(value, "USD")` → "$0.001"
</Card>

<Card title="Backward Compatibility">
  If display currency is **not set** or rate is **0**, costs default to **USD**.
  
  No data migration or schema changes needed — the setting is purely cosmetic.
</Card>

## Exchange Rate Management

The USD-to-TRY exchange rate is **static** and manually entered by the admin:

- **Manual entry**: Click **AI Cost Settings** and type the current rate (e.g., `38.5`)
- **Update frequency**: Set by admin as often as needed (daily, weekly, etc.)
- **Fallback**: If rate is not set, TRY display is disabled and USD is used

:::warning
The exchange rate is **not fetched automatically** from an external service. Updates are manual. Plan to refresh the rate regularly if using TRY display.
:::

## Cost in Reports

Once configured, costs appear in **AI Reports** with the selected currency:

- **Dashboard** → shows **Cost YTD** with currency badge
- **Reports** → tabs like "By Model" show cost breakdown per model
- **Export (XLSX)** → columns show cost in the display currency

## Permissions

**AI Cost Settings** can only be viewed and edited by **system administrators**. Regular users see costs in reports but cannot change pricing.

:::note
Non-admin roles can still see usage and cost breakdowns in **AI Reports** — they just cannot edit the rate configuration.
:::

## Troubleshooting

**Exchange Rate Not Updating**
- Confirm you saved the form (green success message)
- Clear browser cache (`Cmd + Shift + R` on Mac, `Ctrl + Shift + R` on Windows)
- Try again

**Costs Still Showing in USD**
- Confirm display currency is set to **TRY** (not USD)
- Verify exchange rate is **not zero**
- Refresh the report page

**Per-Model Pricing Not Applied**
- Check that the model name matches exactly (case-sensitive)
- Verify price is greater than zero
- New requests after the save will use the updated price; old logs retain original cost
