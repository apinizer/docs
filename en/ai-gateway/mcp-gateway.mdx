## What Is MCP Gateway

MCP (Model Context Protocol) is the standard AI agents use to discover and call external tools. Apinizer AI Gateway participates on **both sides** of that exchange:

- **Inbound MCP Server** — exposes your API proxies as MCP tools that any MCP-compatible agent can discover and call
- **Outbound MCP Client** — connects to external MCP servers so your AI flow can call their tools as part of a request

:::info
New to the MCP protocol itself? See [AI Fundamentals](/en/concepts/core-concepts/ai-fundamentals) for a primer — this page focuses on what Apinizer does with it.
:::

## Inbound: Apinizer as an MCP Server

<CardGroup cols={2}>
  <Card title="Tool Registry and Discovery" icon="search">
    Once enabled, an inbound MCP server publishes a catalog of tools that an agent can list and call through standard MCP discovery calls.
  </Card>
  <Card title="Tool Allow-List" icon="shield-check">
    Only explicitly permitted tools are callable — secure by default. A tool that isn't on the allow-list is rejected outright rather than silently exposed.
  </Card>
  <Card title="Authentication" icon="key">
    Inbound connections require OAuth2 or JWT authentication before any tool call is accepted.
  </Card>
  <Card title="Per-Tool Access Control" icon="lock">
    Each tool can require specific scopes. A caller that's authenticated but lacks the required scope is denied that particular tool.
  </Card>
</CardGroup>

Tool results also pass back through the same output filtering used elsewhere in the gateway — personal data detected in a tool's output is masked before an agent sees it, using the same rules as your other guardrail policies.

## Outbound: Apinizer as an MCP Client

Connect Apinizer to an external MCP server the same way you'd connect to an LLM provider — as a stored, encrypted connection — then let an AI proxy call the server's tools as part of a request.

<CardGroup cols={2}>
  <Card title="Automatic Tool Discovery" icon="search">
    Once an outbound connection is saved, Apinizer fetches the target server's tool catalog automatically.
  </Card>
  <Card title="Health Probe" icon="activity">
    Outbound MCP connections are probed periodically; each connection reports **Healthy**, **Down**, or **Unknown** status with its last response time.
  </Card>
  <Card title="Egress Protection" icon="shield">
    Outbound calls (health probes and tool calls) are protected against reaching private or internal network addresses. This protection is a system-admin-only setting.
  </Card>
  <Card title="Tool-Call Loop" icon="repeat">
    Tool results feed back into the model's context automatically as part of the [agentic tool-call loop](/en/ai-gateway/routing-and-failover#tool-call-loop-agentic-requests).
  </Card>
</CardGroup>

## Setting Up an Inbound MCP Server

<Steps>
  <Step title="Open MCP Settings">
    In the Apinizer UI, go to **AI Gateway** → **MCP** → **Inbound Servers**.
  </Step>

  <Step title="Register Tool Sources">
    Choose which API proxy operations should be published as MCP tools.
  </Step>

  <Step title="Configure the Allow-List">
    Explicitly enable only the tools an agent should be able to call. Anything not enabled is rejected.
  </Step>

  <Step title="Set Authentication and Scopes">
    Require OAuth2 or JWT for inbound connections, and set the required scope for each tool.
  </Step>

  <Step title="Save and Deploy">
    Click **Save**, then deploy the server so it becomes reachable to agents.
  </Step>
</Steps>

## Setting Up an Outbound MCP Connection

<Steps>
  <Step title="Add an MCP Connection">
    Go to **AI Gateway** → **MCP** → **Outbound Connections** and enter the target server's address and authentication.
  </Step>

  <Step title="Review the Discovered Tools">
    Apinizer fetches and previews the target server's tool catalog automatically once the connection is saved.
  </Step>

  <Step title="Add the MCP Tool Call Policy">
    Attach the MCP tool call policy to your AI proxy and select which discovered tool(s) it may use.
  </Step>

  <Step title="Monitor Health">
    Check connection status (Healthy / Down / Unknown) and last response time from the Tool Registry screen.
  </Step>
</Steps>
