## What is Apinizer AI Gateway?

Apinizer AI Gateway is a unified, OpenAI-compatible proxy and gateway that sits between your applications and multiple large language model (LLM) providers. Instead of managing direct connections to each provider separately, you connect once to Apinizer and route requests to any supported LLM without changing your client code.

Key benefits:
- **Single endpoint** — OpenAI SDK drop-in compatibility via configurable `base_url`
- **Multi-provider support** — connect to OpenAI-compatible services (OpenAI, Azure, Groq, Mistral, vLLM, Ollama) as well as **non-OpenAI-format** providers (Anthropic, Google Vertex/Gemini, AWS Bedrock), whose responses are automatically normalized to OpenAI format
- **True streaming** — per-chunk Server-Sent Events (SSE) with no buffering or artificial latency
- **Built-in controls** — request filtering, PII masking, semantic caching, token quotas, and cost tracking
- **Enterprise visibility** — detailed usage reports by user, team, model, and deployment type (cloud vs. on-premise)

## Supported LLM Providers

Apinizer AI Gateway works with:
- **Cloud providers**: OpenAI, Anthropic (Claude), Azure OpenAI, Google Vertex AI, AWS Bedrock
- **Self-hosted**: vLLM, Ollama, Hugging Face TGI
- **Custom**: Any OpenAI-compatible API endpoint

Each provider is configured through a **connection** with encrypted API credentials and deployment metadata.

:::note Request and response format
The client side **always uses the OpenAI format** — there is no native Anthropic Messages or Gemini inbound format; the single interface is OpenAI. Provider **responses** are normalized to the OpenAI canonical format, while the **request body** is largely forwarded as-is (passthrough). As a result, OpenAI-compatible providers get a full round-trip, while native-format providers such as Anthropic/Gemini are supported through the provider's OpenAI-compatible endpoint or response normalization. **AWS Bedrock (SigV4) and Google Vertex (OAuth2) authentication will be completed in Phase 2.**
:::

## How It Works

### Request Pipeline

Every request flows through:

1. **Intake** — Request validated and parsed as OpenAI-format JSON
2. **Guardrails** (optional) — PII masking, content filters, prompt inspection
3. **Semantic Cache** (optional) — Similar requests reuse cached responses
4. **Rate Limiting** — Token quota enforcement (per-minute, hour, day, month, or USD budget)
5. **Routing** — Provider connection selected based on model ID
6. **Inference** — Request forwarded to LLM provider
7. **Response** — Streamed back (chunk-by-chunk) with usage metrics

### OpenAI SDK Integration

Use Apinizer AI Gateway with the Python OpenAI SDK or any compatible client:

```python
from openai import OpenAI

client = OpenAI(
    api_key="your-apinizer-credential-key",
    base_url="https://your-apinizer-gateway.com/api/ai/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello"}],
    stream=True
)

for chunk in response:
    print(chunk.choices[0].delta.content, end="")
```

No client-side code changes needed — just point `base_url` to your Apinizer gateway.

## Key Concepts

### Deployment Type

Each LLM provider connection has a **deployment type**:
- **Cloud** (default for OpenAI, Anthropic, Azure, Vertex, Bedrock) — hosted by the provider
- **On-Premise** (default for vLLM, Ollama, custom endpoints) — self-hosted in your infrastructure

Deployment type is tracked in logs and usage reports for cost attribution and capacity planning.

### Credentials & Secrets

All API keys and secrets are **encrypted at rest** and **never logged**. Credentials are managed through Apinizer's credential system.

### Token Quotas

Control costs and prevent runaway usage with per-model, per-scope quotas:
- **Minute / Hour / Day / Month** — token buckets with configurable limits
- **Monthly budget** — USD spending cap with automatic soft-limit or hard-block
- **Per-model limits** — apply different quotas to GPT-4o vs. GPT-4 Turbo, etc.

### Reports & Analytics

Track usage across dimensions:
- By person, team, organization, or application
- By model and provider
- By cloud vs. on-premise deployment
- Cost breakdown with optional multi-currency display (USD or TRY)
- Performance latency (guardrail time, inference time, overhead)
