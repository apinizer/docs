## Overview

In addition to text-based chat completion endpoints, Apinizer AI Gateway now supports three new modalities:

- **Speech to Text (STT)** â€” `/v1/audio/transcriptions`
- **Text to Speech (TTS)** â€” `/v1/audio/speech`
- **Image Generation** â€” `/v1/images/generations`

All three endpoints use the **OpenAI-compatible request/response format**; your existing OpenAI SDK clients work without code changes beyond pointing `base_url` at Apinizer.

:::info

**Video modality** is not included in this release â€” it is planned for a future phase.

:::

## Speech to Text (STT)

Transcribes an audio file into text:

```python
from openai import OpenAI

client = OpenAI(
    api_key="your-apinizer-credential-key",
    base_url="https://your-apinizer-gateway.com/api/ai/v1"
)

with open("recording.mp3", "rb") as audio_file:
    transcript = client.audio.transcriptions.create(
        model="whisper-1",
        file=audio_file
    )

print(transcript.text)
```

- The request is sent as **multipart/form-data**
- Usage is metered based on **audio duration**, rather than token count
- The response text can be passed through the existing **PII masking** policy â€” personal data detected in the transcript is masked using the same rules as other text endpoints

## Text to Speech (TTS)

Converts text into speech; the response is returned as a **binary audio stream**:

```python
response = client.audio.speech.create(
    model="tts-1",
    voice="alloy",
    input="Hello, this is a test message."
)

response.stream_to_file("output.mp3")
```

:::note

The TTS response is not JSON, but an audio file (binary) returned directly â€” the gateway passes this response type through transparently, preserving the content type.

:::

## Image Generation

Generates an image from a text prompt:

```python
response = client.images.generate(
    model="gpt-image-1",
    prompt="Sunset over the ocean, digital art",
    n=1,
    size="1024x1024"
)

print(response.data[0].url)
```

## Pricing and Usage Metrics

The new modalities are integrated into the existing **AI Cost Settings** and **Reports** infrastructure:

| Modality | Pricing Unit | Usage Metric |
|----------|--------------|--------------|
| STT (whisper) | Per minute | Audio duration (seconds/minutes) |
| TTS | Per character | Input character count |
| Image generation | Per image (by size) | Number of images generated |

These metrics are visible in **Reports and Analytics** alongside other LLM models, broken down by person/team/model.

:::tip

New models (such as whisper, TTS, and gpt-image) ship with **pre-configured prices** in the catalog; you can customize model pricing from the **AI Cost Settings** page if needed.

:::

## Endpoint Routing

The gateway automatically recognizes the target path of the incoming request (`/v1/chat/completions`, `/v1/audio/transcriptions`, `/v1/audio/speech`, `/v1/images/generations`) and routes it to the modality-specific processing pipeline (including binary response support, multipart parsing, and duration-based usage calculation) â€” no additional configuration is required, and all modalities can be served through the same AI proxy.
