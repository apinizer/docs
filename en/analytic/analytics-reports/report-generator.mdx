Fields used for Report Generator configuration are shown in the table below.

| Field | Description |
|------|----------|
| Scheduling Section<br/>*(Trigger Every)* | Settings for how often the report will run are made in this section. |
| Query Section<br/>*(Query)* | Which query should be run for the report is specified in this section. Previously created ready queries can be used. If there is no configuration in existing queries that will contain the desired result, a new query can also be created on this screen. |
| Filter Section<br/>*(Filter)* | Which filter should be run for the report is specified in this section. Previously created ready filters can be used. If there is no configuration in existing filters that will contain the desired result, a new filter can also be created on this screen. |
| Report Fields Section<br/>*(Report Fields)* | Which fields from the fields stored in logs should be included when the report is generated is selected in this section. Thus, unnecessary data appearing in the report is prevented. |
| Test Section<br/>*(Environment)* | Whether the desired result comes when the report is run can be checked in this section. |
| Action Section<br/>*(Action)* | Where the report result will be sent is set in this section. For example, when mail connection is selected, the report result is sent to the specified users via mail. For detailed information about actions, you can visit [this page](/en/integrations/connectors/overview). |

<Info>
The maximum result size that can be sent to an action in a report is 25MB.
</Info>
