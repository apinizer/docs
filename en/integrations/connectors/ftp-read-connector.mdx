## FTP File Read Connector Usage

<Info>
  Thanks to the FTP file read connector you will create, you can perform file reading operations from FTP server. This connector provides flexible search criteria such as searching by file name start, exact match, or a specific pattern, and the ability to produce output with various output formats, allowing you to produce solutions suitable for your needs.
</Info>

## HTTP Request

```bash
curl --location --request POST "https://<APINIZER_ACCESS_URL>/<RELATIVE_PATH>" \
  -H "Content-Type: application/json" \
  --data-raw "{\"caseInsensitive\": true, \"searchType\": \"STARTS_WITH\", \"searchFileName\": \"invoice_2025_\", \"returnType\": \"FIRST_MATCH\", \"maxFileSizeKB\": 10240, \"outputFormat\": \"JSON_BASE64\"}"
```

Request body (Body) is optional. If request body is sent empty, default values are used.

## Example Scenarios

### Example 1: JSON Format - First Match

**Usage Scenario**: Find the first file starting with "invoice_2025_" and return in JSON format. Accept files up to 10 MB in size.

```json
{
  "caseInsensitive": true,
  "searchType": "STARTS_WITH",
  "searchFileName": "invoice_2025_",
  "returnType": "FIRST_MATCH",
  "maxFileSizeKB": 10240,
  "outputFormat": "JSON_BASE64"
}
```

### Example 2: ZIP - All Matches

**Usage Scenario**: Find all files containing ".pdf" (case sensitive) and download as ZIP archive.

```json
{
  "caseInsensitive": false,
  "searchType": "CONTAINS",
  "searchFileName": ".pdf",
  "returnType": "ALL_MATCHES",
  "outputFormat": "ZIP"
}
```

### Example 3: MTOM (Message Transmission Optimization Mechanism)

**Usage Scenario**: Find the file named "payload.xml" with exact match and return in MTOM format.

```json
{
  "searchType": "EXACT_MATCH",
  "searchFileName": "payload.xml",
  "returnType": "FIRST_MATCH",
  "outputFormat": "MTOM"
}
```

## ENUM Values

<AccordionGroup>
  <Accordion title="SearchType ENUM Values">
    SearchType value specifies how the given file name value (searchFileName) will be searched in the relevant directory.

    <CardGroup cols={3}>
      <Card title="STARTS_WITH" icon="arrow-right">
        Checks if the file name value starts with a specific pattern.
      </Card>
      
      <Card title="EXACT_MATCH" icon="equals">
        Checks the exact match status of the target file name with the given file name.
      </Card>
      
      <Card title="CONTAINS" icon="search">
        Checks whether the file name contains the given pattern.
      </Card>
    </CardGroup>
  </Accordion>
  
  <Accordion title="ReturnType ENUM Values">
    <CardGroup cols={2}>
      <Card title="FIRST_MATCH" icon="arrow-down">
        Only the first matching file is returned.
      </Card>
      
      <Card title="ALL_MATCH" icon="list">
        All matching files are returned.
      </Card>
    </CardGroup>
  </Accordion>
  
  <Accordion title="OutputFormat ENUM Values">
    <CardGroup cols={3}>
      <Card title="JSON_BASE64" icon="file-code">
        File content is returned in JSON format, Base64 encoded.
      </Card>
      
      <Card title="MTOM" icon="file">
        Multipart response is returned in MTOM (Message Transmission Optimization Mechanism) format.
      </Card>
      
      <Card title="ZIP" icon="file-zipper">
        Matching files are returned in ZIP archive.
      </Card>
    </CardGroup>
  </Accordion>
</AccordionGroup>
